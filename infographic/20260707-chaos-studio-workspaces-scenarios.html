Workspaces and Scenarios

アプリケーション中心のカオスエンジニアリング -- Workspace でリソース自動検出、Scenario テンプレートで即座にテスト開始

In Preview 2026-07-07

概要

Chaos Studio に Workspaces と Scenarios が新たに追加された。アプリケーション中心のアプローチで、Workspace にスコープ (サブスクリプション、リソースグループ、サービスグループ) を指定すると、リソースを自動検出し実行可能な Scenario を提示する。従来の専門知識が必要だった実験設計が、事前構成テンプレートで大幅に簡素化される。

Before (課題)

  • カオス実験の設計にステップ、ブランチ、アクション、ターゲット、セレクタを個別構成する専門知識が必要
  • 障害パターン再現には個々のフォールトを手動で組み合わせる必要
  • 対象リソースのオンボーディングやターゲット設定を手動で行う必要
  • テスト結果のレポートを別途作成する必要

After (改善)

  • Workspace のスコープ指定だけでリソース自動検出
  • 事前構成 Scenario テンプレートにより設定不要でテスト開始
  • Scenario designer でカスタムシナリオを視覚的に作成
  • Scenario report による自動レポート生成
  • マネージド ID による自動ロール割り当て

Workspace でスコープを設定し、リソースを自動検出。Scenario ライブラリまたは Designer からシナリオを選択・作成し、ターゲットリソースに対して実行。結果は Scenario Report で自動レポート化。

Key Features

🔧

Workspace (ワークスペース)

スコープを定義しリソース自動検出。チーム/環境ごとに整理し、マネージド ID による自動ロール割り当てで権限管理を簡素化。

⚡

Scenario (シナリオ)

実際の障害パターンを再現する事前構成テスト。アクション組み合わせ・シーケンシングを自動処理し、複雑な実験設計が不要に。

🎨

Scenario Designer

Azure ポータル上のビジュアルデザイナー。テンプレートベースでカスタマイズし、コード不要で独自シナリオを構築可能。

🤖

Chaos Studio AI Plugin

GitHub Copilot CLI / MCP サーバーとして動作。Azure Monitor との相関分析で障害影響の可視化を支援。

Bicep サンプル

カスタム Scenario の定義例 (Zone Down + DB Failover):

resource customScenario 'Microsoft.Chaos/workspaces/scenarios@2026-05-01-preview' = {
  parent: workspace
  name: 'zone-down-then-db-failover'
  properties: {
    description: 'Shut down a zone, then fail over the database.'
    parameters: [
      {
        name: 'zone'
        type: 'string'
        required: true
      }
    ]
    actions: [
      {
        name: 'shutdown-zone'
        actionId: 'microsoft-compute-shutdown/1.0'
        duration: 'PT10M'
        parameters: [
          { key: 'zones', value: '%%{parameters.zone}%%' }
        ]
      }
    ]
  }
}

Use Cases

UC1: ゾーン障害への対応検証

マルチゾーン構成の E コマースアプリケーションにおいて、可用性ゾーン障害時のフェイルオーバー動作を検証する。RTO (目標復旧時間) 内にサービスが回復するかを確認。

-- Compute Zone Down シナリオを実行し、ゾーン障害をシミュレート。Scenario Report で RTO 達成状況を自動的にレポート化。

UC2: CI/CD パイプラインへの組み込み

デプロイメントパイプラインのゲートとして Chaos Studio Scenario を実行し、新しいリリースがレジリエンス要件を満たしているかを自動検証する。

-- Bicep/ARM テンプレートで Scenario を定義し、デプロイ後に自動実行。レジリエンスリグレッションを早期検出し品質ゲートとして機能。

UC3: キャッシュ障害耐性の検証

Redis Cache 障害時にバックエンドデータベースが過負荷にならないか、キャッシュスタンピード対策が正しく機能しているかを検証する。

-- Cache Stampede シナリオで Redis 障害を注入し、バックエンドの負荷状況と回復パターンを確認。Azure Monitor との相関分析で影響範囲を可視化。
