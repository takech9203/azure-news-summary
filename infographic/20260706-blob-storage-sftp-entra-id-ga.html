Azure Blob Storage Launched (GA) 2026-07-06

SFTP における Microsoft Entra ID ベースのアクセス
(一般提供開始)

Azure Blob Storage の SFTP エンドポイントにおける Microsoft Entra ID ベースのアクセスが全リージョンで GA。従来のローカルユーザー (パスワード/SSH 鍵) に加え、Entra ID の ID (ユーザー、サービスプリンシパル、ゲストユーザー) で SFTP 接続が可能に。

📋 概要
Azure Blob Storage の SFTP エンドポイントが Microsoft Entra ID による認証に対応。Azure RBAC / ABAC / ACL による統一的なアクセス制御、多要素認証 (MFA) によるセキュリティ強化、Entra External Identities によるゲストユーザーアクセスが可能になり、企業レベルのセキュリティと運用効率を両立。

Before

  • ストレージアカウントごとにローカルユーザーの作成・管理が必要
  • パスワードや SSH 鍵の配布・ローテーションの運用負荷が高い
  • ローカルユーザーは Azure RBAC や ABAC と相互運用できない
  • 外部ユーザーへの SFTP アクセス付与にローカルユーザー管理が必要
  • プレビュー段階では本番ワークロード非推奨

After

  • Entra ID の既存 ID をそのまま SFTP 認証に利用可能
  • Azure RBAC、ABAC、ACL による統一的なアクセス制御
  • 多要素認証 (MFA) によるセキュリティ強化
  • Entra External Identities によるゲストユーザーアクセス
  • GA により SLA 適用、本番ワークロード正式サポート
  • 全リージョンで利用可能

⭐ 主要機能
🔒

Microsoft Entra ID 認証の統合

ユーザー、サービスプリンシパル、ゲストユーザーが Entra ID で SFTP 接続可能。既存のディレクトリサービスの ID をそのまま活用し、ローカルユーザーの個別管理が不要に。マネージド ID は非サポート。

📜

OpenSSH 証明書ベースの認証

Entra ID 認証後、65 分間有効な OpenSSH 証明書が発行される。RSA 鍵のみサポート。証明書の有効期限により、長期間有効なクレデンシャルの漏洩リスクを低減。

🛡

Azure RBAC / ABAC / ACL との統合

Storage Blob Data Contributor 等の既存ロールが SFTP にも適用。属性ベースのアクセス制御 (ABAC) やコンテナレベルの ACL を組み合わせたきめ細かな認可が可能。

🤝

外部コラボレーション (Entra External Identities)

ゲストユーザーを招待し SFTP アクセスを付与。外部パートナーや委託先との安全なファイル共有を、ローカルユーザーの個別作成なしに実現。

💡 ユースケース

🏭 レガシーシステムとの統合

既存の SFTP ベースのバッチ処理やファイル転送システムをクラウドに移行する際、サービスプリンシパルで認証し自動化パイプラインを構築。ローカルユーザーの管理が不要になり、Azure RBAC による統一的な権限管理が実現。

効果: サービスプリンシパルによる認証で、パスワードや SSH 鍵のローテーション運用を排除。証明書の短い有効期限 (65分) によりクレデンシャル漏洩のリスクを最小化。

🌐 外部パートナーとのセキュアなファイル共有

取引先やサプライヤーとの定期的なデータ交換で、Entra External Identities を活用してゲストユーザーに SFTP アクセスを付与。個別のローカルユーザー作成や鍵配布が不要に。

効果: MFA による追加認証で外部アクセスのセキュリティを確保しつつ、ABAC で特定コンテナ・ディレクトリのみへのアクセスを制限。オンボーディング/オフボーディングが Entra ID 上で完結。

🏢 マルチテナント環境での統一的なアクセス管理

複数のストレージアカウントを持つ大規模組織で、各テナントのユーザーに一貫したアクセスポリシーを適用。Entra ID の条件付きアクセスや RBAC により、全環境を統一的に管理。

効果: ストレージアカウントごとのローカルユーザー管理が不要になり、組織全体で一貫したセキュリティポリシーの適用とコンプライアンス監査が容易に。
💰 料金
SFTP エンドポイントの有効化には時間単位のコストが発生します。
Entra ID ベースのアクセス自体に追加料金は発生しません。

従来のローカルユーザー認証と比較して、Entra ID 認証による追加コストはないため、セキュリティ強化のための追加投資なしに移行可能です。
⚠ 制約事項・制限
  • パスワード認証は非サポート (証明書ベースのみ)
  • RSA 鍵のみサポート、ECDSA は利用不可
  • マネージド ID は非サポート
  • ホームディレクトリ設定は非サポート
  • 証明書の有効期限が 65 分のため、長時間のファイル転送には証明書の更新が必要
  • 階層型名前空間 (HNS) の有効化が必須前提条件
