Microsoft Entra ID トークンリフレッシュサポート

Python, .NET, JavaScript クライアントライブラリによる自動トークンリフレッシュで認証管理を簡素化

Preview (プレビュー) 2026-05-20

概要

Azure Database for PostgreSQL Flexible Server において、Microsoft Entra ID のトークンリフレッシュが Python、.NET、JavaScript のクライアントライブラリでネイティブサポートされました。これにより、開発者はトークンのライフサイクル管理を手動で実装する必要がなくなり、長時間稼働するアプリケーションでも接続が途切れることなく安定した認証を維持できます。OAuth 2.0 / JWT ベースの認証フローが自動化され、運用負荷が大幅に軽減されます。

Key Points

  • クライアントライブラリがトークン有効期限を自動監視し、期限切れ前にリフレッシュを実行
  • ユーザートークンは最大 1 時間、マネージド ID トークンは最大 24 時間で期限切れ
  • Python、.NET、JavaScript の 3 言語でサポート
  • 対象: Flexible Server のみ
  • Token Resource URI: https://ossrdbms-aad.database.windows.net/.default

Before

  • 開発者がトークンの有効期限を手動で追跡する必要があった
  • リフレッシュロジックを独自に実装する必要があった
  • 長時間稼働アプリケーションでトークン切れによる接続エラーのリスク
  • ボイラープレートコードが増加し保守コストが高い

After

  • クライアントライブラリが自動的にトークンリフレッシュを処理
  • 接続が途切れることなくシームレスに再認証
  • ボイラープレートコードの大幅削減
  • 開発者はビジネスロジックに集中可能

コードサンプル

.NET での接続例

C# 
using Azure.Identity;
using Npgsql;

var tokenCredential = new DefaultAzureCredential();
var accessToken = tokenCredential.GetToken(
    new Azure.Core.TokenRequestContext(new[] { "https://ossrdbms-aad.database.windows.net/.default" })
);

var connectionString = $"Host={host};Database={database};Username={user};Password={accessToken.Token};SSL Mode=Require;Trust Server Certificate=true";

using var connection = new NpgsqlConnection(connectionString);
connection.Open();

ユースケース

⚙

長時間稼働するバックグラウンドサービス

ETL 処理やバッチジョブなど、数時間から数日にわたって実行されるサービスで、トークン切れによる処理中断を防ぎます。自動リフレッシュにより、再接続ロジックの実装が不要になります。

🛠

マイクロサービスアーキテクチャ

複数のサービスがマネージド ID で PostgreSQL に接続する環境で、各サービスが個別にトークン管理を行う必要がなくなります。統一的な認証パターンでアーキテクチャを簡素化します。

⚡

サーバーレスアプリケーション

Azure Functions や App Service で動作するアプリケーションが、接続プールを長時間保持する際の認証を自動的に維持。コールドスタートからの復帰時も安定した接続を確保します。
