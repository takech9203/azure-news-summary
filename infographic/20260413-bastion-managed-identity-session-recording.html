SAS トークン管理から解放 -- マネージド ID でセッション記録のストレージアクセスをよりセキュアに

📋

概要

Azure Bastion のグラフィカルセッション記録機能において、ストレージアカウントへの書き込みアクセスにマネージド ID (システム割り当て / ユーザー割り当て) が利用可能になった。従来の SAS URL に代わる認証方式として、セキュリティと運用効率の両方が向上する。

Before

  • SAS URL の生成・管理が必要
  • SAS トークンの有効期限管理とローテーション作業が運用負荷
  • SAS トークンの漏洩リスクがあり、セキュリティ上の懸念
  • 期限切れによりセッション記録が失敗する可能性

After

  • マネージド ID により SAS トークンの管理が不要に
  • Azure RBAC でアクセス制御、セキュリティポスチャ向上
  • 有効期限管理やローテーション作業から解放
  • システム割り当て / ユーザー割り当てを選択可能

Azure Bastion がマネージド ID を使用してストレージアカウントに認証し、セッション録画データを Blob コンテナに書き込む。管理者は Azure Portal の統合ビューアからセッション録画を再生できる。

主要機能

🔐

マネージド ID によるストレージ認証

システム割り当て / ユーザー割り当てマネージド ID でストレージアカウントに認証。SAS URL の生成・管理・ローテーションが不要になり、Azure RBAC (Storage Blob Data Contributor ロール) を通じたアクセス制御が可能に。

⚙

システム割り当てマネージド ID の設定

Bastion リソースの Identity ブレードでシステム割り当てマネージド ID を有効化し、ストレージアカウントに対して Storage Blob Data Contributor ロールを割り当て。Configuration で Blob コンテナ URI を指定して使用開始。

👤

ユーザー割り当てマネージド ID の設定

事前に作成したユーザー割り当てマネージド ID を Bastion に関連付け。Bastion リソースのライフサイクルとは独立した ID 管理が可能で、複数リソース間での共有にも対応。

🔄

既存の SAS URL 方式との共存

マネージド ID は SAS URL の代替として提供される。既存の SAS URL 方式も引き続き利用可能であり、段階的な移行が可能。

設定方法

前提条件

  • ⚠ Azure Bastion が Premium SKU でデプロイされていること
  • ⚠ セッション記録用のストレージアカウントと Blob コンテナが作成済みであること
  • ⚠ ストレージアカウントに CORS ポリシーが設定されていること
  • ⚠ ストレージアカウントに不変ストレージポリシーが設定されていないこと
  • ⚠ Blob バージョニングが無効であること

Azure Portal (マネージド ID 設定手順)

  1. Azure Portal で Bastion リソースに移動する
  2. 左ペインから Identity ブレードを選択する
  3. システム割り当てマネージド ID のステータスを On に変更し、構成の完了を待つ
  4. Azure role assignments を選択し、Add role assignment (Preview) をクリックする
  5. ロール割り当てを追加: スコープ: Storage / リソース: ストレージアカウント名 / ロール: Storage Blob Data Contributor
  6. Save をクリックしてロール割り当てを保存する
  7. Bastion リソースに戻り、左ペインから Configuration を選択する
  8. Session Recording Configuration で System Assigned Managed Identity を選択する
  9. Blob Container URI にストレージコンテナの URI を入力する
  10. Apply をクリックして設定を適用する (反映に約 10 分)

ストレージアカウントの CORS 設定

ストレージアカウントの Resource sharing (CORS) で Blob service に以下のポリシーを設定:

項目 値
Allowed origins https:// + Bastion の完全な DNS 名 (bst- で始まる)
Allowed methods GET
Allowed headers *
Exposed headers *
Max age 86400

ユースケース

🏦 金融機関の特権アクセス監査

金融機関において、本番環境の仮想マシンへの特権アクセスを記録し、監査証跡を残す必要がある。SAS トークンの管理を排除し、セキュリティポスチャを強化したい。

期待される効果
マネージド ID で SAS トークン漏洩リスクを排除。全セッションの録画を安全にストレージに保存し、Azure RBAC と組み合わせて監査要件を満たす堅牢なセッション管理を実現。

👥 マルチチーム環境でのセッション記録管理

複数のチームが共有する Bastion ホストで、セッション記録用ストレージのアクセス管理を簡素化したい。SAS URL の共有・ローテーション管理が煩雑になっている。

期待される効果
ユーザー割り当てマネージド ID で Bastion リソースのライフサイクルとは独立した ID 管理が可能に。Azure RBAC でストレージアクセスを制御し、SAS URL 管理作業から解放。

Bastion からストレージアカウントへのデータ転送料金

⚠

制限事項・注意点

  • ⚠ パブリックプレビュー段階であり、SLA の対象外
  • ⚠ Premium SKU が必須。Basic や Standard SKU では利用不可
  • ⚠ Entra ID による RDP セッション認証とグラフィカルセッション記録は同時に利用できない
  • ⚠ ネイティブクライアント経由のセッション記録は現時点で非サポート
  • ⚠ セッション記録は 1 コンテナ / 1 ストレージアカウントの制限あり
  • ⚠ セッション記録が有効化されると、Bastion ホスト経由の全セッションが記録対象 (個別選択不可)
  • ⚠ アクティブなセッション中にストレージコンテナを変更するとセッションが中断される可能性あり
