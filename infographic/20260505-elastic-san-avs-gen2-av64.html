Azure Elastic SAN Launched (GA)

Azure Elastic SAN: Azure VMware Solution (AVS) Gen2 および AV64 SKU サポート

2026-05-05

概要

Azure Elastic SAN が Azure VMware Solution (AVS) Gen2 Private Cloud および AV64 SKU に対応し、一般提供 (GA) となった。Gen2 Private Cloud では ExpressRoute ゲートウェイが不要となり、単一の Private Endpoint で構成が可能に。iSCSI セッションのクローンが自動的に行われるため、接続構成が大幅に簡素化される。AV64 SKU での Elastic SAN データストアのサポートにより、より高いスケールとパフォーマンスのストレージオプションが利用可能に。

Before

  • Gen1 環境では ExpressRoute ゲートウェイ経由の接続が必要で、ゲートウェイメンテナンス時に一時的な接続問題が発生する可能性があった
  • 複数の Private Endpoint を作成・管理して iSCSI セッションをスケールする必要があった
  • AV64 SKU で Elastic SAN データストアが利用できず、高スケール環境でのストレージ拡張オプションが制限されていた

After

  • Gen2 では ExpressRoute ゲートウェイが不要となり、ゲートウェイメンテナンスの影響を受けなくなった
  • 単一の Private Endpoint で接続構成が完了し、iSCSI セッションは自動的にクローンされる
  • AV64 SKU で Elastic SAN データストアが利用可能になり、高パフォーマンス環境でのストレージ拡張が実現

ユースケース

大規模 VMware ワークロードのストレージ拡張

ホスト追加なしでストレージ容量を拡張、コスト効率の高いストレージ拡張を実現

パフォーマンス集約型データベースワークロード

SQL Server や Oracle で高 IOPS を活用

⚠️ 注意点・制約事項

  • Gen2 は AV64 SKU のみ対応
  • 同一リージョン・同一アベイラビリティゾーンに配置必要
  • データストアあたり最大接続数 128
  • CRC 保護は無効化が必要
  • 転送中の暗号化は非対応
