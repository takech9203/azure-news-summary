Azure Elastic SAN

Azure Elastic SAN: Windows VM 向け VM 拡張機能による接続が一般提供開始

Azure Portal から VM 拡張機能を追加するだけで、iSCSI と MPIO の構成が自動化され、Elastic SAN ボリュームへの接続をシンプルに実現

GA 2026-04-28

概要

Azure Elastic SAN で Windows VM 向けの VM 拡張機能 (VM Extension) による接続が一般提供 (GA) となった。Azure Portal から Elastic SAN ボリュームへの接続を直接構成できるようになり、iSCSI イニシエーターの有効化、MPIO のインストールと構成、接続スクリプトの実行が自動化される。VM 作成時のデプロイと同時にストレージ接続を確立でき、既存 VM への接続・切断も Portal の UI から直接操作可能になった。

Key Points

  • Azure Portal から VM 拡張機能を追加するだけで iSCSI と MPIO の構成が自動化
  • VM 作成時 (デプロイ時) に Elastic SAN ボリュームへの接続を同時に構成可能
  • 既存 VM への接続・切断も Azure Portal の UI から直接操作可能
  • ARM デプロイスクリプトを使用して大規模な展開にも対応
  • Virtual Machine Scale Sets での自動接続をサポート

Before

  • iSCSI サービスの有効化、MPIO のインストール、接続スクリプトの実行など複数の手動手順が必要だった
  • VM デプロイ後にストレージ接続を別途構成する必要があり、デプロイの自動化が困難だった
  • 大規模環境で複数の VM に対して一貫した接続設定を行うのが煩雑だった

After

  • Azure Portal から VM 拡張機能を追加するだけで、iSCSI と MPIO の構成が自動化される
  • VM 作成時 (デプロイ時) に Elastic SAN ボリュームへの接続を同時に構成できる
  • 既存 VM への接続・切断も Azure Portal の UI から直接操作可能
  • ARM デプロイスクリプトを使用して大規模な展開にも対応

主要機能

🖥️

VM 作成時の接続構成

VM デプロイと同時に Elastic SAN ボリュームへのストレージ接続が自動確立される。デプロイ完了後すぐにストレージを利用でき、追加の手動操作が不要。

🔌

既存 VM への接続・切断

Azure Portal の UI から Connect/Disconnect 操作が可能。既にデプロイ済みの VM に対しても、ポータルから簡単にボリューム接続を追加・削除できる。

⚙️

自動構成項目

iSCSI サービスの有効化、MPIO (Multipath I/O) のインストールと構成、接続スクリプトの実行がすべて VM 拡張機能により自動的に行われる。

📈

Virtual Machine Scale Sets 対応

スケールセット内の各 VM が自動的に Elastic SAN ボリュームへ接続。ARM デプロイスクリプトを使用して大規模な展開にも対応し、スケールアウト時の接続も自動化される。

ユースケース

大規模データベース環境のストレージ統合

複数の SQL Server VM が高 IOPS を必要とする環境で、Elastic SAN のパフォーマンス共有モデルを活用。VM 拡張機能により各 VM のストレージ接続を自動化し、運用負荷を大幅に削減。

効果: 接続構成の自動化により、大規模環境でも一貫したストレージ設定を迅速に展開可能

Virtual Machine Scale Sets でのスケーラブルストレージ

スケールアウト時に新規 VM が自動的に Elastic SAN ボリュームへ接続。ARM デプロイスクリプトと VM 拡張機能の組み合わせにより、スケーリングイベント発生時のストレージ接続もシームレスに処理。

効果: スケーリング操作後の手動接続が不要になり、完全自動化されたストレージ接続を実現
