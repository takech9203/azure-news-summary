Foundry Toolkit for Visual Studio Code が一般提供開始

旧 AI Toolkit for VS Code -- Microsoft Agent Framework を活用した AI エージェント開発の統合ツールセット。テンプレート、GitHub Copilot でのエージェント作成、ローカルテスト・デバッグ、Microsoft Foundry への直接デプロイをサポート。

Launched (GA) 2026-04-24

概要

Foundry Toolkit for VS Code (旧 AI Toolkit for VS Code) が一般提供 (GA) となった。Microsoft Agent Framework を活用した AI エージェント開発の統合ツールセットであり、テンプレートや GitHub Copilot を使ったエージェント作成、Agent Inspector によるローカルテスト・デバッグ、Microsoft Foundry への直接デプロイを VS Code 内で完結させる。インストール数は 100 万件を超え、モデルカタログ (Foundry, GitHub Models, ONNX, Ollama, OpenAI, Anthropic, Google 等) の統合や、ノーコードビルダーと SDK による 2 つの開発アプローチ、マルチエージェントワークフローの可視化、モデル評価・ファインチューニング・最適化まで幅広くカバーする。

Key Points

  • 旧 AI Toolkit for VS Code が Foundry Toolkit にリブランド、GA リリース
  • Agent Inspector による F5 デバッグ、リアルタイムストリーミング、マルチエージェント可視化
  • ノーコードビルダーと Agent Framework SDK (Python, C#) の 2 つの開発アプローチ
  • Foundry, GitHub Models, ONNX, Ollama, OpenAI, Anthropic, Google 等のモデルカタログ統合
  • Microsoft Foundry への直接デプロイとホステッドエージェント
  • MCP ツール統合、モデル評価・ファインチューニング・最適化機能を搭載

Before

  • モデル選択・テスト・デプロイが別々のツールに分散
  • マルチエージェントワークフローの可視化手段が限定的
  • ローカルからクラウドへの一貫したパイプラインがない
  • AI Toolkit はプレビュー段階で本番利用に不安

After

  • VS Code 内で開発ライフサイクル全体を統合管理
  • Agent Inspector でリアルタイムデバッグ・可視化
  • Microsoft Foundry への直接デプロイが可能に
  • GA で本番サポート・安定性が保証

1. モデルカタログとモデル探索

Foundry, GitHub Models, ONNX, Ollama, OpenAI, Anthropic, Google, HuggingFace 等の多様なモデルソースを統合。VS Code 内でモデルの検索・選択・テストが可能。

🤖

2. エージェント開発 (2 アプローチ)

ノーコードビルダーによる直感的な構築と、Agent Framework SDK (Python, C#) によるコードベースの開発の 2 つのアプローチをサポート。テンプレートや GitHub Copilot でのエージェント作成も可能。

🔍

3. Agent Inspector

F5 キーでデバッグ開始、リアルタイムストリーミング表示、マルチエージェントワークフローの可視化を提供。エージェントの動作をステップごとに追跡・検証できる。

🎮

4. インタラクティブ Playground

テキスト・画像などのマルチモーダル入力をサポートする対話型環境。モデルの応答をリアルタイムで確認しながら、プロンプトやパラメータを調整可能。

📊

5. モデル評価

F1 スコア、関連性、類似性、一貫性などのメトリクスでモデルのパフォーマンスを評価。モデル間の比較や品質保証に活用。

⚙

6. ファインチューニング

ローカル GPU または Azure Container Apps を使用してモデルのファインチューニングが可能。カスタムデータセットに基づく精度向上をサポート。

🚀

7. モデル最適化とデプロイ

モデルの変換・量子化と、Windows ML (CPU / GPU / NPU) へのデプロイをサポート。エッジデバイスやローカル環境での効率的な推論を実現。

🔧

8. MCP ツール統合

Model Context Protocol (MCP) ツールとの統合により、エージェントが外部ツールやデータソースと連携。拡張性の高いエージェント構築が可能。

Use Cases

UC1: カスタマーサポートエージェントの迅速な開発

ノーコードビルダーまたは Agent Framework SDK を使い、FAQ 応答、チケット分類、エスカレーション判断を行うカスタマーサポートエージェントを VS Code 内で構築。テンプレートから開始し、Playground で対話テスト、Agent Inspector でフロー確認後、Microsoft Foundry へ直接デプロイできる。

-- 開発から本番デプロイまでを VS Code 内で完結し、リリースサイクルを大幅短縮。

UC2: マルチエージェントワークフローの開発・デバッグ

複数のエージェントが連携するワークフロー (例: データ収集エージェント、分析エージェント、レポート生成エージェント) を構築する際、Agent Inspector のリアルタイムストリーミング・マルチエージェント可視化機能で各エージェント間のデータフローやハンドオフを詳細にトレースできる。

-- マルチエージェント間の問題を即座に特定し、複雑なワークフローの品質を向上。

UC3: ローカル環境でのモデル評価と最適化

複数のモデル候補を F1 スコア、関連性、類似性、一貫性のメトリクスで比較評価し、最適なモデルを選定。さらにモデルの量子化・変換を行い、Windows ML (CPU / GPU / NPU) でのローカル推論パフォーマンスを最適化する。

-- ローカル環境でのモデル選定から最適化までを一貫して実施し、コスト効率の高い推論を実現。

Limitations / Caveats

  • Foundry リソースアクセスには Azure サブスクリプションが必要
  • Windows ML プロファイリングは Windows 環境限定
  • クラウドモデル利用やデプロイには別途課金が発生
  • 名称変更 (AI Toolkit から Foundry Toolkit) に伴い、ドキュメント更新が途中の可能性がある
