Edge Actions

Azure Front Door PoP 上で JavaScript を実行するサーバーレスエッジコンピューティング

2026-07-14

📋 概要

Azure Front Door に Edge Actions がパブリックプレビューとして追加された。Microsoft のグローバル Azure Front Door PoP ロケーション上でカスタム JavaScript ロジックを直接実行できるサーバーレスエッジコンピューティング機能であり、オリジンサーバーへのラウンドトリップなしに A/B テスト、URL リライト、ヘッダー操作、カスタムルーティングなどを実現する。Hyperlight サンドボックスによるセキュアな実行環境で、超低レイテンシのビジネスロジック実行が可能。

Before

  • カスタマイズにはオリジンへのラウンドトリップが必要でレイテンシ増大
  • A/B テストに別途サーバーサイドインフラが必要
  • ルールエンジンだけでは複雑な条件分岐が困難
  • サードパーティのエッジコンピューティングへの依存

After

  • PoP 上で直接 JavaScript 実行、超低レイテンシ処理
  • エッジレベルで A/B テスト・ヘッダー操作を即時実装
  • Hyperlight サンドボックスでセキュアに実行
  • ゼロダウンタイムのバージョン管理デプロイ

⚙ 主要機能

⚡

クライアントリクエスト呼出

リクエスト到着時に JavaScript ロジックを実行し、リアルタイムの意思決定を実行。プレビュー期間中はクライアントリクエストの処理をサポート。

🛡

Hyperlight サンドボックス実行

各リクエストに対してセキュアなサンドボックスが生成され、イミュータブルなコンテキスト変数（サーバー変数、ヘルシーオリジン一覧、国コード、デバイスタイプ、日時情報）を提供。

🔄

バージョン管理

最大 3 バージョンのコードを管理可能。デフォルトバージョンの切り替えによるゼロダウンタイム更新と、問題発生時の即座のロールバックを実現。

🎯

実行フィルター (Execution Filters)

HTTP ヘッダーに基づいて実行バージョンを制御。カナリアリリース、A/B テスト、ヘッダーベースルーティングによる安全なバージョン移行を実現。

📝

診断ログ (Edge Action Logs)

console.log ステートメントが EdgeActionConsoleLog テーブルに出力され、Log Analytics ワークスペースでの一元管理とモニタリングが可能。

💻 コード例: A/B テスト

JavaScript Edge Action - A/B Testing 
async function handleRequest(context) {
    // Cookie から既存のバケット割当を確認
    const bucket = context.request.headers['x-ab-bucket']
        || (Math.random() < 0.5 ? 'A' : 'B');

    if (bucket === 'B') {
        // バケット B: 新デザインのオリジンへルーティング
        context.route.originGroup = 'new-design-origin';
    }

    // レスポンスヘッダーにバケット情報を付与
    context.response.headers['x-ab-bucket'] = bucket;
    return context.forward();
}

💡 ユースケース

🔀 A/B テストとカナリアリリース

シナリオ: 新しい Web ページデザインを一部のユーザーにのみ表示し、コンバージョン率を比較したい。
効果: オリジンサーバー側の実装変更なしに、エッジレベルで高速にトラフィック分割を実現。レイテンシの追加なく A/B テストが可能。

🌍 地理ベースのカスタムルーティング

シナリオ: ユーザーの接続元国に基づいて、異なるオリジンサーバーやコンテンツを提供したい。
効果: CDN 設定の変更なしに、エッジロジックのみで地理ベースのルーティングを実現。新規リージョン追加時もコードの更新のみで対応可能。

🔒 JWT トークン検証とアクセス制御

シナリオ: API リクエストの JWT トークンをエッジで検証し、不正なリクエストをオリジンに到達させたくない。
効果: オリジンサーバーへの不正リクエストの到達を防ぎ、オリジン負荷の軽減とセキュリティの向上を同時に実現。

💰 料金体系

Edge Actions は従量課金モデルを採用。以下の 2 要素で課金される。

呼出回数

Edge Action が呼び出された回数に基づく課金

実行時間

各呼出で 1 ms を超えた実行時間に基づく課金

⚠ 制限事項・注意点

  • パブリックプレビュー段階のため SLA の対象外。本番ワークロードへの適用は慎重に判断
  • JavaScript のみサポート (TypeScript / WebAssembly 等は利用不可)
  • コードサイズ 16 KB / 実行時間 10 ms の制限。超過時は Edge Action なしで転送
  • プレビュー期間中はクライアントリクエストの呼出のみ (レスポンス処理は未対応)
  • コードアップロードに最大 10 分かかる場合がある (プレビュー期間中)
