Draft & Deploy

Azure Firewall Policy - 2 Phase Policy Management

Release: 2026-03-03 Service: Azure Firewall SKU: Basic / Standard / Premium

📋 Overview

Azure Firewall Policy に新たに導入された Draft & Deploy 機能は、ファイアウォールポリシーの管理に 2 フェーズアプローチを採用し、デプロイ時間の短縮と運用中断の最小化を実現する新機能である。変更内容を「ドラフト」として事前に準備・検証し、準備が整った段階で一括して「デプロイ」する運用が可能となった。
📝

ドラフトフェーズで複数の変更を事前に蓄積し、一括デプロイが可能

🔍

デプロイ前にドラフト内容をレビュー・検証できるため、誤設定リスクを低減

⏱

デプロイタイミングを制御でき、メンテナンスウィンドウに合わせた適用が可能

🤝

変更管理ワークフロー（レビュー、承認、デプロイ）を自然に組み込める

Before

  • 個々の小さな変更ごとにフルデプロイメントが発生し、デプロイ時間が長期化
  • 変更途中の不完全な状態がプロダクション環境に反映されるリスク
  • レビューや承認プロセスをワークフローに組み込むことが困難
  • 変更のたびにネットワーク接続への影響が発生する可能性

After

  • 複数の変更を事前に蓄積し、一括デプロイでデプロイ回数と所要時間を大幅削減
  • デプロイ前にドラフト内容をレビュー・検証し、誤設定リスクを低減
  • 変更管理のワークフロー（レビュー、承認、デプロイ）を自然に統合
  • メンテナンスウィンドウに合わせた変更適用が可能

Draft & Deploy の 2 フェーズワークフロー。管理者がドラフトフェーズでルール変更を作成し、レビュアーが検証・承認した後、デプロイフェーズでアクティブな Azure Firewall Policy に一括適用される。

⚙ Key Features

📝

Draft Phase

ファイアウォールポリシーへの変更をドラフトとして作成・保存。プロダクション環境に影響を与えることなく、複数のルール変更を蓄積できる。

  • DNAT ルール、ネットワークルール、アプリケーションルールをドラフト内に蓄積可能
  • ドラフト状態の変更はプロダクション環境に影響しない
  • 変更内容をチーム間で共有・レビューするためのワークフローを支援
🚀

Deploy Phase

ドラフト内のすべての変更を一括してアクティブなポリシーに適用。デプロイ回数を最小化し、ファイアウォールへの影響を低減する。

  • すべての変更を一括適用
  • デプロイ回数の最小化
  • デプロイタイミングを管理者が制御可能
🔄

2 Phase Policy Management

ポリシーの「編集」と「適用」を明確に分離し、変更管理プロセス（Change Management）との統合を容易にする。

  • 編集と適用の明確な分離
  • Change Management との統合が容易
  • 複数の管理者が協調してポリシー変更を準備可能
⚡

Deploy Time Optimization

複数の変更を個別にデプロイする代わりに一括デプロイすることで、累積的なデプロイ時間を大幅に短縮。ネットワーク中断の回数を最小化する。

🛠 Setup (Azure Portal)

Prerequisites: Azure Subscription, Azure Firewall Policy (Basic/Standard/Premium), RBAC role (Network Contributor or higher)

1

Azure Portal にサインインし、対象の Azure Firewall Policy に移動する

2

Draft & Deploy セクション（またはポリシー編集画面）から「新しいドラフトを作成」を選択する

3

ドラフト内でルールコレクショングループ、ルールコレクション、個別ルールの追加・変更・削除を行う

4

変更内容をレビューし、問題がないことを確認する

5

「デプロイ」を選択して、ドラフトの変更を一括適用する

💡 Use Cases

1

Batch Rule Updates

企業のネットワークチームが、四半期ごとのセキュリティレビューに基づき、50 以上のファイアウォールルールを一括で更新する必要がある。

Implementation
  1. Draft & Deploy のドラフトフェーズで、すべてのルール変更を作成・蓄積
  2. セキュリティチームがドラフト内容をレビュー・承認
  3. 計画されたメンテナンスウィンドウ内で一括デプロイを実行
✅ 従来 50 回のデプロイが必要だった変更を 1 回のデプロイで完了でき、デプロイ時間を大幅に短縮。レビュープロセスで設定ミスのリスクも低減。
2

Multi-Team Policy Management

ネットワークチーム、セキュリティチーム、アプリケーションチームがそれぞれ異なるルール変更を要求する環境で、変更を安全に統合する。

Implementation
  1. 各チームがドラフトフェーズで必要なルール変更を提案
  2. ネットワークアーキテクトがすべての変更を統合・レビュー
  3. 承認後、統合されたドラフトを一括デプロイ
✅ チーム間の変更が競合するリスクを低減し、デプロイ前に全体的な整合性を確認できる。
3

Pre-Production Validation

本番環境のファイアウォールポリシー変更を、まずドラフトとして準備し、テスト環境で検証してから本番に適用する。

Implementation
  1. 本番環境のポリシーに対してドラフトを作成
  2. 同等の変更をテスト環境に先行適用して影響を確認
  3. テスト結果が問題なければ、本番環境のドラフトをデプロイ
✅ 本番環境への変更適用前にリスクを最小化できる。

💰 Pricing

Draft & Deploy 機能自体に追加料金は発生しない。Azure Firewall の既存の料金体系がそのまま適用される。
Item Description
Deployment Charge ファイアウォールデプロイごとの固定時間単位料金
Data Processing ファイアウォールで処理されたデータ量（GB 単位）に応じた料金
Capacity Unit Standard / Premium で利用可能な時間単位のキャパシティ料金
Note: 部分的な時間（1 時間未満）は 1 時間として課金される。SKU（Basic, Standard, Premium）により料金が異なる。Firewall Policy の料金は、ファイアウォールとの関連付けに基づいて課金される。0 または 1 つのファイアウォールに関連付けられたポリシーは無料。複数のファイアウォールに関連付けられたポリシーは固定料金で課金される。

⚠ Limitations / Constraints

⚠️ ドラフト機能を使用する場合、ドラフトとアクティブポリシーの間で状態の不整合が発生しないよう、運用手順の整備が必要
⚠️ 複数の管理者が同時にドラフトを編集する場合の競合管理に注意が必要
⚠️ Azure Firewall Policy の料金は従来と変わらず、ファイアウォールとの関連付けに基づいて課金される（ドラフト機能自体に追加料金は発生しない見込み）
