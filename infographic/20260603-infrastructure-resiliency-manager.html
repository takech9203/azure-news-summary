Azure Infrastructure Resiliency Manager

Azure Infrastructure Resiliency Manager パブリックプレビュー

2026-06-03

In Preview

概要

Azure Infrastructure Resiliency Managerがパブリックプレビューとして公開。Azure上のアプリケーションのレジリエンシー（回復性）を設計・評価・改善するための統合的かつゴール駆動型のエクスペリエンスを提供。Microsoft Build 2026で発表。Availability Zones、Azure Advisor、Chaos Studioなどのレジリエンシー関連機能を一元管理する統合プラットフォーム。

🛡 統合レジリエンシー管理 - Availability Zones、Azure Advisor、Chaos Studioの機能を単一のインターフェースから一元管理
🎯 ゴール駆動型アプローチ - ビジネス要件に基づいたレジリエンシー目標（RTO/RPO）を設定し、達成に向けたアクションプランを自動生成
📊 統合ダッシュボード - アプリケーション全体のレジリエンシー状態を一元的に可視化し、改善アクションを優先度付きで提示
🔬 Microsoft Build 2026 発表 - レジリエンシー関連の分散した機能を統合する新しいプラットフォームとして登場

Before / After

Before

  • レジリエンシー関連の機能が個別のサービスとして分散しており、統合的な管理が困難
  • アプリケーション全体のレジリエンシー目標に対する達成状況を一元的に可視化する手段がなかった
  • どのサービスをどの順番で活用すべきか判断が難しかった
  • 複数リソースにまたがるレジリエンシー設定の整合性確認に手作業が必要

After

  • 統合ダッシュボードでアプリケーション全体のレジリエンシー状態を一元可視化
  • ゴール駆動型アプローチでビジネス要件に基づいたレジリエンシー目標を設定
  • Availability Zones、Azure Advisor、Chaos Studioの機能が統合
  • 推奨事項と改善アクションが優先度付きで提示

アーキテクチャ

Infrastructure Resiliency Manager は既存の Azure レジリエンシー関連サービスを統合し、ユーザーのレジリエンシー目標に基づいて評価・テスト・改善を一貫して管理する。

主要機能

📊

統合レジリエンシーダッシュボード

アプリケーション全体のレジリエンシー状態を一元的に可視化。Availability Zonesの活用状況、Advisorの推奨事項、Chaos Studioのテスト結果を統合表示。

  • Availability Zones の活用状況をリアルタイムで表示
  • Azure Advisor の推奨事項を統合的に確認
  • Chaos Studio のテスト結果をダッシュボードに反映
🎯

ゴール駆動型のレジリエンシー設計

ビジネス要件に基づいたレジリエンシー目標（RTO/RPOなど）を設定。目標達成に向けた具体的なアクションプランを自動生成。

  • RTO/RPO などのビジネス要件に基づく目標設定
  • 目標達成に向けたアクションプランの自動生成
  • 達成状況のリアルタイム追跡と可視化
🔎

レジリエンシー評価

Azure Advisorの信頼性推奨事項との連携。Availability Zonesの構成状況評価。リソース間の依存関係を考慮した包括的な評価。

  • Azure Advisor の信頼性推奨事項との連携
  • Availability Zones の構成状況評価
  • リソース間の依存関係を考慮した包括的な評価
📚

改善アクションの優先順位付け

影響度と実装コストに基づく推奨事項の優先度付け。ステップバイステップの改善ガイダンス。

  • 影響度と実装コストに基づく優先度スコアリング
  • ステップバイステップの改善ガイダンス
  • 改善前後の効果を定量的に可視化
🔬

Chaos Studio連携によるレジリエンシー検証

障害注入テストで設計されたレジリエンシーが実際に機能するか検証。テスト結果をダッシュボードに統合し改善サイクルを回す。

  • 障害注入テストによる実際のレジリエンシー検証
  • テスト結果のダッシュボード統合
  • 検証結果に基づく改善サイクルの推進

ユースケース

🏢 ミッションクリティカルアプリケーションのレジリエンシー評価

金融系アプリのAvailability Zones活用状況、冗長性確認、Advisorの推奨事項遵守状況を一元的に評価。複数リソースにまたがるレジリエンシー設定の整合性を自動チェック。

統合ダッシュボードにより、アプリケーション全体のレジリエンシー達成率を即座に把握し、優先すべき改善ポイントを特定できる。

🛠 新規アプリケーションのレジリエンシー設計

目標RTO/RPOを満たすレジリエンシー設計を効率的に行う。ゴール駆動型アプローチにより、ビジネス要件から逆算した設計パターンを提案。

設計段階からレジリエンシー目標を組み込むことで、後からの改修コストを削減し、要件を満たす構成を初期段階で実現。

🕘 定期的なレジリエンシー検証

Chaos Studio連携で障害注入テストを実施し、統合ダッシュボードで結果を確認。設計されたレジリエンシーが実際の障害シナリオで機能するかを継続的に検証。

定期的な検証により、構成ドリフトや新たな脆弱性を早期に発見し、レジリエンシー品質を継続的に維持。
