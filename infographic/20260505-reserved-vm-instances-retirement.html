Azure Reserved VM Instances:
旧世代 VM シリーズの予約廃止

2026年7月1日以降、旧世代VMシリーズ (Av2、Amv2、Bv1、D、Ds、Dv2、Dsv2、F、Fs、Fsv2、G、Gs、Ls、Lsv2) の1年間 Azure Reserved VM Instances の新規購入・更新が不可。Dv3、Dsv3、Ev3、Esv3 も1年/3年 RI 停止。既存予約は契約期間終了まで有効。

2026-05-05 | Azure Virtual Machines / Azure Reserved VM Instances | 予約廃止
📋

概要

Microsoft Azure は、旧世代 VM シリーズの Reserved VM Instances (RI) の新規購入および更新を 2026年7月1日 に停止することを発表した。対象は Av2、Amv2、Bv1、D、Ds、Dv2、Dsv2、F、Fs、Fsv2、G、Gs、Ls、Lsv2 (1年 RI) と Dv3、Dsv3、Ev3、Esv3 (1年+3年 RI) であり、既存の予約は契約期間満了まで有効。ユーザーは最新世代 VM への移行と Azure Savings Plan for Compute の活用が推奨される。
  • 予約購入停止日: 2026年7月1日以降、対象 VM シリーズの RI 新規購入・更新が不可
  • 既存 RI: 契約期間満了まで有効。自動更新される RI は停止日以降に更新不可
  • 対象範囲 (1年 RI 停止): Av2, Amv2, Bv1, D, Ds, Dv2, Dsv2, F, Fs, Fsv2, G, Gs, Ls, Lsv2
  • 対象範囲 (1年+3年 RI 停止): Dv3, Dsv3, Ev3, Esv3
  • VM 廃止日: 2028年5月〜11月にかけて対象 VM シリーズの完全廃止が予定

変更の影響

Before (変更前)

  • 旧世代 VM シリーズの予約インスタンスを購入・更新できたため、古いハードウェア上のワークロードに長期コミットメントを行うリスクがあった
  • 旧世代 VM はパフォーマンスやコスト効率の面で最新世代に劣る

After (変更後)

  • 新規予約が停止されることで、ユーザーは最新世代 VM への移行を促される
  • Azure Savings Plan for Compute への移行により、VM ファミリーやリージョンを跨いだ柔軟なコスト最適化が可能
  • 最新世代 VM (v5/v6/v7) への移行で、より高いパフォーマンス、NVMe 対応、広いリージョン可用性を享受

マイグレーション手順

推奨アクション

1

予約インスタンスの確認

Azure Portal で現在保有している RI を確認し、対象シリーズの予約を特定する。有効期限と自動更新設定を確認。

2

コスト最適化オプションの選択

RI 交換 (新世代 VM の RI へ)、Azure Savings Plan for Compute、または新規 RI 購入のいずれかを選択。ワークロードの柔軟性に応じて最適な選択肢を検討。

3

VM のリサイズ

VM を停止し、新世代 VM サイズにリサイズして再起動する。ダウンタイムが発生するため計画メンテナンスウィンドウでの実施を推奨。

4

クォータの確認

移行先の VM サイズのクォータが十分であることを確認し、必要に応じてクォータ引き上げをリクエストする。

Azure Savings Plan for Compute

VM ファミリーやリージョンを跨いだ柔軟なコスト最適化が可能。特定の VM シリーズに縛られず、ワークロードの変化に対応できるため、RI よりも柔軟性が高い。

🔧

新世代 VM への移行メリット

最新世代 VM (v5/v6/v7) への移行により、NVMe 対応、より高いパフォーマンス、広いリージョン可用性を享受。同等以上のコスト効率を実現可能。

⚠

制限事項・注意点

  • v6 シリーズの要件: NVMe 対応 OS、Generation 2 VM、MANA (Microsoft Azure Network Adapter) 対応が必要
  • 一時的ダウンタイム: VM リサイズには停止→リサイズ→再起動のプロセスが必要で、一時的なダウンタイムが発生する
  • リージョン制限: 一部リージョンでは v6 シリーズが未対応の場合がある。事前にリージョン可用性を確認すること
  • クォータ制限: 移行先の VM サイズのクォータが不足している場合、引き上げリクエストに時間がかかる場合がある
