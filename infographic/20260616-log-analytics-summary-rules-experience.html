Azure Portal から Summary Rules のフルライフサイクル管理が可能に

リリース日: 2026-06-16 | API version: 2025-07-01

概要

  • Log Analytics ワークスペースの Settings > Summary rules からアクセス
  • ウィザードベースのルール作成で REST API / PowerShell / ARM テンプレートの知識が不要に
  • KQL テスト、ルール管理、実行履歴確認、失敗 bin の再実行がポータル GUI で完結
  • displayName サポートによりルールの識別性が向上

  • ステータス確認に CLI が必要

  • ウィザードベースのルール作成

  • フィルター付きリストビュー

主要機能

1. ルール作成ウィザード

4 ステップのガイド付きウィザード: 基本情報 → ルールロジック (KQL エディター) → スケジュール / binSize 設定 → レビュー & 作成。対応する binSize は 20, 30, 60, 120, 180, 360, 720, 1440 分。

2. ルール一覧とフィルター

Settings > Summary rules からアクセス。Active / Inactive ステータスでフィルター可能。全ルールの状態を一覧で確認でき、運用状況の把握が容易に。

3. ルール管理オペレーション

各ルールの省略記号メニュー (...) から Edit / Start / Stop / Delete を実行。CLI や API を使わずにポータルから直接ルールのライフサイクルを管理。

4. 実行履歴と Rerun bin

View runs で実行履歴を確認。Failed でフィルターし、失敗した bin を GUI から「Rerun this bin」で再実行可能。障害復旧が大幅に簡素化。

5. displayName サポート

ルールにわかりやすい表示名を設定可能。API 名 (リソース名) とは別に日本語などの表示名でルールを識別でき、運用時の視認性が向上。

KQL クエリ例

Container Log Aggregation Query

ContainerLogV2
| summarize Count = count() by Computer, ContainerName, PodName, PodNamespace, LogSource, LogLevel

制約: クロスリソースクエリ不可 / union * 不可 / isfuzzy=true 不可 / パブリッククラウドのみ

ユースケース

1. コンテナログ集約

ContainerLogV2 テーブルから Computer / ContainerName / PodName 等でログを集約し、 ContainerLogsSummary_CL テーブルに出力。binSize 60 分で定期集約することで、 大量のコンテナログをコスト効率よく要約・保持。

2. セキュリティログ月次レポート

サインインログを月次で集約し、セキュリティレポートを自動生成。 失敗した bin は Rerun 機能で再実行し、データ欠損を防止。 Basic / Auxiliary テーブルプランのログもデータスキャンコストのみで集約可能。
