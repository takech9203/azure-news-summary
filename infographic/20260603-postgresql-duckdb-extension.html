📋 概要

Azure Database for PostgreSQL - Flexible Server において、DuckDB 拡張機能 (pg_duckdb) が一般提供 (GA) となった。PostgreSQL のトランザクション処理 (OLTP) 環境内に DuckDB の列指向ベクトル化分析エンジンを統合し、単一データベースインスタンス上で OLTP と OLAP (分析処理) の両方を実行可能に。Microsoft Build 2026 で発表。
列指向エンジン統合: DuckDB のベクトル化エンジンで分析クエリが大幅に高速化
HTAP 実現: 同一インスタンスで OLTP と OLAP を同時処理、データ複製不要
データレイクアクセス: Parquet/CSV/JSON を Azure Blob Storage から直接クエリ
オープンフォーマット対応: Apache Iceberg、Delta Lake 形式に対応
追加料金なし: MIT License、Flexible Server の通常料金のみ
幅広い互換性: PostgreSQL 14, 15, 16, 17, 18 に対応

Before / After 比較

❌ Before

  • PostgreSQL 上で大規模集計・分析クエリを実行するとパフォーマンスが低かった
  • 分析のために別途 Azure Synapse Analytics や DWH を構築し ETL が必要だった
  • Parquet/Iceberg などのデータレイク形式に PostgreSQL から直接アクセスする手段がなかった
  • HTAP アーキテクチャの構築が困難だった

✅ After

  • DuckDB の列指向ベクトル化エンジンで分析クエリが大幅に高速化
  • 同一 PostgreSQL インスタンス内で OLTP と OLAP を同時処理、データ複製不要
  • read_parquet() 等で Azure Blob Storage 上の Parquet/CSV/JSON を直接クエリ可能
  • Apache Iceberg、Delta Lake 形式にも対応しデータレイクハウスを実現

pg_duckdb がクエリを自動ルーティング。トランザクション操作は PostgreSQL エンジン、分析クエリは DuckDB エンジンが処理。外部ストレージ上のファイルも DuckDB 経由で直接アクセス可能。

⚡

透過的クエリアクセラレーション

duckdb.force_execution=true の設定で分析クエリが DuckDB エンジンを経由。既存クエリがそのまま高速化され、SQL 構文の変更は不要。

💾

データレイク読み書き

Parquet、CSV、JSON、Apache Iceberg、Delta Lake 対応。Azure Blob Storage / S3 / GCS 上のファイルを直接クエリ可能。read_parquet() 関数でファイルをテーブルのように扱える。

🔗

PostgreSQL テーブルとの統合クエリ

ローカル PostgreSQL テーブルと外部データレイクファイルを単一クエリで JOIN 可能。データの移動やコピーなしにリアルタイム分析を実現。

🧩

DuckDB 内部拡張機能のサポート

duckdb.install_extension('iceberg') 等で DuckDB 自体の拡張機能をインストール可能。Iceberg テーブルのタイムトラベル (バージョンスナップショット) にも対応。

💡 ユースケース

🛒 UC1: トランザクションデータのリアルタイム分析

EC サイトの注文データに対してリアルタイムの売上集計・傾向分析を実行。別途分析基盤を構築せず、トランザクションデータベース上で直接集計クエリを高速実行。

効果: ETL 遅延ゼロ。データ鮮度がリアルタイムで分析コスト削減。

📊 UC2: データレイクハウスクエリ

Azure Blob Storage の過去データ (Parquet) と PostgreSQL テーブルを結合分析。過去データをデータレイクに低コスト保存しつつ、現行データとシームレスに結合分析可能。

効果: データウェアハウスの構築が不要。アーキテクチャのシンプル化とコスト削減。

🧊 UC3: Iceberg テーブルによるタイムトラベル分析

データレイク上の Apache Iceberg テーブルのスナップショットをクエリし、データの変遷を追跡。PostgreSQL から Iceberg テーブルに直接アクセス。

効果: タイムトラベル機能を活用した監査・コンプライアンス分析が可能。

コードサンプル

SQL 
-- 拡張機能のインストール
CREATE EXTENSION pg_duckdb;

-- DuckDB エンジンの有効化
SET duckdb.force_execution = true;

-- 分析クエリの実行例
SELECT
    date_trunc('month', order_date) AS month,
    COUNT(*) AS order_count,
    SUM(amount) AS total_amount
FROM orders
GROUP BY 1
ORDER BY 1;

-- Azure Blob Storage 上の Parquet ファイルをクエリ
SELECT * FROM read_parquet('az://<container>/<path>/data.parquet');
