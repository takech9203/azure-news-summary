概要

Azure Premium SSD v2 が US Gov Arizona リージョンで一般提供 (GA) を開始した。US Gov Arizona は Availability Zone (AZ) を持たないリージョンであり、Azure Government クラウドにおいて Premium SSD v2 が利用可能となる重要なアップデートである。Azure Premium SSD v2 は、Azure 仮想マシン向けの次世代汎用ブロックストレージであり、サブミリ秒のレイテンシと優れたコストパフォーマンスを提供する。容量・IOPS・スループットを個別に調整できるため、ワークロードに応じた柔軟なパフォーマンスチューニングとコスト最適化が可能である。

Before (アップデート前)

  • US Gov Arizona のユーザーは Premium SSD v2 利用のために US Gov Virginia など他リージョンにデプロイが必要でレイテンシ増加
  • 米国政府機関向けデータレジデンシー・コンプライアンス要件を満たしつつ高性能ストレージを利用するリージョン選択肢が限定
  • Premium SSD v1 では IOPS やスループットがディスクサイズに固定、過剰プロビジョニングによるコスト増

After (アップデート後)

  • US Gov Arizona リージョンで Premium SSD v2 をローカルにデプロイ可能、Government クラウドのストレージ選択肢が拡大
  • サブミリ秒のディスクレイテンシにより、政府機関のデータベースやトランザクション処理の応答性が向上
  • 容量・IOPS・スループットの個別設定により、必要な性能のみを購入できコスト最適化が可能

Azure VM のストレージアーキテクチャにおける Premium SSD v2 の位置付けと、Azure Government での利用可能リージョンを示す。Premium SSD v2 はデータディスクとしてのみ利用可能であり、OS ディスクとしては使用できない。US Gov Arizona は AZ なしのリージョンとして新たに対応が追加された。

主要機能

🏛 US Gov Arizona リージョンでの一般提供

AZ なし Government リージョンでの Premium SSD v2 提供。Azure Government クラウドとしては US Gov Virginia に続く 2 リージョン目の対応。AZ なしリージョンではインフラストラクチャ冗長性なし (No infrastructure redundancy required) でデプロイされる。

⚙ 柔軟なパフォーマンス調整

容量 (1 GiB - 64 TiB)、IOPS (最大 80,000)、スループット (最大 1,200 MB/s) を個別に設定可能。24 時間以内に最大 4 回のパフォーマンス変更が可能 (ディスク作成を含む)。VM の再起動なしでパフォーマンスを動的に調整可能。

💰 コスト効率の高い課金モデル

ベースライン IOPS: 3,000 IOPS (無料)、ベースラインスループット: 125 MB/s (無料) がすべてのディスクサイズに含まれる。容量は GiB 単位の従量課金で、必要な性能のみを購入可能。過剰プロビジョニングによるコスト増を回避。

設定方法

US Gov Arizona は AZ なしリージョンのため、--zone パラメータを指定せずにデプロイする。

Azure CLI 
# Premium SSD v2 ディスクの作成 (US Gov Arizona)
az disk create \
  --name myPremiumSSDv2 \
  --resource-group myResourceGroup \
  --size-gb 100 \
  --disk-iops-read-write 5000 \
  --disk-mbps-read-write 150 \
  --location usgovarizona \
  --sku PremiumV2_LRS \
  --logical-sector-size 4096
パフォーマンスの動的調整 
# IOPS とスループットの変更 (24 時間以内に最大 4 回)
az disk update \
  --resource-group myResourceGroup \
  --name myPremiumSSDv2 \
  --disk-iops-read-write 10000 \
  --disk-mbps-read-write 300

高 IOPS・低レイテンシが要求されるミッションクリティカルアプリケーション。パフォーマンスの動的調整でコスト最適化も実現。

制約事項・注意点

  • OS ディスクとして使用不可: Premium SSD v2 はデータディスク専用。OS ディスクには Premium SSD v1 や Standard SSD を別途使用する必要がある。
  • ホストキャッシュ非対応: ディスクキャッシュが利用できない。ただし、サブミリ秒のレイテンシにより多くの場合はキャッシュなしでも十分なパフォーマンスを得られる。
  • LRS のみ対応: ローカル冗長ストレージ (LRS) のみ。ゾーン冗長ストレージ (ZRS) は非対応。
  • パフォーマンス変更回数の制限: 24 時間以内に最大 4 回まで (ディスク作成を 1 回としてカウント)。
  • Azure Compute Gallery 非対応: イメージの管理・共有に Azure Compute Gallery を使用できない。
