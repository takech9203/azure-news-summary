⚠️ 緊急対応が必要: Preview API

Preview API バージョン (2021-09-30-preview, 2022-01-30-preview, 2022-06-30-preview) は 2026年6月30日 に廃止予定。至急 v4.0 への移行が必要。

概要

Azure Document Intelligence v3.0 API (REST API バージョン 2022-08-31) は 2029年3月30日 に廃止される。最新の GA バージョンである v4.0 API (REST API バージョン 2024-11-30) への移行が必要。

v3.0 (廃止予定)

  • 個別ドキュメント処理のみ
  • Searchable PDF 未サポート
  • 限定的なプリビルトモデル
  • 分析結果削除 API なし

v4.0 (移行先)

  • Batch API で大量一括処理
  • Searchable PDF (日本語対応)
  • 10+ 新プリビルトモデル
  • GDPR 対応の削除 API

v4.0 の主要な新機能

📦

Batch API

全モデル (Read, Layout, プリビルト, カスタム) で大量ドキュメントの一括分析をサポート。LIST/DELETE 機能で運用管理も容易。

📄

Searchable PDF

Read モデルで画像形式からテキスト埋め込み PDF を生成。中国語・日本語・韓国語に対応拡大。

🔄

カスタム分類モデルの強化

インクリメンタルトレーニング対応。トレーニングページ上限が 25,000 ページに拡大。モデルコピー操作によるバックアップ対応。

✍️

カスタムニューラルモデル署名検出

署名検出機能の追加。有料トレーニングによる長時間トレーニング対応 (月20回まで無料)。

🆕

新プリビルトモデル (10+)

銀行小切手、銀行明細書、給与明細、住宅ローン書類、クレジットカード、結婚証明書、米国税務書類など多数追加。

移行時の注意点 (破壊的変更)

⚠️

General Document モデルの廃止

prebuilt-document は v4.0 で廃止。Layout モデルの keyValuePairs オプションに移行が必要。

⚠️

カスタム分類モデルのデフォルト動作変更

v4.0 ではドキュメント分割がデフォルトで無効化。splitMode プロパティを明示的に auto に設定する必要あり。

⚠️

SDK メジャーバージョンアップ

Python, .NET, Java, JavaScript の SDK がメジャーバージョンアップ。コード修正が必要。

⚠️

Business Card モデル非推奨

Business Card モデルは v4.0 で deprecated。代替の処理方法を検討すること。

ユースケース

📦 大量請求書処理の移行

シナリオ: v3.0 の Invoice モデルで個別に請求書を処理していたシステムを、v4.0 の Batch API を活用した一括処理に移行。

効果: 大量の請求書を一度の API 呼び出しで処理可能になり、スループット向上。運用の可視性も向上。

🇯🇵 日本語ドキュメントの Searchable PDF 化

シナリオ: スキャンした日本語ドキュメントを v4.0 の Read モデルで処理し、テキスト検索可能な PDF として出力。

効果: v3.0 では対応していなかった日本語 Searchable PDF が利用可能に。ドキュメントアーカイブの検索性が大幅に向上。
