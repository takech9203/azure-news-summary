量子耐性暗号化への第一歩 - Transparent Data Encryption に AES 対称キーを導入

Preview Security 2026-06-02

概要

Azure SQL Database の Transparent Data Encryption (TDE) で、カスタマーマネージドキー (CMK) として AES 256 対称キーがサポートされるようになった。従来は RSA 非対称キーのみが利用可能だったが、Microsoft の量子安全セキュリティ戦略の一環として、量子コンピューティングに対して耐性のある AES 対称暗号が導入された。十分なキーサイズの AES 対称暗号は量子耐性があるとされる一方、RSA は将来の量子コンピューティングに対して脆弱になる可能性がある。

Key Points

  • AES 対称キー (128/192/256 bit) を TDE プロテクターとして使用可能
  • Azure Key Vault Managed HSM に格納された AES キーを利用
  • RSA キーと AES キー間のオンラインローテーションが数秒で完了
  • 量子コンピューティング脅威への先行対策としての暗号アジリティを実現

Before

  • RSA 非対称キー (2048/3072 bit) のみ CMK TDE に対応
  • 対称キーオプションが存在しない
  • 暗号アジリティが限定的

After

  • AES 対称キー (128/192/256 bit) を TDE プロテクターとして利用可能
  • 量子耐性のある暗号化を実現
  • RSA と AES 間のオンラインローテーション (数秒)

🔑

AES 対称キーによる TDE プロテクター構成

Managed HSM に格納された AES キーを TDE プロテクターとして構成可能。キーサイズは 128/192/256 bit から選択でき、最大セキュリティには 256 bit を推奨。

🔄

キータイプ間のローテーション

RSA 非対称キーから AES 対称キーへの切り替え、またはその逆方向のローテーションをオンラインで実行可能。ダウンタイムなしで数秒以内に完了。

🛡️

量子耐性セキュリティ

AES 256 は量子コンピューティングに対して耐性があるとされる暗号アルゴリズム。将来の量子コンピュータによる攻撃に対する先行対策として有効。

✅

既存の TDE 機能との互換性

自動キーローテーション、バージョンレスキー識別子、Azure Policy による監査など、既存の TDE CMK 機能がそのまま利用可能。

コードサンプル

Azure CLI - TDE AES キー構成

# サーバーの作成 (マネージド ID 付き)
az sql server create --name <servername> --resource-group <rgname> \
  --location <location> --admin-user <user> --admin-password <password> \
  --assign-identity

# TDE プロテクターとして設定
az sql server tde-key set --server <servername> \
  --server-key-type AzureKeyVault --resource-group <rgname> --kid <keyID>

# TDE を有効化
az sql db tde set --database <dbname> --server <servername> \
  --resource-group <rgname> --status Enabled

ユースケース

UC1: 量子耐性暗号化への移行

金融機関が将来の量子コンピューティング脅威に備え、データベース暗号化を RSA から AES 対称キーへ段階的に移行。規制要件への先行対応として暗号化方式を更新。

-- オンラインローテーションにより業務中断なしで量子耐性暗号化へ移行可能

UC2: 暗号アジリティの実現

キータイプ間のオンライン切り替えにより、将来の暗号化標準の変更や規制要件の追加に迅速に対応。ダウンタイムなしでキータイプを柔軟にローテーション。

-- RSA と AES 間を数秒で切り替え可能。暗号化ポリシーの変更に即座に対応

料金

TDE CMK 機能自体は無料

TDE のカスタマーマネージドキー機能自体に追加料金は発生しません。関連コストとして Managed HSM の使用料およびキー操作トランザクション料金が発生します。
