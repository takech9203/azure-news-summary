FileShares リソースプロバイダー GA
& macOS Entra ID 認証プレビュー

ファイル共有をトップレベル Azure リソースとして独立管理 + macOS からの Entra ID 認証アクセス

GA (Launched) In Preview 2026-06-02

概要

Microsoft Build 2026 において、Azure Files に関する 2 つの重要なアップデートが発表された。Microsoft.FileShares リソースプロバイダーの一般提供 (GA) により、ファイル共有をストレージアカウントから独立したトップレベル Azure リソースとして管理可能に。また、macOS からの Entra ID 認証によるアクセスがパブリックプレビューとして提供開始された。

Key Points

  • Microsoft.FileShares リソースプロバイダーが GA - ファイル共有がトップレベル ARM リソースに昇格
  • NFS 4.1 on SSD ストレージで利用可能、サブスクリプションあたりリージョンごとに最大 10,000 共有
  • 各ファイル共有に独立した RBAC、ネットワーク、パフォーマンス、課金を設定可能
  • macOS から Entra ID (Kerberos) 認証で Azure Files にアクセス可能に (プレビュー)

Before

  • ファイル共有はストレージアカウント配下のサブリソースであり、個別のトップレベル ARM リソースとして管理できなかった
  • 独立した RBAC、ネットワーク、課金を実現するにはストレージアカウント単位での分割が必要
  • サブスクリプションあたりのファイル共有数がストレージアカウントの制限に依存
  • macOS ユーザーは Entra ID 認証を使用できず、ストレージアカウントキーや SAS トークンに依存

After

  • ファイル共有が Microsoft.FileShares RP 配下のトップレベル Azure リソースとなり、ストレージアカウント不要で独立管理が可能
  • 個々のファイル共有に専用のパフォーマンス、セキュリティ、ネットワーク、課金を設定可能
  • サブスクリプションあたりリージョンごとに最大 10,000 共有をサポート
  • macOS から Entra ID 認証で Azure Files にアクセス可能、Windows SMB 共有と同等のエクスペリエンスを実現

📁

トップレベル Azure リソースとしてのファイル共有

ARM の完全な管理機能を直接適用可能。ストレージアカウントを介さずにファイル共有を独立してプロビジョニング・管理できる。ARM テンプレート、Bicep、CI/CD パイプラインで直接デプロイ。

🔒

専用のパフォーマンス・セキュリティ・ネットワーク・課金

各ファイル共有に対して独立した RBAC ポリシー、ネットワークルール、パフォーマンス設定、課金を構成可能。マルチテナント環境での分離を実現。

📈

大規模スケール (10,000 共有/サブスクリプション/リージョン)

ストレージアカウントの制限に依存しないスケーラビリティ。ARM テンプレート、Bicep、CI/CD パイプラインによる自動化に完全対応。

🍎

macOS Entra ID 認証 (パブリックプレビュー)

ストレージアカウントキー不要のパスワードレス認証。macOS ユーザーが Entra ID Kerberos を使用して SMB 経由で Azure Files にアクセス可能。オンプレミス AD 不要。

⚡

NFS 4.1 on SSD (プレミアムストレージ)

GA 時点では NFS 4.1 プロトコルの SSD ストレージで利用可能。高パフォーマンスなファイルアクセスを Linux ワークロードに提供。

ユースケース

UC1: マルチテナント SaaS

テナントごとに独立した NFS ファイル共有をトップレベルリソースとして管理。各テナントに専用の RBAC、ネットワーク分離、課金を適用し、10,000 共有まで拡張可能。

-- ストレージアカウント分割不要で大規模マルチテナント環境を簡潔に管理

UC2: クロスプラットフォーム開発チーム

Windows、macOS、Linux 全プラットフォームで統一的な ID ベース認証を実現。macOS ユーザーも Entra ID 認証で SMB 共有にアクセスし、ストレージアカウントキーの共有が不要に。

-- セキュリティ向上と運用負荷の大幅削減、パスワードレスアクセスの実現

UC3: CI/CD パイプライン

Bicep/ARM テンプレートでファイル共有を直接定義・デプロイ。ストレージアカウントの作成ステップを省略し、Infrastructure as Code のシンプルさを向上。

-- デプロイパイプラインの簡素化とプロビジョニング時間の短縮
