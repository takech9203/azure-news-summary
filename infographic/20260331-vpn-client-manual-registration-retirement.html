手動登録 Azure VPN クライアントのリタイア

Azure Government / 21Vianet クラウドにおける手動登録 Azure VPN クライアントが 2029 年 3 月 31 日にリタイア。全クラウドで統一された Audience 値への移行が必要

Release: 2026-03-31 Service: VPN Gateway (P2S) Deadline: 2029-03-31

📋 Overview

Microsoft は、Azure Government および Microsoft Azure operated by 21Vianet クラウドにおいて、Microsoft Entra ID 認証を使用した P2S VPN 接続で使用されている手動登録 Azure VPN クライアントを 2029 年 3 月 31 日にリタイアすることを発表。従来は手動でアプリ登録し、クラウドごとに異なる Audience 値を使用していたが、新しい Microsoft 登録済み Azure VPN クライアント App ID では、全クラウドで統一された Audience 値 (c632b3df-fb67-4d84-bdcf-b95ad541b5c8) を使用し、手動登録が不要になる。

Before (手動登録方式)

  • テナント管理者が Azure VPN クライアントアプリを手動で登録する必要
  • Cloud App Administrator ロールでのアプリ承認やアクセス許可の割り当てが必要
  • クラウドごとに異なる Audience 値を管理
  • Windows、macOS のみサポート

After (Microsoft 登録済み方式)

  • 全クラウドで統一された Audience 値
  • 手動でのアプリ登録プロセスが不要
  • セキュリティ向上と設定手順の簡素化
  • Linux クライアントのサポート追加

手動登録方式から Microsoft 登録済み方式への移行フロー。クラウドごとに異なっていた Audience 値が、全クラウドで統一された単一の値に集約される。移行猶予期間は約 3 年間。

⚙ Key Features

🌐

統一された Audience 値

全クラウド (Azure Public, Government, Germany, 21Vianet) で共通の Audience 値を使用

  • 移行先 Audience 値: c632b3df-fb67-4d84-bdcf-b95ad541b5c8
  • クラウドごとの設定管理が不要になり、運用負荷を大幅に軽減
🔒

手動登録プロセスの撤廃

テナント管理者による手動アプリ登録が不要に

  • Cloud App Administrator ロールでのアプリ承認が不要
  • 不必要な権限付与のリスクを軽減し、セキュリティが向上
🐧

Linux クライアントサポート

Microsoft 登録済み方式では新たに Linux クライアントをサポート

  • Ubuntu 20.04 / 22.04 に対応
  • 従来の手動登録方式では Windows / macOS のみだった対応範囲が拡大
📋

一貫した管理体験

Azure Public、Azure Government、21Vianet で同一の設定手順を適用可能

  • トンネルタイプ: OpenVPN (SSL)
  • 認証方式: Microsoft Entra ID 認証

🛠 Migration Steps

1
Azure Portal で VPN Gateway の「ポイント対サイトの構成」ページを開く
2
Audience フィールドを c632b3df-fb67-4d84-bdcf-b95ad541b5c8 に変更
3
Tenant フィールドはクラウドに応じた URL を維持
4
Issuer フィールドに末尾のスラッシュが含まれていることを確認
5
VPN クライアント構成パッケージを再ダウンロード
6
各クライアントマシンの Azure VPN Client を更新

💡 Use Cases

1

Azure Government を利用する政府機関

職員のリモートアクセスの移行が必要。Microsoft 登録済み方式への移行により、手動登録プロセスが不要になり、セキュリティが向上。Linux ワークステーション対応の拡大により、より多くのデバイスからの安全なアクセスが可能に。

✅ 手動登録の撤廃によるセキュリティ向上と、Linux サポートによるアクセス手段の拡大。
2

21Vianet クラウドを利用する中国拠点の企業

従業員のリモート VPN アクセスの移行が必要。クラウドごとの Audience 値管理が不要になることで、グローバル展開している企業のマルチクラウド管理が簡素化される。

✅ 統一された Audience 値により、マルチクラウド環境の VPN 管理を大幅に簡素化。

⚠ Limitations / Caveats

⚠️ 全クライアント同時更新が必要 - ゲートウェイは同時に 1 つの Audience 値のみサポートするため、全クライアントを同時に更新する必要がある。
⚠️ VPN クライアントプロファイル更新 - 全リモートユーザーの VPN クライアントプロファイル更新が必要。
⚠️ ARM プロセッサ非サポート - Azure VPN Client for Windows は ARM プロセッサをサポートしていない。
⚠️ Linux サポートの制限 - Linux サポートは Ubuntu 20.04 / 22.04 に限定される。
