Microsoft Foundry Launched (GA)

組み込み RBAC ロールの名称変更と機能強化

リリース日: 2026年5月15日

📋 概要

Microsoft Foundry の組み込み RBAC (ロールベースアクセス制御) ロールの名称が、製品ブランドに合わせて更新された。従来「Azure AI」を冠していたロール名が「Foundry」ブランドに統一され、Microsoft Foundry のブランドアイデンティティとの一貫性が確保された。

ポイント: ロール ID は変更されないため、既存のロール割り当ては自動的に新しい名称に移行される。ID ベースで参照している IaC テンプレートやスクリプトへの影響はない。

❌ Before

  • ロール名に「Azure AI」が残存し、リブランディング後も旧ブランド名が混在
  • ドキュメントやポータルでの新旧ブランド名混在による混乱
  • プロジェクト管理ロールの権限が十分に明確化されていなかった

✅ After

  • 全ロールが「Foundry」ブランドに統一され、製品名と一貫性確保
  • ロール名から権限の範囲が直感的に理解可能に
  • Foundry Project Manager により管理と制約付きロール割り当てが明確化

⚡ 主要機能 (ロール詳細)

👑

Foundry Account Owner

AI プロジェクトとアカウントの管理に対するフルアクセス。ABAC 条件により Foundry User ロールのみを他のユーザーに割り当て可能。

ID: e47c6f54-e4a2-4754-9501-8e0985b135e1
🔑

Foundry Owner

AI プロジェクトとアカウントのフルアクセス。データプレーン操作 (Microsoft.CognitiveServices/*) へのアクセスとアラート管理権限を含む。

ID: c883944f-8b7b-4483-af10-35834be79c4a
📁

Foundry Project Manager

プロジェクトスコープでの開発者・管理アクションの実行が可能。ABAC 条件付きで Foundry User ロールの割り当てが可能。データプレーン操作へのアクセスを含む。

ID: eadc314b-1a2d-4efa-be10-5d325db5065e
👤

Foundry User

AI プロジェクトおよびアカウントへの読み取りアクセス。データプレーン (Cognitive Services) へのフルアクセスを含み、推論やモデル呼び出しが可能。

ID: 53ca6127-db72-4b80-b1b0-d745d6d5456d

💡 ユースケース

🏢 エンタープライズ AI プラットフォームの権限管理

シナリオ: 大規模組織で複数の AI プロジェクトを運用し、チームごとに適切なアクセス権限を付与する必要がある場合

効果: Foundry Account Owner がアカウントレベルで全体管理を行い、各プロジェクトには Foundry Project Manager を配置。チームメンバーに Foundry User ロールを委任割り当てする階層的な権限管理が実現できる。

🛠️ 開発者セルフサービス環境の構築

シナリオ: AI 開発者に必要最小限の権限を付与し、データプレーン操作 (モデルの呼び出し、推論の実行) を自律的に行わせたい場合

効果: Foundry User ロールにより、コントロールプレーンは読み取り専用としつつ、データプレーン (Cognitive Services) へのフルアクセスを付与。安全なセルフサービス環境が構築できる。

💻 設定方法 (Azure CLI)

bash Azure CLI 
# Foundry Account Owner ロールの割り当て
az role assignment create \
  --assignee <user-principal-name-or-object-id> \
  --role "Foundry Account Owner" \
  --scope /subscriptions/<subscription-id>/resourceGroups/<resource-group>/providers/Microsoft.CognitiveServices/accounts/<foundry-account>

# Foundry User ロールの割り当て
az role assignment create \
  --assignee <user-principal-name-or-object-id> \
  --role "Foundry User" \
  --scope /subscriptions/<subscription-id>/resourceGroups/<resource-group>/providers/Microsoft.CognitiveServices/accounts/<foundry-account>
Azure Portal での設定: 対象の Microsoft Foundry リソース → アクセス制御 (IAM) → ロールの割り当ての追加 → ロール選択 → ユーザー/グループ選択 → レビューと割り当て

⚠️ 制限事項・注意点

⚠️ ロール名で参照している既存の IaC テンプレート (ARM、Bicep、Terraform) やスクリプトは、新しいロール名への更新が必要になる可能性がある
⚠️ Foundry Account Owner と Foundry Project Manager が割り当て可能なロールは Foundry User に限定されており、他のロールの委任はできない
⚠️ Azure AI Administrator および Azure AI Developer ロールは引き続き「Azure AI」名称を維持。これらは Azure Machine Learning と Foundry ハブの両方に適用される別カテゴリのロール
