Microsoft Foundry Agent Service

Agent-to-Agent (A2A) 通信サポート

A2A プロトコルによる標準的なエージェント間連携をマネージド環境で実現

In Preview

リリース日: 2026-06-03

📋概要

  • Foundry Agent Service に Agent-to-Agent (A2A) 通信機能がパブリックプレビューとして追加
  • Prompt agent および Hosted agent が A2A プロトコルを通じて他のエージェントを名前で呼び出し可能
  • A2A プロトコルはオープン標準仕様 (a2a-protocol.org) に準拠、v1.0 および v0.3 をサポート
  • エージェントカードにより自身の能力を宣言し、呼び出し側がカードを検出して通信を開始
  • Microsoft Entra ID ベースの認証が組み込みで提供、RBAC によるアクセス制御

Before / After 比較

Before（従来）

  • エージェント同士が連携するには独自のインテグレーションコードや中間レイヤーを構築する必要があった
  • マルチエージェントシステムにおける認証・認可の管理が複雑だった
  • エージェント間通信の可観測性が個別に実装が必要だった

After（改善後）

  • A2A プロトコルに準拠した標準的な方法でエージェント間通信が可能に
  • Microsoft Entra ID ベースの認証が組み込みで提供され、RBAC によるアクセス制御が利用可能
  • Foundry プラットフォームの観測性機能が A2A 通信にも適用される

💻設定例 (Python SDK)

from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    AgentEndpoint,
    AgentEndpointProtocol,
)

PROJECT_ENDPOINT = "https://{account}.ai.azure.com/api/projects/{project}"
AGENT_NAME = "your_agent_name"

project_client = AIProjectClient(
    endpoint=PROJECT_ENDPOINT,
    credential=DefaultAzureCredential(),
)

endpoint_config = AgentEndpoint(
    protocols=[
        AgentEndpointProtocol.RESPONSES,
        AgentEndpointProtocol.A2A,
    ],
)

patched_agent = project_client.beta.agents.patch_agent_details(
    agent_name=AGENT_NAME,
    agent_endpoint=endpoint_config,
)

💡ユースケース

🎯 専門エージェントへのタスク委譲

フロントエンドの汎用アシスタントが専門エージェント（データ分析、ドキュメント検索、コード生成）にタスクを委譲。各エージェントがエージェントカードで能力を宣言し、呼び出し側が最適なエージェントを自動選択してA2Aプロトコルで通信。

🌍 クロスプロジェクト・クロス組織連携

異なる Foundry プロジェクトや組織のエージェントが A2A プロトコルでセキュアに連携。Microsoft Entra ID による認証と RBAC により、組織境界を超えたエージェント間通信を安全に実現。

⚠制約事項

  • テキストモダリティのみ対応（ファイルデータや画像は非対応）
  • ストリーミングレスポンス（SSE）は非対応
  • Responses protocol を使用しないエージェントは A2A エンドポイントとして公開不可
  • Foundry ポータルからの A2A 有効化は未対応（REST API または SDK が必要）
  • パブリックプレビューのため本番非推奨
