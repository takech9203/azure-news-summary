Toolbox ツール検索機能

Tool search in Foundry toolboxes | リリース日: 2026-06-04

概要

Microsoft Foundry の Toolbox にツール検索機能がパブリックプレビューとして追加。大規模なマルチチームカタログ内からクエリに関連するツールのみを選択的に取得し、トークンコストの最適化とエージェントパフォーマンスの向上を実現。

❌ Before

  • すべてのツール定義を毎ターン LLM に送信
  • カタログ肥大化でコンテキスト圧迫
  • モデルのパフォーマンス低下
  • ツール発見手段が限定的

✅ After

  • 関連ツールのみを選択的に取得
  • トークンコスト大幅削減
  • エージェントの応答精度向上
  • 大規模カタログでも高速検索

エージェントがリクエストを処理する際、従来はすべてのツール定義を LLM に送信していたが、ツール検索機能によりクエリに関連するツールのみが選択的に返却される。これにより、トークン使用量の最適化とエージェントの応答精度向上が実現される。

主要機能

🔎

Toolbox 内ツール検索

大規模カタログ内から関連ツールを検索・発見する機能。開発者およびエージェントの両方が利用可能。マルチチーム環境での効率的なツール発見を支援。

💸

トークンコスト最適化

すべてのツール定義を毎ターン送信する代わりに、関連ツールのみを選択的に取得。Toolbox の規模に関係なく、効率的なトークン使用を維持。

👥

マルチチームカタログ対応

複数チームが共有する大規模 Toolbox での利用を想定。プロジェクト横断でのツール検索・発見が可能。

  • Toolbox の規模拡大でも効率的な運用を維持

🔧 技術面

  • 関連ツールのみを LLM コンテキストに含めトークン消費削減
  • コンテキストウィンドウの効率的活用で応答精度向上
  • 送信ツール定義数減少でプロンプト処理時間短縮

ユースケース

🏢 大規模エンタープライズのマルチチーム AI エージェント

100 以上のツールが登録された Toolbox を持つ大企業で、複数チームが共有エージェントプラットフォームを運用するケース。エージェントが各ターンで関連ツールのみを使用することで、トークンコストの大幅削減とエージェント応答品質の向上が期待される。

💰 コスト最適化が求められる高頻度エージェント実行

大量のビジネスプロセスを自動化するエージェントが高頻度で実行される環境。各リクエストでのトークン消費削減が累積的に大きなコスト削減につながる。1 ターンあたりのトークン使用量を最適化し、月間の API コストを抑制。

📈 段階的なツール拡張

新しいチームや機能が追加されるたびに Toolbox のツール数が増加する成長期の組織。ツール数の増加がエージェントパフォーマンスやコストに直接影響しないため、スケーラブルなツール管理が可能。

設定方法

前提条件: アクティブな Azure サブスクリプション + Microsoft Foundry プロジェクト + Toolbox にツールが登録済み
1️⃣

Foundry ポータルにアクセス

https://ai.azure.com にアクセスし、対象の Microsoft Foundry プロジェクトを開く

2️⃣

Control Plane の Assets ペインを開く

サブスクリプション内のすべてのプロジェクトにわたるツールの統合的な検索・管理画面にアクセス

3️⃣

ツール検索を使用

検索クエリを入力し、関連ツールを発見・選択。エージェントからの利用も可能

制限事項・注意点

⚠️

パブリックプレビュー

本番ワークロードでの利用は推奨されない。プレビュー機能には SLA が適用されない。

📝

検索精度はツール定義品質に依存

検索精度はツールの名前・説明の品質に依存すると考えられる。適切な命名規則と詳細な説明の記述が重要。

技術詳細は非公開
