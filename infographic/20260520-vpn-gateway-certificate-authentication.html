2026-05-20

概要

Azure VPN Gateway のサイト間 (S2S) VPN 接続で、証明書ベースの認証が一般提供 (GA) となった。従来の事前共有キー (PSK) に代わり、X.509 証明書を用いた非対称信頼モデルによる相互認証が可能になり、エンタープライズレベルのセキュリティとコンプライアンス要件を満たす VPN 接続を構築できる。Azure Key Vault とマネージド ID の統合により、証明書のライフサイクル管理も安全かつ効率的に行える。

Before (従来)

  • PSK (事前共有キー) のみ利用可能
  • 鍵漏洩リスクが高く管理が複雑
  • コンプライアンス要件を満たせない

After (証明書認証)

  • X.509 証明書による非対称信頼モデル
  • Key Vault + マネージド ID 連携
  • Azure RBAC による権限制御
  • 異なるルート CA への対応

主要機能

🔒

X.509 証明書ベースの相互認証

IKE ハンドシェイク時にデジタル証明書を使用した相互認証を実現。非対称信頼モデルにより、事前共有キーの漏洩リスクを排除し、暗号学的に強固な認証基盤を提供する。

🗝

Azure Key Vault 統合

アウトバウンド証明書を Azure Key Vault に安全に格納し、マネージド ID を通じてアクセスする。証明書のライフサイクル管理、ローテーション、アクセス制御を一元化し、運用負荷を削減する。

🔁

双方向の証明書フロー

アウトバウンド (Azure からオンプレミスへの認証) とインバウンド (オンプレミスから Azure への検証) の双方向で証明書認証を実施。それぞれ異なるルート CA を使用可能で、柔軟な PKI 構成に対応する。

技術仕様

認証方式 X.509 証明書 (相互認証)
IKE プロトコル IKEv2
最小鍵長 RSA 2048-bit
証明書形式 PFX (PKCS#12)
保管先 Azure Key Vault (Certificates)
対応 SKU VpnGw2 以上
対応クラウド Azure パブリック / Azure Government

コードサンプル

PowerShell
# 自己署名ルート CA 証明書の作成
$params = @{
    Type = 'Custom'
    Subject = 'CN=AzRootCA1'
    KeySpec = 'Signature'
    KeyExportPolicy = 'Exportable'
    KeyUsage = 'CertSign'
    KeyUsageProperty = 'Sign'
    KeyLength = 2048
    HashAlgorithm = 'sha256'
    NotAfter = (Get-Date).AddMonths(120)
    CertStoreLocation = 'Cert:\CurrentUser\My'
    TextExtension = @('2.5.29.19={critical}{text}ca=1&pathlength=4')
}
$cert = New-SelfSignedCertificate @params

# アウトバウンドリーフ証明書の作成
$params = @{
    Type = 'Custom'
    Subject = 'CN=az-outbound-cert1'
    KeySpec = 'Signature'
    KeyExportPolicy = 'Exportable'
    KeyLength = 2048
    HashAlgorithm = 'sha256'
    NotAfter = (Get-Date).AddMonths(120)
    CertStoreLocation = 'Cert:\CurrentUser\My'
    Signer = $cert
    TextExtension = @(
        '2.5.29.37={text}1.3.6.1.5.5.7.3.2,1.3.6.1.5.5.7.3.1')
}
New-SelfSignedCertificate @params

ユースケース

🏦 金融機関のコンプライアンス対応

PCI DSS や金融規制において、VPN 接続の認証に証明書ベースの認証が求められるケース。事前共有キーでは監査要件を満たせない環境。

X.509 証明書による相互認証で PCI DSS / 金融規制の要件を充足し、監査対応を容易にする。

🌐 大規模マルチサイト接続

数十〜数百の拠点を Azure に接続する必要があり、各拠点ごとの PSK 管理が運用負荷となっているケース。

PKI による一元管理で、サイト追加時の鍵管理を簡素化。証明書のローテーションも Key Vault で自動化可能。

🛡 ゼロトラストアーキテクチャ

ネットワーク接続においても「信頼しない、常に検証する」原則を適用し、デバイス認証を暗号学的に行いたいケース。

証明書による暗号学的相互認証でゼロトラストの原則を VPN レイヤーにも適用。デバイス単位の信頼性を担保する。
