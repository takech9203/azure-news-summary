Foundry エージェントセキュリティ機能の Microsoft Agent 365 ライセンスへの移行

2026-06-08 | ライセンス移行 / セキュリティ統合

概要

2026 年 7 月 1 日より、Microsoft Defender for Cloud で提供されていた Microsoft Foundry エージェントのセキュリティ機能が、Microsoft Agent 365 ライセンスに移行。エージェントレベルの検出・セキュリティポスチャ・脅威検出は Agent 365 ライセンスが必要となる。

❌ Before

  • ・Foundry エージェントのセキュリティが Defender CSPM/AI Services プランの一部
  • ・エージェントインベントリは AIAgentInfo テーブルで管理 (分散)
  • ・Copilot Studio と Foundry のセキュリティが別サービスに分散

✅ After

  • ・Agent 365 がエージェントインベントリの Single Source of Truth に
  • ・オブザーバビリティログによるリアルタイム脅威検出の統合
  • ・統一ライセンスモデルで Copilot Studio + Foundry を一元管理

主要機能

Agent 365 ライセンスが必要となる機能

🔍

エージェント検出とセキュリティポスチャ

クラウドホスト型エージェントの検出 (マルチクラウド含む)。従来は Defender CSPM プランで提供されていた機能。

🛡️

エージェント脅威保護

エージェントに対するリアルタイム脅威検出。Agent 365 オブザーバビリティログベースのアラートに移行。

🔎

Advanced Hunting 統合

新しい AgentInfo テーブルによる統合的なエージェント管理。従来の AIAgentInfo テーブルは廃止。

📋

統合エージェントレジストリ

Microsoft 365 Agent Registry が Single Source of Truth として、サードパーティエージェントのレジストリ同期にも対応。

エクスペリエンスの変更点

項目 変更内容
Advanced Hunting AIAgentInfo テーブル廃止 → 新 AgentInfo テーブルに移行
脅威検出アラート Agent 365 オブザーバビリティログベースの同等アラートに置換
リアルタイム保護 レガシーアラート → BehaviorInfo テーブルの Behaviors に移行
サードパーティエージェント Defender for Cloud コネクタ終了 → Agent 365 レジストリ同期
Azure ポータル Data and AI ダッシュボード / Cloud Security Explorer から Foundry エージェントデータ削除

ユースケース

🏢 Foundry エージェントを運用するエンタープライズ

Microsoft Foundry で複数の AI エージェントを本番運用し、Defender for Cloud でセキュリティポスチャと脅威検出を行っている組織。Agent 365 ライセンスを取得し、Advanced Hunting クエリを AgentInfo テーブルに移行することで、統合されたオブザーバビリティログによる一元管理が実現する。

🔄 マルチプラットフォームのエージェント管理

Copilot Studio と Microsoft Foundry の両方でエージェントを構築・運用し、セキュリティ監視が分散している組織。Agent 365 のもとでセキュリティ管理が統合され、Security for AI Agents トグルの有効化で一元管理が可能に。

Defender for Cloud に残る機能

🔒

Defender CSPM

Microsoft Foundry のアカウントとプロジェクトの検出は引き続き利用可能。エージェントレベルの機能は Agent 365 に移行。

🤖

Defender for AI Services

Azure OpenAI などの Foundry Models に対する脅威保護は継続。エージェント固有のアラートのみ Agent 365 に移行。

制限事項・注意点

  • 7 月 1 日までに Agent 365 ライセンス未取得の場合、エージェントセキュリティ機能が完全に無効化
  • 既存の Block モードのリアルタイム保護ルールが 7/1 に停止 → 新ポリシーでの再定義が必要
  • AIAgentInfo テーブル廃止 → 既存クエリ・カスタム検出・ワークブックの更新が必須
  • サードパーティクラウドエージェントは Defender for Cloud コネクタでの検出不可 → レジストリ同期の再構成が必要
  • Defender CSPM はアカウント/プロジェクトレベルの検出のみ継続
