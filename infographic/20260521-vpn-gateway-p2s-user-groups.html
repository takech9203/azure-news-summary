Azure VPN Gateway GA

P2S 接続のユーザーグループと IP アドレスプール

リリース日: 2026-05-21 | カテゴリ: Networking

📋概要

Azure VPN Gateway の Point-to-Site (P2S) 接続において、ユーザーの資格情報やグループに基づいて異なる IP アドレスプールを割り当てる機能が GA (一般提供) となりました。これにより、単一の VPN ゲートウェイで複数のセキュリティゾーンを実現し、NSG によるきめ細かいアクセス制御が可能になります。

❌ Before

  • 全 P2S ユーザーに単一のアドレスプールを割り当て
  • ネットワークレベルでユーザーを区別できない
  • グループベースのアクセス制御には別々のゲートウェイが必要

✅ After

  • グループに基づく IP プールの割り当てが可能
  • 単一ゲートウェイで複数のセキュリティゾーンに対応
  • Entra ID / RADIUS / 証明書によるグループ識別をサポート

ユーザーの認証方式とグループに基づいて異なる IP プールが割り当てられ、NSG ルールにより各プールからのアクセス先を制御できます。

⚡主要機能

👥

ユーザーグループ (ポリシーグループ)

ユーザーの論理的なグループ化を定義し、各グループに固有の IP アドレスプールを割り当て。最大 90 グループをサポート。

🎫

グループメンバー (ポリシーメンバー)

認証方式に応じた条件でグループ所属を判定。Entra ID では AADGroupID、RADIUS では VSA、証明書では CN を使用。

🔢

優先度によるグループ解決

複数グループに一致した場合、最低優先度値 (数値が小さい方) のグループに割り当て。明確なルールでグループ解決を実現。

🔐

複数認証方式対応

Microsoft Entra ID (AADGroupID)、RADIUS (AzureRADIUSGroupID)、証明書認証 (AzureCertificateID / CN) の 3 方式に対応。

💡ユースケース

🏢 部門別アクセス制御

財務部門、エンジニアリング部門、人事部門にそれぞれ異なる IP プールを割り当て、NSG ルールで各部門がアクセスできるリソースを制限。単一ゲートウェイで部門ごとのネットワーク分離を実現。

✨ ゲートウェイ数を削減しつつ、部門ごとのセキュリティ境界を維持

🤝 外部パートナーと社内ユーザーの分離

外部パートナーと社内ユーザーに異なる IP プールを割り当て、パートナーのアクセスを特定のサブネットやサービスに限定。同じゲートウェイ上で信頼レベルの異なるユーザーを安全に共存。

✨ 外部アクセスのリスクを最小化しながらコラボレーションを促進

🏦 規制環境でのデータ分離

金融機関や医療機関で、規制対象データにアクセスできるユーザーを IP レベルで分離。コンプライアンス要件を満たすネットワーク分離をシンプルに実装。

✨ 規制要件に対応したネットワーク分離を複雑なインフラなしで実現

⚠️制約事項

Basic SKU は非対応 (VpnGw1 以上が必要)
Entra ID 認証は OpenVPN のみ対応
最大 90 グループ / ゲートウェイ
最大 390 ポリシーメンバー合計
IKEv2 は証明書/RADIUS 認証のみ
Default グループは優先度による上書き不可

💡 グループに一致しないユーザーは Default グループの IP プールに割り当てられます。Virtual WAN でも同様の機能が利用可能です。
