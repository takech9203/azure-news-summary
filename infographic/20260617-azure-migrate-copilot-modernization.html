概要

Azure Migrate に GitHub Copilot Modernization との統合機能がパブリックプレビューとして追加されました。大規模なコードベースに対するコードインサイトをスケーラブルに提供できるようになります。Azure Migrate のポートフォリオレベルのディスカバリーおよびアセスメント機能と、GitHub Copilot のコンテキスト認識型コード分析を組み合わせ、インフラストラクチャの評価だけでなくアプリケーションコードレベルでのモダナイゼーション評価が可能になります。

Before / After 比較

❌ Before (課題)

  • Azure Migrate の評価はインフラレベルが中心、コード分析には別ツールが必要
  • 数百〜数千アプリケーションのコードベースを個別評価するのは膨大な工数
  • インフラ評価とコード評価が分断、統合的なモダナイゼーション計画策定困難

✅ After (改善)

  • ポートフォリオレベルのディスカバリー + AI コード分析の統合
  • コンテキスト認識型で依存関係やモダナイゼーション難易度を自動評価
  • インフラ+コード評価を一元管理、統合的なマイグレーション計画策定支援

主要機能

📦

スケーラブルなコードインサイト

大規模アプリケーションポートフォリオに対して一括コード評価を実施します。個別リポジトリ単位ではなくポートフォリオレベルでの分析が可能です。

  • 大規模アプリケーションポートフォリオに対する一括コード評価
  • 個別リポジトリ単位ではなくポートフォリオレベルでの分析
  • 評価結果のダッシュボード化と優先順位付け
🔗

ポートフォリオレベルのディスカバリーとの統合

Azure Migrate の既存ディスカバリー機能（サーバー、DB、Web アプリ）と連携し、インフラ情報とコード分析結果を包括的に評価します。

  • Azure Migrate の既存ディスカバリー機能（サーバー、DB、Web アプリ）と連携
  • インフラ情報+コード分析結果の包括的評価
  • Wave Planning への直接フィード
🧠

コンテキスト認識型コード分析

GitHub Copilot AI で依存関係、アーキテクチャパターン、モダナイゼーション複雑さを分析し、アプリケーション固有のコンテキストを考慮した推奨事項を提供します。

  • 依存関係の自動検出と影響分析
  • アーキテクチャパターンの認識と分類
  • モダナイゼーション複雑さのスコアリング
  • アプリケーション固有のコンテキストを考慮した推奨事項

メリット

💼 ビジネス面

  • 大規模マイグレーションのコード評価工数を大幅削減
  • ポートフォリオ全体の優先順位付け・バッチ分けの意思決定支援
  • モダナイゼーション計画精度向上によるリスク低減

🛠 技術面

  • AI で手動レビューでは見落としがちな依存関係・技術的負債を検出
  • Wave Planning と組み合わせて評価結果を直接実行計画に反映可能
  • リフト&シフトとモダナイゼーションの最適な組み合わせ判断

ユースケース

🏢 大規模エンタープライズのクラウドマイグレーション

数百のオンプレミスアプリケーションを Azure に移行するプロジェクトで、各アプリの最適戦略（リフト&シフト vs リファクタリング vs リビルド）を効率的に判断します。Azure Migrate のインフラ評価 + GitHub Copilot のコード分析を統合し、Wave Planning で段階的マイグレーション計画を策定します。

ポートフォリオ全体のインフラ+コード分析により、数百アプリの最適な移行戦略を効率的に決定

📚 レガシーアプリケーションのモダナイゼーション優先度判定

技術的負債を抱えた多数のレガシーアプリの中から投資対効果が高いものを特定します。コード分析で技術的負債を定量評価し、インフラ評価でランニングコストを分析することで、ポートフォリオ全体の優先順位付けを実現します。

技術的負債の定量評価 + コスト分析で投資対効果の高いモダナイゼーション対象を特定
