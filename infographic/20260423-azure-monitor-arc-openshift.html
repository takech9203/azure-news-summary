Azure Monitor

Azure Arc 対応 Kubernetes での OpenShift / ARO サポート

ハイブリッド・マルチクラウド環境で統一された可観測性を実現する Container Insights / マネージド Prometheus の GA

Launched (GA) 2026-04-23 Compute / Monitoring

概要

Azure Monitor が Azure Arc 対応 Kubernetes クラスター上で動作する OpenShift および Azure Red Hat OpenShift (ARO) に対する監視サポートを一般提供 (GA) 開始した。Container Insights、マネージド Prometheus、Log Analytics などの機能が OpenShift 環境でも利用可能になり、AKS と同じダッシュボード、アラート、分析ツールで一元管理が可能となった。これにより、ハイブリッド・マルチクラウド環境で統一された可観測性を実現する。

Key Points

  • Container Insights、マネージド Prometheus、Log Analytics の全機能が OpenShift で GA
  • ARO と AKS を同一の Azure Monitor ダッシュボードで一元管理
  • ハイブリッド・マルチクラウド環境全体の Kubernetes クラスターを単一画面で確認
  • SLA に裏付けられた本番ワークロード向け監視機能

Before

  • OpenShift クラスターの監視には独自の Prometheus + Grafana スタックを個別に構築・運用する必要
  • オンプレミスや他クラウドの OpenShift のメトリクスやログを Azure 監視基盤に集約する統一的手段が限定的
  • AKS と OpenShift で異なる監視ツールのため、ハイブリッド環境全体の一元把握が困難
  • ARO クラスターに対する Azure Monitor の統合監視が完全にはサポートされていなかった

After

  • Azure Arc を介して接続した OpenShift に対して Container Insights、マネージド Prometheus、Log Analytics の全機能が GA
  • ARO と AKS を同一の Azure Monitor ダッシュボードで一元管理
  • ハイブリッド・マルチクラウド環境全体の Kubernetes クラスターの正常性を単一画面で確認
  • SLA に裏付けられた本番ワークロード向け監視機能が OpenShift でも利用可能

CLI Setup

# Azure Arc に OpenShift クラスターを接続
az connectedk8s connect --name <cluster-name> --resource-group <resource-group>

# Azure Monitor 拡張機能のインストール
az k8s-extension create \
  --name azuremonitor-containers \
  --cluster-name <cluster-name> \
  --resource-group <resource-group> \
  --cluster-type connectedClusters \
  --extension-type Microsoft.AzureMonitor.Containers \
  --configuration-settings \
    logAnalyticsWorkspaceResourceID=<workspace-resource-id> \
    amalogs.useAADAuth=true

Use Cases

UC1: ハイブリッドクラウド Kubernetes 統合監視

オンプレミス OpenShift + Azure AKS + ARO を単一ダッシュボードで監視するケース。

-- 異なる環境の K8s クラスターを Azure Monitor で一元管理し、運用コストとツール分散を削減。

UC2: OpenShift から ARO へのマイグレーション監視

移行前後の両環境を同一 Azure Monitor で監視し、パフォーマンス劣化を早期検出するケース。

-- 移行中の比較分析と劣化検出を同一プラットフォームで実現。

UC3: マルチクラウド OpenShift の統合運用

AWS / GCP 上の OpenShift も Arc 経由で Azure Monitor に統合監視するケース。

-- クラウドプロバイダーを問わない統一的な可観測性を実現。

Constraints / Caveats

  • Azure Arc エージェントのデプロイと維持が必要、ネットワーク要件あり
  • OpenShift 固有モニタリング機能との二重管理の可能性
  • 大規模クラスターではログ収集プロファイルやサンプリングの適切な設定が重要(コスト)
  • オンプレミスから Azure へのネットワーク接続が必要
