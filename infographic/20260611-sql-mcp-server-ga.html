概要

SQL MCP Server が一般提供 (GA) となった。SQL MCP Server は Data API builder (DAB) 2.0 に搭載された Model Context Protocol (MCP) の実装であり、AI エージェントがデータベースと安全かつ高性能にやり取りするためのプロトコル準拠ツールを提供する。データベースを直接公開したり、脆弱な NL2SQL に依存したりすることなく、AI エージェントを本番データに対して制御されたアクセスで接続する。NL2DAB モデルにより決定論的なクエリ生成が保証される。

Key Points

  • Data API builder (DAB) 2.0 をベースに構築された MCP プロトコル実装
  • SQL Server, Azure SQL Database, PostgreSQL, Azure Cosmos DB, MySQL に対応
  • 7 つの DML ツールによる CRUD + 集計操作 (DDL は意図的に非サポート)
  • NL2DAB モデルによる決定論的クエリ生成 (生の SQL を直接生成しない)
  • OBO 認証、RBAC、行レベルセキュリティによるエンタープライズ対応
  • Custom MCP Tools でストアドプロシージャを名前付きツールとして公開可能

Preview (DAB 1.7)

  • DML ツール 6 種のみ。集計クエリには全行読み取りが必要
  • ストアドプロシージャは汎用 execute_entity のみで呼び出し
  • 大規模 DB では手動でエンティティを 1 つずつ構成
  • MCP シナリオでの実ユーザー識別が不透明
  • パブリックプレビュー (本番利用非推奨)

GA (DAB 2.0)

  • aggregate_records ツール追加 (COUNT, SUM, AVG, MIN, MAX)
  • Custom MCP Tools で SP を名前付きツールに直接公開
  • Auto Configuration でパターンマッチ自動検出
  • OBO 認証で実ユーザー ID をDB まで透過伝搬
  • GA: 本番環境利用を正式サポート

DML ツール (7種)

SQL MCP Server が公開する 7 つのデータ操作言語ツール。エージェントはこれらのツールを通じてデータベースを操作する。

describe_entities 利用可能なエンティティと操作を返す
create_record テーブルに新しいレコードを挿入
read_records フィルタ、ソート、ページネーション付きクエリ
update_record 既存レコードを更新
delete_record 既存レコードを削除
execute_entity ストアドプロシージャを実行
aggregate_records COUNT, SUM, AVG, MIN, MAX 集計 (NEW)

GA 新機能

📊

aggregate_recordsNEW

COUNT, SUM, AVG, MIN, MAX の集計関数をネイティブサポート。GROUP BY, HAVING, DISTINCT に対応。全行読み取りを回避し、パフォーマンスとコンテキストウィンドウの効率を大幅に向上。query-timeout (1-600秒) による長時間クエリの制御も可能。

🔧

Custom MCP ToolsNEW

ストアドプロシージャを名前付き MCP ツールとして直接登録。エンティティに "custom-tool": true を設定するだけで tools/list および tools/call に自動公開。エージェントが目的に応じたツールを名前で発見・呼び出し可能。

⚙️

Auto ConfigurationNEW

autoentities パターンによるデータベースオブジェクトの自動検出と公開。include/exclude パターンで対象を制御し、DAB 起動ごとにパターンを再評価して新しいテーブルを自動追加。dab auto-config-simulate による事前プレビューも可能。

🔒

OBO (On-Behalf-Of) 認証NEW

Microsoft Entra ID を使用したパススルー認証。呼び出し元ユーザーのトークンをデータベース用トークンに交換し、行レベルセキュリティを正確に適用。MCP シナリオでの「誰が操作しているか」の透明性を確保。

📋

ロール継承NEW

named-role -> authenticated -> anonymous の継承チェーンにより、権限設定の重複を排除。dab configure --show-effective-permissions で解決済み権限を確認可能。構成ファイルの保守性が大幅に向上。

🔍

OpenTelemetry MCP トレーシングNEW

MCP ツール実行が REST / GraphQL と同様に OpenTelemetry トレースに含まれる。dab init でデフォルトの OTEL セクションが自動生成。全エンドポイントを単一ダッシュボードで監視可能。

設定例

DAB 初期化
dab init --database-type mssql \
  --connection-string "<your-connection-string>" \
  --config dab-config.json \
  --host-mode development
Auto Configuration (パターンによる自動検出)
dab auto-config my-def \
  --patterns.include "dbo.%" \
  --patterns.exclude "dbo.internal%" \
  --patterns.name "{schema}_{object}" \
  --permissions "anonymous:read"

# シミュレーションで結果を事前確認
dab auto-config-simulate
Custom MCP Tool (ストアドプロシージャ登録)
dab add GetProductById \
  --source dbo.get_product_by_id \
  --source.type "stored-procedure" \
  --permissions "anonymous:execute" \
  --mcp.custom-tool true \
  --description "Returns full product details for a given product ID"
MCP ランタイム構成 (dab-config.json)
{
  "runtime": {
    "mcp": {
      "enabled": true,
      "path": "/mcp",
      "dml-tools": {
        "describe-entities": true,
        "create-record": true,
        "read-records": true,
        "update-record": true,
        "delete-record": true,
        "execute-entity": true,
        "aggregate-records": {
          "enabled": true,
          "query-timeout": 30
        }
      }
    }
  }
}

ユースケース

Enterprise Copilot with OBO

社内 Copilot が本番データベースに対して、実際のユーザー権限 (OBO 認証) で安全にデータ照会を行う。行レベルセキュリティによりユーザーは自分が閲覧可能なデータのみにアクセスでき、フィールドレベルの exclude で機密情報を保護。コンプライアンス監査で実際のユーザー操作を追跡可能。

OBO 認証 行レベルセキュリティ エンタープライズ

Analytics Agent with aggregate_records

BI エージェントが売上データの集計 (SUM, AVG, COUNT) を実行し、マネージャーにサマリーを提供する。全行を読み取る必要なくデータベース側で集計処理が実行されるため、パフォーマンスが向上しエージェントのコンテキストウィンドウを節約できる。GROUP BY による多次元分析も可能。

aggregate_records GROUP BY パフォーマンス最適化

Custom Tools for Business Logic

複雑な在庫確認ロジックをストアドプロシージャとして実装し、注文処理エージェントに Custom MCP Tool として公開する。エージェントは名前でツールを発見し直接呼び出せる。ビジネスロジックはデータベース側にカプセル化され、エージェントの実装から分離される。

制約事項

  • DDL (スキーマ変更) 操作はサポートされない (意図的設計)
  • JOIN 操作は read_records ツールで直接サポートされない (ビューまたは SP で対応)
  • stdio トランスポートのリクエストサイズは 1 MB に制限
  • Auto Configuration (autoentities) は現時点で Microsoft SQL データベースのテーブルのみ対応
  • OBO 認証使用時はキャッシュを無効にする必要がある
