📋概要

Azure Functions における PowerShell 7.6 のサポートがパブリックプレビューとして利用可能になりました。PowerShell 7.6 は .NET 10 ランタイム上で構築されており、最新の .NET エコシステムの恩恵を受けることができます。Windows ホスティングプラン（Consumption、Premium、Dedicated）でのみ利用可能です。

⚡主要機能

🚀

PowerShell 7.6 ランタイムサポート

.NET 10 ベースの PowerShell 7.6 を Azure Functions Runtime 4.x 上で利用可能。最新のパフォーマンス改善とセキュリティ更新を享受できます。

⌨️

タブ補完の大幅な改善

コマンドレットやパラメータの補完精度が向上し、開発効率が大幅にアップ。PSReadLine v2.4.5 との組み合わせでさらに快適に。

📦

最新モジュール対応

PSResourceGet v1.2.0、PSReadLine v2.4.5 など最新のモジュールが標準で利用可能。モジュール管理が大幅に改善されています。

⚙️

エンジン最適化

PSRP プロトコルの更新やエンジン内部の改善により、実行パフォーマンスが向上。サーバーレス環境でのコールドスタート改善にも寄与。

🔧

新しいコマンドレット機能

Get-Clipboard -Delimiter、Get-Command -ExcludeModule など新しいパラメータが追加。実験的機能（PSFeedbackProvider、PSNativeWindowsTildeExpansion）が正式化。

⚠️破壊的変更

⚠️ 注意が必要な変更点

  • Join-Path の -ChildPath パラメータが string[] に変更

🎯ユースケース

🏗️ インフラ自動化

Azure リソースのプロビジョニング・設定をサーバーレスで自動化。PowerShell の Az モジュールを活用したインフラ管理。

🔄 運用タスクの自動化

Timer トリガーによる定期的なメンテナンスタスクの実行。リソースのクリーンアップやレポート生成を自動化。

📁 イベント駆動処理

Queue/Blob トリガーによるファイル処理やデータ変換。イベント発生時に自動的に PowerShell スクリプトを実行。

🌐 API エンドポイント

HTTP トリガーによる PowerShell ベースの REST API の構築。管理タスクや監視データの提供に最適。

🔒 セキュリティ監査

Azure リソースのコンプライアンスチェックの自動化。ポリシー準拠状況の定期確認と通知。

📌制約事項

⚠️ 現在の制約

  • プレビュー段階: 本番環境での使用は推奨されない
  • Windows のみ: Linux は非対応
  • 破壊的変更あり: 既存コードの互換性確認が必要
  • Linux Consumption 非対応: Linux ベースのプランでは利用不可
  • サポート期間未定: GA 時期やサポートライフサイクルは未発表
