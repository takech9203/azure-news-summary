Azure Kubernetes Service (AKS) Retirement 2026-04-16

Ubuntu 22.04 サポート終了

AKS における Ubuntu 22.04 のサポートが 2027 年 6 月 30 日に終了。移行先は Ubuntu 24.04 (containerd 2.0 がデフォルト)。Kubernetes 1.35 以降では自動移行されるが、バージョン固定の場合は手動対応が必要。

🔄 概要: 何が変わるのか

AKS における Ubuntu 22.04 のサポートが終了し、Ubuntu 24.04 への移行が必要になります。containerd 2.0 への移行やカーネル更新を含む大きな変更となるため、事前の検証が重要です。

ポイント: --os-sku Ubuntu を使用している場合、Kubernetes 1.35 以降へのアップグレード時に自動的に Ubuntu 24.04 へ移行されます。--os-sku Ubuntu2204 でバージョン固定している場合は手動での OS SKU 変更が必要です。

Before (Ubuntu 22.04)

  • K8s 1.25~1.34 で --os-sku Ubuntu 指定時のデフォルト OS
  • containerd 1.7.x がデフォルト
  • 2027/6/30 以降セキュリティパッチ停止
  • --os-sku Ubuntu2204 でバージョン固定可能 (K8s 1.36 まで)

After (Ubuntu 24.04)

  • K8s 1.35+ で --os-sku Ubuntu 指定時のデフォルト OS
  • containerd 2.0 がデフォルト (互換性検証推奨)
  • 最新カーネルとセキュリティ改善
  • --os-sku Ubuntu2404 で K8s 1.32+ から利用可能

🔑 主要な変更ポイント
📦

containerd 2.0 への移行

Ubuntu 24.04 では containerd 2.0 がデフォルトのコンテナランタイムとなります。containerd 1.7.x との動作の違いがあるため、ワークロードの互換性検証が必要です。

🔄

K8s 1.35+ での自動移行

--os-sku Ubuntu を使用している場合、Kubernetes 1.35 以降へのアップグレード時にデフォルト OS が自動的に Ubuntu 24.04 に切り替わります。追加の移行作業は不要です。

📌

バージョン固定の場合は手動対応が必要

--os-sku Ubuntu2204 でバージョン固定している場合は、Ubuntu2404 への OS SKU 変更または Ubuntu (デフォルト) への変更と K8s アップグレードが必要です。

⏰

2028/4/30 にノードイメージ完全削除

セキュリティパッチ停止後も猶予期間がありますが、2028 年 4 月 30 日にはノードイメージと関連コードが AKS から完全に削除され、スケーリングやノード修復の操作が失敗するようになります。

🔒

FIPS 非サポート (Ubuntu2404)

--os-sku Ubuntu2404 を使用する場合、FIPS (Federal Information Processing Standards) がサポートされません。FIPS 要件がある場合は移行方法の検討が必要です。

現在の OS SKU とノードイメージの確認

bash ノードプール情報を確認 
# クラスターのノード プール情報を確認
az aks nodepool show \
    --resource-group $RESOURCE_GROUP \
    --cluster-name $CLUSTER_NAME \
    --name $NODE_POOL_NAME \
    --query "{osSku:osSku, nodeImageVersion:nodeImageVersion, kubernetesVersion:kubernetesVersion}"

方法 1: Kubernetes アップグレードによる自動移行

bash os-sku が Ubuntu の場合に推奨 
# 方法1: Kubernetes アップグレードによる自動移行
az aks upgrade \
    --resource-group $RESOURCE_GROUP \
    --name $CLUSTER_NAME \
    --kubernetes-version 1.35.0

方法 2: OS SKU の直接更新

bash os-sku が Ubuntu2204 の場合 
# 方法2: OS SKU の直接更新
az aks nodepool update \
    --resource-group $RESOURCE_GROUP \
    --cluster-name $CLUSTER_NAME \
    --os-sku Ubuntu2404 \
    --kubernetes-version 1.32.0 \
    --name $NODE_POOL_NAME \
    --node-count 1

💡 ユースケース

ユースケース 1: デフォルト OS SKU を使用している標準的な AKS クラスター

--os-sku Ubuntu でノードプールを作成し、Kubernetes 1.33 を実行しているクラスター。Kubernetes 1.35 以降へのアップグレードで自動的に Ubuntu 24.04 がデフォルトとして適用されるため、追加の移行作業が不要。

移行方法: az aks upgrade --kubernetes-version 1.35.0

ユースケース 2: バージョン固定 (Ubuntu2204) を使用しているクラスター

特定のアプリケーション要件により --os-sku Ubuntu2204 を明示的に指定してノードプールを運用しているケース。Kubernetes バージョンを変更せずに OS のみを移行でき、アプリケーションへの影響を最小限に抑えられる。

移行方法: az aks nodepool update --os-sku Ubuntu2404

ユースケース 3: LTS を利用予定のクラスター

Kubernetes 1.33 以降で Long Term Support (LTS) を有効にして長期運用を計画しているケース。LTS を有効にする前に Ubuntu 24.04 への移行が必要。移行後は LTS による長期サポートと最新 OS の組み合わせで安定した運用基盤を確保。

移行方法: Ubuntu 24.04 に移行後、LTS を有効化
⚠️ デメリット・制約事項
🔍

containerd 2.0 互換性の検証が必要

Ubuntu 24.04 では containerd 2.0 がデフォルトとなるため、コンテナランタイムの動作に依存するワークロードは事前の互換性検証が必要です。

🔒

FIPS 非サポート

--os-sku Ubuntu2404 を使用する場合、FIPS (Federal Information Processing Standards) がサポートされません。FIPS 要件がある環境では代替手段の検討が必要です。

🔄

ノード再イメージによる一時的な中断

移行に伴いノードの再イメージ (reimage) が発生するため、ワークロードの一時的な中断が発生する可能性があります。PodDisruptionBudget の設定を推奨します。

⛔

サポート終了後のロールバック不可

Ubuntu 22.04 のサポート終了後は、Ubuntu 24.04 から Ubuntu 22.04 へのロールバックは実質的に不可能となります。移行前の十分な検証が重要です。

🔗 関連サービス・機能
🔄

AKS ノードイメージ アップグレード

ノードプールの OS イメージを最新バージョンに更新する機能。OS SKU の移行後も定期的なノードイメージアップグレードが推奨されます。

⚙️

AKS 自動アップグレード

クラスターの Kubernetes バージョンやノードイメージを自動的にアップグレードする機能。自動アップグレードチャネルの設定により、OS 移行後の継続的な更新を自動化できます。

📅

AKS Long Term Support (LTS)

通常の 1 年間のコミュニティサポートに加え、さらに 1 年間の延長サポートを提供する機能。K8s 1.33 以降で LTS を利用する場合は Ubuntu 24.04 への移行が前提条件です。

🐧

Azure Linux (AzureLinux)

AKS で利用可能な別の Linux ディストリビューション。Ubuntu からの移行先として Azure Linux 3.0 も選択肢の一つとなります。
