Azure Site Recovery In Preview

Linux VM の NVMe ディスクコントローラーサポート

2026-06-08 | ディザスターリカバリー / BCDR

概要

Azure Site Recovery が、NVMe 対応の Gen2 VM ファミリー (Da/Ea/Fa v6, Ebsv5/Ebdsv5) で稼働する Linux VM のレプリケーションおよびディザスターリカバリーをサポート。Azure-to-Azure シナリオでのリージョン間 DR が構成可能に。

❌ Before

  • ・NVMe Linux VM は ASR レプリケーション非対応
  • ・高性能 v6 シリーズ VM で DR の選択肢が限定的
  • ・NVMe VM 移行時に DR 非対応がブロッカー

✅ After

  • ・NVMe Gen2 Linux VM で ASR レプリケーション可能
  • ・Da/Ea/Fa v6 で リージョン間 DR を構成
  • ・高性能 + 事業継続性の両立

主要機能

🔄

NVMe ディスクコントローラー対応レプリケーション

NVMe インターフェースを使用する Gen2 Linux VM のディスクレプリケーションをサポート。Azure-to-Azure シナリオでのリージョン間 DR。

🐧

主要 Linux ディストリビューション対応

Red Hat Enterprise Linux 9、SUSE Linux Enterprise Server 15、Ubuntu 24 をサポート。企業で広く利用されるディストリビューションをカバー。

⚡

最新 VM ファミリー対応

Da/Ea/Fa v6 シリーズ、Ebsv5/Ebdsv5 など NVMe インターフェースを使用する最新の高性能 VM で DR を構成可能。

🛡️

Azure Boost 連携

Azure Boost によるストレージ処理のハードウェアオフロードと組み合わせ、ストレージ性能とレジリエンスを両立。

ユースケース

🔀 v6 シリーズ VM への移行と DR 構成

旧世代 (v5) SCSI ベース VM から高性能 Da/Ea/Fa v6 NVMe VM への移行を計画中。移行後も Azure Site Recovery による DR を維持し、性能向上と事業継続性を両立。

🗄️ 高 I/O データベースワークロードの保護

RHEL 9 上で稼働する DB サーバーが Ebsv5 VM で NVMe ストレージを活用。リージョン障害時のフェイルオーバーが必要な環境。

📋 コンプライアンス要件への対応

規制要件により全本番 VM にリージョン間 DR が必須。新規採用した v6 シリーズ VM も含めて DR 要件を満たすことが可能に。
