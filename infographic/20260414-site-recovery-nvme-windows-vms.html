概要

Azure Site Recovery が、NVMe 対応の Gen2 VM ファミリー (Da/Ea/Fa v6, Ebsv5/Ebdsv5) 上の Windows VM のレプリケーションおよび DR をサポート開始。NVMe は SCSI より高い I/O パフォーマンスを提供する次世代ディスクインターフェースであり、v6 シリーズではデフォルトで採用されている。レプリケーション中の SCSI から NVMe へのインプレース移行も自動検出され、中断なく継続可能。
  • NVMe 対応 Gen2 Windows VM の Azure-to-Azure レプリケーションおよび DR をサポート

Before

  • NVMe Windows VM は Site Recovery レプリケーション非サポート

  • NVMe 対応 Gen2 Windows VM の Azure-to-Azure レプリケーション可能
  • SCSI から NVMe 移行時の自動検出・中断なし継続
  • 新世代 VM ファミリーへの移行と DR 戦略を両立

NVMe 対応 Windows VM のソースリージョンからターゲットリージョンへの Azure Site Recovery レプリケーションフロー。Mobility Service エージェントが変更データを追跡し、キャッシュストレージ経由で Recovery Services Vault を通じてレプリカディスクに書き込む。フェイルオーバー VM は NVMe 構成を維持する。

⭐

主要機能

💾

NVMe Windows VM レプリケーション

Gen2 Windows VM (Da/Ea/Fa v6, Ebsv5/Ebdsv5) の NVMe ディスクコントローラーを使用した Azure-to-Azure レプリケーションをサポート。NVMe の高 I/O パフォーマンスを維持したまま DR 環境を構築可能。

🔄

SCSI から NVMe への自動検出

レプリケーション中に SCSI から NVMe へのインプレース移行が発生した場合、自動的に検出し、レプリケーションを中断することなく継続。手動での再構成が不要で運用負荷を削減。

✅

ターゲット SKU 検証

ターゲットリージョンで NVMe 非対応 SKU が選択された場合、事前にバリデーションエラーとして通知。フェイルオーバー時のトラブルを未然に防止し、確実な DR 構成を実現。

ユースケース

🚀

v6 シリーズ VM 移行と DR

v5 シリーズから v6 シリーズへの移行を検討しているが、DR 要件がブロッカーとなり移行が進まない状況。
効果: v6 シリーズ VM への移行時にも Azure Site Recovery による DR 構成が可能となり、DR 要件がブロッカーにならなくなる。最新世代 VM のパフォーマンスメリットと DR 戦略を両立。
🔄

既存レプリケーション環境での SCSI から NVMe 移行

既に Site Recovery でレプリケーションを構成済みの環境で、SCSI から NVMe へのディスクコントローラー移行を実施する必要がある。
効果: SCSI から NVMe へのインプレース移行を自動検出し、レプリケーションを中断なく継続。手動での再構成が不要で、ダウンタイムなしに移行を完了可能。
⚠

制限事項・注意点

  • パブリックプレビュー: 本番利用は要検討。SLA の適用対象外となる可能性あり
  • Windows のみ: Linux VM は非サポート
  • Gen2 のみ: Gen1 VM は対象外
  • エフェメラル OS ディスク: 非サポート
  • ローカル NVMe ディスク: 非サポート
  • 混合コントローラー VM (Lsv3 等): 非サポート
  • ターゲットリージョン: NVMe 対応 SKU が必須。非対応の場合はバリデーションエラー
  • Azure CLI 非サポート: Portal または PowerShell を使用する必要あり
