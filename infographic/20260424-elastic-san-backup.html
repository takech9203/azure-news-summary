Azure Backup: Elastic SAN バックアップ (パブリックプレビュー)

Elastic SAN ボリュームのバックアップとリストアをフルマネージドで実行可能に。Backup vault を通じた一元管理と増分スナップショットによる効率的なデータ保護

In Preview 2026-04-24

概要

Azure Backup が Azure Elastic SAN のバックアップをサポートするパブリックプレビューが開始された。Elastic SAN ボリュームのバックアップとリストアをフルマネージドで実行可能になり、Backup vault を通じてバックアップを管理する。バックアップは増分スナップショット(オペレーショナルティア)としてエクスポートされる仕組みで、一時スナップショットを作成後、変更データのみを増分スナップショットとしてエクスポートすることで、ストレージコストを最適化しつつ信頼性の高いデータ保護を実現する。

Key Points

  • Backup vault を通じたフルマネージドなバックアップ・リストア
  • 日次・週次のスケジュールバックアップとオンデマンドバックアップをサポート
  • 最大 450 のリカバリポイントを保持、柔軟な保持ポリシー
  • ZRS / LRS 冗長性を選択可能
  • Backup Center からの一元的な監視・管理

  • ポイントインタイムリカバリが困難
  • 監視やコンプライアンスレポートが一元化されていない

  • 日次・週次のスケジュールバックアップとオンデマンドバックアップをサポート

スナップショットエクスポートによるバックアップ

バックアップ処理では、まず対象ボリュームの一時スナップショットを作成し、その後変更データを増分スナップショットとしてスナップショットリソースグループにエクスポートする。増分スナップショットにより、変更分のみを保存するため、ストレージコストを最適化できる。

📅

柔軟なバックアップスケジュールとリテンション

日次および週次のスケジュールバックアップに対応。最大 450 のリカバリポイントを保持可能で、デフォルトの保持期間は 7 日間。ビジネス要件に合わせた柔軟な保持ポリシーを設定できる。

🛡

ストレージ冗長性の選択

Backup vault のストレージ冗長性として ZRS (ゾーン冗長ストレージ) と LRS (ローカル冗長ストレージ) を選択可能。可用性要件やコストに応じて最適な冗長性レベルを選択できる。

🔄

代替場所へのリストア (ALR)

バックアップからのリストアは、同一サブスクリプション内だけでなく異なるサブスクリプションの Elastic SAN ボリュームへのリストアにも対応。Import API を使用して新しいボリュームとしてリストアするほか、ポータルからマネージドディスクとして直接作成することも可能。

Use Cases

UC1: ミッションクリティカルなデータベースワークロードのデータ保護

Elastic SAN 上で稼働する SQL Server や Oracle などのミッションクリティカルなデータベースに対して、スケジュールバックアップによる定期的なデータ保護を実現。最大 450 のリカバリポイントにより、細かい粒度でのポイントインタイムリカバリが可能になり、データ損失リスクを最小化する。

-- Backup vault による自動化されたバックアップで、データベースの継続的なデータ保護を実現。

UC2: AKS ワークロードの永続ボリュームバックアップ

AKS クラスターで使用される Elastic SAN の永続ボリュームに対して、フルマネージドなバックアップを提供。コンテナ化されたステートフルアプリケーションのデータを定期的にバックアップし、障害時には代替場所へのリストア (ALR) により迅速な復旧を実現する。

-- コンテナワークロードのデータ保護をフルマネージドで自動化。

UC3: コンプライアンス要件への対応

規制要件によりデータの定期バックアップと一定期間の保持が求められる環境において、Backup Center からの一元的な監視・管理と柔軟な保持ポリシーにより、コンプライアンス要件を満たすバックアップ運用を構築する。

-- Backup Center による一元管理で、コンプライアンスレポートと監視を実現。

Pricing

プレビュー期間中の料金

プレビュー期間中は保護インスタンス料金が無料。増分スナップショットの料金はマネージドディスク料金に準拠する。GA 後の料金体系については、別途発表される予定。

Limitations / Caveats

  • 4 TB 以下のボリュームのみサポート
  • Vault ティア非対応 -- オペレーショナルティアのみ
  • OLR (元の場所へのリストア) 非対応 -- ALR のみ
  • 時間単位のバックアップ非対応 -- 日次・週次のみ
  • プレビュー段階のため、本番ワークロードでの利用は慎重に検討が必要
  • オンデマンドバックアップは 1 日 10 回まで、リストアも 1 日 10 回まで
