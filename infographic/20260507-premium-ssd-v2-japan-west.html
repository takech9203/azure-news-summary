概要

Azure Premium SSD v2 が Japan West (西日本) リージョンの全 3 Availability Zone で GA。これにより Japan East と Japan West の両方で Premium SSD v2 の全 AZ サポートが実現し、日本国内で対称的な HA/DR アーキテクチャの構築が可能に。

❌ Before

  • Japan West で一部 AZ のみ Premium SSD v2 対応
  • 国内 DR 構成でストレージ性能に非対称性
  • ゾーン冗長構成の選択肢が制限

✅ After

  • Japan West の全 3 AZ で Premium SSD v2 利用可能
  • 国内 2 リージョン間の対称的 DR 設計が実現
  • 完全なゾーン冗長アーキテクチャを構築可能

Japan West の 3 つの Availability Zone に Premium SSD v2 を分散配置し、Japan East との間でリージョン間 DR を構成。国内 2 リージョンの両方で全 AZ 対応となったことで、対称的な HA/DR アーキテクチャが構築可能に。

主要機能

🌐

全 3 AZ サポート

Japan West リージョンの 3 つすべての Availability Zone で Premium SSD v2 が利用可能に。完全なゾーン冗長構成が実現。

⚡

独立したパフォーマンスチューニング

容量 (1 GiB ~ 64 TiB)、IOPS (最大 80,000)、スループット (最大 2,000 MB/s) をそれぞれ独立して設定可能。ダウンタイムなしで調整可能。

🚀

サブミリ秒レイテンシ

99.9% の時間でサブミリ秒のレイテンシを提供。トランザクション集約型ワークロード (SQL Server, Oracle, SAP HANA 等) に最適。

💰

コスト効率の高い料金モデル

ベースライン IOPS (3,000) とスループット (125 MB/s) が無料。必要なパフォーマンスのみに課金される従量制モデル。

設定方法 - Azure CLI

bash Japan West AZ 1 にディスク作成 
# Premium SSD v2 ディスクを Japan West の AZ 1 に作成
az disk create -n myPremiumSSDv2Disk -g myResourceGroup \
  --size-gb 100 \
  --disk-iops-read-write 5000 \
  --disk-mbps-read-write 150 \
  --location japanwest \
  --zone 1 \
  --sku PremiumV2_LRS \
  --logical-sector-size 4096

# パフォーマンスの動的調整 (ダウンタイム不要)
az disk update --resource-group myResourceGroup \
  --name myPremiumSSDv2Disk \
  --disk-iops-read-write 10000 \
  --disk-mbps-read-write 300

ユースケース

🏦 西日本拠点の金融機関向けデータベース基盤

シナリオ: 大阪に本拠を置く金融機関が、Japan West リージョンで SQL Server Always On 可用性グループを 3 つの AZ に分散配置。

効果: 3 AZ にまたがる構成により、単一 DC 障害時も自動フェイルオーバーでサービス継続可能。セカンダリの IOPS を抑えてコスト最適化しつつ、必要時に動的引き上げ可能。

🔄 Japan East / Japan West 間の国内 DR 構成

シナリオ: 日本国内でデータ主権要件を満たしつつ、リージョン障害に備えた DR 構成を構築。プライマリを Japan East、DR サイトを Japan West に配置。

効果: 両リージョンで Premium SSD v2 の全 AZ サポートが利用可能になったことで、DR サイトでも同等のストレージ性能を確保。DR サイトの IOPS を通常時は低く設定し、フェイルオーバー時に動的に引き上げ。

💡 Premium SSD v1 と比較して、必要なパフォーマンスのみに課金される従量制モデルであるため、ワークロードによっては大幅なコスト削減が期待できる。

制限事項・注意点

⚠️

OS ディスク非対応

Premium SSD v2 はデータディスクのみ。OS ディスクには Premium SSD v1 を使用すること。

⚠️

ゾーナル VM 必須

Availability Zone を指定して VM を作成する必要がある。リージョナル VM にはアタッチ不可。

⚠️

LRS のみ

現時点では LRS (ローカル冗長ストレージ) 構成のみサポート。ホストキャッシュ非対応。
