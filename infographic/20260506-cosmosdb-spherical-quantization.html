ベクトル検索のための球面量子化 (Spherical Quantization)

高次元ベクトルにおいてより高い検索精度 (Recall) とインデックス作成速度の向上を実現する新しい量子化方式

Preview 2026-05-06

概要

Azure Cosmos DB for NoSQL において、ベクトルインデックスの量子化方式として「球面量子化 (Spherical Quantization)」がパブリックプレビューで利用可能に。従来の積量子化 (Product Quantization) と比較して、高次元ベクトルにおいてより高い検索精度 (Recall) とインデックス作成速度の向上を実現。

Key Points

  • 高次元ベクトルでの Recall 向上
  • インデックス作成速度の改善
  • DiskANN / quantizedFlat 両インデックスタイプで利用可能
  • quantizerType: "spherical" で簡単に設定可能

Before

  • 積量子化 (Product Quantization) のみが利用可能で、高次元ベクトルでは Recall が安定しない場合があった
  • 大規模ベクトルデータセットのインデックス作成時間が長い
  • 高次元エンベディング使用時、量子化による精度低下が課題

After

  • 球面量子化により、高次元ベクトルでもより高い Recall を安定的に維持
  • インデックス作成速度の向上
  • DiskANN / quantizedFlat 両インデックスタイプで利用可能
  • quantizerType: "spherical" で簡単に設定可能

Index Policy 設定例

{
    "vectorIndexes": [
        {
            "path": "/contentVector",
            "type": "diskANN",
            "quantizerType": "spherical"
        }
    ]
}

ベクトル検索クエリ

SELECT TOP 10 c.title, VectorDistance(c.contentVector, @queryVector) AS SimilarityScore
FROM c
ORDER BY VectorDistance(c.contentVector, @queryVector)

Use Cases

UC1: 大規模 RAG アプリケーション

OpenAI text-embedding-3-large (3072次元) で数百万件のドキュメント検索。高次元ベクトルでの高い Recall を維持しつつ、大規模データセットで高速な近似最近傍検索を実現。

UC2: マルチモーダル AI 検索

IoT センサーデータと画像データのベクトル化、異常検知。多様なデータソースからのエンベディングを統合し、球面量子化による効率的なインデックスで類似パターンを高速検出。

UC3: リアルタイムレコメンデーション

EC サイトの商品レコメンデーション。ユーザー行動と商品特徴のベクトル化により、低レイテンシでパーソナライズされたレコメンドを提供。

Pricing

追加料金なし

球面量子化自体に追加料金は不要。通常の Cosmos DB 課金 (RU/s + ストレージ) が適用されます。量子化によりストレージコストと RU 消費が削減されるため、コスト最適化に貢献します。

Limitations / Caveats

  • プレビュー段階のため、本番ワークロードでの利用は慎重に検討
  • 最低 1,000 件のベクトルが必要
  • 最大 4,096 次元まで
  • 共有スループットアカウントでは利用不可
  • ベクトルインデックスを有効にすると無効化できない
