Ingestion Volume Change ダッシュボード

Metrics Usage Insights | リリース日: 2026-06-05

概要

Azure Monitor の Metrics Usage Insights に「Ingestion Volume Change」ダッシュボードがパブリックプレビューとして追加。メトリクスのインジェスト量を 2 つの時点間で比較し、タイムシリーズ数やイベントインジェストレートの変化を視覚的に把握可能に。

❌ Before

  • インジェスト量の変動調査は手動で実施
  • 請求額増減の原因メトリクス特定に時間がかかる
  • 時系列比較を行う簡便な手段がなかった

✅ After

  • 2 日付間のインジェスト量を並列比較し変化率を把握
  • メトリクスごとの差分をテーブルで即座に確認
  • コスト変動の原因特定が大幅に効率化

Azure Monitor Workspace に取り込まれたメトリクスデータは、診断設定を通じて Log Analytics Workspace に使用状況データとして送信される。Ingestion Volume Change ダッシュボードは、この使用状況データを基に 2 時点間のインジェスト量比較を提供する。

主要機能

📅

時点間比較 (Comparison dates)

ドロップダウンで開始日と終了日を選択し、比較期間を定義。選択期間の最初の日 (Initial) と最後の日 (Final) のメトリクスデータを比較。60 日や 90 日の長期レンジから短期レンジまで柔軟に分析可能。

📊

メトリクス比較テーブル

Namespace、Metric、Dimensions ごとの詳細な比較データ。イベントレートの差分と変化率、タイムシリーズ数の差分と変化率をテーブル形式で即座に確認可能。新規メトリクスは変化率 100 として表示。

⚡

スパイク・ドロップ検出

タイムシリーズ数やイベントインジェストレートの急激な変動を迅速に検出。コスト変動の原因特定を加速する。

比較テーブルのカラム構成

カラム名 説明
Namespace Azure Monitor Workspace 内の Namespace
Metric メトリクス名
Dimensions ラベル/ディメンション
Initial / Final Event Rate 開始日・終了日のイベントインジェストレート
Event Rate Difference 開始日と終了日の差分
Event Rate Change % イベントレートの変化率
Initial / Final Timeseries 開始日・終了日のアクティブなタイムシリーズ数
Timeseries Difference タイムシリーズ数の差分
Timeseries Change % タイムシリーズ数の変化率

設定方法

前提条件: Azure Monitor Workspace (AMW) + Log Analytics Workspace (LAW) が作成済みであること
1️⃣

診断設定を有効化

Azure Monitor Workspace の診断設定で、使用状況データを Log Analytics Workspace に送信するよう構成

2️⃣

データ流入を待機

約 24 時間待機してデータが流入するのを確認

3️⃣

ダッシュボードを使用

Metrics Usage Insights を開き、「Ingestion Volume Change」を選択。比較期間を設定して分析開始

ユースケース

💰 月次コストレビュー

月次の Azure 請求額が前月比で大幅に増加した場合、原因がメトリクスインジェスト量の増加にあるか調査

  1. Comparison dates で前月の初日と末日を選択
  2. Event Rate Difference が大きいメトリクスを特定
  3. 該当メトリクスの Namespace と Dimensions から発生元を絞り込む

☸️ AKS クラスタースケール後の影響確認

AKS クラスターのノード数を増加させた後、Prometheus メトリクスのインジェスト量がどの程度増えたか確認

  1. スケールアウト前後の日付を Comparison dates で選択
  2. Timeseries Difference でタイムシリーズ数の増加量を確認
  3. 増加率が想定範囲内か評価

📈 長期トレンドの分析

過去 90 日間でメトリクスインジェスト量がどのように変化しているか、長期トレンドを把握

  1. Comparison dates で 90 日間のレンジを設定
  2. Event Rate Change % で増加傾向にあるメトリクスを特定
  3. 将来のコスト予測とリソース計画に反映

制限事項・注意点

⚠️

パブリックプレビュー

GA 前に仕様が変更される可能性がある。本番運用環境での利用は十分なテストの上で判断すること。

⏰

データ反映に約 24 時間

診断設定を有効化してからデータが反映されるまで約 24 時間かかる。即時分析には事前の準備が必要。

📋

比較対象は 2 時点のみ

選択期間の最初の日と最後の日のみの比較であり、期間中の日次推移は別途確認が必要。
