Microsoft Foundry Agent Service

Bring Your Own AI Gateway

外部 AI ゲートウェイのモデルを Foundry Agent Service のエージェントから利用可能に

Launched (GA) 2026-04-27

概要

Foundry Agent Service に「Bring Your Own AI Gateway」(BYO AI Gateway) 機能が一般提供 (GA) として追加された。Azure API Management などのエンタープライズ AI ゲートウェイの背後にホストされた AI モデルを、Foundry Agent Service のエージェントから利用できるようになる。Azure API Management 接続と、非 Azure の Model Gateway 接続の 2 種類をサポートし、既存のモデル管理インフラを維持したまま Foundry のエージェント機能を活用可能。

Key Points

  • Azure API Management と非 Azure ゲートウェイの両方に対応
  • OpenAI 互換のチャット補完 API を実装するモデルを接続可能
  • API キー、マネージド ID、OAuth 2.0 による柔軟な認証
  • Code Interpreter、ファイル検索、MCP 等の組み込みツールと併用可能
  • 既存のエンタープライズセキュリティポリシーをそのまま活用
  • パブリック / プライベートネットワーキング両対応

Before

  • Foundry モデルカタログ内のモデル (Azure OpenAI, Llama, DeepSeek 等) のみ利用可能
  • 既存の API Management ガバナンス基盤を Foundry と統合不可
  • サードパーティ / セルフホストモデルの利用には独自基盤が必要

After

  • AI ゲートウェイ背後のモデルをエージェントから直接利用可能
  • 既存のセキュリティポリシー・トークン管理をそのまま活用
  • API キー / マネージド ID / OAuth 2.0 による柔軟な認証
  • Foundry 組み込みツールを外部モデルと組み合わせ可能

🏢

Azure API Management 接続

既存の Azure API Management リソースを選択し、OpenAI 互換チャット補完 API を実装するモデルデプロイメントに接続。API キーまたはマネージド ID で認証。インテリジェントなデフォルト設定により、デプロイメント一覧エンドポイント (/deployments) を自動検出。

🌐

Model Gateway 接続 (非 Azure)

セルフホスト、非 Azure ホスト、カスタムソリューションへの接続をサポート。ベース URL を指定して任意のゲートウェイエンドポイントに接続。API キーまたは OAuth 2.0 (クライアント資格情報) で認証。静的・動的モデルディスカバリーに対応。

📦

モデルデプロイメント自動管理

接続ウィザードで追加したモデルは Foundry にデプロイメントとして自動登録。エージェント構成時に admin-connected デプロイメントとして選択可能で、リクエストはゲートウェイ経由で自動ルーティング。

🧰

Foundry 組み込みツール対応

Code Interpreter、Functions、ファイル検索、OpenAPI、Foundry IQ、SharePoint Grounding、Fabric Data Agent、MCP、Browser Automation が BYO モデルと組み合わせ可能。

🔒

ネットワーク分離サポート

パブリックネットワーキングと完全なネットワーク分離の両方に対応。API Management は専用 Bicep テンプレート、セルフホストゲートウェイは VNet 内アクセスで構成。

認証方式

🔑

API キー

API Management サブスクリプションキーまたはカスタム API キーによる認証。最もシンプルな接続方式。

👤

マネージド ID

Microsoft Entra ID によるマネージド ID 認証。API Management 接続で利用可能。シークレット管理が不要。

🔏

OAuth 2.0

クライアント ID、クライアントシークレット、トークン URL、スコープを指定。Model Gateway 接続で利用可能。

セットアップ手順 (Azure API Management)

1

Microsoft Foundry にサインイン

ai.azure.com にアクセスし、Operate > Admin console を選択

2

プロジェクトの Parent resource を選択

All projects タブから対象プロジェクトの Parent resource リンクをクリック

3

Admin-connected models で Add を選択

Connection Type で Azure API Management を選択し、既存の API Management リソースとモデルデプロイメントを指定

4

認証とモデル設定

API キーまたはマネージド ID を選択し、デプロイメント名・モデル名・表示名を設定

5

接続を作成

Advanced で API バージョンやカスタムヘッダーを構成し、Add をクリック

Code Sample

Python SDK でのエージェント作成 (BYO モデルを使用)

import os
from azure.ai.projects import AIProjectClient
from azure.identity import DefaultAzureCredential

# 環境変数の設定
# FOUNDRY_PROJECT_ENDPOINT: プロジェクトエンドポイント URL
# FOUNDRY_MODEL_DEPLOYMENT_NAME: <接続名>/<モデル名> (例: my-apim-connection/gpt-4o)

client = AIProjectClient(
    endpoint=os.getenv("FOUNDRY_PROJECT_ENDPOINT"),
    credential=DefaultAzureCredential(),
)

# エージェントの作成 (BYO モデルを使用)
agent = client.agents.create_version(
    model=os.getenv("FOUNDRY_MODEL_DEPLOYMENT_NAME"),
    name="my-byom-agent",
    instructions="You are a helpful assistant.",
)

Use Cases

UC1: エンタープライズ AI ガバナンスの統合

Azure API Management で複数の AI モデルデプロイメントを一元管理し、部門ごとのトークン制限、コンテンツセーフティポリシー、アクセスログの監査を行っている組織が、既存のガバナンス基盤を維持しつつ Foundry Agent Service でエージェントを構築。

-- API Management の既存ポリシー（トークンレート制限、セマンティックキャッシュ、ロードバランシング）をそのまま活用しながら、Foundry のエージェントオーケストレーション機能を組み合わせ、ガバナンスとエージェント機能を両立。

UC2: マルチプロバイダーモデルの活用

AI ゲートウェイの背後に Azure OpenAI の GPT モデル、セルフホストのオープンソースモデル、サードパーティプロバイダーのモデルなど、複数モデルを運用し、用途に応じて最適なモデルをエージェントに割り当て。

-- Model Gateway 接続で複数のモデルエンドポイントを Foundry に登録し、コストパフォーマンスや精度の要件に応じた最適なモデル選択が可能に。

料金

BYO AI Gateway 機能自体の追加料金は明示されていない。料金は以下の要素で構成される。

🤖

Foundry Agent Service

エージェントが使用するモデルデプロイメントの従量課金 (トークン料金)

🏢

Azure API Management

APIM インスタンス料金。AI Gateway 無料枠あり。v2 ティア (Basic/Standard/Premium v2) が必要

🌐

外部モデル

BYOM モデルの利用料金はサードパーティプロバイダーのライセンス条項に従う

Limitations / Caveats

  • BYOM モデルは「非 Microsoft 製品」として扱われ、Responsible AI の軽減策は利用者が実装する必要がある
  • データ処理要件の遵守は利用者の責任であり、データが Azure のコンプライアンス・地理的境界を越える可能性がある
  • 現時点では Agent SDK のプロンプトエージェントのみ対応。ホステッドエージェントやワークフローエージェントは未サポート
  • モデルは OpenAI 互換のチャット補完 API を実装している必要がある

関連サービス

Azure API Management

エンタープライズ AI ゲートウェイとして、トークン管理、アクセス制御、ロードバランシング、セマンティックキャッシュ、コンテンツセーフティを提供。

Microsoft Foundry Agent Service

AI エージェントの構築、デプロイ、スケーリングのフルマネージドプラットフォーム。BYO AI Gateway はこのサービスの拡張機能。

Microsoft Entra ID

マネージド ID を使用した AI ゲートウェイへの認証を提供。シークレット管理不要のセキュアな接続。

Azure AI Content Safety

API Management のポリシーを通じて LLM プロンプトのコンテンツ安全性チェックを適用。
