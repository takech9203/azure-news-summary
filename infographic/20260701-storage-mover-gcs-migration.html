📋 概要
Azure Storage Mover に、Google Cloud Storage (GCS) から Azure Blob Storage へのクラウド間データ移行機能がパブリックプレビューとして追加。S3 互換インターフェースを利用することで、GCS バケット内のデータを Azure Blob Storage に移行でき、マルチクラウド環境から Azure への統合を簡素化。

Before

  • GCS から Azure Blob Storage へのデータ移行には、手動でのデータパイプライン構築やサードパーティツールの利用が必要だった
  • マルチクラウド環境でのデータ統合に一貫した移行ツールが存在しなかった
  • GCS から Azure への移行を一元管理・監視する仕組みが限られていた

After

  • Azure Storage Mover のフルマネージドサービスとして GCS からの移行が可能になった
  • S3 互換インターフェースを利用するため、追加のカスタム開発が不要
  • プロジェクト単位・ジョブ単位での移行進捗管理と監視が一元化された
  • マルチクラウド統合を Azure 上に集約するシナリオが簡素化された

⭐ 主要機能
🔗

S3 互換インターフェースによる GCS 接続

Google Cloud Storage が提供する S3 互換 API (GCS Interoperability API) を利用してデータにアクセス。追加のカスタム開発なしで GCS バケット内のデータを読み取り可能。

☁

クラウド間データ移行

GCS バケットから Azure Blob コンテナへの直接的なクラウド間データ転送を実現。マルチクラウド環境のデータを Azure に集約するシナリオを大幅に簡素化。

💻

フルマネージドオーケストレーション

プロジェクト・ジョブ定義・ジョブ実行の階層構造で移行を管理。進捗状況の追跡と監視を一元化し、大規模な移行プロジェクトも効率的に運用可能。

🔄

差分転送サポート

初回のフルコピー後、差分のみを転送する繰り返し実行に対応。変更分のみを効率的に同期することで、転送時間とコストを最適化。

💡 ユースケース

🌐 Google Cloud から Azure へのクラウド移行

GCS バケットに蓄積された大量の非構造化データ (画像、動画、ログ、バックアップなど) を Azure Blob Storage に移行し、Azure 上のサービスで活用する。

効果: フルマネージドの移行サービスにより、手動パイプライン構築が不要。差分転送で段階的な移行が可能で、ダウンタイムを最小化しながら大規模データ移行を実現。

🏢 マルチクラウドデータ統合

GCS 上のデータを Azure Blob Storage に集約し、Azure Synapse Analytics や Azure Databricks などのデータ分析基盤で統合的に活用する。

効果: S3 互換インターフェース経由で追加開発不要のデータ集約を実現。プロジェクト単位の管理で複数のデータソースからの移行を効率的にオーケストレーション可能。
