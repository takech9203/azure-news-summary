Flex Consumption プランにおける TLS/SSL 証明書サポート

リリース日: 2026-05-21

📋概要

Azure Functions の Flex Consumption プランに、新しいサイトスコープ証明書モデルによる TLS/SSL 証明書サポートが追加されました。従来の Web スペーススコープモデルとは異なり、証明書が個々の Function App に紐づけられるため、アプリ間の証明書分離が実現されます。全 Flex Consumption リージョンで利用可能です。

Before

  • Flex Consumption プランでは TLS/SSL 証明書をカスタムドメインに使用できなかった
  • Web スペーススコープモデルでは同一リージョン・リソースグループ内のアプリ間で証明書が共有されていた
  • 証明書の分離が困難でセキュリティ境界が曖昧

After

  • サイトスコープモデルにより証明書がアプリごとに分離
  • マネージド証明書、App Service 証明書、Key Vault インポート、PFX/CER アップロードに対応
  • 全 Flex Consumption リージョンで利用可能
  • Key Vault 連携で自動同期 (24時間以内)

従来の Web スペーススコープモデルでは証明書がリージョン+リソースグループ単位で共有されていましたが、新しいサイトスコープモデルではアプリごとに証明書が独立して管理されます。

⚡主要機能

🔒

サイトスコープ証明書モデル

証明書が個々の Function App に紐づけられる新モデル。同じリージョン・リソースグループ内の他アプリとは共有されず、セキュリティ境界が明確に分離されます。

📜

複数の証明書タイプのサポート

App Service マネージド証明書 (無料・自動更新)、App Service 証明書、Azure Key Vault からのインポート、PFX ファイルアップロード、CER ファイルアップロードに対応。

🔑

Key Vault 連携

マネージド ID による認証で Key Vault から証明書をインポート。証明書の更新は 24 時間以内に自動的に同期されます。

🔄

マネージド証明書の自動更新

App Service マネージド証明書は無料で提供され、プラットフォームにより自動的に更新されます。カスタムドメインの HTTPS 化を最小限の運用負荷で実現。

💡ユースケース

🌐 サーバーレス API のカスタムドメイン HTTPS 化

Flex Consumption プラン上の Function App にカスタムドメインを設定し、マネージド証明書を利用して HTTPS を有効化。無料かつ自動更新で運用負荷ゼロ。

マネージド証明書により証明書管理が完全に不要、コストゼロで HTTPS を実現

🔑 Key Vault による証明書の一元管理

組織全体の証明書を Azure Key Vault で一元管理し、各 Function App にマネージド ID 経由でインポート。証明書の更新は 24 時間以内に全アプリに自動反映。

証明書のライフサイクル管理を Key Vault に集約、更新忘れによるサービス停止リスクを排除

🔏 クライアント証明書認証 (mTLS)

パブリック証明書を /var/ssl/private パスから読み込み、クライアント証明書認証 (相互 TLS) を実装。B2B API 連携のセキュリティを強化。

Flex Consumption プランでも mTLS ベースの高セキュリティ通信を実現

💻利用可能リージョンの確認

Azure CLI で Flex Consumption プランが利用可能なリージョン一覧を確認できます:

bash Azure CLI 
az functionapp list-flexconsumption-locations --query "sort_by(@, &name)[].{Region:name}" -o table

⚠️制約事項・注意点

E2E 暗号化は未サポート
プライベート証明書はアプリあたり最大 3 個
パブリック証明書はアプリあたり最大 3 個
Key Vault 同期は最大 24 時間のラグあり
プライベート証明書は PFX 形式のみ
プレビュー段階のため仕様変更の可能性あり

証明書の即座な反映が必要な場合は Key Vault 同期の 24 時間ラグに注意してください。緊急の証明書更新が必要な場合は直接アップロードを検討してください。
