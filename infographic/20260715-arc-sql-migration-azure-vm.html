Azure Arc-enabled SQL Server Launched (GA)

SQL Server on Azure VM へのマイグレーション対応が一般提供開始

2026年7月15日 | Azure Arc / SQL Server on Azure Virtual Machines

📋 概要

Azure Arc マイグレーションソリューションが SQL Server on Azure Virtual Machines をターゲットとしてサポート開始。Azure Arc 対応 SQL Server から Azure SQL Managed Instance だけでなく、SQL Server on Azure VM (IaaS) への移行も統合マイグレーションエクスペリエンスで実行可能に。

Before

  • Azure SQL Managed Instance のみがターゲット
  • SQL VM への移行は DMS を直接構成する必要あり
  • Arc ポータルからシームレスな操作不可
  • アセスメントと移行実行が別ツール

After

  • SQL VM と SQL MI の両方をターゲット選択可能
  • 統一マイグレーションウィザードで移行実行
  • Arc ポータルから一貫したワークフロー
  • アセスメントからカットオーバーまで統合

Azure Arc 対応 SQL Server は Azure Connected Machine Agent を通じて Arc サービスに接続。統合マイグレーション UI からアセスメントと移行を実行し、Azure DMS がバックアップファイルを利用してターゲット (SQL Server on Azure VM または Azure SQL Managed Instance) にデータベースを復元します。

⭐ 主要機能

🔄

統合マイグレーションエクスペリエンス

Azure Arc ポータルから SQL Server on Azure VM と Azure SQL MI の両方をターゲットとして選択し、同一のウィザードで移行を実行。アセスメントからカットオーバーまで一貫したワークフロー。

⚡

オンライン / オフライン移行モード

オフラインモードでは一括移行、オンラインモードではトランザクションログの継続的復元により最小限のダウンタイムで移行完了。ワークロード要件に応じた柔軟な選択が可能。

🔍

Migration Readiness Assessment

Arc 対応 SQL Server で自動実行される移行準備アセスメントに基づき、最適なターゲット (SQL VM vs SQL MI) の推奨と SKU サイズの提案を提供。

🔒

Self-hosted Integration Runtime

オンプレミスのネットワーク共有にあるバックアップファイルを安全に Azure Storage にアップロード。セキュアなデータ転送アーキテクチャを提供。

🛡

TDE / Always Encrypted 対応

Transparent Data Encryption (TDE) で保護されたデータベースの移行に対応。Always Encrypted キーは自動的にターゲットに移行。

💡 ユースケース

🏢 レガシー SQL Server のリフト & シフト移行

オンプレミスの SQL Server 2016 で稼働する基幹業務アプリケーションを、OS レベルのカスタマイズや SQL Server Agent ジョブへの依存があるため、SQL Server on Azure VM に移行するケース。
Arc ポータルからの統一ワークフローにより、アセスメントから移行完了までをシームレスに実行。オンラインモードで最小ダウンタイムを実現。

🌐 ハイブリッド環境からの段階的クラウド移行

複数のオンプレミス SQL Server インスタンスを持つ企業が、ワークロード特性に応じて SQL Server on Azure VM と Azure SQL Managed Instance に振り分けて移行するケース。
統一されたマイグレーションエクスペリエンスで複数ターゲットへの移行を効率管理。アセスメントに基づく最適ターゲット選定でコストとパフォーマンスを最適化。

⚠ 制限事項・注意点

  • Windows OS 上の SQL Server のみ対応 (Linux では Database migration 機能は利用不可)
  • SQL Server Agent ジョブ、SSIS パッケージなどのサーバーオブジェクトは自動移行されない (手動再構成が必要)
  • 同一ターゲット VM への移行は最大 100 DB、100 DB 完了後 30 分の待機が必要
  • Azure Arc 対応 SQL Server は Azure VM 上の SQL Server では非サポート (オンプレミスまたは他クラウドのみ)
  • 高可用性/災害復旧構成のターゲットへの自動構成は非サポート
