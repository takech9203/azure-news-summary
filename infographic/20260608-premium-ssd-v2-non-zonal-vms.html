Premium SSD v2: 非ゾーン仮想マシンのサポート開始

2026-06-08 | ストレージ / パフォーマンス

概要

Azure Premium SSD v2 ディスクが、可用性ゾーンを指定せずにデプロイされた非ゾーン (Non-zonal) VM へのアタッチに対応。シンプルな単一インスタンス構成でも最大 80,000 IOPS / 2,000 MB/s の高性能ストレージが利用可能に。

❌ Before

  • ・Premium SSD v2 にはゾーナル VM が必須
  • ・既存の非ゾーン VM には後からアタッチ不可
  • ・シンプルな単一 VM 構成では利用不可

✅ After

  • ・対象リージョンで非ゾーン VM にもアタッチ可能
  • ・Azure が自動でゾーンを選択
  • ・VM 再作成なしでストレージアップグレード

主要機能

💿

非ゾーン VM への Premium SSD v2 アタッチ

可用性ゾーンを指定せずにデプロイされた単一インスタンス VM に Premium SSD v2 をアタッチ可能。Azure がバックエンドで適切な AZ を自動選択。

🔄

バックグラウンドゾーンアラインメント

VM とディスクのゾーンが異なる場合、バックグラウンドコピーでアラインメントを実行。最大 24 時間かかる可能性あり。

⚡

パフォーマンス特性の維持

非ゾーナルデプロイでも Premium SSD v2 の全パフォーマンス仕様は変わらない。IOPS / スループット / 容量の個別調整も継続可能。

💰

コスト効率の高い料金体系

容量 + IOPS (3,000 超過分) + スループット (125 MB/s 超過分) の 3 要素課金。必要な分だけ払う柔軟な設計。

ユースケース

🧪 開発・テスト環境の高性能化

非ゾーン VM で構成されたテスト環境で、本番相当のストレージパフォーマンスでテスト可能。AZ を意識せずに Premium SSD v2 をアタッチ。

🗄️ 単一インスタンスのデータベースサーバー

高可用性が不要だが高い I/O が求められる DB (小規模 SQL Server, MongoDB 等) を単一 VM で運用。最大 80,000 IOPS を活用。

⬆️ 既存非ゾーン VM のストレージアップグレード

Premium SSD のパフォーマンスが不足する非ゾーン VM に、再作成なしで Premium SSD v2 データディスクを追加し性能向上。

制限事項・注意点

  • 対応リージョンが限定的 (一部リージョンのみ)
  • バックグラウンドコピーに最大 24 時間かかる可能性
  • コピー中のデタッチ/リアタッチ操作は失敗する
  • 停止/割当解除 VM への非ゾーナルディスクアタッチは VM 再起動失敗の可能性
  • コピー中のディスクサイズ増加・CMK 変更不可
  • OS ディスクとしての使用不可 (データディスクのみ)
