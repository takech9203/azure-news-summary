Azure Files

ファイル共有中心管理モデル GA と macOS Entra ID 認証プレビュー

2026-06-02 GA Public Preview

📋 概要

Azure Files に2つの重要なアップデートが発表されました。ファイル共有をトップレベル ARM リソースとして独立管理できる Microsoft.FileShares リソースプロバイダー が GA となり、共有単位の RBAC・ポリシー・タグ適用が可能になりました。また、macOS からの Entra ID 認証がパブリックプレビューとして利用可能になり、Windows・Linux・macOS 全プラットフォームで統一的な ID ベース認証が実現します。

⚠️ Before

  • ファイル共有はストレージアカウント配下のサブリソース、個別ARM管理不可
  • 共有ごとの独立RBAC/タグ/ポリシーにはストレージアカウント分割が必要
  • macOSユーザーはEntra ID認証が使えず、共有キー/SASトークンに依存
  • macOSだけ認証方法が異なりセキュリティポリシーの統一困難

✅ After

  • ファイル共有がトップレベルARMリソースとなり独立管理可能
  • 個々の共有にRBAC、Azure Policy、タグ、ロックを直接適用
  • macOSからEntra ID認証でAzure Filesにアクセス可能
  • Windows、Linux、macOS全主要プラットフォームでEntra ID認証利用可能

⭐ 主要機能

GA Microsoft.FileShares リソースプロバイダー
📦

トップレベル ARM リソース化

ファイル共有がストレージアカウントに依存しない独立リソースとして管理可能に。サブスクリプション直下でのリソース管理が実現します。

🔒

個別 RBAC 適用

共有単位でロールベースアクセス制御を設定。チームごと・プロジェクトごとの権限分離が容易になります。

📜

Azure Policy 直接適用

ファイル共有に対してガバナンスポリシーを直接適用。コンプライアンス要件を共有レベルで管理できます。

🛡

独立ライフサイクル管理

リソースロック、タグによる分類、独立した削除保護を共有単位で実現。NFS 4.1 + SSD が GA 時点でサポートされます。

Preview macOS Entra ID 認証
🔑

ID ベースのアクセス制御

共有キー不要でセキュアなアクセスを実現。Entra ID のアイデンティティベースの認証でファイル共有にアクセスできます。

📁

SMB プロトコル統合

macOS Finder の標準的な SMB 接続から Azure Files にアクセス。ユーザーにとって透過的な認証体験を提供します。

🌐

クロスプラットフォーム統一認証

Windows、Linux、macOS すべてのプラットフォームで Entra ID 認証が利用可能に。セキュリティポリシーの統一管理を実現します。

🛡

ゼロトラスト対応

MFA（多要素認証）、条件付きアクセスポリシーを macOS クライアントにも適用可能。ゼロトラストセキュリティモデルを全プラットフォームで統一できます。

💻 コードサンプル

Azure CLI 
# Microsoft.FileShares リソースプロバイダーの登録
az provider register --namespace Microsoft.FileShares

# NFS 4.1 ファイル共有の作成
az fileshare create \
  --resource-group myResourceGroup \
  --name my-nfs-share \
  --protocol NFS \
  --storage-type SSD \
  --quota 1024 \
  --location eastus
Bicep 
resource nfsShare 'Microsoft.FileShares/fileShares@2026-01-01' = {
  name: 'my-nfs-share'
  location: 'eastus'
  properties: {
    protocol: 'NFS'
    storageType: 'SSD'
    quotaInGiB: 1024
  }
  tags: {
    environment: 'production'
    team: 'platform'
  }
}

💡 ユースケース

1

クロスプラットフォーム開発チームの統一ファイル共有

Windows、Linux、macOS が混在する開発チームで、Entra ID による統一認証を活用し、全員が同じファイル共有にセキュアにアクセス。プラットフォームごとの認証方法の違いを意識する必要がなくなります。

2

IaC によるファイル共有のライフサイクル管理

Microsoft.FileShares をトップレベルリソースとして Bicep/Terraform で宣言的に管理。チームごとに独立した RBAC・タグ・ポリシーを適用し、ガバナンスを自動化します。

3

macOS 開発者のゼロトラストアクセス

macOS を利用する開発者が Entra ID 認証で Azure Files にアクセス。MFA・条件付きアクセスを適用し、共有キーの漏洩リスクを排除したゼロトラスト環境を実現します。
