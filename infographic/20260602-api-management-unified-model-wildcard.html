Unified Model API プレビュー & ワイルドカードホスト名 GA

Build 2026 で発表された 2 つの重要アップデート: マルチ LLM 統合エンドポイントとワイルドカードドメインサポート

Unified Model API: Public Preview Wildcard Hostname: GA

リリース日: 2026-06-02

📋概要

  • Unified Model API (パブリックプレビュー): 単一の OpenAI 互換エンドポイントで複数の LLM プロバイダー (OpenAI, Anthropic, Google Vertex AI) を統合管理
  • クライアントは OpenAI Chat Completions 形式でリクエストを送信し、APIM がバックエンドへのフォーマット変換を自動処理
  • ワイルドカードカスタムホスト名 (GA): *.api.contoso.com パターンでワイルドカードドメインをサポート
  • Premium v2 および Standard v2 ティアで利用可能

Before / After 比較

Before（従来）

  • プロバイダーごとに異なる API 形式・SDK が必要
  • フェイルオーバーやガバナンスの実装が困難
  • テナントごとにサブドメイン＋証明書を手動構成

After（改善後）

  • 全プロバイダーに対して単一の OpenAI 互換エンドポイント
  • モデルエイリアスにより透過的なバックエンド変更が可能
  • 1 つのワイルドカード証明書で全サブドメインをカバー

💻コードサンプル

Python クライアントから Unified Model API を利用する例:

from openai import OpenAI

client = OpenAI(
    base_url="https://<apim-instance>.azure-api.net/llm/v1",
    api_key="<api-management-subscription-key>",
)

# モデルエイリアスを指定してリクエスト
response = client.chat.completions.create(
    model="gpt",  # "claude-sonnet" や "gemini" など設定済みエイリアスも可
    messages=[{"role": "user", "content": "What can you do?"}],
)
print(response.choices[0].message.content)

💡ユースケース

🤖 マルチモデル AI チャットボットの構築

GPT-4o を汎用会話に、Claude を分析タスクに、Gemini をマルチモーダル処理に使い分け。Unified Model API により単一エンドポイントからモデルエイリアスで切り替え、フェイルオーバーも自動化。

🏢 マルチテナント SaaS のサブドメイン管理

customer1.api.contoso.com、customer2.api.contoso.com のようなテナント別サブドメインを、1 つのワイルドカード証明書で一括管理。テナント追加時の証明書設定が不要に。
