Microsoft Build 2026 発表 - 開発者向けツールの 3 つの主要アップデート

GA Public Preview 2026-06-03 / 2026-06-04

概要

Microsoft Build 2026 にて、Microsoft Foundry の開発者向けツールに関する 3 つのアップデートが発表された。VS Code 拡張機能の一般提供 (GA)、Foundry Toolbox のツール検索 (パブリックプレビュー)、モデルカタログのドメインフィルター (パブリックプレビュー) により、開発者はより効率的に AI アプリケーションの開発・デプロイが可能になる。

主要ポイント

  • VS Code 拡張機能の一般提供 (GA) - VS Code 内からフルモデルカタログ、プレイグラウンド、Hosted Agent デプロイが可能
  • Foundry Toolbox のツール検索 (パブリックプレビュー) - 大規模なマルチチームカタログ内から適切なツールを高速に発見
  • モデルカタログのドメインフィルター (パブリックプレビュー) - ロボティクス、生物医学、材料科学などの分野に特化したモデルを発見

VS Code 拡張ステータス

Before

  • VS Code 拡張はプレビューでモデルカタログ全体へのアクセスが制限されていた
  • Toolbox 内のツール数増加で全ツール定義を毎ターン送信するコストが高かった
  • 1,900 以上のモデルから特定の業界に適したモデルを絞り込む手段が限定的だった

After

  • VS Code 内からフルモデルカタログ、モデルプレイグラウンド、Hosted Agent デプロイが利用可能 (GA)
  • ツール検索により、大規模なマルチチームカタログ内から適切なツールを高速に発見
  • ドメインフィルターにより特定分野に特化したモデルを効率的に発見

主要機能

💻

Microsoft Foundry for Visual Studio Code GA

VS Code 内から Foundry のフル機能にアクセス可能な拡張機能が一般提供に

  • フルモデルカタログへのアクセス (1,900+ モデル)
  • モデルプレイグラウンド (対話的チャット、システムプロンプト調整、View Code)
  • Hosted Agent デプロイ (VS Code から直接デプロイ)
  • プロジェクト管理 (新規作成、リソースグループ選択)
  • コード生成 (SDK 選択、認証方式の選択)
🔎

Tool Search in Foundry Toolboxes Public Preview

大規模なマルチチーム環境でのツール管理を効率化

  • 大規模マルチチームカタログ内からツールを高速検索
  • 必要なツールのみをエージェントに渡しトークン消費を抑制
🔬

Domain Filter for Model Catalog Public Preview

特定分野に特化したモデルを効率的に発見

  • 対応ドメイン: ロボティクス、生物医学、材料科学など
  • 既存フィルター (Collection, Industry, Capabilities, Inference tasks) との組み合わせ

技術仕様

VS Code 拡張名 Microsoft Foundry Toolkit for Visual Studio Code
認証方式 Azure アカウント (Microsoft Entra 認証)
対応モデル数 1,900 以上
デプロイオプション Serverless API / Managed Compute

ユースケース

VS Code 内でのエンドツーエンド AI アプリ開発

モデル探索からデプロイ、プロンプトテスト、コード統合まで完結
  • モデルカタログからモデルを選択
  • プレイグラウンドでプロンプトをテスト
  • View Code で SDK コードを生成
  • Hosted Agent として直接デプロイ
効果: IDE を離れることなくフルサイクルの AI 開発が可能

大規模マルチチーム環境でのエージェントツール管理

ツール検索で関連ツールのみを動的に選択
  • Toolbox に多数のツールを登録
  • ツール検索で必要なツールを自動選択
  • エージェントに最小限のツール定義のみを送信
効果: トークン消費の削減とエージェントの応答精度向上

専門分野向けモデル選定

生物医学研究チームがドメインフィルターで専門モデルを発見
  • ドメインフィルターで「生物医学」を選択
  • 既存フィルターと組み合わせて絞り込み
  • 専門分野に最適なモデルを迅速に特定
効果: 1,900+ モデルから分野特化モデルを効率的に発見

制約事項

  • ツール検索およびドメインフィルターはパブリックプレビュー
  • モデルデプロイにはサブスクリプションのクォータ制限が適用
  • Managed Compute には対応する VM クォータが必要
  • Hugging Face モデルなどにはハブベースプロジェクトが必要
