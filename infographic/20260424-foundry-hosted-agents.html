Microsoft Foundry Agent Service

Hosted Agents がパブリックプレビューとして提供開始

任意のフレームワークでビルドしたエージェントをコンテナイメージとしてパッケージ化し、マネージドインフラ上にデプロイ・実行

In Preview 2026-04-24

概要

Hosted Agents は、開発者が独自のコードをコンテナイメージとしてパッケージ化し、Foundry Agent Service のマネージドインフラ上にデプロイ・実行できる新機能。Agent Framework、LangGraph、Semantic Kernel など任意のフレームワークを使用でき、分離された実行サンドボックスで各セッションが実行される。ホスティング、スケーリング、ID 管理をプラットフォームが自動管理するため、開発者はエージェントロジックに集中できる。

Key Points

  • 任意のフレームワーク (Agent Framework, LangGraph, Semantic Kernel 等) でエージェントを構築可能
  • コンテナイメージとして ACR 経由でデプロイ
  • セッションごとに分離されたサンドボックスで実行
  • Microsoft Entra ID による専用エージェント ID を自動作成
  • イミュータブルバージョニングとカナリア/ブルーグリーンデプロイをサポート
  • Foundry Toolbox (MCP 経由) で Code Interpreter, Web Search 等を統合

Before

  • オープンソースフレームワークでエージェント構築時にインフラ管理が必要
  • セッション間データ分離が複雑
  • Prompt Agent ではカスタムオーケストレーション不可
  • バージョン管理・ロールバック等の運用管理を個別実装

After

  • 任意フレームワークでコンテナデプロイ可能
  • セッションごとの分離サンドボックス自動提供
  • ホスティング・スケーリング・ID 管理をプラットフォームが自動管理
  • イミュータブルバージョン管理とトラフィック分割

🔒

分離された実行サンドボックス

各セッションに専用のセキュアランタイムを提供。クロスセッションのデータ漏洩を防止し、セキュアなマルチテナント実行を実現する。

📦

コンテナベースデプロイメント

Azure Container Registry (ACR) 経由でコンテナイメージをデプロイ。Python および C# をサポートし、任意のフレームワークでエージェントを構築可能。

🌐

プロトコルサポート

Responses、Invocations、Activity、A2A の 4 種類のプロトコルをサポート。OpenAI 互換 SDK からのアクセスやエージェント間通信に対応。

🔑

専用エージェント ID

Microsoft Entra ID によるエージェント専用の ID を自動作成。エージェントが Azure サービスやリソースに安全にアクセスするための認証基盤を提供。

💾

セッション状態永続化

$HOME ディレクトリおよび /files パスにセッション状態を永続化。最大 30 日間保持され、セッション再開時に状態を復元可能。

🔄

イミュータブルバージョニング

デプロイごとにイミュータブルなバージョンを作成。カナリアデプロイやブルーグリーンデプロイによるトラフィック分割で安全なリリースを実現。

🧰

Foundry Toolbox 統合

MCP (Model Context Protocol) 経由で Code Interpreter、Web Search、File Search 等のツールをエージェントから利用可能。機能拡張が容易。

Code Samples

azd AI Agent 拡張機能を使用した Hosted Agent のデプロイフロー

# azd ai agent 拡張機能のインストール
azd ext install azure.ai.agents

# 新しい Hosted Agent プロジェクトの初期化
azd ai agent init

# Azure リソースのプロビジョニング
azd provision

# ローカルでのテスト実行
azd ai agent run

# Azure へのデプロイ
azd deploy

# デプロイ済みエージェントの状態確認
azd ai agent show --output table

Use Cases

UC1: カスタムオーケストレーションによる社内業務自動化エージェント

LangGraph や Semantic Kernel 等のフレームワークで複雑なワークフローを構築し、社内の申請処理、データ集計、レポート生成などの業務を自動化するエージェントをデプロイ。インフラ管理不要で、エージェントロジックの開発に集中できる。

-- マネージドインフラにより運用負荷を大幅に削減し、開発サイクルを短縮。

UC2: Webhook 受信による外部サービス連携エージェント

Invocations プロトコルを活用して外部サービスからの Webhook を受信し、イベント駆動でタスクを処理するエージェントを構築。CRM、チケットシステム、モニタリングツール等との連携を実現。

-- プロトコルの柔軟性により、多様な外部システムとのシームレスな統合を実現。

UC3: Teams 統合による全社向け AI アシスタント

Microsoft Teams と連携し、社員が自然言語で問い合わせできる AI アシスタントをデプロイ。セッション分離により各ユーザーのデータを安全に管理しつつ、Foundry Models を活用した高精度な応答を提供。

-- セッション分離と Entra ID 認証により、セキュアな全社展開が可能。

Limitations / Caveats

  • パブリックプレビュー段階であり、本番ワークロードでの使用は推奨されない
  • 最大同時アクティブセッション数は 50 / サブスクリプション / リージョン
  • サンドボックスの上限は 2 vCPU / 4 GiB メモリ
  • サポート言語は Python と C# のみ (Java, Go 等は未対応)
  • 利用可能リージョンは 4 リージョンに限定 (Australia East, Canada Central, North Central US, Sweden Central)
  • プライベート ACR (VNet 統合 ACR) は非対応
