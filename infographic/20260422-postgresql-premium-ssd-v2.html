Azure Database for PostgreSQL

Premium SSD v2 ストレージの一般提供開始

最大 4 倍の IOPS、柔軟なパフォーマンス調整、エンタープライズ機能フルサポート

Launched (GA) 2026-04-22

概要

Azure Database for PostgreSQL Flexible Server において、Premium SSD v2 ストレージオプションが一般提供 (GA) となった。Premium SSD v2 は、従来の Premium SSD と比較して最大 4 倍の IOPS (80,000)、大幅に低いレイテンシ、および I/O 集約型ワークロードに対する優れたコストパフォーマンスを提供する。容量・IOPS・スループットをそれぞれ独立して調整可能であり、ワークロードの特性に応じたきめ細かなパフォーマンスチューニングが実現する。

Key Points

  • 最大 80,000 IOPS (従来の 4 倍)
  • 容量・IOPS・スループットを独立して調整可能
  • 最大 64 TiB のディスク容量
  • HA, Geo冗長バックアップ, CMK 等のエンタープライズ機能対応

Before

  • 最大 IOPS 20,000、最大スループット 900 MB/s
  • 固定 SKU でサイズとIOPSが紐づき
  • IOPS・スループット個別調整不可
  • 最大 32 TiB

After

  • 最大 80,000 IOPS、1,200 MB/s
  • 1 GiB 単位でサイズ設定可能
  • 容量・IOPS・スループット独立調整
  • 最大 64 TiB

⚡

柔軟なパフォーマンス調整

容量・IOPS・スループットをそれぞれ独立して設定可能。24 時間以内に最大 4 回のパフォーマンス変更が可能で、ワークロードの変動に応じてリアルタイムにパフォーマンスを最適化できる。

🎁

無料ベースライン IOPS

399 GiB 以下のディスク: 3,000 IOPS / 125 MB/s が無料で提供。400 GiB 以上のディスク: 12,000 IOPS / 500 MB/s が無料で提供。ベースラインを超えた分のみ追加課金。

📏

きめ細かなストレージサイズ

1 GiB 単位でストレージサイズを設定可能 (1 GiB - 64 TiB)。従来の Premium SSD のような固定 SKU サイズの制約がなく、無駄のないリソースプロビジョニングを実現。

🏢

エンタープライズ機能フルサポート

High Availability (HA)、Geo 冗長バックアップ、Geo レプリカ、メジャーバージョンアップグレード、カスタマーマネージドキー (CMK)、Geo DR (ディザスタリカバリ) をサポート。

📈

大幅なパフォーマンス向上

Premium SSD 比で IOPS 4 倍 (20,000 → 80,000)、スループット約 1.3 倍 (900 → 1,200 MB/s)、最大容量 2 倍 (32 → 64 TiB)。サブミリ秒のレイテンシを実現。

Use Cases

UC1: 高頻度トランザクション処理

EC サイトやフィンテックアプリケーションなど、大量のトランザクションを処理するデータベースで、高い IOPS と低レイテンシが求められるケース。

Implementation:
  • Premium SSD v2 で最大 80,000 IOPS をプロビジョニング
  • サブミリ秒レイテンシで高負荷トランザクション処理を実現
  • HA 構成で高可用性を確保

-- 従来の Premium SSD (最大 20,000 IOPS) では対応困難だった高負荷処理を実現。

UC2: ピーク負荷変動ゲームアプリ

ゲームバックエンドのデータベースで、イベント時やピーク時間帯にのみ高い IOPS が必要になるケース。

Implementation:
  • ピーク時に IOPS を動的に引き上げ
  • オフピーク時に IOPS を下げてコスト最適化
  • 24 時間以内に最大 4 回のパフォーマンス調整が可能

-- 動的パフォーマンス調整により、ピーク対応とコスト最適化を両立。

UC3: 大規模データウェアハウス

数十 TiB のデータを保持する分析用データベースで、大容量ストレージと高スループットが求められるケース。

Implementation:
  • 最大 64 TiB のディスク容量を活用
  • 最大 1,200 MB/s のスループットで大規模データセットの読み書きを効率化
  • 1 GiB 単位のサイズ設定で無駄のないプロビジョニング

-- Premium SSD の 32 TiB 制限を超える大容量と高スループットを実現。

Pricing

3 要素による従量課金

Premium SSD v2 の料金は、ストレージ容量、プロビジョニングされた IOPS、およびプロビジョニングされたスループットの 3 つの要素で構成される。無料ベースラインを超えた分のみ追加課金。

Limitations / Caveats

  • ストレージ自動拡張 (autogrow) 非対応 -- 手動でのディスク容量管理が必要
  • Burstable ティア非対応 -- General Purpose / Memory Optimized のみ
  • PostgreSQL バージョン 13 非対応 -- バージョン 14 以降が必要
  • 長期保存バックアップ非対応 -- Long-term backup は現時点で未サポート
  • オンライン移行不可 -- Premium SSD からの移行は PITR またはレプリカが必要
  • ストレージスケールダウン不可 -- 増やすことはできるが減らすことはできない
  • パフォーマンス変更は24時間で最大4回 -- 新規作成ディスクは最初の24時間で3回
  • Geo冗長バックアップとCMKの同時使用非対応 -- それぞれ個別には対応
