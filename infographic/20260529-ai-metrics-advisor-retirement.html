Azure AI Metrics Advisor: サービス廃止

時系列データの異常検知を AI で自動化するサービスが完全に廃止。代替サービスへの移行が必要。

Retirement 2026-05-29

概要

Azure AI Metrics Advisor が 2026 年 5 月 18 日をもって正式に廃止された。Metrics Advisor は、時系列データの監視と異常検知を AI を活用して自動化するサービスであり、機械学習の知識がなくてもデータの取り込み、異常検知、診断が可能であった。Microsoft は代替サービスとして Azure Monitor、オープンソースの Anomaly Detector、および Microsoft Fabric の多変量異常検知機能を推奨している。

要点

  • 2026 年 5 月 18 日にサービスが完全廃止 (当初予定の 10 月 1 日より前倒し)
  • 2023 年 9 月 20 日以降、新規リソースの作成は不可であった
  • 代替として Azure Monitor、OSS Anomaly Detector、Microsoft Fabric を推奨
  • 現在利用中の場合は早急に移行が必要

❌ 廃止されたサービス

  • Metrics Advisor による時系列異常検知
  • 専用ポータルでの可視化と診断
  • 多次元メトリクスの自動監視
  • 根本原因分析とリアルタイム通知

✅ 推奨される代替手段

  • Azure Monitor KQL ベースの異常検知
  • OSS Anomaly Detector (セルフホスト)
  • Microsoft Fabric 多変量異常検知
  • Azure Data Explorer の時系列分析

廃止タイムライン

代替サービス比較

📊

Azure Monitor

KQL の機械学習オペレーターで時系列分析と異常検知。既存の Log Analytics ワークスペースを活用可能。

🔓

OSS Anomaly Detector

Metrics Advisor と同じアルゴリズムをセルフホスト。クラウド依存度を低減し完全な制御を実現。

🧠

Microsoft Fabric

GAT ベースの多変量異常検知。リアルタイムストリーミングと高度な相関分析に対応。

廃止されたサービスの主要機能

🔍

時系列データの異常検知

多次元メトリクスの自動監視、最適な検知モデルの自動選択 (機械学習の知識不要)、パラメータチューニングとインタラクティブフィードバックを提供。

🔔

リアルタイム通知

メール、Webhook、Teams、Azure DevOps のフックに対応した柔軟なアラート設定。

🧩

スマート診断インサイト

特定のディメンションへの根本原因分析とメトリクスグラフによるクロスメトリクス分析。

🔌

多様なデータソース接続

SQL Server、Azure Blob Storage、MongoDB 等のデータソースに接続し、データを自動取り込み。

移行例: Azure Monitor KQL 異常検知

Azure Monitor の Log Analytics で KQL の機械学習オペレーターを使用して時系列分析と異常検知を行う。主な KQL 関数: make-series、series_decompose_anomalies()、diffpatterns()。

KQL Azure Monitor Log Analytics 
// 時系列の作成と異常検知の例
let starttime = 21d;
let endtime = 0d;
let timeframe = 1d;
Usage
| where TimeGenerated between (startofday(ago(starttime))..startofday(ago(endtime)))
| make-series ActualUsage=sum(Quantity) default = 0
    on TimeGenerated step timeframe by DataType
| extend(Anomalies, AnomalyScore, ExpectedUsage)
    = series_decompose_anomalies(ActualUsage)
| mv-expand ActualUsage to typeof(double),
    TimeGenerated to typeof(datetime),
    Anomalies to typeof(double)
| where Anomalies != 0

ユースケース別推奨移行先

AIOps (IT 運用の異常検知)

サーバーメトリクス (CPU、メモリ、ディスク I/O) の異常を検知し、障害を早期発見する

推奨: Azure Monitor (KQL 異常検知) - 既にメトリクスが収集されている場合、追加のデータ移動なしに異常検知を実行可能

IoT システムの多変量監視

複数センサーの値を相関分析し、個別には正常範囲内でも組み合わせとして異常なパターンを検知する

推奨: Microsoft Fabric 多変量異常検知 - GAT ベースのアルゴリズムにより変数間の相関関係を学習

ビジネスメトリクスの監視

売上、トランザクション数、ユーザーアクティビティなどのビジネス KPI の異常を監視する

推奨: Azure Monitor またはオープンソース Anomaly Detector

移行時の注意点

  • Metrics Advisor のような専用 UI (ポータル) は代替サービスには存在しない
  • Azure Monitor の KQL ベースの異常検知は単変量分析が中心であり、多次元メトリクスの自動相関分析とは異なるアプローチが必要
  • Microsoft Fabric の多変量異常検知は Fabric ライセンスが別途必要
  • オープンソース版はセルフホストのためインフラ管理が必要

関連サービス

Azure のネイティブ監視サービス。KQL による時系列分析と異常検知機能を提供。

⚡
