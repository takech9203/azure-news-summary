📝 概要

Azure Functions の Durable Task Scheduler に Consumption SKU が一般提供 (GA) として追加されました。Durable Task Scheduler は、Durable Functions および Durable Task SDK のオーケストレーション状態を永続化するためのマネージドバックエンドサービスです。従来の BYO (Bring Your Own) ストレージプロバイダー (Azure Storage、MSSQL など) に代わる専用バックエンドとして構築されています。

Consumption SKU は従量課金 (Pay-per-use) モデルを採用し、アクション単位での課金となります。初期費用、最低利用コミットメント、基本料金が不要で、アイドル時のコストも発生しません。

Before

  • BYO ストレージプロバイダーの手動管理が必要
  • キャパシティプランニングやパーティション管理のオーバーヘッド
  • Dedicated SKU のみで小規模ワークロードでも固定費発生

After

  • ストレージ管理完全不要、自動永続化
  • 実行アクション分のみ課金、アイドルコストゼロ
  • 500 アクション/秒まで自動スケール

⭐ 主要機能

💰

従量課金 (Pay-per-use)

アクション単位での課金。初期費用、最低コミットメント、基本料金なし。アイドル時のコストゼロを実現し、使った分だけ支払うシンプルなモデル。

📦

フルマネージドストレージ

ストレージアカウントの管理不要。オーケストレーション状態はサービス内部で自動的に永続化。インメモリストアと永続ストアの組み合わせによる最適化。

🚀

プッシュベースのワークアイテム配信

gRPC 接続によるプッシュモデルでワークアイテムをストリーミング。ポーリング不要でエンドツーエンドのレイテンシを削減。

📊

Durable Task Scheduler ダッシュボード

オーケストレーションとエンティティインスタンスの監視・管理。フィルタリング、詳細確認、サブオーケストレーションの追跡。一時停止・終了・再起動などの管理操作に対応。

🔐

マネージド ID 認証

接続文字列ではなくマネージド ID による認証のみサポート。RBAC によるきめ細かいアクセス制御で高いセキュリティを確保。

🔧 技術仕様

項目 詳細
最大アクション/秒 500
データ保持期間 最大 30 日
スケジューラー数上限 10 (リージョンあたり、サブスクリプションあたり)
タスクハブ数上限 5 (リージョンあたり、サブスクリプションあたり)
最大ペイロードサイズ 1 MB

エンドポイント形式 {scheduler-name}.{region}.durabletask.io

💻 セットアップ (Azure CLI)

bash 
# Durable Task Scheduler CLI 拡張機能のインストール
az extension add --name durabletask
az extension update --name durabletask

# Consumption SKU でスケジューラーを作成
az durabletask scheduler create \
  --name "YOUR_SCHEDULER" \
  --resource-group "YOUR_RESOURCE_GROUP" \
  --location "LOCATION" \
  --ip-allowlist "[0.0.0.0/0]" \
  --sku-name "consumption"

# タスクハブの作成
az durabletask taskhub create \
  --resource-group YOUR_RESOURCE_GROUP \
  --scheduler-name YOUR_SCHEDULER \
  --name YOUR_TASKHUB

💡 ユースケース

AI エージェントオーケストレーション

マルチステップの AI エージェントワークフローを実行。各ステップで外部 API 呼び出しやモデル推論を行い、結果に基づいて次のアクションを決定。ワークロードは不定期に発生するため、アイドル時のコストゼロにより不定期に実行される AI エージェントワークフローのコストを最小化。

開発・テスト環境

Durable Functions アプリケーションの開発・テストで、ストレージインフラの管理を避けたい場合に最適。ストレージアカウントの管理不要で、すぐに開発開始可能。使用した分のみ課金。

イベント駆動型ワークフロー

EC サイトのプロモーションイベント時にオーダー処理のオーケストレーションが急増し、通常時はほぼゼロになる変動的なワークロード。従量課金によりピーク時のみコスト発生。

⚠️ 制約事項・注意点

  • 最大スループットが 500 アクション/秒 (Dedicated SKU は 2,000 アクション/秒/CU)
  • データ保持期間が最大 30 日 (Dedicated SKU は 90 日)
  • スケジューラー数がリージョンあたり 10 個まで
  • タスクハブ数がリージョンあたり 5 個まで
  • Extended Sessions 機能は未対応
  • マネージド ID 認証のみサポート (接続文字列は不可)
  • 最大ペイロードサイズが 1 MB に制限
