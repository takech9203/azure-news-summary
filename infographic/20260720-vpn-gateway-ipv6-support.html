Azure VPN Gateway

IPv6 デュアルスタックサポートの一般提供 (GA)

VPN トンネル内で IPv6 インナートラフィックをデュアルスタック構成で実行可能に

Launched GA 2026-07-20

概要

Azure VPN Gateway における IPv6 サポートが一般提供 (GA) として正式にリリースされました。VPN トンネル内で IPv6 インナートラフィックをデュアルスタック構成で実行することが可能になり、スタンダードパブリック IP を使用するすべてのプロダクション SKU ゲートウェイでサポートされます。

Before

  • VPN Gateway 経由のトラフィックは IPv4 のみに制限
  • オンプレミスの IPv6 ネットワークと Azure VNet 間で IPv6 トラフィックを直接転送する手段がなかった
  • IPv6 移行を進める企業では、VPN 接続部分が IPv4 ボトルネック

After

  • VPN トンネル内で IPv6 インナートラフィックをネイティブに転送可能
  • デュアルスタック (IPv4 + IPv6) 構成によりシームレスな移行パスを提供
  • サイト間、ポイント対サイト、VNet 間のすべての主要接続タイプで IPv6 に対応
  • Standard パブリック IP を使用するプロダクション SKU (VpnGw1AZ - VpnGw5AZ) で利用可能

主要機能

🌐

IPv6 デュアルスタック VPN トンネル

IPv4 と IPv6 を同時に転送可能。既存の IPv4 通信を維持しながら、IPv6 トラフィックをネイティブにサポートするデュアルスタック構成を実現します。

🔗

複数接続タイプのサポート

S2S (IKEv2)、P2S (IKEv2, OpenVPN)、VNet-to-VNet のすべての主要接続タイプで IPv6 デュアルスタックに対応します。

🏢

プロダクション SKU 対応

VpnGw1AZ から VpnGw5AZ まで、Standard パブリック IP を使用するすべてのプロダクション SKU で利用可能です。

🛠

Azure Portal / PowerShell / CLI 対応

Azure Portal、PowerShell、Azure CLI の複数ツールで構成可能。既存のワークフローに統合しやすい管理体験を提供します。

技術仕様

サポート SKU VpnGw1AZ, VpnGw2AZ, VpnGw3AZ, VpnGw4AZ, VpnGw5AZ
非対応 SKU Basic SKU
パブリック IP 要件 Standard SKU パブリック IP アドレス
サポートプロトコル (S2S) IKEv2 (IKEv1 は非サポート)
サポートプロトコル (P2S) IKEv2, OpenVPN (SSTP は非サポート)
IPv6 トラフィック範囲 インナートラフィックのみ
VPN タイプ Route-based
デュアルスタック→IPv4 のみへの変更 不可 (不可逆的な変更)
IPv6 デュアルスタック構成による追加料金なし

料金は SKU に基づく時間料金 + データ転送料金で構成されます。IPv6 デュアルスタック構成を有効化しても追加のコストは発生しません。

ユースケース

1. IPv6 移行中の企業のハイブリッド接続

オンプレミスで IPv6 移行を進めている企業が、Azure との VPN 接続でもデュアルスタックを活用するケース。

デュアルスタック構成で移行期間中の運用複雑性を低減し、段階的な IPv6 移行を実現

2. IPv6 ネイティブ IoT デバイスのリモートアクセス

IPv6 アドレスのみを持つ IoT デバイスに対して、P2S VPN 経由でセキュアなリモートアクセスを提供するケース。

IKEv2 / OpenVPN プロトコルで IPv6 ネイティブデバイスへ直接アクセス可能

3. マルチリージョン VNet 間のデュアルスタック接続

複数の Azure リージョンにまたがる VNet 間で、IPv6 を含むデュアルスタック通信を実現するケース。

VNet-to-VNet 接続で IPv6 通信をネイティブにサポートし、グローバル展開を簡素化

制限事項・注意点

  • 外部トンネルの IPv6 非対応 - インナートラフィックのみ IPv6 をサポート
  • Basic SKU 非対応 - Standard パブリック IP を使用するプロダクション SKU のみ
  • SSTP プロトコル非対応 - P2S では IKEv2 または OpenVPN を使用
  • IKEv1 非対応 - S2S 接続では IKEv2 のみサポート
  • 不可逆的な変更 - デュアルスタック構成を有効化すると IPv4 のみに戻すことは不可
  • 非 AZ SKU の廃止予定 - 今後は AZ 対応 SKU への移行を推奨
