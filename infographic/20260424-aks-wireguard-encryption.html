WireGuard In-Transit Encryption

概要

AKS で WireGuard プロトコルによるインターポッド転送中暗号化が一般提供 (GA) となった。Azure CNI powered by Cilium と Advanced Container Networking Services (ACNS) を使用するクラスターで利用可能であり、ノード間を移動するポッド間トラフィックを透過的に暗号化する。アプリケーション側の変更は一切不要で、eBPF データプレーンと統合された WireGuard が自動的に暗号化を処理する。

アップデート前の課題

  • ノード間ポッドトラフィックがデフォルトで暗号化されていなかった
  • 仮想ネットワーク暗号化は同一物理ホスト上のノード間トラフィックを暗号化できない制約
  • WireGuard 暗号化はプレビュー段階であり、本番利用が困難
  • マルチクラウド環境で一貫した暗号化が困難

アップデート後の改善

  • WireGuard 暗号化が GA、本番ワークロードで正式サポート
  • ノード間のポッドトラフィックが透過的に暗号化
  • 同一物理ホスト上のノード間トラフィックも保護
  • VM SKU 非依存のソフトウェアレベル暗号化

主要機能

🔒

透過的なノード間ポッドトラフィック暗号化

アプリケーションの変更は一切不要。eBPF データプレーンと統合された WireGuard が、ノード間を移動するすべてのポッドトラフィックを自動的に暗号化する。既存のワークロードにそのまま適用可能。

🔑

自動鍵管理

WireGuard の暗号化鍵は 120 秒ごとに自動ローテーションされ、メモリ内にのみ保持される。管理者による手動の鍵管理は不要で、セキュリティを継続的に維持する。

🔗

専用ネットワークインターフェース (cilium_wg0)

各ノードに cilium_wg0 という専用の WireGuard ネットワークインターフェースが作成され、暗号化トンネルのエンドポイントとして機能する。Cilium Agent が WireGuard Agent と連携してインターフェースを管理。

🌎

クラウド非依存の暗号化ソリューション

WireGuard はクラウドプロバイダーに依存しないオープンソースの暗号化プロトコル。マルチクラウド環境やハイブリッド環境でも一貫した暗号化ポリシーを適用可能。

Azure CLI コマンド例

Bash - ACNS を有効にして新規 AKS クラスターを作成
az aks create \
    --resource-group <リソースグループ名> \
    --name <クラスター名> \
    --network-plugin azure \
    --network-plugin-mode overlay \
    --network-dataplane cilium \
    --advanced-networking-observability-enabled \
    --location <リージョン>
Bash - 既存クラスターで ACNS を有効化
az aks update \
    --resource-group <リソースグループ名> \
    --name <クラスター名> \
    --advanced-networking-observability-enabled

ユースケース

💳 ユースケース 1: 金融サービスにおける PCI DSS コンプライアンス対応

クレジットカード情報を処理する金融サービスのマイクロサービスアーキテクチャにおいて、PCI DSS の転送中暗号化要件を満たす必要がある。WireGuard 暗号化により、ノード間のポッドトラフィックを透過的に暗号化し、アプリケーション側の TLS 実装に依存せずにネットワークレベルでのコンプライアンスを確保する。

効果: アプリケーションの変更なしにネットワークレベルでの転送中暗号化が実現され、PCI DSS 監査要件への対応が簡素化される。

🏥 ユースケース 2: ヘルスケア業界における患者データ保護 (HIPAA)

電子カルテ (EHR) システムを AKS 上で運用するヘルスケア企業において、HIPAA の技術的安全措置として転送中の患者データを暗号化する必要がある。WireGuard により、マイクロサービス間で送受信される患者データがノードを跨ぐ際に自動的に暗号化される。

効果: 患者データの転送中保護が自動化され、HIPAA の技術的安全措置要件を効率的に満たすことができる。

🌐 ユースケース 3: マルチクラウド環境での統一暗号化

Azure と他のクラウドプロバイダーの両方で Kubernetes クラスターを運用する企業において、クラウドプロバイダー固有の暗号化メカニズムに依存せず、統一的な暗号化ポリシーを適用したい。WireGuard はクラウド非依存のプロトコルであるため、マルチクラウド環境全体で一貫した暗号化アプローチを実現できる。

効果: クラウドプロバイダーに依存しない一貫した暗号化ポリシーにより、マルチクラウド運用のセキュリティ管理が統一される。

制約事項

  • FIPS 非準拠 - FIPS 140-2/140-3 認証を必要とする環境では使用不可
  • 同一ノード内のポッドトラフィックは暗号化されない (ノード間トラフィックのみが対象)
  • ホストネットワーキング (hostNetwork: true) を使用するポッドでは非対応
  • パフォーマンス影響あり - MTU 1500 で約 1.5 Gbps、MTU 3900 で約 3.75 Gbps がスループット上限
  • Azure CNI powered by Cilium および ACNS の有効化が必須
