概要

Azure Databricks Lakebase の一般提供 (GA) が発表された。Lakebase は、Databricks Data Intelligence Platform に統合されたフルマネージドの PostgreSQL データベースサービスであり、Lakehouse にオンライントランザクション処理 (OLTP) 機能を提供する。従来の分析ワークロードに加えて、リアルタイムのトランザクショナルアプリケーションを同一プラットフォーム上で構築可能となる。次世代のストレージとコンピュートの分離アーキテクチャを採用しており、オートスケーリング、Scale-to-Zero、ブランチ (インスタントクローン)、インスタントリストアといった高度な機能群が利用可能。PostgreSQL 16/17 互換で、既存アプリケーションからシームレスに接続できる。

Before (課題)
  • Lakehouse 上に OLTP のマネージドソリューションが存在しない
  • 環境ごとに個別の DB サーバーを運用 (高コスト・管理負荷)
  • コンピュートとストレージが密結合で独立スケール困難
  • DB クローン作成に時間がかかり開発テストが遅延
After (改善)
  • 分析ワークロードと OLTP を統合的に管理可能
  • Copy-on-Write でインスタントブランチ (変更分のみ課金)
  • Scale-to-Zero で非稼働時のコストをゼロに削減
  • オートスケーリングで手動リソース管理が不要

Lakebase アーキテクチャ: ストレージとコンピュートの分離、Copy-on-Write による共有ストレージレイヤー

主要機能

オートスケーリング

ワークロードの需要に応じてコンピュートリソースが自動スケール。CPU 負荷、メモリ使用量、ワーキングセットサイズの 3 メトリクスを監視。ユーザー定義の min/max CU 範囲内 (最大 32 CU) で動作し、ダウンタイムや接続中断なしでスケーリングが実行される。

Scale-to-Zero

非アクティブ状態が続くとコンピュートが自動サスペンド。デフォルト 5 分 (最小 60 秒)。サスペンド中のコンピュートコストはゼロ。新しいクエリ到着時に数百ミリ秒で自動再アクティベーション。データ、接続文字列、認証情報はサスペンド中も保持される。

ブランチ (インスタントクローン)

Git のブランチに類似したデータベース環境の分岐機能。Copy-on-Write 技術により DB サイズに関係なく瞬時にブランチを作成。変更データのみストレージコストが発生 (例: 100 GB DB で 1 GB 変更 = 追加 ~1 GB)。親子階層構造をサポートし、ブランチリセットで親の最新状態を即時反映。

インスタントリストア (ポイントインタイムリカバリ)

設定したリストアウィンドウ内 (0 - 30 日) の任意の時点にデータベースを復元可能。誤ったデータ削除やスキーマ変更からの迅速な復旧。監査やコンプライアンス対応のための過去データへのアクセス。

Read Replica

同一ストレージレイヤーを共有する読み取り専用コンピュートエンドポイント。データ複製なしで瞬時に作成可能。読み取りワークロードのスケールアウトに有効。

Data API (PostgREST 互換)

HTTP 経由でデータベースに直接アクセス可能な RESTful インターフェース。PostgREST 互換の API で、既存のツールやフレームワークと統合可能。

Unity Catalog 統合

Lakebase データベースを Unity Catalog に登録可能。Delta Lake テーブルとの Reverse ETL (Delta -> Postgres 同期) および Query Federation によるクロスデータソースクエリをサポート。

ユースケース

AI エージェント / リアルタイムアプリ

AI エージェントやチャットボットのバックエンド DB として利用。業務時間帯は高トラフィック、夜間・週末はほぼアクセスなし。Production Branch はオートスケーリング (2-16 CU) で常時稼働、Development Branch は Scale-to-Zero 有効。

トラフィックスパイクに自動対応しつつ、非稼働時コストゼロ。本番データを使ったテスト環境を瞬時に構築可能。

マイクロサービスの OLTP 層

マイクロサービスごとに独立した DB が必要なアーキテクチャ。各サービス (users / orders / inventory) を個別の Project として管理し、Unity Catalog で全 Project を統合管理。Query Federation で分析ワークロードからクロスサービスクエリを実行。

独立した PostgreSQL DB を持ちつつ、統合的なデータガバナンスを実現。分析チームが全データを横断参照可能。

ブランチベースの CI/CD

PR ごとに production ブランチからインスタントブランチを作成 -> マイグレーション適用 -> 自動テスト -> マージ後に本番へ適用 -> テスト用ブランチ削除。本番データの完全コピーに対してスキーマ変更を事前検証。

ブランチ作成は DB サイズに関係なく瞬時。Scale-to-Zero によりテスト中以外のコンピュートコストはゼロ。

制約事項・注意点

高可用性 (Readable Secondaries) 未サポート - Autoscaling 版では利用不可。必要な場合は Provisioned 版を選択
Compliance Security Profile 未対応 - Autoscaling 版ではコンプライアンスセキュリティプロファイルがサポートされていない
Postgres -> Delta 同期が Autoscaling 版で未サポート - Provisioned 版では Private Preview として利用可能
Feature Store / Stateful AI Agent 連携が Autoscaling 版で未サポート - これらの統合は Provisioned 版のみ
Billing タグ / Budget Policy が Autoscaling 版で未サポート - コスト管理のきめ細かい制御に制限あり
Scale-to-Zero 再アクティベーション時のセッションリセット - インメモリ統計、一時テーブル、準備済みステートメント、セッション設定がリセットされる
Provisioned / Autoscaling 版間の直接マイグレーション不可 - pg_dump/pg_restore または Reverse ETL が必要
Databricks Apps UI 統合が Autoscaling 版で未サポート - 接続認証情報を手動で設定する必要がある
