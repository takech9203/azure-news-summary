Azure Files Launched (GA) Security

Managed Identity Support for SMB Access

2026年5月14日 リリース | Microsoft Build 2026

概要

Azure Files が SMB アクセスにおけるマネージド ID (Managed Identity) 認証の一般提供 (GA) を開始。アプリケーションやサービスが静的な資格情報やストレージアカウントキーを保存することなく、Azure Files SMB ファイル共有に対して認証を行うことが可能になった。

Zero Trust 原則に準拠: ワークロードが Microsoft Entra ID が発行するトークンを使用して Azure Files にアクセスでき、静的な資格情報の管理が不要に。

Before / After

❌ Before

  • SMB アクセスにストレージアカウントキーや静的資格情報が必要
  • AD DS や Microsoft Entra Domain Services の複雑な構成が前提条件
  • アプリケーションコード内に資格情報を保持する必要あり
  • Kerberos チケット取得のための複雑なインフラ設定が必要

✅ After

  • マネージド ID による資格情報不要の認証が可能
  • Zero Trust 原則に準拠した認証フローでセキュリティ向上
  • トークンベース認証で資格情報ローテーションが自動化
  • 追加のインフラ構成なしに SMB ファイル共有へアクセス

マネージド ID を使用した Azure Files SMB 認証フロー。ワークロードがマネージド ID を通じて Microsoft Entra ID からトークンを取得し、そのトークンを使用して SMB ファイル共有にアクセスする。

主要機能

🔑

マネージド ID による SMB 認証

Azure リソースに割り当てられたマネージド ID (システム割り当ておよびユーザー割り当て) を使用して、SMB プロトコル経由で Azure Files にアクセス。ストレージアカウントキーや静的資格情報の保存が不要。

🛡️

Zero Trust 原則への準拠

「信頼しない、常に検証する」の原則に基づき、すべてのアクセスでトークンベースの認証を実施。最小権限の原則に沿ったアクセス制御が可能。

🎫

Microsoft Entra トークンベース認証

Microsoft Entra ID が発行する OAuth 2.0 トークンを使用した認証フロー。トークンの有効期限とリフレッシュがプラットフォームにより自動管理される。

技術仕様

項目 詳細

メリット

ビジネス面

  • 静的資格情報の漏洩リスクを排除し、セキュリティインシデントの可能性を大幅に低減
  • Zero Trust 原則への準拠により、セキュリティ監査やコンプライアンス要件への対応が容易
  • 資格情報のローテーション管理やシークレット管理の運用負担が軽減

技術面

  • マネージド ID のライフサイクルは Azure プラットフォームが管理し手動管理が不要
  • ドメインコントローラーやドメイン参加の構成が不要でシンプル
  • シークレット取得コードが不要になりアプリケーションのセキュリティ表面積が縮小

ユースケース

Azure VM 上のアプリケーションからのファイルアクセス

Azure VM 上で動作するアプリケーションが、設定ファイルやデータファイルを Azure Files SMB 共有から読み書きする必要がある。従来はストレージアカウントキーをアプリケーション設定に保存していた。

✅ マネージド ID により、VM にシステム割り当て ID を有効化するだけで、キーレスでファイル共有にアクセス。資格情報の漏洩リスクがゼロに。

コンテナワークロードからの共有ストレージアクセス

Azure Kubernetes Service (AKS) や Azure Container Apps で動作するコンテナが、複数のインスタンス間で共有するファイルストレージとして Azure Files を使用する。

✅ ポッドに割り当てられたマネージド ID を使用して、シークレットやキーなしに SMB マウントが可能。Kubernetes Secret にストレージキーを保存する必要がなくなる。

Zero Trust アーキテクチャの実現

組織全体で Zero Trust セキュリティモデルを導入しており、すべてのサービス間通信から静的な資格情報を排除する方針を策定している。

✅ Azure Files へのアクセスにおいても資格情報レスの認証が実現し、組織の Zero Trust 戦略を完全に適用できる。

制約事項・注意点

  • Azure リソース上で動作するワークロードのみが対象 (オンプレミスのクライアントからはマネージド ID を直接使用できない)
  • マネージド ID は Azure リソース単位で割り当てられるため、アクセス制御の粒度に注意が必要
  • 既存のファイル共有で AD DS ベースの認証を使用している場合、マネージド ID 認証への移行計画が必要
