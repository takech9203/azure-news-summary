🔄 概要: Before / After 比較

Azure DevOps の 2026 年 3 月アップデートでは、AI 統合の強化、セキュリティの改善、ボード表示の最適化、アクセシビリティの向上など、開発者体験を大幅に改善する複数のアップデートが提供されます。

Before

  • MCP Server の利用にはローカルサーバーのインストールと管理が必要
  • ビルドサービス ID による API アクセスのセキュリティ管理が不十分
  • Kanban/Sprint ボードでフィールドやタグが多い場合、画面スペース不足
  • Team Calendar のカラーパレットやコントラストが見づらい

After

  • Remote MCP Server で Streamable HTTP トランスポート、ローカルサーバー管理不要
  • Microsoft Foundry 統合で GitHub Copilot Chat から Azure DevOps データアクセス
  • ビルドサービス ID のセキュリティ強化、サービスプリンシパルへの移行推奨
  • コンデンスドビューにより効率的に作業項目を確認
🏗️ アーキテクチャ図: Remote MCP Server 統合

Remote MCP Server により、開発者は Visual Studio / VS Code の GitHub Copilot Chat から直接 Azure DevOps のサービスにアクセスできるようになります。ローカルサーバーのインストールは不要です。

✨ 主要アップデート
🤖

Remote MCP Server (パブリックプレビュー)

ホスト型 MCP Server によりローカルインストールが不要に。Streamable HTTP トランスポートを使用し、Microsoft Foundry 統合と GitHub Copilot Chat 連携で AI を活用した Azure DevOps との統合が実現します。

🔒

ビルドサービス ID のセキュリティ強化

ビルドサービス ID の API アクセスに制限が導入され、過剰な権限のリスクを軽減。サービスプリンシパルへの移行が推奨されています。Advanced Security: Read alerts 権限の付与が必要です。

📋

Kanban / Sprint ボードのコンデンスドビュー

フィールドやタグが多い場合でもコンパクトな表示で一覧性が向上。限られた画面スペースでも効率的に作業項目を確認できます。

🎨

Team Calendar ビジュアル改善

カラーパレットの更新とコントラスト改善により視認性が向上。アクセシビリティも大幅に強化されています。

🔧

Azure DevOps Server パッチ

グループメンバーシップの非アクティブ化問題を修正するパッチがリリースされました。オンプレミス環境を運用中の場合は適用を推奨します。

💡 ユースケース
🗣️

AI 支援によるプロジェクト管理

GitHub Copilot Chat から Azure Boards の作業項目を自然言語で検索・更新できます。Remote MCP Server を介して、コードエディタから離れることなくプロジェクト管理タスクを実行可能です。

🔐

セキュアな CI/CD パイプライン

ビルドサービス ID からサービスプリンシパルへ移行することで、最小権限の原則に基づいた CI/CD パイプラインを構築。セキュリティリスクを軽減しつつ、パイプラインの自動化を維持できます。

🧪

Remote MCP Server はパブリックプレビュー

プロダクション環境での使用は推奨されません。ローカル MCP Server と比較して機能差がある可能性があります。

🔄

ビルドサービス ID の API 制限は一時ロールバック

ビルドサービス ID の API アクセス制限は一時的にロールバックされていますが、今後再適用される予定です。早期にサービスプリンシパルへの移行を計画してください。

📦

プレビュー版の機能ギャップ

Remote MCP Server のプレビュー版では、ローカル MCP Server と比較して一部の機能が利用できない場合があります。
