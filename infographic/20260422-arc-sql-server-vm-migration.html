Azure Arc Database

SQL Server on Azure Virtual Machines を移行ターゲットとして追加

Azure Arc の統合マイグレーションエクスペリエンスに IaaS 型移行先を追加、PaaS / IaaS を単一 UI で選択可能に

In Preview 2026-04-22

概要

Azure Arc のマイグレーション機能に SQL Server on Azure VM が新たな移行ターゲットとして追加された（パブリックプレビュー）。これまで Azure SQL Managed Instance のみだった移行先に、IaaS 型の SQL Server on Azure VM が加わり、同一の統合マイグレーションエクスペリエンスで実行可能になった。Azure Database Migration Service (DMS) との統合により、オンライン移行（最小ダウンタイム）およびオフライン移行の両方をサポートする。

Key Points

  • Azure Arc の「Database Migration」ペインから SQL Server on Azure VM を直接選択可能
  • Azure SQL Managed Instance と SQL Server on Azure VM の両ターゲットに統一 UI で対応
  • オンライン移行（最小ダウンタイム）とオフライン移行の両モードをサポート
  • Azure Database Migration Service (DMS) との統合による自動化された移行ワークフロー

Before

  • Azure Arc を通じた SQL Server の移行ターゲットは Azure SQL Managed Instance のみに限定
  • SQL Server on Azure VM への移行は Azure Arc の統合エクスペリエンスとは別に DMS を個別構成する必要
  • IaaS 型ターゲットでは移行ツールが分散し管理が複雑化

After

  • Azure Arc の「Database Migration」ペインから SQL Server on Azure VM を直接選択可能
  • Azure SQL Managed Instance と SQL Server on Azure VM の両方に対応した統一的マイグレーション
  • 評価・選択・実行・監視・カットオーバーを単一インターフェースで完結
  • ワークロード互換性要件に応じて最適な移行先を柔軟に選択可能

Key Features

🔄

SQL Server on Azure VM への統合マイグレーション

Azure Arc の「Database Migration」ペインから直接選択可能。既存の Managed Instance 移行と同じ UI / ワークフローで IaaS 型ターゲットへの移行を実行できる。

🔍

移行先の評価と推奨

ソース SQL Server の構成やワークロード特性を自動評価。Managed Instance vs Azure VM の最適推奨と互換性問題の事前検出を行う。

⚡

オンライン移行とオフライン移行のサポート

オンライン: 継続的リストアで最小ダウンタイム。オフライン: バックアップ/リストア方式。SHIR 経由 SMB 共有または Blob Storage 直接対応。

🎯

Azure Portal からのエンドツーエンドの移行管理

評価 → 実行 → 監視 → カットオーバーを一元管理。ステータス追跡により移行進捗をリアルタイムで確認可能。

Migration Workflow

1

Azure Arc エージェントのインストールと SQL Server の登録

オンプレミスの SQL Server に Azure Arc Agent (Azure Connected Machine Agent) をインストールし、Azure Extension for SQL Server を有効化する。

2

Azure Portal で「Database Migration」ペインを開く

Arc 対応 SQL Server のリソースページから「Database Migration」セクションに移動し、移行ウィザードを起動する。

3

移行先ターゲットの選択

Azure SQL Managed Instance または SQL Server on Azure VM のいずれかを移行先として選択する。ワークロード評価結果に基づき最適なターゲットを推奨。

4

移行モードとバックアップ設定の構成

オンライン(最小ダウンタイム)またはオフライン移行を選択。バックアップ格納先として Azure Blob Storage または SMB 共有(SHIR 経由)を設定する。

5

移行の実行と監視

DMS が自動でバックアップの転送とリストアを実行。Portal からリアルタイムで移行ステータスを追跡できる。

6

カットオーバーの実行

オンライン移行の場合、最終同期後にカットオーバーを実行して移行を完了。アプリケーション接続先を新しいターゲットに切り替える。

Use Cases

UC1: 高い互換性要件を持つレガシーアプリケーションの移行

SQL Agent ジョブ、リンクサーバー、CLR アセンブリなど、Managed Instance で制限がある機能に依存するアプリケーションの移行。

Implementation:
  • Azure Arc のワークロード評価で Managed Instance との非互換項目を検出
  • SQL Server on Azure VM を移行先として選択し、オンプレミス同等環境を実現
  • SQL Agent ジョブやリンクサーバーの設定をそのまま移行可能

-- IaaS 環境でオンプレミス同等の互換性を維持しつつクラウドのメリットを享受。

UC2: 段階的なクラウドモダナイゼーション

まずリフト&シフトで Azure VM に移行し、将来的に PaaS 化を検討する段階的アプローチ。

Implementation:
  • 第 1 段階: SQL Server on Azure VM にオンライン移行（最小ダウンタイム）
  • 第 2 段階: Azure VM 上で動作検証とパフォーマンスチューニング
  • 第 3 段階: 準備が整い次第、Azure SQL Managed Instance へ再移行

-- リスクを最小化しながら段階的にクラウドネイティブ化を推進。

UC3: マルチクラウド / エッジ環境からの集約

Azure Arc で管理された複数拠点の SQL Server を、Azure VM に集約・統合するケース。

Implementation:
  • 各拠点の Arc 対応 SQL Server から統一 UI で移行を管理
  • 複数データベースを同一 Azure VM インスタンスに集約
  • 集約後のパフォーマンスを Azure Monitor で統合監視

-- 分散した SQL Server 環境を Azure VM に統合し、運用管理コストを大幅に削減。

Constraints / Limitations

  • パブリックプレビューのため SLA 対象外
  • SQL Server on Azure VM は IaaS のため、OS パッチ適用やバックアップ管理の運用責任がユーザーに
  • SHIR 構成が必要な場合、追加ネットワーク要件あり
  • TDE 保護データベースの移行には事前に証明書移行が必要
  • Managed Instance Link 方式は Azure VM 移行では利用不可の可能性
