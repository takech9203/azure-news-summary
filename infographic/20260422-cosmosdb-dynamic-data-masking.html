Dynamic Data Masking (DDM) の一般提供開始

サーバーサイドのポリシー設定で機密データを保護、アプリケーションコード変更不要

Launched (GA) 2026-04-22

概要

Azure Cosmos DB の Dynamic Data Masking (DDM) が一般提供 (GA) となった。DDM はサーバーサイドのポリシーベースのセキュリティ機能であり、クエリ実行時にリアルタイムでデータをマスキングする。格納されたデータ自体は一切変更されず、ユーザーの RBAC 権限に応じて自動的にマスク済みデータまたは元データが返される。アプリケーションコードの変更は不要で、コンテナーレベルのポリシー設定のみでデータ保護を実現できる。

Key Points

  • サーバーサイドのリアルタイムマスキング
  • 3種類のマスキング戦略 (Default, Custom String, Email)
  • RBAC 連携による権限ベースの自動マスキング
  • アプリケーションコード変更不要

Before

  • アプリケーション側でマスキングロジック実装が必要
  • ユーザー権限に応じたデータ表示制御がサーバーサイドで不可
  • コンプライアンス対応に追加開発工数

After

  • サーバーサイドのポリシー設定のみで実現
  • RBAC連携で権限レベルに応じた自動マスキング
  • 複数戦略の柔軟な組み合わせ

Masking Policy Configuration

{
  "dataMaskingPolicy": {
    "includedPaths": [
      { "path": "/" },
      { "path": "/profile/contact/email", "strategy": "Email" },
      {
        "path": "/employment/history/[]/company",
        "strategy": "MaskSubstring",
        "startPosition": 2,
        "length": 4
      }
    ],
    "excludedPaths": [
      { "path": "/id" },
      { "path": "/department" }
    ]
  }
}

Use Cases

UC1: 医療データの保護

HIPAA コンプライアンス要件に対応し、患者の個人情報や医療記録をマスキングするケース。

Implementation:
  • 患者名・住所・電話番号に Default マスキングを適用
  • メールアドレスに Email 戦略を適用
  • 医師・管理者には unmask 権限を付与
  • 一般スタッフはマスク済みデータのみ参照可能

-- HIPAA コンプライアンスをサーバーサイドで自動的に実現し、開発工数を大幅削減。

UC2: IoT デバイスデータの機密保護

IoT デバイスから収集されるメトリクスは表示しつつ、個人情報やデバイスオーナー情報はマスクするケース。

Implementation:
  • デバイスメトリクス (温度・湿度等) のパスを excludedPaths に設定
  • オーナー情報・位置情報に Custom String マスキングを適用
  • 運用チームには部分的な unmask 権限を付与

-- デバイスメトリクスは全スタッフが確認でき、個人情報は権限に応じて自動保護。

UC3: マルチテナント SaaS

サポートチームが顧客データを確認する際に、機密情報を部分的にマスクして表示するケース。

Implementation:
  • 顧客のクレジットカード情報・SSN に Default マスキングを適用
  • メールアドレスに Email 戦略で部分表示
  • サポートチームは読み取り専用 (マスク済み)、管理者は unmask 権限付与

-- サポート業務に必要な情報は確認可能、機密データは権限レベルに応じて自動制御。

Limitations / Caveats

  • NoSQL API のみ (他 API 非対応)
  • 有効化後の無効化不可
  • アカウントキー認証非対応
  • 配列の特定インデックスへのマスキング非対応
  • Change Feed は unmask 権限なしユーザーには利用不可
  • Fabric ミラーリング・Analytical Store はデフォルト非対応
