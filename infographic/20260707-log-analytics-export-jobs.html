Log Analytics Export Jobs
(Public Preview)

Log Analytics ワークスペースから過去に蓄積された履歴データを Azure Blob Storage にエクスポートできる新機能。KQL クエリと時間範囲を指定して、必要なデータのみを Parquet 形式でオンデマンドエクスポートできる非同期ジョブ機能。数百テラバイト規模のデータを並列処理で効率的にエクスポートする設計。

📋 概要
Log Analytics ワークスペースに蓄積された大量の履歴データを、KQL クエリで柔軟にフィルタリングしながら Azure Blob Storage に Parquet 形式でエクスポートする非同期ジョブ機能。独立したコンピュートリソースで並列処理を行うため、ワークスペースのパフォーマンスに影響を与えず、数百テラバイト規模のデータを効率的にエクスポート可能。

Before

  • 過去の履歴データを大規模にエクスポートする専用の仕組みがなかった
  • Data Export Rules は新規到着データのみ対象
  • REST API や Logic Apps では大規模エクスポートに適していなかった
  • PowerShell スクリプトではスケーラビリティに制約

After

  • KQL クエリと時間範囲で選択的エクスポート可能
  • Parquet 形式で出力、分析ツールとの連携容易
  • 数百テラバイト規模の並列エクスポート
  • ワークスペースのパフォーマンスに影響しない独立コンピュート
  • リトライ機構によるデータ完全性確保

KQL クエリで指定したデータを並列ビン処理で Parquet 形式に変換し、Azure Blob Storage に出力。出力されたデータは各種分析基盤で活用可能。

⭐ 主要機能
⚡

KQL クエリベースのフィルタリングエクスポート

テーブル全体だけでなく、KQL クエリで特定条件のレコードのみをエクスポート。必要なデータだけを効率的に抽出し、ストレージコストを最適化。

📦

Parquet 形式での出力

列指向フォーマットで高い圧縮率とクエリ性能を実現。Azure Data Explorer、Synapse Analytics、Fabric など主要分析ツールとの連携が容易。

🚀

大規模データの並列エクスポート

数百テラバイト規模に対応。独立したコンピュートリソースでワークスペースのパフォーマンスに影響を与えずに大量データを処理。

🔄

リトライ機構によるデータ完全性

失敗時の自動リトライにより確実にデータをエクスポート。重複防止メカニズムで一貫性のあるデータ出力を保証。

💾

複数ストレージアカウント対応

1 ジョブあたり最大 10 ストレージアカウントを指定可能。大規模データの分散配置やリージョン別出力に対応。

💻 サンプルコード

Azure CLI を使用した Export Job の作成例:

bash Azure CLI - REST API 
az rest --method post --url "$url" --body '{
  "startTime": "2026-01-01T00:00:00",
  "endTime": "2026-03-01T00:00:00",
  "query": "CommonSecurityLog",
  "destinationStorageAccounts": ["<storage-account-resource-id>"],
  "containerName": "export-container",
  "outputDataFormat": "Parquet"
}'
💡 ユースケース

⚖ 法的コンプライアンス・監査対応

監査機関から特定期間のセキュリティログ提出要求への対応。KQL クエリで必要な期間とログタイプを指定し、Parquet 形式で効率的にエクスポート。監査要件に応じた柔軟なデータ抽出が可能。

効果: 従来は PowerShell スクリプトで数日かかっていた大規模ログの抽出が、並列処理により大幅に短縮。フォーマット変換も不要で監査対応の工数を削減。

🧠 機械学習モデルのトレーニングデータ準備

過去のパフォーマンスログや異常検知データを ML モデルの学習データとして活用。Parquet 形式で出力されるため、Spark や Azure Machine Learning から直接読み込み可能。

効果: 大規模な履歴データを列指向フォーマットで取得でき、データサイエンティストが分析に集中可能。ETL パイプラインの構築工数を削減。

🔍 セキュリティインシデント調査

サードパーティ SIEM でのフォレンジック分析用に、特定期間のセキュリティイベントをエクスポート。KQL で関連するイベントのみを抽出し、外部ツールでの詳細分析に活用。

効果: インシデント発生時に迅速に関連ログを外部分析基盤に提供。独立コンピュートで実行されるため、調査作業が本番環境のパフォーマンスに影響しない。
