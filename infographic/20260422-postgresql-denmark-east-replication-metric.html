Denmark East GA + logical_replication_slot_sync_status メトリックのパブリックプレビュー

Launched (GA) In Preview 2026-04-22

概要

Azure Database for PostgreSQL Flexible Server に関する 2 つのアップデートが発表された。1 つ目は Denmark East リージョンでの一般提供開始 (GA) で、デンマーク国内でのデータレジデンシー要件を満たすデプロイが可能になった。2 つ目は logical_replication_slot_sync_status メトリックのパブリックプレビューで、高可用性 (HA) 構成における論理レプリケーションスロットの同期状態を Azure Monitor で監視できるようになり、フェイルオーバー時のデータ一貫性リスクを事前に評価可能になった。

Key Points

  • Denmark East で PostgreSQL Flexible Server デプロイ可能に
  • 論理レプリケーションスロット同期ステータスメトリック追加
  • Azure Monitor でのフェイルオーバー安全性監視
  • PostgreSQL 17 以上でネイティブスロット同期対応

Before

  • デンマーク国内でデータレジデンシー要件を満たすには近隣リージョンにデプロイ必要
  • HA構成での論理レプリケーションスロット同期確認手段なし
  • フェイルオーバー時のデータ一貫性リスク

After

  • Denmark East で直接デプロイ可能
  • メトリック (1=同期済み、0=未同期) で監視可能
  • フェイルオーバーの安全性を事前評価可能

Denmark East リージョン GA

コペンハーゲンにデプロイ可能になり、デンマーク国内のデータレジデンシー要件に対応。Intel Compute ベースの PostgreSQL Flexible Server を利用可能。

📊

論理レプリケーションスロット同期メトリック

HA 構成のスロット同期状態を Azure Monitor で監視。logical_replication_slot_sync_status メトリックにより、プライマリとスタンバイ間のスロット同期状態をリアルタイムで把握可能。

⚡

PostgreSQL 17 ネイティブスロット同期

sync_replication_slots によるフェイルオーバー後のスロット自動保持。PostgreSQL 17 以上で利用可能なネイティブ機能により、フェイルオーバー後も論理レプリケーションスロットが自動的に維持される。

🔔

アラート連携

同期が途切れた際の即時通知が可能。スロット名ディメンション別に監視でき、特定のスロットの同期状態をきめ細かく追跡・アラート設定できる。

🔧

簡単有効化

metrics.collector_database_activity を on にするだけでメトリック収集を開始。サーバーの再起動は不要で、既存の運用環境にも即座に適用可能。

Use Cases

UC1: デンマーク国内のデータレジデンシー対応

デンマークの規制要件に準拠し、国内にデータを保持する必要があるケース。

Implementation:
  • Denmark East リージョンに PostgreSQL Flexible Server をデプロイ
  • Intel Compute ベースのインスタンスを選択
  • データレジデンシーポリシーに準拠したバックアップ構成

-- デンマーク規制準拠のデータ保管を実現し、コンプライアンスリスクを排除。

UC2: HA 構成でのフェイルオーバー安全性監視

高可用性構成で論理レプリケーションを使用しており、フェイルオーバー前にスロットの同期状態を確認したいケース。

Implementation:
  • metrics.collector_database_activity を on に設定
  • logical_replication_slot_sync_status メトリックを Azure Monitor で監視
  • 値が 0 (未同期) の場合にアラートを発報するルールを設定

-- メトリックアラートで即時通知し、フェイルオーバーの安全性を事前に評価可能。

UC3: 論理レプリケーションの運用監視

複数の論理レプリケーションスロットの同期状態を一元的に可視化し、運用効率を高めたいケース。

Implementation:
  • Azure Monitor からメトリックデータを取得
  • スロット名ディメンションでフィルタリング
  • Grafana ダッシュボードに統合して可視化

-- Grafana ダッシュボード統合で運用チームの監視効率を大幅に向上。

Limitations / Caveats

  • メトリックはデフォルト無効、手動有効化必要
  • ネイティブスロット同期は PostgreSQL 17 以上のみ
  • 論理レプリケーションの DDL 自動レプリケーション非対応
  • 未使用スロット放置で WAL ログ蓄積リスク
