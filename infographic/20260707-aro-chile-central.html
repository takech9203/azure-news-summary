Azure Red Hat OpenShift Launched (GA)

Azure Red Hat OpenShift:
Chile Central リージョンで一般提供開始

2026年7月7日 | Microsoft 初のチリにおける Azure リージョン

📋概要

Azure Red Hat OpenShift (ARO) が Azure Chile Central リージョンで一般提供 (GA) を開始。Microsoft 初のチリにおける Azure リージョンであり、南米地域での OpenShift デプロイメントのリージョン可用性が拡大された。

ARO はフルマネージドの OpenShift サービスであり、Red Hat と Microsoft が共同でエンジニアリング、運用、サポートを提供する。顧客は仮想マシンの運用やパッチ適用を行う必要がなく、コントロールプレーン、インフラストラクチャ、アプリケーションノードは Red Hat と Microsoft が代わりにパッチ適用、更新、監視を行う。

🔄Before / After

❌ Before

  • 南米でARO利用時、ブラジル南部など他リージョンが必要でチリからのレイテンシーが高い
  • チリ国内のデータレジデンシー要件を満たすためにAROを利用できなかった
  • 南米西海岸にフルマネージドOpenShiftサービスを提供するリージョンがなかった

✅ After

  • チリ国内でフルマネージドのOpenShiftクラスターを直接デプロイ可能
  • チリ国内のデータレジデンシー要件に準拠したコンテナワークロードの実行が可能
  • 南米西海岸地域のユーザーに低レイテンシーでサービスを提供可能

Azure Red Hat OpenShift は Chile Central リージョン内の仮想ネットワークにデプロイされ、Red Hat と Microsoft が共同でクラスターの管理・運用を行う。

⚡主要機能

🔧

フルマネージドの OpenShift クラスター

コントロールプレーン、インフラストラクチャ、アプリケーションノードのパッチ適用・更新・監視を Red Hat と Microsoft が代行。仮想マシンの運用が不要。

🔐

Microsoft Entra ID との統合

統合されたサインオンエクスペリエンスと Kubernetes RBAC によるアクセス制御を提供。

🌐

柔軟なデプロイオプション

既存の仮想ネットワークへのデプロイ、可用性ゾーン間の自動分散、カスタムドメインの指定が可能。

🛡️

エンタープライズグレードの SLA

99.95% の可用性 SLA を提供。高い信頼性でミッションクリティカルなワークロードに対応。

🚀設定方法 (Azure CLI)

前提条件: Azure CLI 2.67.0 以上、最低 44 コアの vCPU クォータ、必要なリソースプロバイダーの登録、Contributor および User Access Administrator 権限

bash Azure CLI 
# リソースプロバイダーの登録
az provider register --namespace Microsoft.RedHatOpenShift --wait
az provider register --namespace Microsoft.Compute --wait
az provider register --namespace Microsoft.Storage --wait
az provider register --namespace Microsoft.Authorization --wait

# 変数の設定
LOCATION=chilecentral
RESOURCEGROUP=aro-chile-rg
CLUSTER=my-aro-cluster
VIRTUALNETWORK=aro-vnet

# リソースグループの作成
az group create \
  --name $RESOURCEGROUP \
  --location $LOCATION

# 仮想ネットワークの作成
az network vnet create \
  --resource-group $RESOURCEGROUP \
  --name $VIRTUALNETWORK \
  --address-prefixes 10.0.0.0/22

# マスターノード用サブネットの作成
az network vnet subnet create \
  --resource-group $RESOURCEGROUP \
  --vnet-name $VIRTUALNETWORK \
  --name master-subnet \
  --address-prefixes 10.0.0.0/23

# ワーカーノード用サブネットの作成
az network vnet subnet create \
  --resource-group $RESOURCEGROUP \
  --vnet-name $VIRTUALNETWORK \
  --name worker-subnet \
  --address-prefixes 10.0.2.0/23

# クラスターの作成
az aro create \
  --resource-group $RESOURCEGROUP \
  --name $CLUSTER \
  --vnet $VIRTUALNETWORK \
  --master-subnet master-subnet \
  --worker-subnet worker-subnet

💡ユースケース

🏦 チリ国内のデータレジデンシー準拠

シナリオ: チリの金融機関がデータ主権要件に準拠しながら、コンテナベースのアプリケーションを運用する必要がある。

効果: Chile Central リージョンで ARO を利用することで、データがチリ国内に留まり、規制要件を満たしながらフルマネージドの OpenShift 環境でアプリケーションを実行できる。

🌐 南米西海岸向け低レイテンシーサービス

シナリオ: チリ、ペルー、コロンビアなど南米西海岸の顧客にサービスを提供する SaaS プロバイダーが、低レイテンシーでアプリケーションを配信したい。

効果: Chile Central リージョンにデプロイすることで、南米西海岸のエンドユーザーに対するレスポンス時間を短縮し、ユーザー体験を向上させることができる。

✨メリット

📈 ビジネス面

  • チリ国内のデータレジデンシー要件への準拠が可能
  • 南米西海岸地域の顧客への低レイテンシーサービス提供
  • Red Hat と Microsoft の統合サポートによるエンタープライズグレードの運用

⚙️ 技術面

  • フルマネージドによる運用負荷の軽減
  • 99.95% SLA による高可用性の保証
  • Microsoft Entra ID との統合による統一的なアイデンティティ管理
  • 既存の仮想ネットワークへのデプロイによる柔軟なネットワーク設計

⚠️制限事項・注意点

  • クラスター作成に最低 44 コアの vCPU クォータが必要（新規サブスクリプションのデフォルトでは不足する場合がある）
  • 同一クラスター内で複数の Azure リージョンにノードを配置することは不可
  • テナント間やサブスクリプション間のクラスター移動は非対応
  • サービスプリンシパルからマネージド ID への既存クラスターの移行は非対応（新規作成が必要）
  • コントロールプレーンノードの削除・置換・追加・変更はサポート対象外

💰料金情報

ARO の料金はアプリケーションノード（ワーカーノード）単位で課金される。コントロールプレーンノードおよびインフラストラクチャノードの ARO サービス料金は無料。基盤となる Azure インフラストラクチャ（VM、ストレージ、ネットワーク）の料金が別途発生する。Red Hat OpenShift のライセンス費用は ARO サービス料金に含まれている。

📝まとめ

Azure Red Hat OpenShift の Chile Central リージョンでの一般提供開始は、Microsoft 初のチリにおける Azure リージョン開設に伴うサービス拡張の一環。南米地域、特にチリを拠点とする企業にとって、データレジデンシー要件を満たしながらフルマネージドの OpenShift 環境を活用できる重要なアップデート。
推奨アクション:
  • チリまたは南米西海岸地域にワークロードがある場合、Chile Central リージョンへの ARO デプロイを検討
  • vCPU クォータの増加申請が必要な場合は事前に対応
  • 既存の他リージョンの ARO クラスターから移行を検討する場合、新規作成が必要であることに注意
