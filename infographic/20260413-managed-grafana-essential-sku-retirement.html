Azure Managed Grafana Retirement Notice

Essential SKU の廃止
Standard SKU / Azure Monitor Dashboards with Grafana への移行

Essential (プレビュー) SKU は廃止されます。早急な移行計画の策定が必要です。

2026-04-13
⚠️

Essential SKU は既に機能停止しています

2026年3月31日をもって Essential ティアのワークスペースは機能停止済みです。移行を行わない場合、2026年8月1日にワークスペースが削除されます。最終廃止日は 2027年3月30日です。

📋

概要

Azure Managed Grafana の Essential (プレビュー) SKU が 2027年3月30日に廃止されます。Essential SKU はもともと評価・テスト用途を想定したプレビュー段階のティアであり、SLA が提供されず、ゾーン冗長やプライベートエンドポイント、アラート機能などが利用できませんでした。Microsoft は移行先として Standard SKU へのアップグレード、または無料の Azure Monitor Dashboards with Grafana への移行を提供しています。

1

Essential SKU の廃止 - 評価・テスト用途のプレビューティアが廃止。新規ワークスペース作成は既に無効化済み。

2

Standard SKU へのインプレースアップグレード - Azure Portal から数ステップで変更可能。ダッシュボード、データソース構成、ユーザー設定がそのまま引き継がれる。

3

Azure Monitor Dashboards with Grafana - 無料の Azure ネイティブ代替サービス。Essential SKU と同等の機能を提供し、Azure Monitor との直接統合が可能。

Essential SKU から 2 つの移行パスが存在します。推奨はインプレースで Standard SKU にアップグレードする方法で、追加機能と SLA が提供されます。もう一方は、ダッシュボードをエクスポートして Azure Monitor Dashboards with Grafana にインポートする方法で、無料で利用できます。

移行オプション

オプション 1: Standard SKU へアップグレード 推奨
  • 既存ワークスペースをそのまま Standard SKU に変更
  • ダッシュボード、データソース、ユーザー設定を引き継ぎ
  • SLA 保証で可用性を担保
  • ゾーン冗長による耐障害性向上
  • アラート、レポーティング機能の追加
  • プライベートエンドポイント対応
  • 全コアプラグイン利用可能
  • Grafana Enterprise オプション
注意: Standard から Essential へのダウングレードは不可
オプション 2: Azure Monitor Dashboards with Grafana
  • 無料で利用可能
  • Essential SKU と同等の機能を提供
  • Azure Monitor (メトリクス、ログ、トレース、Prometheus) との直接統合
  • スポット VM に依存しない高可用性
  • ダッシュボードのエクスポート・インポートで移行
注意: プレビュー段階のため SLA なし。データソースの再構成が必要な場合あり。

ユースケース

🏗️

本番環境の監視ダッシュボード運用

Essential SKU で Azure Monitor のメトリクスを可視化する監視ダッシュボードを運用していた組織が、SLA とゾーン冗長が求められる本番環境で継続利用するケース。

推奨移行先: Standard SKU
SLA 保証とゾーン冗長により監視基盤の可用性が向上。アラート機能により閾値超過時の自動通知が可能になり、インシデント検知から対応までの時間が短縮されます。
💻

開発・テスト環境での軽量な監視

開発チームが Essential SKU を使って開発・テスト環境のメトリクスを確認していたが、コストを抑えつつ監視を継続したいケース。

推奨移行先: Azure Monitor Dashboards with Grafana
無料で Grafana ダッシュボードを継続利用でき、Azure Monitor との統合が維持されます。開発・テスト環境ではプレビュー段階のサービスでも許容可能なケースが多いです。
🔒

セキュリティ要件の高い環境での監視

プライベートネットワーク内のリソースを監視するため、プライベートエンドポイント経由でのアクセスが必要な環境。

推奨移行先: Standard SKU
プライベートエンドポイントと確定的送信 IP により、ネットワーク分離された環境でも Grafana ダッシュボードにセキュアにアクセスでき、データソースへの通信もファイアウォールで制御可能になります。

設定・移行手順

前提条件

  • Azure サブスクリプション
  • 既存の Azure Managed Grafana Essential ワークスペース
  • ワークスペースに対する所有者または共同作成者のロール

Azure Portal での手順

1

ワークスペースへ移動

Azure Portal で対象の Azure Managed Grafana ワークスペースに移動します。

2

価格プランを選択

左メニューの「設定」>「構成」>「価格プラン」を選択します。

3

Standard SKU を選択

「Standard SKU」を選択します。必要に応じてインスタンスサイズを X2 に変更します。

4

変更を保存

「保存」を選択して変更を適用します。

注意: Standard SKU から Essential SKU へのダウングレード、および大きいインスタンスサイズから小さいインスタンスサイズへの変更はサポートされません。

手順 1: ダッシュボードのエクスポート

Grafana UI を使用する場合:

1

Grafana UI でエクスポートするダッシュボードを開く

2

ダッシュボード上部の「Share」を選択する

3

表示されるダイアログで「Export」タブを選択する

4

「Save to file」を選択してダッシュボード JSON ファイルをダウンロードする

Azure CLI を使用する場合:

bash 
# ダッシュボード、データソース、フォルダーのバックアップ
az grafana backup --name <grafana-workspace-name>

# 詳細オプション: 特定のコンポーネントを指定フォルダーにバックアップ
az grafana backup \
    --name MyGrafana \
    --resource-group MyResourceGroup \
    --directory "/tmp/grafana-backup" \
    --folders-to-exclude "General" "Azure Monitor" \
    --components datasources dashboards folders

手順 2: Azure Monitor Dashboards with Grafana へのインポート

1

Azure Portal で Azure Monitor を開く

2

サービスメニューから「Dashboards with Grafana (プレビュー)」を選択し、「New」>「Import」を選択する

3

「Import dashboard from File」で、エクスポートした JSON ファイルをアップロードする

4

「Load」を選択する

5

ダッシュボード名を入力し、サブスクリプション、リソースグループ、リージョンを選択して作成する

移行のメリット

Standard SKU へのアップグレード

ビジネス面

  • SLA 保証でビジネスクリティカルな監視ダッシュボードの可用性を担保
  • ゾーン冗長でデータセンター障害時の耐障害性が向上
  • アラート機能で閾値ベースの自動通知が可能、インシデント対応の迅速化

技術面

  • プライベートエンドポイントで VNet 内からのセキュアアクセス
  • 確定的送信 IP でファイアウォール許可リスト設定が容易
  • Azure Data Explorer, GitHub, JSON API 等の追加プラグイン
  • SMTP 設定によるメール通知・レポート配信
  • Grafana Enterprise オプション

Azure Monitor Dashboards with Grafana への移行

💲

コスト削減

追加コストなしで Grafana ダッシュボードを利用可能。Azure Monitor との直接統合とスポット VM に依存しない高可用性が提供されます。

⚠️

制約事項・注意点

  • Standard SKU へのアップグレードは追加コストが発生する (Essential SKU は無料だったのに対し、Standard SKU は有料)
  • Standard SKU から Essential SKU へのダウングレードはサポートされない
  • Azure Monitor Dashboards with Grafana はプレビュー段階であり、SLA が提供されない
  • Azure Monitor Dashboards with Grafana への移行はダッシュボードの手動エクスポート・インポートが必要。データソースの再構成が必要になる場合がある
  • 2026年3月31日以降は Essential ワークスペースが機能停止するため、移行の時間的猶予は限られている
