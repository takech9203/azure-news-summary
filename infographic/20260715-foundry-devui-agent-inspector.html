DevUI Agent Inspector

エージェント実行をナビゲート可能なツリー構造で可視化するローカルデバッグサーフェス

2026-07-15 | Microsoft Foundry

📋 概要

Microsoft Agent Framework の DevUI に新機能「Agent Inspector」がパブリックプレビューとして追加された。Agent Inspector は、エージェントの実行をモデル呼び出し、ツール呼び出し、中間状態を含むツリー構造として可視化するローカルデバッグサーフェスである。一時停止 (Pause)、リプレイ (Replay)、プロンプト編集 (Edit Prompts Mid-Run) 機能により、エージェント開発のイテレーション速度を大幅に向上させる。

Agent Inspector は DevUI の Web インターフェース上で動作し、エージェント実行をツリー構造として可視化する

⭐ 主要機能

🌳

ナビゲート可能なツリービュー

エージェントの実行をモデル呼び出し、ツール呼び出し、中間状態のツリー構造として表示。各ノードをクリックして詳細な入出力データを確認し、マルチエージェントワークフローの実行パスを視覚的に追跡可能。

⏸

一時停止 (Pause) 機能

エージェントの実行を任意のポイントで一時停止し、中間状態やコンテキストの内容を詳細に検査。ステップバイステップでの実行制御が可能。

🔁

リプレイ (Replay) 機能

過去の実行を再生して動作を確認。特定のステップから再実行を開始し、異なるパスを探索可能。問題の再現と修正の検証に活用。

✏

プロンプト編集 (Edit Prompts Mid-Run)

実行中のプロンプトをその場で編集して即座に再実行。システムプロンプト、ユーザープロンプトの両方を編集可能。コード変更なしでプロンプトエンジニアリングのイテレーションを高速化。

📡

OpenTelemetry トレーシング統合

実行のトレースデータを OpenTelemetry 形式で記録。レイテンシ、トークン使用量などのメトリクスを可視化し、Grafana や Jaeger など既存の可観測性スタックと統合可能。

🛠 セットアップ

bash インストールと起動 
# DevUI のインストール (プレリリース版)
pip install agent-framework-devui --pre

# CLI による起動 (トレーシング有効)
devui ./agents --port 8080 --tracing
python プログラムによる起動 
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient
from agent_framework.devui import serve

agent = Agent(
    name='MyAgent',
    client=OpenAIChatClient(),
    tools=[]
)

serve(entities=[agent], auto_open=True)

💡 ユースケース

🔍 マルチツールエージェントのデバッグ

複数のツール (データベースクエリ、API 呼び出し、ファイル操作) を組み合わせたエージェントが期待通りの順序でツールを呼び出しているか検証する。

ツリービューでツール呼び出しの順序と各ステップの入出力を確認し、一時停止して中間状態を検査可能

📝 プロンプト最適化

エージェントのシステムプロンプトを調整し、回答の品質や一貫性を向上させたい場合に活用。

プロンプト編集機能により、コード変更・再起動なしで複数のプロンプトバリエーションを迅速に試行可能

🔀 ワークフロー実行パスの検証

グラフベースのマルチエージェントワークフローにおいて、条件分岐が正しく動作しているか確認する。

ツリービューで実行パスを可視化し、リプレイ機能で特定の分岐条件を変更した場合の動作を探索可能

⚠ 制限事項・注意点

  • Preview: パブリックプレビュー段階であり、本番環境での利用は非推奨
  • Python のみ: 現時点では Python のみ完全サポート (C# 準備中、Go 開発中)
  • ローカル限定: ローカル開発環境での使用を前提、リモートデバッグには追加構成が必要
  • 仕様変更の可能性: プレビュー期間中に機能仕様が変更される可能性あり
