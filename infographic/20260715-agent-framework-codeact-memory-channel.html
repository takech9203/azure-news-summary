Microsoft Agent Framework

CodeAct / Episodic Memory / Agent Channel (Public Preview)

Microsoft Agent Framework に 5 つのパブリックプレビュー機能が追加 -- 実行効率・記憶・通信・可観測性を強化

In Preview 2026-07-15

概要

Microsoft Agent Framework (Microsoft Foundry) に 5 つのパブリックプレビュー機能が追加された。CodeAct パターンは複数ステップのプランを単一の実行可能なコードブロックに集約しレイテンシを削減。Hyperlight コンテナは CodeAct が生成するコードのセキュアなサンドボックス実行環境を提供。エピソード手続き記憶はエージェントが成功したタスクパターンを記憶・再利用するための組み込みストアを実現。Agent Channel はカスタムルーティングコード不要でエージェント間の型付きメッセージ通信を可能にする。Tracing は OpenTelemetry ベースの可観測性をエージェントの推論とツール呼び出しに提供する。

Key Points

  • CodeAct パターン: マルチステッププランを単一コードブロックに集約し、ツール呼び出しのレイテンシを大幅に削減
  • Hyperlight コンテナ: CodeAct 生成コードを安全に実行するセキュアサンドボックス
  • エピソード手続き記憶: 成功したタスクパターンを自動記録し、次回以降の実行を高速化
  • Agent Channel: エージェント間の構造化メッセージ通信を組み込みで提供 (Request/Response, Fire-and-forget)
  • Tracing (Python/.NET): OpenTelemetry ベースのトレーシングで推論・ツール呼び出しを可視化

Before

  • ツールを 1 つずつ呼び出すため、往復のレイテンシが累積していた
  • エージェントが過去の成功パターンを記憶できず、毎回ゼロから推論していた
  • マルチエージェント通信にカスタムルーティングコードの実装が必要だった
  • エージェントの推論やツール呼び出しの可観測性が限定的だった

After

  • CodeAct が複数ステップを単一コードブロックに集約し、レイテンシを削減
  • エピソード手続き記憶が成功パターンを学習し、繰り返しタスクを高速化
  • Agent Channel が組み込みの型付きメッセージ通信を提供
  • OpenTelemetry トレーシングで推論・ツール呼び出しを完全に可視化

📝

CodeAct パターン

複数ステップのプランを単一の実行可能コードブロックに集約。ツール呼び出しの往復回数を削減し、レイテンシを大幅に改善。Python および .NET で利用可能。

🔒

Hyperlight コンテナ

CodeAct が生成したコードをセキュアに実行するためのサンドボックス環境。隔離された実行空間でコードを安全に実行し、ホスト環境を保護する。

🧠

エピソード手続き記憶

エージェントが成功したタスクパターンを自動的に記録・保存する組み込みストア。繰り返し実行されるタスクでは記憶されたパターンを再利用し、推論コストとレイテンシを削減。

🔗

Agent Channel

エージェント間の構造化されたメッセージ通信を実現。Request/Response および Fire-and-forget パターンをサポートし、カスタムルーティングコード不要で型付きメッセージ通信を提供。

📊

Tracing (Python/.NET)

OpenTelemetry ベースの可観測性をエージェントの推論とツール呼び出しに提供。Azure Monitor Application Insights にトレースデータを送信し、90 日間保持。

Use Cases

複雑なデータ分析タスク

CodeAct パターンにより複数のデータ処理ステップを単一コードブロックに集約し、ツール呼び出しの往復レイテンシを削減。エピソード手続き記憶が過去の成功パターンを記録し、繰り返し実行される分析タスクの処理時間を短縮する。

CodeAct Episodic Memory

マルチエージェントによる業務自動化

Agent Channel が構造化されたエージェント間通信を提供し、カスタムルーティングコードなしで複数エージェントの協調動作を実現。Request/Response パターンで同期的な委任、Fire-and-forget パターンで非同期的な通知を使い分け可能。
