概要

Azure SQL Managed Instance の Business Critical サービスティアにおいて、Flexible Memory 機能がパブリックプレビューとして提供開始。vCore 数を変更せずにインスタンスに割り当てるメモリ量を柔軟に調整でき、過剰プロビジョニングを回避しながらパフォーマンスを維持できる。

❌ Before

  • メモリ量が vCore 数に固定 (7 GB/vCore)
  • メモリ増加には不要な vCore 追加購入が必要
  • メモリ不要でも vCore 付随分のコスト発生

✅ After

  • vCore 数を変更せずにメモリ量を調整可能
  • ワークロードに応じた適正なメモリ配分
  • Azure Portal / REST API から設定変更可能

Business Critical ティアの 4 ノードクラスター構成において、各ノードのメモリ割り当てを vCore 数とは独立して調整可能。変更はインスタンス全体に適用され、最終操作としてフェイルオーバーが実行される。

主要機能

🔧

メモリの独立スケーリング

vCore 数を変更せずに、メモリ量を最小値 (7 GB/vCore) から最大値 (12 GB/vCore) の範囲で調整可能。vCore あたりのメモリ比率を柔軟に選択できる。

🔄

既存インスタンスへの適用

新規インスタンスだけでなく、既存の Business Critical インスタンスにも適用可能。Azure Portal の「Compute + Storage」ブレードまたは REST API で変更。

🌐

ゾーン冗長構成のサポート

ローカル冗長構成に加え、ゾーン冗長構成のインスタンスでも利用可能。Premium-series ハードウェアで動作。

設定方法 - REST API

bash REST API で memorySizeInGB を設定 
curl -X PATCH \
  "https://management.azure.com/subscriptions/{subscription-id}/resourceGroups/{resource-group}/providers/Microsoft.Sql/managedInstances/{instance-name}?api-version=2024-08-01-preview" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "properties": {
      "memorySizeInGB": 96
    }
  }'

ユースケース

💾 In-Memory OLTP ワークロードのメモリ増強

シナリオ: 8 vCores の Business Critical インスタンスで In-Memory OLTP を利用しているが、デフォルトの 56 GB では不足する場合

効果: vCore を増やさずにメモリを 96 GB まで拡張でき、CPU コストを抑えながらインメモリ処理の容量を確保

💰 コスト最適化のためのメモリ削減

シナリオ: 高い CPU 性能は必要だがメモリ使用量が少ないワークロード（多数の同時接続による軽量クエリ処理など）

効果: メモリを適正化することでコスト効率を改善。不要なリソースへの課金を回避

制限事項・注意点

⚠️

フェイルオーバーの発生

メモリ変更時にフェイルオーバーが発生するため、短時間の接続断が生じる

⚠️

対象ハードウェアの制限

Premium-series ハードウェアのみが対象。Standard-series / Gen5 は非対応

⚠️

大規模構成での制限

48 vCores を超える構成ではメモリの柔軟性が制限される (64 vCores 以上では固定)
