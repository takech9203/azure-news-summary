Azure Backup for Cosmos DB: Vault バックアップ

2026-06-02 Public Preview Microsoft Build 2026

📋 概要

Azure Backup for Cosmos DB が Public Preview として発表されました。Azure Cosmos DB アカウントに対して Vault (コンテナー) ベースのバックアップを有効化し、ミッションクリティカルなデータをセキュアで分離された耐障害性の高いバックアップで保護できるようになりました。既存の継続バックアップ (ポイントインタイムリストア) とは異なり、Azure Backup Vault にデータを保存する方式を採用。偶発的または悪意のある削除を含むシナリオにおいても、サイバーレジリエンスとコンプライアンスの要件を満たすことが可能です。ランサムウェア攻撃やインサイダー脅威からのデータ保護を強化します。

⚠️ Before

  • 継続バックアップは同一ストレージ内に保持、アカウント侵害時にバックアップも影響
  • ランサムウェアやインサイダー脅威に対する論理的分離が不足
  • 長期保持 (30日超) のバックアップを保持する標準的な方法がなかった
  • コンプライアンス要件で求められる Vault 分離型バックアップに非対応

✅ After

  • Azure Backup Vault にデータを分離保存、ソース侵害時もバックアップは安全
  • Vault 分離によりランサムウェアや悪意のある削除から保護
  • 長期保持ポリシー設定可能 (年次、月次、週次の保持ルール)
  • クロスサブスクリプションリストアに対応し災害復旧の柔軟性向上

Azure Cosmos DB のデータが Azure Backup Vault を経由して論理的に分離されたストレージに保存されます。リストア時は別の空の Cosmos DB アカウントにデータを復元します。

⭐ 主要機能

🛡

Vault ベースのバックアップ

Azure Backup Vault にデータを分離保存。ソース Cosmos DB アカウントとは独立した保護レイヤーを提供し、偶発的・悪意のある削除からデータを保護します。

📅

週次バックアップスケジュール

プレビュー期間中は週次バックアップのみサポート。RPO 7日間。オンデマンドバックアップでフルバックアップも実行可能です。

📋

柔軟な保持ポリシー

年次、月次、週次の保持ルールを設定可能。優先順位: 年次 > 月次 > 週次。デフォルト保持期間は1年です。

🔄

クロスサブスクリプションリストア

異なるサブスクリプションのターゲットアカウントへのリストアが可能。災害復旧や組織変更時の柔軟性を確保します。

💡 ユースケース

1

ランサムウェア対策

金融機関がランサムウェア攻撃で Cosmos DB データが暗号化・削除されても、Vault に保存された分離バックアップから復旧。Backup Vault のアクセス制御は独立しており、ソースアカウントの管理者権限が奪取されてもバックアップデータは保護されます。

2

コンプライアンス・長期保持

ヘルスケア業界の規制要件により患者データを7年間保持する必要がある場合、年次保持ルールで対応。既存の PITR (最大30日) では対応できない長期保持要件を Vault バックアップで充足できます。

3

災害復旧 (DR)

本番環境サブスクリプションで障害発生時に DR サブスクリプションの Cosmos DB へクロスサブスクリプションリストア。サブスクリプションレベルの障害でも別サブスクリプションへの復旧パスを確保します。

⚠ 制約事項

  • 週次バックアップのみ (日次非対応)
  • クロスリージョンリストア非対応
  • 継続バックアップ (PITR) モード有効が前提条件
  • リストア先は空の単一リージョン Cosmos DB アカウントが必要
  • 階層パーティションキー非対応
  • Per-Partition Automatic Failover (PPAF) 非対応
  • アイテムレベルのバックアップ/リストア非対応
