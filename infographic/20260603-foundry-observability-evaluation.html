評価ツールスイート

AI エージェントの品質測定・改善サイクルを開発者ワークフロー全体に統合し、本番環境での継続的品質保証を実現

Public Preview 2026-06-02 / 2026-06-03

概要

Microsoft Build 2026 において、Microsoft Foundry の Observability（可観測性）および評価機能に関する 5 つの新しいパブリックプレビューが発表された。エンドユーザーからのフィードバックをトレースに関連付けて記録する機能、ML パイプラインによるインテリジェントなトレースサンプリング、コンテキスト固有の評価基準を定義できるルブリック評価器、VS Code 内でのコードファースト Observability、Azure Developer CLI（azd）での評価体験の 5 つが含まれる。OpenTelemetry 標準に基づいた分散トレーシング基盤の上に構築され、Azure Monitor Application Insights と統合されている。

5 つの新機能 (Public Preview)

  • ユーザーフィードバックロギング - thumbs up/down、レーティング、カスタムアノテーションをトレースに関連付け
  • インテリジェントトレースサンプリング - ML パイプラインで代表的サブセットを選択し評価コスト削減
  • ルブリック評価器 - ドメイン固有のカスタム評価基準をシングルターン/マルチターンに適用
  • VS Code コードファースト Observability - Foundry Plugin "observe skill" でエディタ内評価ループ
  • Azure Developer CLI (azd) - CLI 内で評価体験、コード→デプロイ→評価→改善の一気通貫

Before

  • AI エージェントの品質評価は手動で、全トレースへの評価はコスト大
  • エンドユーザーフィードバックを構造化してトレースに紐付ける標準的方法がなかった
  • 評価基準がビルトイン指標に限定、ドメイン固有の基準定義が困難
  • Observability のセットアップにはポータルへの切り替えが必要で開発者ワークフロー分断

After

  • インテリジェントサンプリングでコストを抑えつつ品質監視
  • 構造化フィードバック (thumbs up/down、レーティング) をトレースに自動関連付け
  • ルブリック評価器でドメイン固有の評価基準をカスタム定義
  • VS Code / azd CLI から直接、評価駆動の最適化ループを実行可能

👍

1. ユーザーフィードバックロギング

エンドユーザーからの構造化フィードバックシグナルを自動的にトレースに関連付けて記録する機能。

  • thumbs up/down、レーティング（数値スケール）、カスタムアノテーションに対応
  • W3C Trace Context による分散トレース相関で元リクエストと紐付け
  • Azure Monitor Application Insights に統合保存
🧪

2. インテリジェントトレースサンプリング

マルチステージ ML パイプラインで代表的なサブセットを選択し、評価コストを削減しながら品質の全体像を正確に把握。

  • 本番トレース全量ではなく代表的サブセットのみ評価
  • LLM ジャッジモデルのトークン消費を大幅に削減
  • 継続的評価 (Continuous Evaluation) で本番品質を常時監視
📋

3. ルブリック評価器

開発者がコンテキスト固有の評価基準を定義できるカスタム評価器。ビルトイン評価器を補完。

  • シングルターンおよびマルチターンのエージェントフローに対応
  • ドメイン固有の品質基準を testing_criteria で柔軟に設定
  • ビルトイン評価器 (Task Adherence, Coherence, Violence 等) と組み合わせ可能
💻

4. VS Code コードファースト Observability

Foundry Plugin for VS Code Copilot Chat 内の "observe skill" として提供。エディタ内で完全な評価ループを実行。

  • トレース確認 → 評価実行 → 改善 → 再評価のサイクルをエディタ内で完結
  • ポータルへの画面切り替え不要で開発ワークフローを維持
  • Foundry Agents 開発者向けに最適化
⌨️

5. Azure Developer CLI (azd) での Observability 体験

azd CLI に評価体験を追加し、ホステッドエージェントのライフサイクルに品質計測ループを統合。

  • コード → デプロイ → 評価 → 改善のサイクルを CLI から一気通貫で実行
  • Microsoft Foundry で作成されたエージェントが対象
  • CI/CD パイプラインの品質ゲートとして統合可能

実装例: 継続的評価の設定

# SDK のインストール
# pip install "azure-ai-projects>=2.0.0"

import os
from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient

endpoint = os.environ["AZURE_AI_PROJECT_ENDPOINT"]
model_deployment = os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"]

credential = DefaultAzureCredential()
project_client = AIProjectClient(endpoint=endpoint, credential=credential)
client = project_client.get_openai_client()

# 評価定義の作成
data_source_config = {"type": "azure_ai_source", "scenario": "responses"}
testing_criteria = [
    {
        "type": "azure_ai_evaluator",
        "name": "Task Adherence",
        "evaluator_name": "builtin.task_adherence",
        "data_mapping": {
            "query": "{{item.query}}",
            "response": "{{sample.output_items}}",
        },
        "initialization_parameters": {"deployment_name": model_deployment},
    },
]

eval_object = client.evals.create(
    name="Continuous Evaluation",
    data_source_config=data_source_config,
    testing_criteria=testing_criteria,
)

Use Cases

1. カスタマーサポートエージェントの品質モニタリング

本番デプロイされたカスタマーサポート AI エージェントの応答品質を継続的に監視し、ユーザー満足度と品質スコアの相関を分析する。ルブリック評価器でドメイン固有の品質基準を定義し、継続的評価ルールで常時モニタリング。

品質スコアの低下をリアルタイムに検知し、Azure Monitor アラートと連携して即座に対応可能。ユーザーフィードバックとの相関分析により改善領域を特定。

2. CI/CD パイプラインでの評価駆動デプロイ

エージェントの変更をデプロイする前に、azd CLI を使用して自動評価を実行し、品質閾値（例: Task Adherence 85% 以上）を満たさない場合はデプロイをブロックする。

品質劣化を本番環境にリリースする前に検知し、安全なデプロイサイクルを確立。GitHub Actions との統合により、PR マージ時の品質ゲートとして機能。
