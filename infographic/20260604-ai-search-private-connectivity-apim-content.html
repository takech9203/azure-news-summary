Azure AI Search:
プライベート接続 GA、APIM 統合、Content Understanding チャンキング

エンタープライズ RAG パイプラインのセキュリティ、ガバナンス、ドキュメント処理能力を大幅強化

GA Public Preview 2026-06-04

概要

Microsoft Build 2026 において、Azure AI Search に関する 3 つの重要なアップデートが発表された。エンタープライズ向け RAG パイプラインのセキュリティ、ガバナンス、およびドキュメント処理能力を大幅に強化するもの。

Key Points

  • プライベート接続が一般提供 (GA) - Shared Private Link または Network Security Perimeter でプライベートネットワーク経由のルーティング
  • APIM サポート (パブリックプレビュー) - Foundry および Azure OpenAI モデルを APIM の背後に配置
  • Content Understanding スキル (パブリックプレビュー) - セマンティックチャンキングと画像言語化

Before

  • Azure AI Search と Foundry サービス間の通信がパブリックネットワーク経由で規制要件を満たせないケースがあった
  • 大規模 RAG パイプラインでモデルエンドポイントのアクセス制御やレート制限が困難だった
  • ドキュメントのチャンキングは固定サイズベースで段落境界やセマンティクスを考慮しない分割が行われていた

After

  • Shared Private Link と NSP で全トラフィックをプライベートチャネルで保護可能
  • APIM 経由でスロットリング、認証、監視、ポリシー適用を一元的に実施可能
  • セマンティックチャンキングで段落境界を尊重した分割、画像言語化で画像のAI記述をチャンク内に埋め込み

プライベート接続の一般提供 (GA)

エンタープライズ環境でのセキュアな RAG パイプライン構築を実現するプライベートネットワーク接続。

  • Shared Private Link: 特定リソースへの 1 対 1 のプライベートエンドポイント接続
  • Network Security Perimeter (NSP): 論理的なネットワーク境界内で暗黙的にプライベート通信を許可
  • サポートトラフィック: インジェスト、エンリッチメント、検索、エージェントトラフィック
🌐

APIM サポート (パブリックプレビュー)

Azure API Management を介したモデルエンドポイントの統合管理とガバナンス。

  • 統合ゲートウェイ管理
  • エンタープライズガバナンス (レート制限、認証・認可の集中管理)
  • Shared Private Link の Group ID "Gateway" で APIM への接続
🧩

Content Understanding チャンキングと画像言語化

AI を活用したインテリジェントなドキュメント分割と画像のテキスト変換。

  • セマンティックチャンキング: method=semantic, unit=tokens, maximumLength 100-8000
  • 画像言語化: ドキュメント内の画像・チャート・図が AI によるテキスト記述に変換

設定例

Content Understanding スキル定義の JSON 構成例。セマンティックチャンキングと画像抽出を組み合わせた設定。

{
  "skills": [
    {
      "@odata.type": "#Microsoft.Skills.Util.ContentUnderstandingSkill",
      "context": "/document",
      "modelName": "gpt-4.1",
      "modelDeployment": "myGpt41Deployment",
      "extractionOptions": ["images", "locationMetadata"],
      "chunkingProperties": {
        "method": "semantic",
        "unit": "tokens",
        "maximumLength": 500
      },
      "inputs": [
        {
          "name": "file_data",
          "source": "/document/file_data"
        }
      ],
      "outputs": [
        { "name": "text_sections", "targetName": "text_sections" },
        { "name": "normalized_images", "targetName": "normalized_images" }
      ]
    }
  ]
}

ユースケース

規制業界での RAG パイプライン

金融機関が社内ドキュメントの AI アシスタントを全プライベートネットワーク内で構築

  • Shared Private Link でインデクサーから Blob Storage/Cosmos DB へのプライベート接続
  • NSP で Azure AI Search と Foundry 間の暗黙的プライベート通信
  • APIM 経由でモデルエンドポイントへのアクセス制御とレート制限

大規模マルチモーダルドキュメント検索

製造業の図面や写真を含む技術文書を AI 検索の対象に

  • Content Understanding スキルで技術文書をセマンティックチャンキング
  • 図面・写真を画像言語化でテキスト記述に変換し検索可能に
  • マルチモーダルなドキュメントを統一的に検索インデックスに格納

制約事項

  • Shared Private Link はティアごとにリソース数上限あり
  • Content Understanding スキルは 5 分以上のドキュメントにはタイムアウト
  • セマンティックチャンキングと画像言語化は 2026-05-01-preview API のみ
  • NSP のデータソースは Blob Storage, Cosmos DB, SQL Database に限定
  • Content Understanding スキルには無料枠なし
