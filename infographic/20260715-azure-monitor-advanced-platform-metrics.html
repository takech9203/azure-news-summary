Azure Monitor

Advanced Platform Metrics

プラットフォームパフォーマンス、リソースヘルス、運用トレンドに対する高度な可視性を提供する新しいテレメトリ機能

Public Preview 2026-07-15

概要

Azure Monitor の Advanced Platform Metrics がパブリックプレビューとして利用可能になった。従来のプラットフォームメトリクスでは取得できなかった、より深いレベルのテレメトリデータにアクセス可能となり、Azure サービスのパフォーマンスとヘルスの可視性が大幅に強化される。データ収集ルール (DCR) を活用したメトリクスエクスポートとの組み合わせにより、柔軟かつ高精度な監視基盤を構築できる。

Before / After

Before

  • リソースの深い内部動作を把握する情報が限定的
  • リソースヘルスの詳細な変化を早期検知する手段が不十分
  • 複数サービスにまたがる運用トレンドの横断分析が困難
  • ディメンション付きメトリクスのエクスポートが制限

After

  • プラットフォームレベルの詳細テレメトリにアクセス可能
  • リソースヘルスと運用トレンドの可視性が強化
  • 対応サービス全体で一貫した高度なメトリクスを提供
  • DCR でディメンション付きエクスポートとフィルタリングが可能

主要機能

📈

拡張テレメトリデータ

従来のプラットフォームメトリクスでは利用できなかった、より深いレベルのパフォーマンス指標やリソースヘルス情報を提供。リソースの内部動作を詳細に理解できる。

💙

リソースヘルス監視の強化

リソースの健全性に関するより細かい粒度の情報を収集し、問題の予兆を早期に検知。プロアクティブな障害対応を支援する。

📊

運用トレンド分析

対応する Azure サービス全体にわたる運用傾向を横断的に分析。長期的なパフォーマンス変化やキャパシティ傾向を把握できる。

🚀

DCR メトリクスエクスポート

データ収集ルールにより、ディメンション付きメトリクスのエクスポートとフィルタリングが可能。約 3 分の低レイテンシで Log Analytics、Storage、Event Hubs に配信。

🌐

複数サービス対応

Azure Blob Storage、Storage Accounts をはじめとする対応サービスで利用可能。一貫した高度なメトリクスをプラットフォーム全体で提供。

設定例 (Azure CLI)

bash Azure CLI 
# データ収集ルールの作成
az monitor data-collection rule create \
  --name "myAdvancedMetricsDCR" \
  --resource-group "myResourceGroup" \
  --location "eastus" \
  --kind PlatformTelemetry \
  --identity "{type:'SystemAssigned'}" \
  --rule-file "./dcr-config.json"

# データ収集ルールの関連付け
az monitor data-collection rule association create \
  --name "myAssociation" \
  --rule-id "/subscriptions/{subId}/resourceGroups/{rg}/providers/Microsoft.Insights/dataCollectionRules/myAdvancedMetricsDCR" \
  --resource "/subscriptions/{subId}/resourceGroups/{rg}/providers/Microsoft.Storage/storageAccounts/{name}"

ユースケース

1

ストレージパフォーマンスの詳細監視

Azure Blob Storage や Storage Accounts のプラットフォームレベルのパフォーマンス指標を深く分析し、ボトルネックを特定する。

2

プロアクティブなリソースヘルス管理

リソースの健全性に関する詳細なメトリクスを活用して、障害の予兆を早期に検知しアラートを設定する。

3

運用トレンドに基づくキャパシティプランニング

長期的な運用トレンドを分析し、将来のリソース増強計画やコスト最適化に活用する。

4

クロスサービスの相関分析

複数の Azure サービスにまたがる Advanced Platform Metrics を Log Analytics に集約し、サービス間の影響を分析する。

⚠ 制限事項・注意点

  • パブリックプレビュー段階のため SLA は提供されない
  • プレビュー中に仕様が変更される可能性がある
  • 1 リソースあたり最大 5 つの DCR まで関連付け可能
  • DCR による時間粒度 (hourly grain) メトリクスのエクスポートは非対応
  • 1 つの DCR につき 1 つの送信先タイプのみ指定可能
