Azure Event Hubs GA Security Networking

Network Security Perimeter サポートの一般提供開始

リリース日: 2026-07-07 機能: Network Security Perimeter (NSP) サポート

🔍 概要

Azure Event Hubs における Network Security Perimeter (NSP) のサポートが一般提供 (GA) となった。Network Security Perimeter は、仮想ネットワーク外の PaaS リソースに対して論理的なネットワーク境界を定義し、パブリックネットワークアクセスを境界ベースのアクセスルールで制御する機能である。

複数の PaaS リソースにまたがる統一的なネットワークセキュリティポリシーを一元管理可能となり、境界内通信はデフォルトで許可、外部アクセスはデフォルトで拒否されることでデータエクスフィルトレーションを防止する。

Before (課題)

  • サービス単位のファイアウォール設定やプライベートエンドポイントに依存、複数サービスの管理が煩雑
  • データエクスフィルトレーション防止の統一的な境界制御が困難
  • 各サービスのネットワーク設定を個別管理、セキュリティポリシーの一貫性維持が困難

After (改善)

  • NSP により複数 PaaS リソースを論理的境界内に配置し一元的にネットワークアクセスを制御
  • 境界内通信はデフォルト許可、外部アクセスはデフォルト拒否でデータエクスフィルトレーション防止
  • プロファイルベースで環境別 (開発/ステージング/本番) のルール管理が可能

Network Security Perimeter は Event Hubs Namespace を含む PaaS リソースの周囲に論理境界を形成する。境界内のリソース間通信はデフォルトで許可され、外部からのアクセスはインバウンドルールで制御される。外部サービスへの通信はデフォルトで拒否される。

⭐ 主要機能

🔐 インバウンドアクセスルール

外部からの Event Hubs アクセスを IP アドレスベースまたはサブスクリプションベースで制御。許可されたクライアントのみが境界外から Event Hubs にアクセス可能。

📤 アウトバウンドアクセスルール

FQDN ベースで通信可能な外部リソースを定義。Event Hubs Capture のストレージ書き込みなど、必要な外部通信のみを明示的に許可。

📋 プロファイルベースの管理

環境別 (開発/ステージング/本番) のルールセットをプロファイルとして管理。異なる環境に対して適切なアクセスポリシーを個別に適用可能。

📊 診断ログ

ネットワークアクセス試行の監視とセキュリティイベント監査。インバウンド/アウトバウンドの許可・拒否をログに記録し、コンプライアンス対応を支援。

💡 ユースケース

ユースケース 1: Kafka ワークロードのセキュア化

Event Hubs の Kafka インターフェースに NSP でセキュリティ境界を確立し、Kafka プロトコル経由のアクセスを境界ベースで制御する。

Kafka ワークロードに対して統一的なネットワークセキュリティポリシーを適用し、不正アクセスを防止

ユースケース 2: Event Hubs Capture とストレージの統合

アウトバウンドルールで特定のストレージアカウントのみへのデータ書き込みを許可し、外部へのデータ流出を防止する。

Capture データの書き込み先を制限し、意図しない外部ストレージへのデータエクスフィルトレーションを防止

ユースケース 3: カスタマーマネージドキー (BYOK) 暗号化

Key Vault と Event Hubs を同一 NSP 内に配置し、暗号化キーへのアクセスをペリメーター内通信として安全に確保する。

Key Vault との通信がペリメーター内でデフォルト許可され、BYOK 暗号化をセキュアに運用可能

⚠ 制約事項・注意点

制約事項

  • Geo-disaster Recovery との非互換: Event Hubs の Geo-DR 機能は現時点で NSP に対応していない
  • SAS 認証の制限: 境界内通信で SAS 認証が動作しない。Microsoft Entra ID による認証が必須
  • 同一リージョン必須: NSP と Event Hubs は同一リージョンに配置する必要がある
  • Microsoft Sentinel との非互換: 現時点で Microsoft Sentinel との連携がサポートされない
