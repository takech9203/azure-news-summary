NFS v4.1 転送中暗号化 (Encryption in Transit)

Azure Files NFS 共有の通信を TLS で透過的に暗号化し、コンプライアンス要件を容易に達成

2026-07-17

📋 概要

Azure Kubernetes Service (AKS) において、Azure Files NFS v4.1 ボリュームの転送中暗号化 (Encryption in Transit: EiT) が一般提供 (GA) となった。Azure File CSI ドライバーを通じて、AKS ワークロードと Azure Files NFS 共有間で転送されるデータが TLS を使用して暗号化される。アプリケーションレベルの変更を必要とせず、データの機密性が確保される。

従来、Azure Files の NFS プロトコルでの接続は暗号化されておらず、仮想ネットワーク内のトラフィックは平文で送受信されていた。StorageClass パラメータに encryptInTransit: "true" を設定するだけで、NFS 通信が TLS で暗号化されるようになった。

Before

  • Azure Files NFS 共有への通信が暗号化されず平文で転送
  • 転送中のデータ暗号化にはアプリケーション側で独自実装が必要
  • コンプライアンス要件を満たすために SMB を選択せざるを得ないケースがあった

After

  • encryptInTransit: "true" を設定するだけで TLS 暗号化が有効
  • アプリケーションコードの変更不要で透過的に暗号化
  • NFS v4.1 のパフォーマンス特性を維持しつつセキュリティ要件を満たす

⭐ 主要機能

🔐

透過的な TLS 暗号化

Azure File CSI ドライバーが NFS v4.1 通信を自動的に TLS で暗号化。アプリケーション側での対応は一切不要で、既存のワークロードをそのまま暗号化通信に移行可能。

⚙

StorageClass パラメータによる簡易設定

encryptInTransit: "true" を設定するだけで有効化。複雑なネットワーク設定やサイドカーの追加は不要。

💾

既存 PV への適用

既存の PersistentVolume にも volumeAttributes で暗号化を有効化可能。新規作成だけでなく、運用中のボリュームにも適用できる。

📐 技術仕様

項目 詳細
プロトコル NFS v4.1
暗号化方式 TLS
CSI ドライバー file.csi.azure.com
最低 AKS バージョン 1.33 以降
サポート OS Linux (Ubuntu 22.04 以降)
非サポート OS Ubuntu 20.04、Windows ノード
ストレージ SKU SSD (PremiumV2_LRS, PremiumV2_ZRS, Premium_LRS, Premium_ZRS)

📝 設定方法 (StorageClass)

YAML StorageClass definition 
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: azurefile-csi-premiumv2-eit
provisioner: file.csi.azure.com
allowVolumeExpansion: true
parameters:
  protocol: nfs
  skuName: PremiumV2_LRS
  encryptInTransit: "true"
mountOptions:
  - nconnect=4
  - noresvport
  - actimeo=30

💡 ユースケース

🏥 医療データ処理パイプライン

シナリオ: HIPAA 準拠のデータ処理で患者データを複数の処理 Pod 間で共有
効果: NFS の高パフォーマンスを活かしつつ、TLS 暗号化により患者データの機密性を確保。コンプライアンス監査に対応可能な暗号化通信を実現。

🏦 金融機関の共有ストレージ

シナリオ: 複数マイクロサービスが取引ログや市場データを読み書き、PCI-DSS 要件に対応
効果: アプリケーション変更なしで転送中暗号化を実現し、PCI-DSS の転送中データ暗号化要件を満たす。NFS v4.1 の並列アクセス性能を維持。

⚠ 制約事項

  • Ubuntu 20.04 ノードは非サポート
  • Windows ノードは非サポート
  • AKS バージョン 1.33 以降が必要
  • NFS ファイル共有には SSD (Premium) が必須
  • TLS のオーバーヘッドによるわずかなレイテンシ増加の可能性
