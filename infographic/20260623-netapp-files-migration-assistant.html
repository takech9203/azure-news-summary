Azure NetApp Files: Migration Assistant の一般提供開始

2026-06-23 | Migration Assistant (with SnapMirror)

概要

Azure NetApp Files の Migration Assistant が一般提供 (GA) として正式リリース。ONTAP の SnapMirror レプリケーションエンジンを活用し、オンプレミスの ONTAP ストレージまたは Cloud Volumes ONTAP から Azure NetApp Files への効率的かつコスト効果の高いデータ移行を実現する。

主要機能

🔄

SnapMirror ベースのブロックレベルレプリケーション

変更されたブロックのみを転送。ベースライン転送後はインクリメンタル更新のみ送信し、ネットワーク帯域幅を最適化。

⚡

完全なデータ保全

ディレクトリ、ファイル、ファイルメタデータ、既存のボリュームスナップショットをすべて保持したまま移行。

📋

ドライラン機能

カットオーバー前にボリュームを読み書き可能にし、アプリケーションテストが可能。問題があれば再開して同期を継続。

🖥️

Azure Portal 統合管理

NetApp アカウントの Migration Assistant ビューから一元管理。移行の作成、同期、ドライラン、カットオーバー、ファイナライズを GUI で実行。

🔧

REST API / PowerShell サポート

REST API による自動化対応。PowerShell サンプルスクリプトで自動化ワークフロー構築が可能。

📅

柔軟なレプリケーションスケジュール

ソースボリュームとターゲットボリューム間の同期頻度を選択可能。ビジネスニーズに合わせた柔軟な設定。

セットアップ手順 (Portal)

  1. NetApp アカウントで Migration Assistant を選択
  2. New migration でソース情報を入力
  3. ターゲットボリュームのプロトコル・エクスポートポリシーを設定
  4. Review + Create で作成
  5. Configure Peering で Intercluster LIF を設定
  6. オンプレミス側で cluster peering を実行
  7. オンプレミス側で SVM peering を実行
  8. ベースライン転送の完了を待機
  9. 任意: Dry run でテスト実行
  10. Cut over でレプリケーション関係を切断
  11. Finalize migration で移行を完了

ユースケース

1. オンプレミス ONTAP から Azure への大規模データ移行

エンタープライズ環境でオンプレミスの NetApp ONTAP 上に数十 TB のファイルデータがある場合、SnapMirror のブロックレベルレプリケーションにより、初回のベースライン転送後は差分のみ同期。最終カットオーバー時のダウンタイムを最小化。

2. Cloud Volumes ONTAP (他クラウド) から Azure への移行

他クラウドプロバイダーで CVO を利用している場合、直接 SnapMirror レプリケーションで移行可能。中間ステップなしでシームレスな移行を実現し、既存のスナップショットやメタデータも保持。

3. DR/テスト目的でのドライラン検証

本番移行前にドライラン機能でアプリケーション動作を検証。ボリュームが読み書き可能になり、パフォーマンステストや接続確認が可能。問題があれば再開して同期を継続。

制約事項・注意点

  • FlexGroup ボリュームは Large Volumes 移行に非対応
  • クラスターピアリングは 60 分以内に承認が必要
  • 単一ソースクラスター: 一度に 1 つの Azure サブスクリプション
  • 移行中はバックアップを有効化しないこと
  • Standard ネットワーク機能のみ対応 (Basic 非対応)
  • SnapMirror ライセンスが必要
  • LDAP/AD 設定はユーザー責任
