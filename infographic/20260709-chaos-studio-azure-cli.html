Azure Chaos Studio In Preview

Azure CLI による Chaos Studio 管理
— az chaos 拡張のパブリックプレビュー

📅 2026年7月9日 リリース

📋 概要

Azure Chaos Studio が Azure CLI からの管理に対応。新しい az chaos 拡張機能を使用することで、REST API の直接呼び出しや JSON ファイルの手動作成が不要になり、コマンドラインから直接レジリエンステストの Workspace 作成、Scenario の構成・実行が可能になる。

最大の特徴: az chaos setup コマンドによるワンステップセットアップ。リソースグループの作成、Workspace のデプロイ、マネージド ID の割り当て、RBAC 権限の付与、リソースの検出、Scenario の推奨評価までを一括で実行。

🔄 Before / After

❌ Before

  • REST API を直接呼び出す必要があり、JSON ペイロードの構築が煩雑
  • 複数の API エンドポイントを順番に呼び出す必要
  • CI/CD パイプラインへの組み込みが困難
  • セットアップ手順が多段階で、手動での権限設定が必要

✅ After

  • az chaos setup でワンステップのセットアップが可能
  • CLI コマンドによる直感的な操作
  • スクリプトによる自動化が容易
  • CI/CD パイプラインとのネイティブな統合
  • RBAC 権限の自動付与機能

⚡ 主要機能

🚀

ワンステップセットアップ (az chaos setup)

リソースグループ作成 → Workspace デプロイ → マネージド ID 割り当て → RBAC 付与 → リソース検出 → Scenario 推奨まで一括実行。

📦

Workspace 管理 (az chaos workspace)

Workspace の作成、表示、更新、削除をコマンドラインから実行。システム割り当て / ユーザー割り当てマネージド ID に対応。

🎯

Scenario 管理 (az chaos scenario)

Scenario の一覧取得、Configuration の作成・検証、実行・キャンセル、レポート取得まで一連のワークフローを CLI で完結。

🔑

権限修復 (fix-permissions)

Scenario Configuration に必要な権限を検証し、不足している場合は自動修復。手動での RBAC 設定が不要に。

🔍

リソース検出 (discovered-resource)

スコープ内の対象リソースを自動検出し、適用可能な Scenario を推奨。テスト設計の出発点として活用可能。

⚙️ az chaos setup の処理フロー

az chaos setup が自動実行する 5 つのステップ:

1️⃣

リソースグループの作成

指定されたリソースグループが存在しない場合、自動作成する。

2️⃣

Workspace の作成

マネージド ID を持つ Workspace をデプロイ。

3️⃣

Reader ロール付与

ターゲットスコープに対する Reader ロールを自動で割り当て。

4️⃣

リソース検出

スコープ内のリソースを検出し、Scenario 推奨の評価を実行。

5️⃣

結果の表示

検出された Scenario と次のステップをターミナルに表示。

💻 コード例

基本的なセットアップ

Bash 
# 基本的なセットアップ (システム割り当てマネージド ID)
az chaos setup \
  --name MyWorkspace \
  --resource-group MyRG \
  --location westus2 \
  --scopes "/subscriptions/<sub-id>/resourceGroups/MyRG"

Scenario の管理と実行

Bash 
# Scenario 一覧の取得
az chaos scenario list \
  --workspace-name MyWorkspace \
  --resource-group MyRG

# Scenario Configuration の検証
az chaos scenario config validate \
  --workspace-name MyWorkspace \
  --resource-group MyRG ...

# Scenario の実行
az chaos scenario config execute \
  --workspace-name MyWorkspace \
  --resource-group MyRG ...

CI/CD パイプラインでの使用

Bash 
# CI/CD 向けオプション: 評価の待機をスキップ
az chaos setup \
  --name MyWorkspace \
  --resource-group MyRG \
  --location westus2 \
  --scopes "/subscriptions/<sub-id>/resourceGroups/MyRG" \
  --skip-evaluation-wait

💡 ユースケース

🚦 デプロイメントゲート

CI/CD パイプラインにレジリエンステストを組み込み、デプロイ前に障害耐性を自動検証する。テストに合格した場合のみデプロイを進行。

📅 定期的なゲームデイ

cron ジョブやスケジュールパイプラインで月次・四半期ごとのレジリエンステストを自動実行。Scenario Report をコンプライアンスエビデンスとして活用。

🌐 マルチ環境管理

開発、ステージング、本番の各環境に対して一貫したレジリエンステスト構成をスクリプトで管理し、環境ごとの差異を排除。

🔥 インシデント再現

障害発生後に CLI で素早く Scenario を構成し、修正の有効性を検証。再発防止策の確認に活用。

📝 コンプライアンスエビデンス収集

定期的なテスト実行と結果レポートの自動収集。DORA などの運用レジリエンスフレームワークへのエビデンスとして活用可能。

⚠️ 制約事項・注意点

  • パブリックプレビュー段階であり、GA までにコマンド体系が変更される可能性がある
  • az chaos setup は GA だが、Scenario 関連コマンドの多くは Preview ステータス
  • --scopes パラメータが必須であり、デフォルトスコープは存在しない (安全のため意図的な設計)
  • Chaos Studio 自体のリージョン制限があり、全リージョンで利用可能ではない
  • Azure Resource Graph の伝搬に時間がかかる場合がある (新規 Reader ロール割り当て後)
  • クラシックな Experiment モデルの CLI 管理は含まれていない (Workspace/Scenario モデルのみ)
