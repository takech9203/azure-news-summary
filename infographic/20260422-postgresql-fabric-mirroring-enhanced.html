Azure Database for PostgreSQL / Microsoft Fabric

Fabric Mirroring 機能強化の一般提供開始

JSON/JSONB データ型サポートで、ゼロ ETL のデータレプリケーションがさらに実用的に

Launched (GA) 2026-04-22

概要

Azure Database for PostgreSQL flexible server の Fabric Mirroring 機能が強化され、JSON/JSONB データ型のサポートが一般提供 (GA) として開始された。これにより、PostgreSQL に格納された JSON/JSONB カラムがそのまま Microsoft Fabric の OneLake にレプリケーションされ、ETL パイプラインを構築することなくリッチな構造化データを分析基盤に直接流入させることが可能になった。大規模実運用ワークロードへの信頼性も強化されている。

Key Points

  • JSON/JSONB ネイティブデータ型のミラーリング対応
  • ゼロ ETL でリッチなデータが OneLake に直接流入
  • 大規模実運用ワークロードへの対応強化
  • SQL Analytics Endpoint の自動生成

Before

  • JSON/JSONB がミラーリング非対応でカラムレプリケーション不可
  • データ変換パイプライン構築が必要
  • データ型制約が本番採用の障壁

After

  • JSON/JSONB がそのまま OneLake にレプリケーション
  • ETL パイプライン不要
  • 大規模ワークロードの信頼性向上

📦

ネイティブ PostgreSQL データ型サポート

JSON/JSONB がそのまま OneLake へレプリケーションされる。データ型変換やカスタム ETL ロジックを構築する必要がなく、PostgreSQL に格納されたリッチな構造化データをそのまま分析基盤で活用可能。

🔄

ゼロ ETL レプリケーション

WAL ベースの Change Data Capture (CDC) により INSERT/UPDATE/DELETE を継続的に検出し、OneLake へリアルタイムに近いレプリケーションを実現。データパイプラインの構築・運用コストを大幅に削減。

📊

SQL Analytics Endpoint 自動生成

ミラーリング設定時に SQL Analytics Endpoint が自動的に生成され、T-SQL クエリ、ビュー、テーブル値関数 (TVF) の作成が可能。既存の SQL スキルセットをそのまま活用した分析が実現。

🌐

柔軟なネットワーク構成

パブリックアクセス、VNet Data Gateway、プライベートエンドポイントに対応。企業のネットワークポリシーに応じた接続方式を選択でき、セキュアなデータレプリケーションを実現。

🔐

認証方式

Basic (PostgreSQL ネイティブ) 認証および Microsoft Entra ID 認証に対応。組織の ID 管理ポリシーに沿ったセキュアな認証構成が可能。

Use Cases

UC1: JSONB データを活用したリアルタイム分析

Web アプリケーションのユーザーイベントデータ (JSONB 形式) を PostgreSQL に蓄積し、Fabric Mirroring で OneLake にレプリケーションして Power BI ダッシュボードで可視化するケース。

Implementation:
  • PostgreSQL にユーザーイベントを JSONB カラムで格納
  • Fabric Mirroring で OneLake へゼロ ETL レプリケーション
  • SQL Analytics Endpoint で T-SQL クエリを作成
  • Power BI Direct Lake モードでリアルタイムダッシュボードを構築

-- Web アプリのユーザーイベントを Power BI ダッシュボード化し、リアルタイムに近い分析を実現。

UC2: IoT データの大規模分析基盤

IoT デバイスから送信される JSON 形式のセンサーデータを PostgreSQL に格納し、Fabric 上で Spark Notebook を使った ML パイプラインに投入するケース。

Implementation:
  • IoT デバイスのセンサーデータを JSON カラムで PostgreSQL に蓄積
  • Fabric Mirroring で Delta Lake テーブルとして OneLake にレプリケーション
  • Spark Notebook で機械学習モデルのトレーニングデータとして活用
  • Data Science ワークスペースで予測モデルを構築・デプロイ

-- JSON センサーデータを Spark Notebook で ML パイプラインに投入し、予測分析基盤を構築。

Limitations / Caveats

  • Burstable ティア非対応
  • Read Replica からのミラーリング非対応
  • テーブル上限 1,000
  • DDL 操作がミラーリング中非対応 (再開必要)
  • ビュー・マテリアライズドビュー・パーティションテーブル非対応
  • bit/inet/interval/macaddr/xml 等の一部データ型は非対応
