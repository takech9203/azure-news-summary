Azure Databricks OpenAI Launched (GA)

OpenAI GPT-5.6 一般提供開始
— Azure Databricks Model Serving

📅 2026年7月9日 リリース

📋 概要

OpenAI GPT-5.6 が Azure Databricks 上で一般提供 (GA) として利用可能に。Microsoft Foundry で購入した GPT-5.6 モデルを、Azure Databricks の Model Serving Endpoint 経由で利用でき、Foundry と Azure Databricks 間のインテグレーションを通じて、AI モデルのセキュアな構築、デプロイ、管理が可能となる。

ポイント: Databricks のエコシステム内で一貫した開発体験を維持しながら、最新の OpenAI モデルを活用。AI Gateway による一元的なガバナンス、レート制限、アクセス制御といった運用面の管理機能も利用可能。

🔄 Before / After

❌ Before

  • GPT-5.6 利用に Azure Databricks 外の別サービスで個別にエンドポイントを構築する必要
  • Foundry で購入したモデルと Databricks ワークロードの連携で認証やネットワーク設定が複雑
  • 複数の LLM プロバイダー利用時に API キー管理やアクセス制御が分散

✅ After

  • Microsoft Foundry で購入した GPT-5.6 を Databricks Model Serving Endpoint から直接利用
  • Foundry と Azure Databricks 間のシームレスなインテグレーションで統一された体験
  • AI Gateway を通じた一元的なガバナンス (レート制限、使用量追跡、アクセス制御)

⚡ 主要機能

📡

Model Serving Endpoint 経由の GPT-5.6 アクセス

REST API および MLflow Deployments SDK を通じた統一的なインターフェースで GPT-5.6 を呼び出し可能。OpenAI 互換の API フォーマットをサポート。

🏭

Microsoft Foundry との統合

Foundry で購入した GPT-5.6 のライセンスを Azure Databricks 上でそのまま利用可能。認証情報の一元管理によりセキュアなモデルアクセスを実現。

🛡️

AI Gateway によるガバナンス

レート制限の設定 (ユーザー単位/エンドポイント単位)、使用量の追跡とモニタリング、ガードレール (入出力フィルタリング) の適用が可能。

🔑

一元的な資格情報管理

API キーを Azure Databricks Secrets に安全に保存。コード内での鍵の露出を防止し、Microsoft Entra ID による認証も対応。

💻 コード例

MLflow Deployments SDK でエンドポイント作成

Python 
import mlflow.deployments

client = mlflow.deployments.get_deploy_client("databricks")

endpoint = client.create_endpoint(
    name="gpt-56-chat-endpoint",
    config={
        "served_entities": [
            {
                "name": "gpt56_chat",
                "external_model": {
                    "name": "gpt-5.6",
                    "provider": "openai",
                    "task": "llm/v1/chat",
                    "openai_config": {
                        "openai_api_key": "{{secrets/my_scope/foundry_api_key}}"
                    }
                }
            }
        ],
        "rate_limits": [
            {
                "calls": 100,
                "key": "user",
                "renewal_period": "minute"
            }
        ]
    }
)

エンドポイントへのクエリ (OpenAI 互換)

Python 
from openai import OpenAI

client = OpenAI(
    api_key="<databricks-personal-access-token>",
    base_url="https://<workspace-url>/serving-endpoints"
)

response = client.chat.completions.create(
    model="gpt-56-chat-endpoint",
    messages=[
        {"role": "user", "content": "Azure Databricks の特徴を教えてください"}
    ]
)
print(response.choices[0].message.content)

⚙️ セットアップ手順 (Serving UI)

1️⃣

Serving タブを開く

Azure Databricks ワークスペースの左サイドバーから Serving タブを選択

2️⃣

エンドポイント作成

Create serving endpoint をクリック、エンドポイント名を入力

3️⃣

モデル選択

Foundation models → External model providers → OpenAI を選択

4️⃣

設定・作成

モデル名に gpt-5.6 を指定、タスク選択、Foundry 認証情報を設定して Create

💡 注意: GPT-5.6 を External Model として利用する場合、Model Serving の DBU 料金に加えて Microsoft Foundry でのモデル購入費用が別途必要。14 日間の無料トライアルが利用可能。コミットメント利用割引あり。

⚠️ 制約事項・注意点

  • External model エンドポイントの served_entities リストにはオブジェクトを 1 つのみ設定可能
  • External model を含むエンドポイントの作成後、external model を削除する更新は不可
  • Microsoft Foundry で事前にモデルを購入する必要がある
  • データがワークスペースのリージョン外で処理される可能性がある (外部モデルの特性)
  • Model Serving はリージョンによって利用可否が異なる
