Azure Event Hubs

Confidential Computing サポート (Dedicated tier)

ハードウェアベースの Trusted Execution Environment (TEE) により、イベントデータのメモリ上処理中も保護を実現

GA 2026-07-07

概要

Azure Event Hubs Dedicated が Confidential Computing をサポートし、GA となった。ハードウェアベースの Trusted Execution Environment (TEE) を活用して、イベントデータがメモリ上で処理されている間も保護する。「使用中のデータ (data in use)」の保護が Event Hubs に追加され、クラウドオペレーターを含む非認可アクターからのアクセスを防止する。

Before

  • イベントストリーミングデータはメモリ上で処理される際に保護されていなかった
  • 処理中データへの非認可アクセスリスクに対応する追加措置が必要
  • クラウドオペレーターレベルでのデータアクセスを技術的に防止する手段がなかった

After

  • TEE によるハードウェアレベルでのメモリ分離により処理中データを保護
  • コード変更不要で namespace レベルで有効化可能
  • CMK + Managed HSM で保存時・転送中・使用中すべてのデータ保護を実現

🔒

ハードウェアベースの TEE

ハードウェアルート・オブ・トラストでデータを保護。メモリ上の非認可アクセスや改ざんを防止し、クラウドオペレーターを含む第三者からのデータアクセスを技術的に排除する。

⚡

コード変更不要の有効化

namespace レベルの設定のみで有効化が可能。既存アプリケーションのコード変更は不要で、透過的に Confidential Computing の保護が適用される。

🔑

既存セキュリティ機能との統合

CMK (Managed HSM) + プライベートエンドポイント + マネージド ID と組み合わせることで、保存時・転送中・使用中のすべてのフェーズでデータ保護を実現。

📋

Azure Policy によるガバナンス

カスタムポリシーで組織全体に Confidential Computing の使用を強制可能。Deny / Audit モードに対応し、コンプライアンス要件の自動適用を実現。

Code Sample (Bicep)

resource eventHubCluster 'Microsoft.EventHub/clusters@2025-05-01-preview' = {
  name: clusterName
  location: location
  sku: {
    name: 'Dedicated'
    capacity: capacity
  }
  properties: {
    supportsScaling: true
    platformCapabilities: {
      confidentialCompute: {
        mode: 'Enabled'
      }
    }
  }
}

Use Cases

UC1: 金融機関のリアルタイムトランザクション処理

決済トランザクションデータの不正検知パイプラインにおいて、処理中のデータをハードウェアレベルで保護。PCI DSS 準拠のイベントストリーミング基盤を構築し、規制要件を満たしながらリアルタイム処理を実現する。

-- PCI DSS 準拠のリアルタイム不正検知基盤を実現。

UC2: 医療データのストリーミング分析

患者バイタルデータのリアルタイムモニタリングにおいて、TEE により処理中の個人健康情報 (PHI) を保護。HIPAA 対応のイベント駆動アーキテクチャを構築し、医療データのプライバシーを確保する。

-- HIPAA 対応のリアルタイム医療データ分析基盤を構築。

UC3: 政府機関の機密データイベント処理

データ主権要件を満たしつつクラウドのスケーラビリティを活用。Confidential Computing によりクラウドオペレーターからもデータを保護し、国家安全保障レベルのデータ処理をクラウド上で実現する。

-- データ主権を確保しながらクラウドスケーラビリティを活用。

Limitations / Caveats

  • Dedicated ティア限定 (最もコストが高いティア)
  • 作成時のみ有効化可能 (後付け不可)
  • Korea Central と UAE North の 2 リージョンのみ対応
  • 既存クラスターの移行不可
