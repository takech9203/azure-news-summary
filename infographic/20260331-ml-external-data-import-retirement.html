外部データインポートおよびデータ接続機能の廃止 (2026 年 9 月 30 日)

Azure Machine Learning の外部データインポート機能 (Preview) および外部データ接続機能 (Preview) が 2026 年 9 月 30 日に廃止されます。Amazon S3、Snowflake、Azure SQL Database からのデータインポートが対象です。代替として Azure ML データストアと Azure Data Factory の組み合わせが推奨されます。

🔄 概要: Before / After 比較

外部データインポート機能は、外部データソースからデータを取得し Azure ML 内にキャッシュする仕組みでした。バックエンドでは ADF パイプラインを使用してデータを Parquet 形式で転送していました。プレビュー段階のまま廃止となるため、GA 済みのデータストアへの移行が必要です。

Before: 外部データインポート (Preview)

  • Preview 機能に依存したパイプライン
  • S3/Snowflake/SQL DB への直接接続
  • Workspace Connections による認証
  • Parquet への自動変換
  • 組み込みスケジュール (Recurrence/Cron)

After: Datastore + ADF (GA)

  • GA 済みの安定したデータアクセス基盤
  • Blob/ADLS Gen2/Files/OneLake 対応
  • Entra ID / SAS / Key / SP 認証
  • ソース形式維持 or ADF で変換
  • ADF トリガー (スケジュール/イベント)
重要: 2026 年 9 月 30 日以降、外部データインポートおよび外部データ接続機能は利用できなくなります。廃止日までに代替構成への移行を完了してください。
🏗️ アーキテクチャ図: 廃止機能と代替手段

従来は Data Connections と Data Import 機能を通じて外部ソースから直接データを取り込んでいましたが、廃止後は Azure Data Factory や Microsoft Fabric で外部データを Azure ストレージに取り込み、データストア経由でアクセスする構成が推奨されます。

🚫 廃止対象の機能
📥

外部データインポート (Import Data from External Sources - Preview)

Amazon S3 からの uri_folder 形式インポート、Snowflake / Azure SQL DB からの mltable 形式インポート、スケジュールベースの定期データインポート、ADF パイプラインを利用したバックエンドデータ転送が廃止対象です。

🔗

外部データ接続 (External Data Connections - Preview)

Amazon S3、Snowflake、Azure SQL Database への外部接続設定が廃止対象です。これらの接続設定を利用しているワークフローは代替手段への移行が必要です。

🚀 移行手順
1

現在の外部データインポート設定の確認

Azure ML Studio の「Data」セクションで「Data Import」タブを確認し、使用中のデータ接続とインポートスケジュールを洗い出します。

2

Azure ストレージの準備

データの格納先となる Azure Blob Storage または Azure Data Lake Storage Gen2 を準備し、適切な RBAC を設定します。

3

Azure Data Factory によるデータ転送パイプラインの構築

外部ソース (S3, Snowflake, Azure SQL DB) から Azure ストレージへのコピーパイプラインを作成し、既存スケジュールに対応するトリガーを設定します。

4

データストアの作成

Azure ML のデータストアを作成し、Azure ストレージへの接続を構成します。Python SDK または Azure CLI で設定可能です。

5

データアセットの作成とジョブの更新

新しいデータストアを参照するデータアセットを作成し、トレーニングジョブの入力設定を更新します。

6

テストと検証

新しいデータパイプラインとデータストア経由のアクセスが正常に動作することを確認し、2026 年 9 月 30 日までに切り替えを完了します。

💻 移行コードサンプル

Python SDK を使用したデータストア作成の例です。

Python Azure ML SDK v2 
from azure.ai.ml.entities import AzureBlobDatastore
from azure.ai.ml import MLClient
from azure.identity import DefaultAzureCredential

ml_client = MLClient.from_config(credential=DefaultAzureCredential())
store = AzureBlobDatastore(
    name="my_datastore",
    account_name="mystorageaccount",
    container_name="mycontainer"
)
ml_client.create_or_update(store)

Azure CLI を使用したデータストア作成の例です。

Bash Azure CLI ml 拡張機能 v2 
az ml datastore create --file my_blob_datastore.yml
前提条件: Azure CLI ml 拡張機能 v2 (バージョン 2.37.0 以降) または Python SDK azure-ai-ml v2 (バージョン 1.31.0 以降) が必要です。
💡 ユースケース: 移行パターン

Amazon S3 データの活用

S3 バケットのデータを Azure ML のトレーニングジョブで使用しているケース

Amazon S3 → ADF → Azure Blob Storage → Datastore

Snowflake データの活用

Snowflake のクエリ結果を ML モデルのトレーニングデータとして利用しているケース

Snowflake → ADF コネクタ → Azure Storage → mltable

Azure SQL DB の定期インポート

Azure SQL Database のデータを定期的にインポートしてモデル再トレーニングを行っているケース

Azure SQL DB → ADF トリガー → Azure Storage → Datastore
⚠️ 制約事項・注意点
🔄

中間転送ステップが必要

外部データインポートが提供していた「直接接続」の利便性が失われ、ADF 経由の中間データ転送ステップが必要になります。

💰

ADF 追加リソースが必要

Azure Data Factory の追加リソースが必要となり、構成の複雑さが増す可能性があります。

⏰

スケジュールの再構築

既存のインポートスケジュールを ADF トリガーとして再構築する作業が発生します。

📄

Parquet 変換の明示設定

外部データインポートが自動的に行っていた Parquet 変換を、ADF パイプライン内で明示的に設定する必要があります。

データストア種別 認証方式
Azure Blob Storage ID ベース / アカウントキー / SAS
Azure Data Lake Storage Gen2 ID ベース / サービスプリンシパル
Azure Files アカウントキー / SAS
