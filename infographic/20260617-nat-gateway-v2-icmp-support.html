概要

Azure StandardV2 NAT Gateway が、アウトバウンド方向の ICMP Echo Request および Echo Reply トラフィックをサポートするようになった。これにより、StandardV2 NAT Gateway の背後にあるワークロードから ping コマンドを使用してアウトバウンド接続性の検証やネットワーク問題のトラブルシューティングが可能になる。

この機能はデフォルトで有効であり、追加の構成は不要。IPv4 および IPv6 の両方で ICMP Echo (ping) をサポートする。

デフォルト有効: StandardV2 NAT Gateway に組み込まれた機能であり、NSG で ICMP をブロックしていない限り、既存デプロイメントで即座に利用可能。追加の設定変更や再デプロイは不要。

Before (課題)

  • NAT Gateway 背後のワークロードから ping による接続性検証が不可
  • TCP ベースのツール (PsPing, curl, nc) に頼る必要あり
  • ICMP ping が使えず問題切り分けに時間を要する

After (改善)

  • StandardV2 NAT Gateway 経由で ICMP ping が可能
  • 追加構成不要 (デフォルト有効)
  • IPv4 / IPv6 デュアルスタック対応

ユースケース

1. アウトバウンド接続性の迅速な検証

NAT Gateway 配下のワークロードから外部 API への接続障害時にネットワーク到達性を即座に確認。従来は TCP ベースのツールが必要だったが、ping で簡潔に診断が可能に。

bash 接続性検証コマンド 
# VM から外部ホストへの接続性を確認
ping -c 4 8.8.8.8

# IPv6 での接続性確認
ping6 -c 4 2001:4860:4860::8888

# 特定の外部 API エンドポイントへの到達性確認
ping -c 4 api.example.com
効果: ネットワーク到達性の問題を即座に切り分け。TCP レイヤーの問題かネットワークレイヤーの問題かを迅速に判別でき、障害対応時間を大幅に短縮。

2. ネットワーク監視の自動化

定期的に外部サービスへの接続性を監視し、到達不能検知時にアラートを発報するスクリプトを実装。NAT Gateway 経由のアウトバウンド接続の健全性を継続的に確認。

bash 監視スクリプト例 
#!/bin/bash
TARGETS=("8.8.8.8" "1.1.1.1" "api.partner.com")
for target in "${TARGETS[@]}"; do
    if ! ping -c 3 -W 5 "$target" > /dev/null 2>&1; then
        echo "ALERT: $target is unreachable via NAT Gateway"
    fi
done
効果: シンプルな ping ベースの監視により、外部サービスへの到達性を継続的に確認。問題発生時の早期検知とプロアクティブな対応が可能に。

料金

ICMP サポートの追加による料金変更なし

StandardV2 と Standard の料金は同一。ICMP トラフィックも通常のデータ処理として課金される。

項目 課金方式
NAT Gateway リソース時間 デプロイ時間に基づく課金
データ処理 NAT Gateway 経由データ量に基づく課金
ICMP トラフィック 通常のデータ処理として課金 (追加料金なし)
⚠

制限事項・注意点

  • Standard NAT Gateway (従来 SKU) では ICMP 非サポート。StandardV2 への移行が必要
  • Standard から StandardV2 への直接アップグレードは不可 (新規作成+切り替えが必要)
  • StandardV2 は StandardV2 SKU パブリック IP のみサポート
  • インバウンド ICMP は非サポート (アウトバウンドのみ)
  • 一部リージョン未対応: Canada East, Chile Central, Indonesia Central, Israel Northwest, Malaysia West, Qatar Central, Sweden South, West India
