統合テキスト翻訳 API 一般提供

単一エンドポイントから NMT、LLM、SLM、Adaptive Custom Translation を統一的に利用

GA (Launched) 2026-06-02

概要

Microsoft Build 2026 にて、Azure AI Translator の統合テキスト翻訳 API (Unified Text Translation API) が一般提供 (GA) となった。新しい API バージョン 2026-06-06 として提供され、単一のエンドポイントから Neural Machine Translation (NMT)、Large Language Model (LLM) 翻訳、Small Language Model 翻訳、Adaptive Custom Translation を統一的に利用できる。

Key Points

  • 単一エンドポイントから NMT、LLM、SLM、Adaptive Custom Translation を統一利用
  • API パラメータで翻訳エンジンを選択可能 (NMT / LLM)
  • LLM 翻訳でトーン制御 (formal/informal/neutral) とジェンダー固有の出力に対応
  • Adaptive Custom Translation: 最大5つの参照翻訳で用語・スタイル制御
  • リクエスト/レスポンスの JSON スキーマ刷新 (inputs/value 構造)

Before

  • v3.0 API では NMT のみ利用可能、翻訳エンジン選択不可
  • LLM を活用した翻訳には別サービスや別エンドポイントが必要
  • トーン (フォーマル/カジュアル) やジェンダー表現の制御不可
  • カスタム翻訳モデル構築に大量の対訳データと長時間トレーニングが必要

After

  • 単一エンドポイントから NMT、LLM、SLM、Adaptive Custom Translation を統一利用
  • モデル選択 (NMT / LLM) を API パラメータで指定可能
  • LLM 翻訳でトーン制御とジェンダー固有の出力に対応
  • 最大5つの参照翻訳で用語・スタイル制御、大規模トレーニング不要

🔀

モデル選択 (NMT / LLM)

API パラメータで翻訳エンジンを選択。品質・コスト・シナリオに応じた使い分けが可能。LLM ベース翻訳には Microsoft Foundry リソースが必要。

🎯

Adaptive Custom Translation

最大5つの参照翻訳またはデータセット ID でスタイル・用語制御。従来のカスタム翻訳のような大量トレーニングデータ不要。

🎨

トーン・ジェンダー制御

LLM ベース翻訳でトーンバリアント (formal/informal/neutral) をサポート。ジェンダー固有の出力制御にも対応。

📄

刷新されたスキーマ

リクエストは inputs 配列で翻訳テキストを指定、レスポンスは value 配列で結果返却。v3.0 からの破壊的変更 (Breaking Changes)。

API リクエスト例

curl -X POST "https://api.cognitive.microsofttranslator.com/translate?api-version=2026-06-06" \
  -H "Ocp-Apim-Subscription-Key: ${TRANSLATOR_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "text": "Hello, how are you?",
        "language": "en",
        "targets": [
          { "language": "ja" }
        ]
      }
    ]
  }'

ユースケース

UC1: ハイブリッド翻訳パイプライン

EC サイトで商品スペックは NMT で高速処理、ブランドメッセージは LLM + トーン制御で高品質翻訳を実施。翻訳コストを最小限に抑えながら、顧客接点の高い部分では LLM の表現力を活用。

-- コストと品質の最適バランスを実現、ブランド体験の一貫性を確保

UC2: Adaptive Custom Translation による迅速カスタマイズ

法律事務所が契約書の翻訳において、参照翻訳を最大5件提供するだけで、大規模トレーニングなしに業界固有の用語や表現スタイルを統一。

-- 従来数万件の対訳データと数十時間のトレーニングが必要だったカスタマイズが、少数の参照例だけで即座に実現
