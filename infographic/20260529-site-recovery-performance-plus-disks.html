Azure Site Recovery

Performance Plus マネージドディスクのレプリケーションサポート

高 IOPS/高スループットのディスクでも Azure Site Recovery による DR レプリケーションが可能に

In Preview 2026-05-29

概要

Azure Site Recovery (ASR) が Performance Plus 対応マネージドディスクを使用する仮想マシンのレプリケーションをサポートした。Performance Plus は、513 GiB 以上の Premium SSD、Standard SSD、Standard HDD マネージドディスクの IOPS とスループットの上限を追加料金なしで引き上げる機能であり、今回のアップデートにより高性能ディスクを使用する本番ワークロードの DR 保護が実現した。

要点

  • Performance Plus 対応ディスクを使用する VM の ASR レプリケーションが可能に
  • Premium SSD、Standard SSD、Standard HDD (513 GiB 以上) に対応
  • Performance Plus の追加料金なしで IOPS/スループット上限が向上
  • レプリケーション時は Premium ストレージアカウントの使用が必要
  • パブリックプレビュー段階

❌ Before

  • Performance Plus ディスクを使用する VM は ASR でレプリケーション不可
  • 高 IOPS ワークロードの DR 保護に制約あり
  • Performance Plus の利用か DR 保護の二者択一

✅ After

  • Performance Plus ディスクのレプリケーションが可能
  • 高性能ワークロードの DR 保護を実現
  • パフォーマンス向上と DR 保護を同時に両立

主要機能

💾

Performance Plus ディスクのレプリケーション

Premium SSD、Standard SSD、Standard HDD の Performance Plus 対応ディスクを使用する VM のレプリケーションが可能。レプリケーション中も Performance Plus のパフォーマンス特性が維持される。

⚡

高チャーンサポート

Premium ストレージアカウントの使用により、VM あたり最大 100 MBps のデータ変更に対応。ディスクあたり最大 20 MBps (Premium SSD 512 GiB 以上、16 KB I/O)。

🔄

フェイルオーバー時のパフォーマンス維持

フェイルオーバー後もディスクの Performance Plus パフォーマンス特性が維持されるため、DR 発動後も本番同等の性能でサービス継続が可能。

🌍

グローバル DR 対応

Azure Site Recovery がサポートされるすべての Azure リージョンで利用可能。任意の 2 つのリージョン間でレプリケーションとリカバリが実行できる。

技術仕様

対応ディスクタイプ Premium SSD、Standard SSD、Standard HDD
最小ディスクサイズ 513 GiB (Performance Plus の要件)
レプリケーション要件 Premium ストレージアカウントの使用
VM あたり最大チャーン 100 MBps (高チャーンサポート利用時)
ディスクあたり最大チャーン 20 MBps (Premium SSD 512 GiB 以上、16 KB I/O)
Performance Plus 追加料金 なし (ディスク自体の料金のみ)
ステータス パブリックプレビュー

設定方法

Azure Portal から Recovery Services コンテナーで Performance Plus ディスクを含む VM のレプリケーションを構成する。

前提条件の確認

513 GiB 以上のマネージドディスクに Performance Plus が有効化されていることを確認。サブスクリプションのディスク数が上限 (3,000) 以内であること。

Recovery Services コンテナーに移動

Azure Portal で Recovery Services コンテナーを開き、対象の VM を選択。

レプリケーションを有効化

対象の VM に対してレプリケーションを有効化する。

ストレージ設定で Premium ストレージアカウントを選択

レプリケーション時に Premium ストレージアカウントを指定。Performance Plus ディスクを含む VM のレプリケーションが開始される。

メリット

ビジネス面

  • 高性能データベースワークロードの DR 保護が実現し事業継続性が向上
  • 追加料金なしでパフォーマンス向上と DR 保護を両立
  • DR 設計のために Performance Plus の利用を断念する必要がなくなった

技術面

  • IOPS/スループット上限が引き上げられたディスクでも ASR レプリケーション可能
  • 既存の ASR ワークフローと同じ方法で管理でき追加の学習コスト少
  • フェイルオーバー後もディスクのパフォーマンス特性が維持

ユースケース

ミッションクリティカルなデータベースの DR 保護

SQL Server や Oracle Database など高 IOPS が求められるデータベースワークロードを Azure 上で運用しており、リージョン障害に備えた DR を構成する必要がある。

効果: Performance Plus により IOPS/スループット上限が向上した状態で DR レプリケーションが可能になり、本番環境のパフォーマンス最適化と DR 保護を両立。

トランザクション処理系アプリケーション

E コマースや金融系のトランザクション処理で高スループットのディスク I/O が必要なアプリケーションの DR 構成。

効果: フェイルオーバー先でも Performance Plus の拡張パフォーマンスが維持されるため、DR 発動後も本番同等のパフォーマンスでサービス継続が可能。

制限事項・注意点

  • レプリケーション時には Premium ストレージアカウントの使用が必須
  • パブリックプレビュー段階であり、SLA は適用されない
  • Performance Plus はディスク作成時にのみ有効化でき、既存ディスクへの後付けは不可 (スナップショットから新規ディスクを作成する回避策あり)
  • VM 作成時に作成されるディスクには Performance Plus を有効にできない (個別にディスクを作成してアタッチする必要がある)

料金情報

Performance Plus: 追加料金なし

Performance Plus 自体にはディスクへの追加料金は発生しない。Azure Site Recovery の料金は公式料金ページを参照。
