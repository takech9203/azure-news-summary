Agent Framework 1.0 -- 安定 API とマルチエージェントオーケストレーション

GA 2026-04-29

概要

Microsoft Agent Framework (Semantic Kernel Agent Framework) がバージョン 1.0 GA として .NET および Python で一般提供開始。"production-ready release: stable APIs, and a commitment to long-term support." Semantic Kernel エコシステム上に構築され、複数の AI プロバイダー (Foundry, Azure OpenAI, OpenAI, Anthropic Claude, Amazon Bedrock, Google Gemini, Ollama) をサポート。A2A および MCP プロトコルによるクロスランタイム相互運用性を実現する。

Key Points

  • .NET および Python で安定 API と長期サポート (LTS) のコミットメントを提供
  • 5 つのマルチエージェントオーケストレーションパターン (Sequential, Concurrent, Handoff, Group Chat, Magentic)
  • 7 つ以上の AI プロバイダーをサポートするマルチプロバイダーアーキテクチャ
  • A2A プロトコルと MCP によるクロスランタイム相互運用性
  • ミドルウェアパイプラインによるプロンプト変更なしのポリシー適用
  • MIT ライセンスによる 100% オープンソース

Before

  • API の安定性が保証されていなかったため、本番環境への導入にリスクがあった
  • マルチエージェントオーケストレーション機能は実験的ステージにあった
  • 異なるフレームワーク間でのエージェント連携に標準化されたプロトコルがなかった
  • エージェントの動作をカスタマイズするにはプロンプトの直接変更が必要だった

After

  • 安定した API と長期サポート (LTS) のコミットメントにより、本番環境での採用が可能
  • 5 つのオーケストレーションパターンが統一インターフェースで利用可能
  • A2A プロトコルによるクロスランタイムエージェント連携と MCP による外部ツール動的検出
  • ミドルウェアパイプラインにより、プロンプト変更なしでポリシー適用可能

from semantic_kernel.agents.orchestration import SequentialOrchestration
from semantic_kernel.agents.runtime import InProcessRuntime

orchestration = SequentialOrchestration(members=[agent_a, agent_b])
runtime = InProcessRuntime()
runtime.start()

result = await orchestration.invoke(task="Your task here", runtime=runtime)
final_output = await result.get()
await runtime.stop_when_idle()
