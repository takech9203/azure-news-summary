HTTP レイヤー DDoS 攻撃に特化したマネージドルールセットによる多層防御

In Preview 2026-04-28

概要

Azure Front Door Premium 上の Azure Web Application Firewall (WAF) に、新たに Microsoft HTTP DDoS Ruleset がパブリックプレビューとして追加された。HTTP レイヤー (L7) の DDoS 攻撃はアプリケーションのダウンタイムを引き起こす主要な原因の一つであり、従来の静的な制御手法では進化するボットネットに十分に対応できなかった。本ルールセットは Microsoft が管理・更新するマネージドルールで、HTTP フラッド攻撃やキャッシュバイパス攻撃、ボットネットによる分散型攻撃を自動的に検出・緩和する。

Key Points

  • HTTP DDoS 攻撃に特化した Microsoft マネージドルールセット
  • Azure Front Door Premium の 192+ エッジ PoP でグローバルに動作
  • 既存の Default Rule Set / Bot Manager Rule Set と組み合わせた多層防御
  • 検出 (Detection) / 防止 (Prevention) モードによる柔軟な運用
  • 攻撃シグネチャを Microsoft が継続的に更新
  • オリジンサーバーへの攻撃到達前にエッジで遮断

Before

  • L7 DDoS 攻撃に対し静的なレート制限や IP ブロックで手動対応が必要
  • 新しい攻撃パターンへの迅速な対応が困難
  • HTTP DDoS 攻撃に特化した防御ルールセットが存在しなかった
  • 大規模 HTTP フラッド攻撃に対する包括的な自動防御が不足

After

  • Microsoft 管理のマネージドルールで自動的に攻撃を検出・緩和
  • 進化するボットネットに対して Microsoft が継続的にルールを更新
  • 既存ルールセットと組み合わせた多層的な防御戦略を構築可能
  • グローバルエッジネットワーク上で攻撃を遮断しオリジン負荷を軽減

HTTP DDoS 攻撃に特化したマネージドルールセット

Microsoft が管理・更新する HTTP DDoS 攻撃防御用ルールセット。HTTP フラッド攻撃やボットネットによる分散型攻撃パターンを検出・緩和。攻撃シグネチャの継続的な更新により、新たな攻撃手法への対応が自動化される。

🛡

既存 WAF 防御機能との多層的な統合

Default Rule Set (OWASP Top 10 等)、Bot Manager Rule Set (悪意のあるボット防御)、カスタムルール (レート制限、IP 制限、Geo フィルタリング) と組み合わせた多層防御を実現。WAF ポリシー内で他のルールセットと並行して適用可能。

🌍

グローバルエッジでの攻撃緩和

Azure Front Door の 192 以上のエッジ PoP 上で動作し、攻撃トラフィックを攻撃元に近いネットワークエッジで遮断。オリジンサーバーに到達する前に攻撃を緩和し、バックエンドの可用性とパフォーマンスを保護。

⚙

WAF モードによる柔軟な運用

検出モード (Detection): ルールに一致するリクエストをログに記録するのみ。本番導入前のテストに活用可能。防止モード (Prevention): ルールに一致するリクエストに対してブロック、リダイレクト等のアクションを実行。

Use Cases

UC1: EC サイトの大規模セール時の DDoS 防御

大規模オンラインセールイベント時に、ボットネットによる HTTP フラッド攻撃でサイトがダウンするリスクがある。既存のレート制限ルールだけでは、分散型の低頻度攻撃を検出できない。

-- HTTP DDoS Ruleset を WAF ポリシーに追加し、Microsoft 管理の攻撃シグネチャによる自動防御を有効化。既存のレート制限では検出困難な攻撃パターンにも対応可能に。

UC2: 金融サービスの API エンドポイント保護

金融機関が Azure Front Door Premium 経由で公開する API エンドポイントが、HTTP レイヤーの DDoS 攻撃を受けてレスポンス遅延が発生し、取引処理に影響が出ている。

-- HTTP DDoS Ruleset と Bot Manager Rule Set を組み合わせ、悪意のあるボットによる API 乱用と HTTP DDoS 攻撃の両方を防御し、API の可用性とパフォーマンスを維持。

UC3: グローバル展開の SaaS アプリケーション防御

複数リージョンに展開された SaaS アプリケーションが、地理的に分散したボットネットからの HTTP DDoS 攻撃を受けている。攻撃元 IP が頻繁に変わるため、IP ベースのブロックでは対応しきれない。

-- 192 以上のグローバル PoP で HTTP DDoS Ruleset が動作し、攻撃元に最も近いエッジで攻撃を遮断。IP に依存しないルールベースの検出により、IP ローテーション型攻撃にも対応可能。

Limitations / Caveats

  • パブリックプレビュー段階であり、SLA の対象外。本番ワークロードへの適用は慎重に検討する必要がある
  • Azure Front Door Premium のみ対応。Standard ティアでは利用できない。Premium ティアへのアップグレードが必要
  • マネージドルールセットの特性上、正規トラフィックが誤ってブロックされる可能性がある。導入初期は検出モードで十分なテストを推奨
  • ルールの詳細やカスタマイズ可能な範囲がプレビュー期間中に変更される可能性がある

関連サービス

Azure Front Door Premium

HTTP DDoS Ruleset の実行基盤となるグローバル CDN/リバースプロキシサービス。Premium ティアで WAF のフル機能が利用可能。

Azure DDoS Protection

ネットワーク層 (L3/L4) の DDoS 防御サービス。HTTP DDoS Ruleset (L7) と組み合わせて全レイヤーの防御を実現。

WAF Default Rule Set

OWASP Top 10 などの一般的な Web 脆弱性に対するマネージドルールセット。HTTP DDoS Ruleset と併用して多層防御を構築。

WAF Bot Manager Rule Set

既知の悪意あるボットに対する防御ルールセット。HTTP DDoS Ruleset と補完的に動作し、ボット攻撃全般をカバー。

Azure Monitor / Log Analytics

WAF ログの収集・分析基盤。HTTP DDoS Ruleset のルール一致状況やブロックされたリクエストの監視に活用。
