Python 2.7/3.8 および PowerShell 7.1/7.2 ランタイムの廃止

📅 2026年7月10日 発表 ｜ ⏰ 廃止日: 2026年9月30日

概要

2026 年 9 月 30 日をもって、Azure Automation における Python 2.7、Python 3.8、PowerShell 7.1、PowerShell 7.2 のランタイムサポートが廃止されます。これらは親プロダクトで既にサポートが終了しているバージョンであり、セキュリティリスクの観点からも速やかな移行が推奨されます。

主要ポイント

📅

ランタイム廃止スケジュール

廃止日: 2026年9月30日。対象: Python 2.7、Python 3.8、PowerShell 7.1、PowerShell 7.2。廃止後も Runbook の実行は可能だがサポートなし。

🔄

推奨移行先

Python Runbook は Python 3.10 へ、PowerShell Runbook は PowerShell 7.4 (LTS) へアップグレード。PowerShell 5.1 は引き続きサポート対象。

🔒

セキュリティリスク低減

親プロダクトで EOL 済みのバージョンを使い続けることで、脆弱性への対応が受けられない状態を解消。LTS 版への移行でセキュリティパッチの継続的な提供を確保。

🚀

新機能の活用

Python 3.10 のパターンマッチング・型ヒント改善、PowerShell 7.4 の最新機能とパフォーマンス改善を活用可能に。

⚠️

移行時の注意点

PowerShell 7.x ではワークフロー (Workflow) 非サポート。ソース管理統合は PowerShell 7.4 未対応。Python 2.7→3.10 は構文変更を伴う大幅な改修が必要。

制限事項・注意点

⚠️ 重大な制約事項

  • PowerShell 7.x ではワークフロー (Workflow Runbook) がサポートされない — PowerShell 5.1 への移行またはワークフロー不使用の書き換えが必要
  • ソース管理統合は現時点で PowerShell 7.4 をサポートしていない
  • Python 2.7 から Python 3.10 への移行は print 文の構文変更、文字列処理の変更、ライブラリ互換性など大幅なコード改修が必要

📍 リージョン制限

  • PowerShell 7.4 / Python 3.10 は Brazil Southeast リージョンで利用不可
  • Gov クラウドは非対応

ユースケース

ユースケース 1: PowerShell Runbook の移行

PowerShell 7.2 で動作するリソース管理用 Runbook を PowerShell 7.4 へ移行する

bash 
# 現在の Runbook のランタイムバージョンを確認
az automation runbook list \
  --resource-group myResourceGroup \
  --automation-account-name myAutomationAccount \
  --query "[?runbookType=='PowerShell72'].{Name:name, Type:runbookType}" \
  --output table

# PowerShell 7.4 ランタイム環境を作成
az automation runtime-environment create \
  --resource-group myResourceGroup \
  --automation-account-name myAutomationAccount \
  --name "PS74-Runtime" \
  --location japaneast \
  --language "PowerShell" \
  --version "7.4"

✅ 効果: セキュリティアップデートの継続的な提供を受けつつ、最新の PowerShell 機能を活用可能

ユースケース 2: Python Runbook の移行

Python 3.8 で動作する監視スクリプトを Python 3.10 へ移行する

bash 
# Python 3.10 ランタイム環境を作成
az automation runtime-environment create \
  --resource-group myResourceGroup \
  --automation-account-name myAutomationAccount \
  --name "Python310-Runtime" \
  --location japaneast \
  --language "Python" \
  --version "3.10"

✅ 効果: Python 3.10 の新機能 (構造パターンマッチング等) を活用した効率的なスクリプト開発が可能

移行手順 (Azure CLI)

bash 
# 1. Automation アカウント内の Runbook 一覧を取得
az automation runbook list \
  --resource-group <リソースグループ名> \
  --automation-account-name <Automationアカウント名> \
  --output table

# 2. 特定の Runbook の詳細情報を確認 (ランタイムバージョン含む)
az automation runbook show \
  --resource-group <リソースグループ名> \
  --automation-account-name <Automationアカウント名> \
  --name <Runbook名> \
  --query "{name:name, runbookType:runbookType, state:state}" \
  --output table

# 3. PowerShell 7.4 ランタイム環境を作成
az automation runtime-environment create \
  --resource-group <リソースグループ名> \
  --automation-account-name <Automationアカウント名> \
  --name "PowerShell-7.4-env" \
  --location <リージョン> \
  --language "PowerShell" \
  --version "7.4"

# 4. Python 3.10 ランタイム環境を作成
az automation runtime-environment create \
  --resource-group <リソースグループ名> \
  --automation-account-name <Automationアカウント名> \
  --name "Python-3.10-env" \
  --location <リージョン> \
  --language "Python" \
  --version "3.10"

Azure Portal での移行手順

  1. Azure Portal で Automation アカウントに移動
  2. 「プロセス オートメーション」>「Runbook」を選択
  3. 各 Runbook のランタイムバージョンを確認
  4. 廃止対象バージョンを使用している Runbook を特定
  5. Runbook を編集し、新しいランタイム環境 (PowerShell 7.4 / Python 3.10) を選択
  6. テスト実行で動作を確認後、公開

推奨アクション

  1. 現在使用中の Runbook のランタイムバージョンを棚卸しする
  2. 廃止対象バージョンを使用している Runbook を特定する
  3. PowerShell 7.4 (LTS) または Python 3.10 への移行計画を策定する
  4. PowerShell ワークフローを使用している場合は代替手段を検討する
  5. テスト環境で移行後の動作を検証し、2026年9月30日までに本番環境へ適用する
