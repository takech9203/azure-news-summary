概要

Microsoft Azure は、Premium SSD v2 および Ultra Disk に対するクロステナント カスタマー マネージド キー (CMK) 暗号化の一般提供 (GA) を発表した。この機能により、マネージド ディスクを別の Microsoft Entra テナントに配置された Azure Key Vault に格納されたカスタマー マネージド キーで暗号化できるようになる。
  • 異なるテナントの Azure Key Vault に格納されたキーで Premium SSD v2 / Ultra Disk を暗号化可能
  • SaaS プロバイダーや MSP が顧客テナントのキーで自社テナント内のディスクを暗号化する BYOK シナリオを実現
  • フェデレーション ID 資格情報を利用したシークレット共有不要の安全なクロステナントアクセス
  • 3 フェーズ構成: SP 側 ID 構成 → 顧客側アクセス許可 → DES 作成・暗号化

Before

  • Premium SSD v2 / Ultra Disk では異なるテナント間の CMK 暗号化が非サポート
  • マルチテナント環境での BYOK シナリオが困難
  • コンプライアンス要件がある場合、テナント分離が障壁

After

  • 異なるテナントの Key Vault キーで暗号化可能
  • SaaS / MSP シナリオでの顧客管理キー利用が容易に
  • フェデレーション ID 資格情報による安全なクロステナントアクセス

サービスプロバイダー (Tenant 1) のディスク暗号化セットが、顧客 (Tenant 2) の Key Vault に格納されたカスタマー マネージド キーを使用してディスクを暗号化するクロステナント構成。フェデレーション ID 資格情報によりシークレット共有なしで安全にアクセスする。

⭐

主要機能

🔐

クロステナント CMK 暗号化

異なる Microsoft Entra テナントの Azure Key Vault に格納されたカスタマー マネージド キーで Premium SSD v2 および Ultra Disk を暗号化できる。暗号化タイプは EncryptionAtRestWithCustomerKey を使用し、テナント境界を超えたキー管理を実現する。

💪

フェデレーション ID 資格情報

ユーザー割り当てマネージド ID にフェデレーション ID 資格情報を設定し、マルチテナント アプリ登録と関連付ける。シークレットの共有や定期的なローテーションが不要な安全なクロステナントアクセスを実現する。

🛠

3 フェーズ構成プロセス

構成はサービスプロバイダー側の ID 構成 (マルチテナントアプリ登録 + マネージド ID + フェデレーション資格情報)、顧客側のアクセス許可 (サービスプリンシパル + Key Vault RBAC ロール割り当て)、DES 作成とディスク暗号化の 3 フェーズで完了する。

🔒

Azure Key Vault RBAC 統合

顧客テナント側の Key Vault は Azure RBAC を使用し、サービスプリンシパルに Key Vault Crypto Service Encryption User ロールを割り当てる。Purge Protection が有効な Key Vault が必須であり、キーの誤削除を防止する。

⚙

構成手順

3 フェーズ構成

1

サービスプロバイダー側: ID 構成

マルチテナント アプリ登録を作成し、ユーザー割り当てマネージド ID を作成。マネージド ID にフェデレーション ID 資格情報を設定してアプリと関連付ける。

2

顧客側: アクセス許可

サービスプリンシパル (アプリのインスタンス) を顧客テナントに作成。Key Vault で Azure RBAC を有効化し、Purge Protection を有効化。サービスプリンシパルに Key Vault Crypto Service Encryption User ロールを割り当てる。

3

サービスプロバイダー側: DES 作成・暗号化

ディスク暗号化セットを作成し、顧客の Key Vault キー URI、マネージド ID、アプリのクライアント ID を指定。ディスクを暗号化セットで暗号化する。

ユースケース

🏦

SaaS BYOK 対応

金融業界の顧客向けに高性能データベース (Premium SSD v2) を提供する SaaS プロバイダーが、規制要件に対応するために顧客管理キーでのディスク暗号化を求められている。
効果: 顧客テナントの Key Vault に格納されたキーでディスクを暗号化し、BYOK 要件を満たす。顧客がキーのライフサイクルを完全に管理でき、金融規制への準拠を実現する。
🏥

MSP Ultra Disk ホスティング

マネージド サービス プロバイダー (MSP) が医療機関向けに低レイテンシストレージ (Ultra Disk) をホスティングし、HIPAA 準拠のキー管理が必要。
効果: 医療機関の顧客テナントでキーを管理し、MSP テナント内の Ultra Disk を暗号化。HIPAA の暗号化要件とキー管理要件を同時に満たし、患者データの保護を強化する。

⚠

制限事項・注意点

  • ディスクと Key Vault は同一リージョンに配置する必要がある
  • フェデレーション資格情報はアプリあたり最大 20 個まで
  • 除外リージョン: US Gov Virginia / Arizona / Texas、China North 3
  • ARM テンプレートでの Entra ID アプリ作成は非推奨
  • Key Vault は Azure RBAC が有効で、Purge Protection が有効化されている必要がある
  • 暗号化タイプは EncryptionAtRestWithCustomerKey を使用する必要がある
