概要

Ephemeral OS Disk with Full Caching が Azure VM および VMSS 向けにパブリックプレビューとして提供開始された。従来の Ephemeral OS Disk では初回読み取り時にリモート Azure Storage からデータを取得する必要があったが、本機能では OS ディスクイメージ全体を VM のローカルストレージ (キャッシュディスク、リソースディスク、NVMe) にキャッシュすることで、OS ディスクのパフォーマンスを大幅に高速化・安定化する。

Before (従来の Ephemeral OS Disk)

  • 初回読み取り時にリモート Azure Storage からデータ取得が必要でレイテンシが発生
  • VM 再イメージやスケールセットのスケールアウト時に OS ディスクデータフェッチに時間がかかる
  • ステートレスワークロードのデプロイ・再イメージ時間の短縮が求められていた

After (Full Caching)

  • OS ディスクイメージ全体がローカル VM ストレージにキャッシュされ、全ディスクアクセスが高速化
  • VM 起動時間・再イメージ時間がさらに短縮
  • リモートストレージ依存を排除し、一貫した低レイテンシ OS ディスクパフォーマンスを実現

従来の Ephemeral OS Disk ではキャッシュミス時にリモート Azure Storage へのフェッチが発生していたが、Full Caching ではデプロイ時に OS イメージ全体をローカルストレージにコピーし、以降のすべてのディスク I/O がローカルで完結する。

主要機能

💾 OS ディスクイメージの完全キャッシュ

OS ディスクイメージ全体をローカルストレージにキャッシュし、初回読み取り時のリモートアクセスを排除。従来のキャッシュミスによるレイテンシスパイクが解消され、すべてのディスク I/O が一貫して高速に処理される。

🛠 VM / VMSS 両対応

個別の Azure VM に加え、Virtual Machine Scale Sets (Flexible / Uniform) でも利用可能。スケールアウト時の新規インスタンス起動速度が向上し、オートスケーリングの応答性が改善される。

⚡ 既存 Ephemeral OS Disk の拡張

既存の Ephemeral OS Disk の配置オプション (NVMe v6、Temp disk v5、Cache disk v2/v3) と連携。ローカルストレージの種類に応じた最適な配置が可能。

制約事項・注意点

  • データ非永続性: VM の再デプロイやサイズ変更を行うとデータが消失する。ステートレスなワークロードでの使用が前提。
  • Stop-deallocate 非対応: VM を停止 (割り当て解除) するとデータが失われる。コスト削減のための停止運用には不向き。
  • バックアップ・DR 非対応: Azure Backup、Azure Site Recovery、ディスクスナップショット、VM イメージキャプチャがいずれも非対応。
  • 暗号化制限: Azure Disk Encryption が使用できない。セキュリティ要件に応じた代替手段の検討が必要。
  • OS ディスクリサイズ: ディスクサイズの変更は作成時のみ可能。運用中のリサイズは不可。
  • ローカルストレージ容量制約: VM サイズのローカルストレージ容量が OS ディスクイメージサイズ以上である必要がある。
  • パブリックプレビュー段階: SLA の対象外。本番ワークロードでの利用は十分な検証が必要。
