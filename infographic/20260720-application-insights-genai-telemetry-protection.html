生成 AI テレメトリのセンシティブデータ保護

GenAIContent 専用テーブルと Protected Tables による AI コンテンツのアクセス制御

2026-07-20

📋 概要

Azure Monitor Application Insights において、生成 AI コンテンツが専用の GenAIContent テーブルに格納されるようになった。センシティブな AI テレメトリに対して Protected Tables 機能でアクセス制御を適用でき、「deny by default」モデルによりプロンプト・レスポンスデータへの不正アクセスを防止する。

追加課金なし
通常の Log Analytics 料金

Before

  • 生成 AI のプロンプト・レスポンスデータが通常のテレメトリと同じテーブルに格納
  • AI コンテンツに含まれる機密情報へのアクセスを選択的に制限する手段がなかった
  • Log Analytics ワークスペースへの読み取り権限を持つ全ユーザーが AI の入出力内容を閲覧可能

After

  • 生成 AI コンテンツが GenAIContent (AppGenAIContent) 専用テーブルに格納・分離
  • Protected Tables 機能により「deny by default」モデルでテーブルレベルのアクセス制御
  • Privileged Monitoring Data Reader ロールや ABAC 条件付きカスタムロール
  • Microsoft Entra PIM と組み合わせた期間限定のジャストインタイムアクセス

⭐ 主要機能

📦

GenAIContent 専用テーブル

生成 AI のプロンプト・レスポンスデータが自動的に GenAIContent / AppGenAIContent テーブルに格納され、通常のテレメトリデータと物理的に分離される。

🔒

Protected Tables によるアクセス制御

「deny by default」モデルにより、明示的な権限付与がない限り Protected Table 内のデータにアクセスできない。テーブルレベルでの細粒度アクセス制御を実現。

👤

Privileged Monitoring Data Reader ロール

保護されたテーブルへのアクセスを許可する組み込み Azure ロール。必要なユーザーにのみこのロールを割り当てることで最小権限の原則を適用。

🎯

ABAC (属性ベースアクセス制御) 条件

カスタムロールに ABAC 条件を設定し、protectionLevel 属性に基づいてアクセスを制御。より柔軟なアクセスポリシーの構築が可能。

🚫

DataActionsOnly モード

コントロールプレーン操作からの暗黙的なデータアクセスを遮断。ARM API 経由でのデータ読み取りを防止し、データプレーン経由のアクセスのみを許可。

💡 ユースケース

💬 エンタープライズ AI チャットボットの監視

シナリオ: 社内の AI チャットボットを運用する組織で、SRE チームはパフォーマンス監視を行い、AI 品質管理チームは会話内容をレビューする必要がある
効果: SRE チームにはパフォーマンスメトリクス (AppRequests, AppTraces) のみアクセスを許可し、AI 品質管理チームのみが AppGenAIContent テーブルの会話内容にアクセスできるよう権限を分離。機密性の高い顧客との会話データを適切に保護。

🚨 インシデント対応時の一時アクセス

シナリオ: AI モデルの出力品質に関するインシデントが発生し、エンジニアが一時的に生成 AI コンテンツを確認する必要がある
効果: Microsoft Entra PIM を通じて Privileged Monitoring Data Reader ロールを一時的に付与。インシデント対応完了後は自動的にアクセスが失効し、不要な常時アクセスを排除。監査ログによりアクセス履歴も記録される。

⚠ 制限事項・注意点

  • 現在プレビュー段階であり、GA 時に仕様が変更される可能性がある
  • クロスワークスペースクエリは未サポート

  • テーブルスキーマは保護レベルに関係なく全ユーザーに可視
