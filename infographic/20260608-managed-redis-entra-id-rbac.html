Azure Managed Redis In Preview

Entra ID ベースの RBAC によるデータアクセス制御

2026-06-08 | セキュリティ / アクセス制御

概要

Azure Managed Redis において、Microsoft Entra ID を使用したロールベースアクセス制御 (RBAC) によるデータアクセス管理機能がパブリックプレビュー開始。Redis ACL 構文を活用したカスタムアクセス文字列により、コマンドレベル・キーパターンレベルでの最小権限アクセスを実現。

❌ Before

  • ・Entra ID 認証でも全ユーザーがフルアクセス権
  • ・共有アクセスキーの漏洩リスク
  • ・マルチテナント環境でのデータ分離手段なし

✅ After

  • ・ユーザーごとに Redis ACL ベースの権限設定
  • ・パスワードレス + 最小権限アクセス
  • ・ARM テンプレート / REST API で IaC 対応

主要機能

🔑

カスタムアクセス文字列

Redis ACL 構文 (+@read, +@write, ~cache:* 等) でコマンドカテゴリとキーパターンを制限。ユーザーごとに精密な権限設定が可能。

🖥️

Azure Portal からの設定

Authentication 画面から「Custom data access policy (preview)」を選択し、アクセス文字列を入力してユーザーに割り当て。

📝

ARM テンプレート / REST API 対応

accessPolicyAssignments リソースとして管理。Infrastructure as Code による一貫した権限管理が可能。

⚡

インプレース権限更新

ユーザーの接続を切断することなく、アクセス文字列を変更して権限を即時更新可能。

サンプルコード

ARM テンプレートによるアクセスポリシー割り当て

JSON 
{
    "type": "Microsoft.Cache/redisEnterprise/databases/accessPolicyAssignments",
    "apiVersion": "2026-05-01-preview",
    "name": "[concat(parameters('cacheName'), '/default/', parameters('assignmentName'))]",
    "properties": {
        "accessPolicyName": "default",
        "accessString": "[parameters('accessString')]",
        "user": {
            "objectId": "[parameters('objectId')]"
        }
    }
}

アクセス文字列の例

Redis ACL 
# テナント A: テナント A のキーのみ全操作可能
+@all ~tenantA:*

# 読み取り専用: cache プレフィックスのキーのみ
+@read ~cache:*

# セッション管理: session キーへの読み書きのみ
+@read +@write ~session:*

ユースケース

🏢 マルチテナント SaaS アプリケーション

複数テナントが同一 Redis インスタンスを共有する SaaS で、テナント間のデータ分離を実現。各テナントは自身のキープレフィックスのみにアクセス可能。

📖 読み取り専用キャッシュクライアント

レポーティングサービスがキャッシュを読み取るが書き込み・削除は禁止。誤操作やバグによるデータの破壊を防止。

🔧 マイクロサービスごとの権限分離

セッション管理サービスは session:* のみ、アプリデータサービスは app:* のみにアクセス。サービス間の関心の分離を強制。

制限事項・注意点

  • プレビュー段階のため、GA 時に仕様変更の可能性あり
  • API バージョン 2026-05-01-preview 以降が必要 (古い API では常にフルアクセス)
  • Microsoft Entra ID グループは非サポート (個々のユーザー/SP 単位)
  • SSL 接続のみでの認証サポート
  • データベースあたりユーザー 1 つのアクセスポリシー割り当てに限定
  • キーパターンは大文字小文字を区別
