Azure Red Hat OpenShift GA (一般提供) AI / ML HPC GPU

NVIDIA H100 / H200 GPU サポートの一般提供開始

リリース日: 2026-04-07 サービス: Azure Red Hat OpenShift (ARO)

概要

Azure Red Hat OpenShift (ARO) が、NVIDIA H100 および H200 GPU を搭載した Azure VM SKU のサポートを GA 開始。フルマネージドの OpenShift サービス上で大規模な AI/ML および HPC ワークロードを実行可能に。

ARO はこれまで T4、V100、A100 をサポートしていたが、最新の Hopper アーキテクチャ (H100) と後継 H200 が追加された。H100 は A100 比で AI トレーニング/推論のパフォーマンスが大幅向上。H200 は HBM3e メモリで LLM 推論に最適化。H100/H200 は「Day-2」オペレーションとしてのみサポートされ、ARO 4.19 以降が必要。クラスター作成後に MachineSet を追加する形で利用する。

Before (従来)

  • 最新 Hopper アーキテクチャ GPU が利用できなかった
  • A100 では GPU メモリやスループットが不足するケースが増加
  • 最新 GPU には ARO 以外の環境を選択する必要があった

After (アップデート後)

  • H100 (80GB HBM3) / H200 (141GB HBM3e) が ARO ワーカーノードとして利用可能
  • OpenShift の統一運用管理で最先端 GPU を活用
  • NVIDIA GPU Operator と NFD で自動 GPU 検出・ドライバ管理
  • InfiniBand 対応で分散トレーニング実現

ARO クラスター上で GPU ワーカーノード (H100/H200) を Day-2 オペレーションとして追加し、NVIDIA GPU Operator と Node Feature Discovery によって GPU リソースを自動管理する構成。AI/ML および HPC ワークロードは nodeSelector を使用して GPU ノードにスケジューリングされる。

主要機能

1. NVIDIA H100 GPU サポート

96 vCPU、1900 GiB RAM、8 基の NVIDIA H100 Tensor Core GPU (各 80 GB HBM3)。400 Gb/s InfiniBand 接続によるマルチノード分散トレーニング対応。FP8 Transformer Engine による AI トレーニング / 推論の高速化。

2. NVIDIA H200 GPU サポート

96 vCPU、1850 GiB RAM、8 基の NVIDIA H200 Tensor Core GPU (各 141 GB HBM3e)。H100 比で約 1.8 倍の GPU メモリ容量により、大規模モデルの推論パフォーマンスが向上。400 Gb/s InfiniBand 接続。

3. Day-2 MachineSet スケーリング

クラスター作成後に GPU ワーカーノードを MachineSet として追加する運用モデル。GPU ノード数の増減を MachineSet のレプリカ数で制御可能。GPU ワークロードの需要に応じた柔軟なスケーリング。

4. NVIDIA GPU Operator / NFD

Node Feature Discovery Operator が GPU ハードウェアを自動検出しノードにラベルを付与。NVIDIA GPU Operator がドライバ、コンテナランタイム、デバイスプラグイン、モニタリングツールを自動デプロイ。

設定コード

GPU MachineSet 作成

# GPU MachineSet 作成
MACHINESET=$(oc get machineset -n openshift-machine-api -o=jsonpath='{.items[0]}' | jq -r '[.metadata.name] | @tsv')
oc get machineset -n openshift-machine-api $MACHINESET -o json > gpu_machineset.json
jq '.spec.replicas = 1' gpu_machineset.json | sponge gpu_machineset.json
jq '.spec.template.spec.providerSpec.value.vmSize = "Standard_ND96isr_H100_v5"' gpu_machineset.json | sponge gpu_machineset.json
oc create -f gpu_machineset.json

GPU 動作確認

# GPU 動作確認
oc describe node | egrep 'Roles|pci-10de' | grep -v master
for i in $(oc get pod -lopenshift.driver-toolkit=true --no-headers | awk '{print $1}'); do
  oc exec -it $i -- nvidia-smi
done

ユースケース

ユースケース 1: LLM ファインチューニング

H100 x 4 ノード = 32 GPU で 70B パラメータモデルのファインチューニングを数時間で完了。InfiniBand でノード間通信最小化。

32 GPU の分散トレーニングにより、大規模モデルのファインチューニング時間を大幅短縮

ユースケース 2: LLM 推論サービング

H200 の 141GB HBM3e で 70B+ モデルを 1 ノード (8GPU) で完全収容。テンソル並列化の分割数削減で推論レイテンシ向上。

大容量 HBM3e メモリにより、モデル分割数を削減し推論スループットが向上

ユースケース 3: HPC / 科学技術計算

製薬企業の分子動力学シミュレーション。OpenShift のジョブスケジューリング + InfiniBand 対応 H100 で効率的に実行。

コンテナ化された HPC 環境で、研究者がセルフサービスで GPU リソースを利用可能

料金

GPU VM はリージョン・契約形態により異なる。リザーブドインスタンス (1 年 / 3 年) で大幅コスト削減可能。ARO OpenShift ライセンス料が別途発生。

制約事項・注意点

制約事項

  • Day-2 オペレーションのみ: クラスター作成時のインストールオプションとしては不可
  • ARO 4.19 以降が必須: 既存クラスターのアップグレードが必要な場合がある
  • Generation 2 VM が必須: MachineSet の image SKU を v2 バージョンに設定する必要がある

注意事項

  • GPU クォータはデフォルト 0: Azure Portal から事前にクォータ増加を申請する必要がある
  • 96 vCPU/ノード: サブスクリプション全体の vCPU クォータへの影響が大きい
  • 高額な GPU VM 料金: コスト管理に十分な注意が必要。リザーブドインスタンスの活用を推奨
