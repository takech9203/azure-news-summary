Azure Infrastructure Resiliency Manager

統合レジリエンシー管理サービス

2026-06-03 | Microsoft Build 2026

In Preview

概要

Azure Infrastructure Resiliency Manager がパブリックプレビューとして提供開始。Azure 上のアプリケーションレジリエンシーを設計、評価、改善するための統合的かつゴール駆動型のエクスペリエンスを提供する。Availability Zones、Azure Advisor、Chaos Studio などの既存機能を横断的に統合し、単一のインターフェースからレジリエンシー態勢を管理できる。

🛡 Resiliency in Azure プラットフォーム - 従来の Azure Business Continuity Center (ABCC) を拡張・発展させたもの
🌐 グローバル (非リージョン) サービス - 任意の Azure リージョンのリソースを横断的に管理・運用
📑 3 つの柱 - Infrastructure Resiliency (ゾーン)、Data Resiliency (バックアップ/DR)、Cyber Recovery (ランサムウェア対策)

Before / After

Before

  • レジリエンシー管理が複数サービスに分散しており統合的な把握が困難
  • アプリケーション単位でのレジリエンシーゴール設定や達成状況追跡ができなかった
  • ゾーン障害に対する準備状況を検証するためのシミュレーションドリルが体系化されていなかった
  • 障害発生時のリカバリー順序を定義しオーケストレーションする仕組みが不足

After

  • Service Group によるアプリケーション単位のリソースグループ化とゴール設定が可能
  • ゾーンレジリエンシーの達成状況を Zone-resilient / Non zone-resilient / Not evaluated で一目で把握
  • Availability Zone Down Drill によりゾーン障害をシミュレートしフェイルオーバーの準備状況を事前検証
  • Recovery Orchestration Plan により複数リソースの復旧順序を定義し自動フェイルオーバーが実行可能

主要機能

🎯

Goals and Recommendations

Service Group を作成しリソースをグループ化。ゾーンレジリエンシーゴールを設定し達成状況をリアルタイムで追跡。

  • サブスクリプションやリソースグループを横断したリソースグループ化
  • ステップバイステップの修復ガイダンスを提供
  • Manual Attestation によるカスタムレジリエンシーソリューションの手動登録
  • ゴール達成状況のリアルタイム追跡
⚡

Availability Zone Down Drills

Azure 推奨のフォールトテンプレートによるゾーン障害シミュレーション。フェイルオーバーの準備状況を事前検証。

  • Azure Runbooks によるカスタムロジックでのフォールトオーバーライド
  • ドリル実行中のリアルタイムヘルスモニタリング
  • フェイルオーバー、リプロテクト、逆フェイルオーバーの完全なライフサイクル管理
  • Azure Chaos Studio 連携による障害注入
🔄

Recovery Orchestration Plan

カスタマイズ可能なグループによるリカバリー順序の定義。自動化されたフェイルオーバーワークフロー。

  • 自動 Readiness Check による 24 時間ごとのドリフト検出
  • Azure Automation Runbooks による pre-script / post-script 実行
  • 手動アクション挿入による柔軟なフェイルオーバーワークフロー
  • カスタマイズ可能なグループでリカバリー順序を定義
📊

Resiliency Posture Dashboard

Zone-resilient / Non zone-resilient / Not evaluated の 3 区分でリソース状態を可視化。全 Service Group 横断のスケーラブルなビュー。

  • Azure Advisor と連携した推奨事項の統合表示
  • 全 Service Group を横断したスケーラブルなビュー
  • 経営層にも分かりやすいレジリエンシー態勢の可視化

ユースケース

🏢 エンタープライズアプリのゾーンレジリエンシー評価

複数サブスクリプションにまたがる基幹業務アプリケーションのゾーン障害耐性を包括的に評価。改善計画を策定。

アプリケーション全体のゾーンレジリエンシー達成率を定量的に把握し、優先順位を付けた改善計画を策定できる。

🕘 定期的なゾーン障害ドリルによる DR 準備状況検証

四半期ごとにゾーン障害シミュレーションを実施しフェイルオーバー手順の有効性を検証。RTO を計測。

実際のゾーン障害発生時の対応手順を事前に検証し、RTO/RPO の実測値に基づく改善が可能。

👥 マルチチーム環境でのレジリエンシーガバナンス

複数チームの環境で組織全体のレジリエンシー基準を一元管理。チームの自律性を維持しながらガバナンスを実現。

組織全体のレジリエンシー基準を強制することなく可視化し、チームの自律性を維持しながらガバナンスを実現。

制約事項

  • パブリックプレビュー段階のため、本番ワークロードでの使用は慎重な検討が必要
  • Recovery Plan はゾーンリカバリーのみ対応 (リージョナルリカバリーは非対応)
  • Service Group あたり 500 リソース上限。大規模アプリケーションでは分割が必要
  • 一部リソースタイプ (ネットワークプリミティブ、ID/アクセス、モニタリング等) は自動的に除外される
  • ゾーン障害ドリルはテスト環境での事前検証を推奨
