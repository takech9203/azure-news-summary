概要: Before / After

Before (旧世代 VM)

  • SCSI ディスクコントローラー (低速 I/O)
  • 標準ネットワークアダプター
  • Generation 1 VM (レガシーブート)
  • NVMe 非対応
  • 価格対性能比が劣る
  • キャパシティ制限 (一部リージョン)

After (v5/v6/v7 シリーズ)

  • NVMe ディスクコントローラー (高速 I/O)
  • MANA ネットワークアダプター (v6+)
  • Generation 2 VM (UEFI / セキュアブート)
  • NVMe ネイティブサポート
  • 改善された価格対性能比
  • 広いリージョン可用性

移行時の制約事項

NVMe OS イメージ必須 (v6)

v6 シリーズへの移行には NVMe 対応 OS イメージが必要。カスタムイメージの再構築が必要になる場合があります。

Gen2 VM 必須 (v6+)

v6 以降は Generation 2 VM のみサポート。Generation 1 VM イメージからの移行作業が発生します。

プール再作成が必要

既存 Batch プールの VM サイズはインプレース変更不可。新プール作成 → ジョブ切り替え → 旧プール削除の手順が必要。

MANA 対応 OS 必要 (v6)

v6 シリーズは Microsoft Azure Network Adapter (MANA) 対応 OS が必要。OS の互換性を事前確認してください。

リージョン可用性

v6 シリーズのリージョン可用性が限定的な場合あり。利用不可の場合は v5 シリーズを検討してください。

クォータ確認

移行先 VM シリーズのクォータが十分か確認。不足の場合はサポートリクエストで増加申請が必要です。

リザーブドインスタンス (RI) への影響

RI 購入・更新への影響

  • 2026年7月1日以降: 対象 VM シリーズの 1 年 RI の新規購入・更新が不可
  • 既存 RI: 元の期間満了まで有効 (途中キャンセル不要)
  • 推奨アクション: Azure Savings Plan for Compute への切り替え、または新世代 VM の RI 購入を検討
  • RI 交換: 既存 RI を新世代 VM シリーズの RI に交換可能 (Azure ポータルから申請)

移行手順 (Azure CLI)

Bash - 現在のプール確認 
# 現在の Batch プールで使用中の VM サイズを確認
az batch pool list --account-name <batch-account-name> \
  --query "[].{Name:id, VMSize:vmSize}" -o table

# リージョンでサポートされている VM SKU と EOL 日を確認
az batch location list-skus --location <region> \
  --query "[?contains(name, 'Standard_D') || contains(name, 'Standard_F')].{Name:name, EOL:batchSupportEndOfLife}" -o table
Bash - 新プール作成と移行 
# 新しいプールを作成 (移行先 VM シリーズを使用)
az batch pool create --account-name <batch-account-name> \
  --id pool-dsv5-migration \
  --vm-size Standard_D4s_v5 \
  --target-dedicated-nodes 10 \
  --image "canonical:0001-com-ubuntu-server-jammy:22_04-lts:latest" \
  --node-agent-sku-id "batch.node.ubuntu 22.04"

# ジョブを新しいプールに移行
az batch job set --job-id <job-id> \
  --pool-id pool-dsv5-migration

# 旧プールの削除 (移行完了後)
az batch pool delete --account-name <batch-account-name> \
  --pool-id <old-pool-id> --yes

💾

ストレージ集約型ワークロード

Ls/Lsv2 シリーズを使用した大量ファイル処理やログ解析バッチ。

移行先: Lsv3/Lsv4 シリーズ。NVMe ローカルストレージにより I/O 性能が大幅向上。

Av2 シリーズを使用した開発/テスト用バッチプールや低コスト処理。

移行先: Bsv2/Basv2 シリーズ。バースト可能な低コスト VM で同等用途をカバー。
