Microsoft Entra Server Principals (Logins)

Azure SQL Database でサーバーレベルのログイン管理が GA - SQL ログインと同等の機能を Entra ID で実現

📅 2026-06-10 🏷 General Availability 🔒 ID & Security

📋 概要

Microsoft Entra サーバー プリンシパル (ログイン) が Azure SQL Database で一般提供 (GA) になりました。CREATE LOGIN ... FROM EXTERNAL PROVIDER を仮想 master データベースで実行することにより、Entra ユーザー、グループ、サービス プリンシパルをサーバーレベルのログインとして作成できます。SQL ログインとの機能パリティを実現し、エンタープライズ環境での一元的なアクセス管理と CI/CD 自動化を強力にサポートします。

❌ Before (従来の方式)

  • 各データベースに含まれるユーザーを個別作成
  • サーバーレベルのロール割り当て不可
  • ログインの一括無効化ができない
  • SQL 認証の完全無効化が困難
  • Geo レプリカ間でユーザー管理が分散
  • サービスプリンシパルによる管理操作に制限

✅ After (Entra Server Principals)

  • 仮想 master DB でサーバーレベルログイン作成
  • サーバーロール (##MS_DefinitionReader## 等) 割当可
  • ALTER LOGIN DISABLE で即座にアクセス遮断
  • Entra-only 認証で SQL 認証を完全無効化
  • ログインベースユーザーで Geo レプリカ一括管理
  • サービスプリンシパルで完全な CI/CD 自動化

⭐ 主要機能

🔑

サーバーレベル ログイン (Server-level Logins)

仮想 master データベースで CREATE LOGIN ... FROM EXTERNAL PROVIDER を実行し、Entra ユーザー・グループ・サービスプリンシパルをサーバーレベルのログインとして作成。SQL ログインと同等の管理粒度を実現します。

T-SQL サーバーレベル ログインの作成 
-- Entra ユーザーのサーバーログインを作成
CREATE LOGIN [alice@contoso.com] FROM EXTERNAL PROVIDER;

-- Entra グループのサーバーログインを作成
CREATE LOGIN [DBAdmins_Group] FROM EXTERNAL PROVIDER;

-- サービスプリンシパルのサーバーログインを作成
CREATE LOGIN [cicd-service-principal] FROM EXTERNAL PROVIDER;
👥

ログインベース ユーザー (Login-based Users)

サーバーログインに紐づくデータベースユーザーを作成。ログイン1つで複数データベースのアクセスを一元管理でき、Geo レプリカ間でも同一ログインで透過的にアクセスできます。

T-SQL ログインベース ユーザーの作成 
-- ユーザーデータベースでログインベースユーザーを作成
CREATE USER [alice@contoso.com] FROM LOGIN [alice@contoso.com];

-- デフォルトスキーマを指定して作成
CREATE USER [alice@contoso.com] FROM LOGIN [alice@contoso.com]
    WITH DEFAULT_SCHEMA = [Sales];
🛡

サーバーロール割り当て (Server Roles)

Entra ログインにサーバーレベルのロールを付与可能。##MS_DefinitionReader##、##MS_ServerStateReader##、##MS_SecurityDefinitionReader## などのビルトインロールを活用できます。

T-SQL サーバーロールの割り当て 
-- サーバーステート読み取りロールを付与
ALTER SERVER ROLE ##MS_ServerStateReader## ADD MEMBER [alice@contoso.com];

-- 定義読み取りロールを付与
ALTER SERVER ROLE ##MS_DefinitionReader## ADD MEMBER [alice@contoso.com];

-- セキュリティ定義読み取りロールを付与
ALTER SERVER ROLE ##MS_SecurityDefinitionReader## ADD MEMBER [DBAdmins_Group];
🚫

ログイン無効化/有効化 (Login Disable/Enable)

ALTER LOGIN ... DISABLE で即座にアクセスを遮断。退職者やセキュリティインシデント発生時に、全データベースへのアクセスを一括で無効化できます。

T-SQL ログインの無効化と有効化 
-- ログインを即座に無効化 (全データベースへのアクセスを遮断)
ALTER LOGIN [alice@contoso.com] DISABLE;

-- ログインを再有効化
ALTER LOGIN [alice@contoso.com] ENABLE;
🔒

Entra-only 認証 (Entra-only Authentication)

SQL 認証を完全に無効化し、Entra 認証のみを許可。パスワードベースの攻撃リスクを排除し、条件付きアクセスや MFA をすべての接続に強制できます。

💼 ユースケース

1

エンタープライズ統合アクセス管理

大規模組織で複数の Azure SQL Database を運用。DBA チームの Entra グループに対してサーバーレベルログインを作成し、全データベースの管理権限を一元付与。退職時は ALTER LOGIN DISABLE で即座に全アクセスを遮断。

✅ 管理工数の大幅削減。セキュリティインシデント時の即座のアクセス遮断。監査証跡の一元化。
2

CI/CD パイプラインの完全自動化

サービスプリンシパルのサーバーログインを作成し、GitHub Actions / Azure DevOps からのデータベースマイグレーション、スキーマデプロイ、テスト実行を完全自動化。パスワード管理が不要で、マネージド ID との統合も可能。

✅ パスワードレスな CI/CD。サービスプリンシパルによるフル管理権限。デプロイの完全自動化。
3

Geo レプリカの読み取り分離

ログインベースユーザーにより、プライマリと Geo レプリカで同一ログインを使用。レポーティング用ユーザーにはリードレプリカのみへのアクセスを付与し、本番書き込み環境へのアクセスを制限。

✅ Geo レプリカ間の透過的アクセス。読み取り専用ワークロードの安全な分離。運用の簡素化。

⚠ 制約事項

⚠️
SA は Entra ログインを作成不可: SQL Server 管理者 (SA) アカウントでは Entra ログインを作成できません。Entra 管理者アカウントまたは既存の Entra ログインが必要です。
⚠️
EXECUTE AS はサポート対象外: EXECUTE AS LOGIN は Entra サーバープリンシパルに対してサポートされていません。権限借用が必要な場合は代替手段を検討してください。
⚠️
グループメンバーシップ上限 2,048: 1 つのユーザーが属できるグループ数は最大 2,048 です。多数のグループに所属するユーザーではトークンサイズに注意が必要です。
⚠️
Graph API 接続が必要: ログイン作成時に Microsoft Graph API への接続が必要です。ネットワーク制限環境では Graph エンドポイントへの接続を許可する必要があります。

💰 コスト

🎉
追加コストなし - Microsoft Entra サーバー プリンシパル機能は Azure SQL Database の既存サブスクリプションに含まれており、追加料金は発生しません。Entra ID (旧 Azure AD) のライセンスで利用可能です。
