Azure Virtual Network / ExpressRoute / VPN Gateway Networking

Summarized Advertised Gateway Prefixes for Route Advertisement

ハブ&スポーク構成でアドバタイズされるプレフィックスを集約し、ExpressRoute/VPN Gateway のルーティングテーブル肥大化を防止

Preview 2026-05-20

概要

Azure Virtual Network の新機能「Summarized Advertised Gateway Prefixes」により、ExpressRoute および VPN Gateway からオンプレミスへアドバタイズされるルートプレフィックスを集約 (サマライズ) できるようになりました。大規模なハブ&スポーク構成において、個別のスポーク VNet アドレス空間を一つの集約 CIDR プレフィックスにまとめることで、BGP プレフィックス上限への到達リスクを大幅に低減します。

Key Points

  • summarizedGatewayPrefixes プロパティで集約ルートを定義
  • スポーク VNet のアドレス空間がカバーされる場合は自動的に抑制
  • ExpressRoute Gateway / VPN Gateway の両方に対応
  • ゲートウェイ未作成の状態でも事前設定が可能

Before

  • ハブ&スポーク構成で多数の個別プレフィックスがオンプレミスにアドバタイズ
  • ExpressRoute プレフィックス上限 (1,000) 到達リスク
  • BGP セッションドロップ
  • ルーティングテーブル肥大化

After

  • サマライズプレフィックスで集約ルートのみアドバタイズ
  • プレフィックス上限リスク低減
  • ルーティングテーブルサイズ削減
  • 事前設定サポート

Key Features

📋

サマライズされたプレフィックスの定義

summarizedGatewayPrefixes プロパティに集約 CIDR プレフィックスを設定。複数のスポーク VNet アドレス空間を一つの集約ルートにまとめてアドバタイズ

🚫

スポーク VNet アドレス空間の自動抑制

サマライズプレフィックスにカバーされているスポーク VNet のアドレス空間は、個別のアドバタイズが自動的に抑制される

🔄

ExpressRoute / VPN Gateway 両対応

同一の summarizedGatewayPrefixes プロパティで、ExpressRoute Gateway と VPN Gateway の両方のアドバタイズ動作を制御可能

⚙

事前設定サポート

ゲートウェイが未作成の状態でもプロパティを設定可能。インフラ構築前にルートアドバタイズメントポリシーを計画・準備できる

Use Cases

大規模ハブ&スポーク環境

100+ スポーク VNet のプレフィックスを集約し、オンプレミスへのアドバタイズ数を大幅に削減。Standard SKU の ExpressRoute でも十分にスケール可能に。

マルチリージョン展開

リージョン単位で集約ルートを定義 (例: 10.1.0.0/16 = East Japan、10.2.0.0/16 = West Japan)。オンプレミスルーターのルーティングテーブルを簡潔に維持。

ExpressRoute Premium なしでのスケーリング

Standard SKU を維持しながらプレフィックス数を削減。Premium へのアップグレードコストを回避しつつ、大規模環境をサポート。

Technical Specs
