Azure Monitor

Azure Monitor Pipeline の一般提供開始

OpenTelemetry ベースのテレメトリ収集・変換・ルーティングの一元制御ポイント

Launched (GA) 2026-04-21

概要

Azure Monitor Pipeline が一般提供開始 (GA) となった。OpenTelemetry Collector をベースとし、Arc 対応 Kubernetes クラスター上にデプロイされるテレメトリの一元的なインジェスト・変換・ルーティング制御ポイントである。データソース近くでのフィルタリング・集約によりインジェストコストを削減し、ローカルバッファリングによる接続断時のデータ損失防止、TLS/mTLS や Private Link によるエンタープライズグレードのセキュリティを提供する。

Key Points

  • GA リリースにより本番環境での SLA 対象
  • データソース近くでのフィルタリング・変換でインジェストコスト削減
  • ローカルバッファリングで接続断時のデータ損失防止
  • TLS/mTLS、BYOC、Private Link のエンタープライズ機能

Before

  • テレメトリ直接送信でインジェストコスト増大
  • 一元的なフィルタリング・変換ソリューション不足
  • 帯域幅制約環境でのボトルネック
  • Preview で SLA なし

After

  • GA で SLA 対象に
  • データソース近くでフィルタリング・集約
  • 永続ストレージへのローカルバッファリング
  • Syslog/CEF の自動スキーマ化

🎛

一元的なテレメトリ制御

フィルタリング・集約・変換・ルーティングを一元管理。データソース近くで処理することで、クラウドへの転送量を最適化し、インジェストコストを削減。

🔒

セキュアインジェスト (TLS/mTLS)

BYOC 対応の暗号化インジェスト。企業独自の PKI 証明書を使用した TLS/mTLS 通信構成が可能。mutualTls / serverOnly / disabled の 3 つのモードから選択。

💾

ローカルバッファリング

永続ストレージへのローカルバッファリングにより、接続断時のデータ損失を防止。接続復旧時に自動バックフィルでデータをクラウドへ転送。

📋

自動スキーマ化

Syslog/CEF データを Azure Monitor テーブル向けに自動変換。手動でのスキーマ定義やデータ変換作業が不要。

📊

ビルトインモニタリング

パイプライン自体の正常性・パフォーマンスシグナルを公開。パイプラインの稼働状況をリアルタイムに監視可能。

🎯

ポッド配置制御

K8s ノードへのスケジューリング制約を設定可能。AZ 指定や maxInstancesPerHost によるノードあたりのインスタンス数制限をサポート。

🔗

Private Link サポート

AMPLS (Azure Monitor Private Link Scope) 経由のプライベートエンドポイント接続。パブリックインターネットを経由せずにデータを転送。

Use Cases

UC1: 帯域幅制約のあるエッジ環境

帯域幅が限られたエッジ環境で大量のテレメトリデータを効率的にクラウドへ転送するケース。

Implementation:
  • データソース近くでフィルタリング・集約を実施し送信量を削減
  • ローカルバッファリングで接続断時のデータ損失を防止
  • 接続復旧時に自動バックフィルでデータを転送

-- 帯域幅コストを削減しつつ、データの完全性を確保。

UC2: セキュリティ要件の高い環境

金融・医療など、厳格なセキュリティ要件を持つ環境でテレメトリデータを安全に収集するケース。

Implementation:
  • BYOC mTLS で企業独自の PKI 証明書による暗号化通信
  • Private Link でパブリックインターネットを経由しないデータ転送
  • ポッド配置制御でセキュリティゾーン内のノードに限定

-- コンプライアンス要件を満たしたセキュアなテレメトリ収集を実現。

UC3: マルチクラウド環境の統合監視

複数のクラウド環境やオンプレミス環境を統合的に監視するケース。

Implementation:
  • 各環境の Arc 対応 K8s クラスターに Pipeline をデプロイ
  • OTLP + Syslog で多様なデータソースを一元収集
  • 統一された変換・ルーティングルールで Azure Monitor に集約

-- 環境を問わない統合的な可観測性を実現。

Pricing

No additional cost for pipeline itself

パイプライン自体に追加料金は発生しない。Azure Monitor の料金体系に従い、Log Analytics Workspace へのデータ取り込み量に応じて課金される。最初の 5 GB/月は無料。

Limitations / Caveats

  • Arc 対応 K8s クラスター必須
  • K8s ディストリビューション限定 (VMware TKGm, K3s, AKS Arc)
  • OTLP は Preview のみ (GA は Syslog/CEF)
  • 6 リージョンに限定
  • cert-manager 必須 (既存インスタンスはアンインストール必要)
  • BYOC 証明書管理は利用者責任
