主要機能

💻

Microsoft Foundry for Visual Studio Code (Build 2026 refresh)

GA

VS Code エディタ内から Foundry プラットフォームの主要機能にアクセス。1,900 以上のモデルの探索、プレイグラウンドでのプロンプトテスト、Hosted Agent のデプロイまでを完結。

  • 1,900+ モデルの検索・フィルタリング (Microsoft, OpenAI, Anthropic, Mistral, Meta, DeepSeek 等)
  • エディタ内プレイグラウンドでプロンプトテスト・パラメータ調整
  • Hosted Agent のコンテナ化・デプロイ・管理
  • Azure アカウント統合でサインインのみで即座にアクセス
  • Foundry プロジェクトの作成・切り替え・リソース管理
🤖

Agent-to-Agent (A2A) Support

Preview

Prompt Agents および Hosted Agents に A2A 通信サポートを追加。エージェントが名前で他のエージェントを呼び出し可能に。

  • 名前ベースの呼び出し - サービス名でエージェントを発見・呼び出し
  • マネージドエンドポイント - ルーティング・負荷分散を Foundry が管理
  • Microsoft Entra ID による相互認証でセキュアな通信
  • コンテンツセーフティ統合 - メッセージに自動フィルタリング
  • A2A 呼び出しのトレース・レイテンシ・エラー率を自動記録
📊

Evaluations with Intelligent Trace Sampling

Preview

多段階 ML パイプラインにより、大量のトレースから代表的なサブセットを自動選別。評価の効率と精度を両立する。

  • トレースの特徴量(レイテンシ、トークン数、エラー有無等)を分析し代表サンプル選出
  • 正常系トレースを間引きつつ、エッジケースは優先的にサンプリング
  • LLM ベース評価コスト(トークン消費)を大幅に削減
  • バイアスの少ない代表サンプルにより評価結果の信頼性を維持
📝

Rubric Evaluator in Microsoft Foundry

Preview

コンテキスト固有の評価基準をプロンプトとスコアリングルーブリックで定義。シングルターン/マルチターン対応。

  • ドメイン固有の品質基準をルーブリック形式(スコアと説明のマッピング)で記述
  • 評価者自体が AI エージェントとして動作しプロンプトで制御
  • 本番トレースデータを評価対象として直接使用可能
  • 複数ターンにわたる会話フローの一貫性・品質を評価
🛠

Observability Developer Experience in Azure Developer CLI (azd)

Preview

azd に評価体験を統合。エージェントに対して測定可能な品質ループを追加し、CI/CD パイプラインに組み込み可能。

  • azd コマンドラインから直接エージェントの品質評価を実行
  • GitHub Actions / Azure DevOps パイプラインに評価ステップを組み込み
  • 品質ゲート - 評価スコアが閾値を下回るとデプロイを自動ブロック
  • ローカル開発環境で即座に評価実行しコード変更の影響を確認
🔍

Domain Filter for Specialized Model Discovery

Preview

モデルカタログにドメインフィルターを追加。1,900+ モデルを業界・ユースケース別に絞り込み、専門分野向けモデルの発見性が大幅に向上。

  • 対応ドメイン: ロボティクス、生物医学、材料科学、その他産業固有ドメイン
  • 汎用キーワード検索では見つけにくい専門モデルを迅速に発見
  • ドメイン知識がなくても適切なモデル候補を絞り込み可能
  • 組織のユースケースに最適なモデル選定を加速

ユースケース

🚀 エディタ内完結のエージェント開発サイクル

開発者が VS Code 内でエージェントの設計・テスト・デプロイ・評価を一気通貫で実行する。

  1. VS Code Foundry 拡張機能でドメインフィルターから適切なモデルを選択
  2. プレイグラウンドでプロンプトをテストし、最適なパラメータを決定
  3. エージェントコードを実装し、Hosted Agent としてデプロイ
  4. azd CLI で評価を実行し、品質スコアを確認
  5. A2A エンドポイントを有効化し、他のエージェントから呼び出し可能に
効果: 開発者がエディタを離れることなく、エージェント開発の全ライフサイクルをカバー

🤝 マルチエージェントによるカスタマーサポート自動化

複数の専門エージェントを A2A で連携させ、カスタマーサポートの問い合わせを自動処理する。

  1. フロントエージェント: ユーザーの意図を分類
  2. FAQ エージェント: 一般的な質問に回答
  3. テクニカルサポートエージェント: 技術的な問題を診断
  4. エスカレーションエージェント: 人間のオペレーターにハンドオフ
効果: エージェント間の連携品質を定量管理しながら、サポート業務を段階的に自動化

🔬 専門分野でのモデル選定と評価

製薬企業の研究者が、生物医学分野に特化したモデルを選定し、薬物相互作用の予測精度を評価する。

  1. ドメインフィルター (Biomedical Sciences) で候補モデルを絞り込み
  2. VS Code プレイグラウンドで各モデルの応答品質を比較
  3. Rubric evaluator で専門家定義の評価基準に基づきスコアリング
  4. 最適なモデルを Hosted Agent としてデプロイ
効果: ドメイン固有の品質基準に基づく客観的なモデル選定プロセスの実現
