GitHub Copilot Modernization:
アプリケーション近代化エージェント GA

2026年6月2日 リリース
GA (一般提供開始)

概要

Azure Migrate の評価機能と GitHub Copilot のコード変換機能を統合した、エンドツーエンドのアプリケーション近代化エージェントが一般提供開始（GA）されました。従来は分離していたアプリケーション評価とコード変換のプロセスを一体化し、近代化タイムラインを数か月から数日に短縮します。

主要ポイント

🔄 Azure Migrate 評価から直接 GitHub Copilot コード変換ワークフローに接続するエンドツーエンド統合
⚡ 近代化タイムラインを数か月から数日に短縮（70% 時間削減）
🤖 エージェントがコードベースを分析し、移行計画を自動生成・実行
📊 ポートフォリオ全体のスケーラブルな評価とアップグレードを実現

Before / After 比較

❌ Before

  • アプリケーション評価（Azure Migrate）とコード変換が分離
  • 近代化に数か月のタイムライン必要
  • アプリごとの個別調査・計画・実装が手動
  • ポートフォリオ全体の一貫したアップグレード戦略実現困難

✅ After

  • Azure Migrate 評価から直接コード変換ワークフローに接続
  • 近代化タイムラインを数か月から数日に短縮（70% 時間削減）
  • エージェントがコードベース分析し移行計画を自動生成
  • ポートフォリオ全体のスケーラブルな評価とアップグレード

主要機能

🔍

スケーラブルなアプリケーション評価

ポートフォリオ全体を一括で評価し、クラウド移行に必要な課題を体系的に分類します。

  • ポートフォリオ一括評価
  • 17 カテゴリによる課題分類
  • ソリューションカバレッジの可視化
🤖

エージェント駆動のコード変換

AI エージェントがコードベースを自律的に分析し、移行計画の生成からコード変換まで自動実行します。

  • 自律的コードベース分析
  • 移行計画の自動生成
  • 設定・ライブラリ・依存関係の一括更新
🔧

カスタムスキル

組織固有のビジネスロジックやアーキテクチャパターンに対応するカスタムスキルを定義できます。

  • 組織固有ビジネスロジックの定義
  • ソフトウェアファクトリーパターンの適用
🛡

セキュリティ統合

移行プロセス全体を通じてセキュリティ課題を検出し、ポリシー準拠を確認します。

  • CVE の早期検出
  • ポリシー準拠確認
  • CI/CD パイプラインでの検証
✅

段階的承認フロー

各変換ステップで開発者が Allow/Skip を選択でき、制御を維持しながら自動化の恩恵を受けられます。

  • Allow/Skip の選択による段階的承認
  • 開発者が制御を維持

ユースケース

📦 レガシー .NET Framework から .NET 8 への移行

既存の .NET Framework アプリケーションを最新の .NET 8 に移行します。NuGet パッケージの互換性確認、API 変更の自動適用、csproj / web.config の変換を含むフルスタックの移行を実現します。

対象: NuGet パッケージ互換性、API 変更、csproj/web.config 変換

☕ Java EE アプリケーションのクラウド最適化

オンプレミスの Java EE アプリケーションを Azure App Service や Container Apps に最適化して移行します。ランタイムとフレームワークのアップグレードを含みます。

対象: Azure App Service / Container Apps への移行

🏢 大規模ポートフォリオの一括近代化

100 以上のアプリケーション、500,000 行以上のコード変更を数週間で完了します。ポートフォリオ全体の評価から移行計画の生成、コード変換までをスケーラブルに実行します。

実績: 100+ アプリ、500,000 行以上のコード変更を数週間で完了
