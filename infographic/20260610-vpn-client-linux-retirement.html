Azure VPN Client for Linux (Preview) 廃止

2026-06-10 | 廃止日: 2026 年 8 月 31 日 | 代替: OpenVPN / strongSwan

概要

Azure VPN Client for Linux (Preview) は 2026 年 8 月 31 日に廃止される。プレビューのまま GA への移行パスがなく、現行のセキュリティ・信頼性基準を満たさないことが廃止理由。代替として OpenVPN クライアント または strongSwan (IKEv2) への移行が必要。

❌ Before (廃止対象)

  • Azure VPN Client for Linux (Preview)
  • Ubuntu 20.04/22.04 のみ対応
  • GA への道筋なし
  • Linux で唯一の Entra ID 認証手段

✅ After (代替手段)

  • OpenVPN / strongSwan を使用
  • 幅広い Linux ディストリビューション対応
  • オープンソースで活発にメンテナンス
  • 証明書/RADIUS 認証で安定接続

代替ソリューション

クライアント トンネルタイプ 認証方式 対応ディストリビューション

🔓

OpenVPN クライアント (証明書認証)

TLS 1.2/1.3、TCP 443。ゲートウェイが既に OpenVPN + 証明書認証で構成済みなら変更不要。

🛡️

strongSwan (IKEv2/IPsec)

AES256-GCM。証明書認証または RADIUS 認証。RADIUS + AD 統合で組織認証を維持可能。

設定例: OpenVPN クライアント

bash OpenVPN インストールと接続 
# OpenVPN クライアントのインストール
sudo apt-get install openvpn
sudo apt-get -y install network-manager-openvpn
sudo service network-manager restart

# VPN クライアントプロファイルを使用して接続
sudo openvpn --config <vpnconfig.ovpn ファイルのパス> &

設定例: strongSwan (IKEv2)

bash strongSwan インストールと接続 
# strongSwan のインストール
sudo apt-get install strongswan
sudo apt install strongswan-pki
sudo apt install libstrongswan-extra-plugins
sudo apt install libtss2-tcti-tabrmd0

# 証明書の配置
sudo cp ${USERNAME}Cert.pem /etc/ipsec.d/certs/
sudo cp ${USERNAME}Key.pem /etc/ipsec.d/private/
sudo chmod -R go-rwx /etc/ipsec.d/private /etc/ipsec.d/certs

# 接続の開始
sudo ipsec restart
sudo ipsec up azure

移行手順

  1. ゲートウェイ構成の更新 — 代替クライアントが必要とするトンネルタイプをサポートするよう P2S ゲートウェイ構成を確認・更新
  2. プロファイルの再生成 — ゲートウェイから新しい VPN クライアントプロファイル構成ファイルを生成
  3. クライアントのインストール — 各 Linux デバイスに代替 VPN クライアントをインストール・構成
  4. 接続テスト — 接続性を検証し、ユーザーベースに展開
  5. 旧クライアントの削除 — Azure VPN Client for Linux をアンインストール

制約事項・注意点

🚫

Microsoft Entra ID 認証の喪失

Linux での Microsoft Entra ID (AAD) 認証は代替手段なし。証明書認証または RADIUS 認証への切り替えが必要。条件付きアクセスポリシーや MFA の直接適用も困難に。

⚠️

廃止後のリスク

2026 年 8 月 31 日以降、バグ修正・セキュリティパッチ・サポートは提供されない。パッケージは Microsoft の Linux リポジトリから削除される。

移行ユースケース

1️⃣

証明書認証で OpenVPN に移行

ゲートウェイが既に OpenVPN で構成済みなら変更不要。vpnconfig.ovpn にクライアント証明書を設定して接続テスト。幅広い Linux ディストリビューションで安定動作。

2️⃣

Entra ID 認証からの移行

RADIUS サーバーを Active Directory と統合すれば組織ドメイン資格情報での認証が可能。Windows/macOS では引き続き Entra ID 認証を利用可能。
