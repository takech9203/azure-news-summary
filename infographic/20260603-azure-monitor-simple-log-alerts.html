Simple Log Alerts - 一般提供開始 (GA)

2026-06-03 | Microsoft Build 2026

GA

概要

Microsoft は Azure Monitor の Simple Log Alerts (シンプルログアラート) の一般提供 (GA) を発表した。従来のログ検索アラートルール (Log Search Alert) に対して、より簡素化された直感的な監視・アラート体験を提供する。各行を個別に評価する仕組みを採用し、エラーや障害が発生した時点で即座にアラートが発火する。Analytics ログと Basic ログの両方のテーブルプランに対応。Microsoft Build 2026 のタイミングでの GA 発表。

Before / After 比較

Before

  • 従来のログ検索アラートでは集計期間設定が必要で設定が複雑
  • 行ごとの即時アラートには1分間隔の評価頻度とカスタムKQLクエリの組み合わせが必要
  • 個別イベントごとのアラート発火シナリオの設定が直感的でなかった
  • Basic ログテーブルに対するアラート設定が限定的

After

  • クエリ結果の各行を個別に評価し、マッチした行ごとにアラートを発火可能
  • 集約設定 (Measure、Aggregation type、Window size) が不要で設定ステップが大幅に簡素化
  • Analytics ログと Basic ログの両テーブルプランをサポート
  • 1分ごとの行単位のイベントマッチングでリアルタイム監視を実現

アーキテクチャ図

ログデータが Log Analytics Workspace に取り込まれ、Simple Log Alert ルールが各行を個別に KQL クエリで評価する。条件にマッチした行が検出されると、アクショングループを通じて即座に通知が送信される。

主要機能

📌

行単位の個別評価 (Single Event モード)

クエリ結果の各行を個別に評価し、条件にマッチした行ごとにアラートを発火。従来の集約ベースの評価とは異なり、個別イベントの検出に最適化されている。

📝

簡素化された KQL クエリ

Transformation KQL 言語に基づくシンプルなクエリ構文。最初の 5 カラムが通知出力に表示される。基本的なフィルタリングと射影に特化した設計。

⚡

柔軟なトリガー閾値

「すべてのマッチ行でアラート」「1分間に1回以上」「1分間にN回以上」から選択可能。シナリオに応じたノイズ制御が可能。

📊

Analytics / Basic ログ両対応

両テーブルプランをサポートし、Basic ログに対するコスト効率の良い監視が可能。ログプランを問わず統一的なアラート設定を実現。

Simple Log Alert vs 従来のログ検索アラート

比較項目 Simple Log Alert 従来の Log Search Alert
評価方式 行単位 (Single Event) 集約ベース (Aggregated)
集約設定 不要 必須
評価頻度 1分固定 1分 - 24時間
ディメンション分割 非対応 対応
設定の複雑さ 低い 高い
ユースケース 即時検出・個別イベント監視 集計分析・トレンド監視

ユースケース

🔥 バックアップジョブ失敗の即時検出

企業の DR (災害復旧) 要件として、すべてのバックアップジョブの失敗を即座に検知し、運用チームに通知する。

AzureDiagnostics
| where Category == "BackupJob" and ResultType == "Failed"
各バックアップ失敗が発生した時点で個別にアラートが発火し、どのジョブがいつ失敗したかを即座に把握可能。

🔒 セキュリティイベントのリアルタイム監視

セキュリティチームがブルートフォース攻撃の兆候をリアルタイムで検出し、各不正ログイン試行を個別に追跡する。

SecurityEvent
| where EventID == 4625
認証失敗イベントごとにアラートが発火し、攻撃パターンの即座な可視化と対応が可能。

💻 アプリケーションエラーの簡易監視

開発チームが新しくデプロイしたアプリケーションのエラーを簡単に監視。複雑なアラートルールの設計に時間をかけずに即座に監視開始。

AppExceptions
| where SeverityLevel >= 3
最小限の設定で重大なアプリケーション例外を即座に検出。デプロイ直後から数分で監視体制を構築可能。

制約事項

⚠️ ディメンション分割非対応
⚠️ ミュートアクション非対応
⚠️ let, print, datatable 使用不可
⚠️ 動的閾値 (Dynamic Thresholds) 非対応
⚠️ ステートフルアラート非対応 (自動解決なし)
⚠️ 評価頻度 1分固定 (変更不可)
