SSMS GitHub Copilot In Preview

GitHub Copilot Agent Mode for SSMS

2026-06-09 | 自律的マルチステップ推論でデータベース管理を革新

概要

SQL Server Management Studio (SSMS) の GitHub Copilot に Agent mode がパブリックプレビューとして追加。データベースに対して自律的にマルチステップの推論を行い、パフォーマンス調査、セキュリティ監査、クエリチューニングなどの複雑なタスクを一貫して遂行する。

❌ Before (Ask mode)

  • 単発の質問応答のみ
  • 複雑な調査には複数回の手動プロンプトが必要
  • DMV クエリを手動で実行・分析
  • 各ステップをユーザーが指示

✅ After (Agent mode)

  • 自律的マルチステップ推論
  • 自然言語で依頼するだけで複雑なタスク実行
  • 診断クエリの生成・実行・分析を自動化
  • Agent が計画・実行を自律的に遂行

メンテナンス・構成レビュー

インデックス断片化、統計情報の鮮度、データベース設定の最適性を確認し、改善提案を実施。

🛡️

セキュリティ上の懸念の特定

権限設定の監査、脆弱な構成の検出。セキュリティベストプラクティスへの準拠状況を確認。

🐛

エラーのトラブルシューティング

エラーメッセージからの根本原因分析。解決手順の自動生成と実施支援。

🤖

運用タスクの支援

日常的な管理タスクの支援。構成変更の影響分析。

Ask mode vs Agent mode

項目 Ask mode (従来) Agent mode (新規)
推論方式 単一ステップの質問応答 マルチステップの自律的推論
タスク実行 ユーザーが各ステップを指示 Agent が自律的に計画・実行
クエリ実行 読み取り専用クエリのみ 複数クエリを連続実行し調査
対象タスク 個別の質問、コード生成 複雑な調査、分析、最適化
コンテキスト 単一の応答内で完結 複数ステップにわたり維持

ユースケース

パフォーマンス問題の調査

本番データベースで応答時間が悪化したとの報告があり、DBA が原因を調査する必要がある。

"このデータベースのパフォーマンス問題を調査してください。応答時間が遅くなっている原因を特定し、改善策を提案してください。"

Agent が待機統計、ブロッキング、リソース使用率、最もコストの高いクエリを自律的に調査し、根本原因と推奨アクションを一括で提示。

セキュリティ監査

定期的なセキュリティレビューとして、データベースの権限設定と構成を確認する。

"このデータベースのセキュリティ状態をレビューしてください。過剰な権限、脆弱な構成、ベストプラクティスからの逸脱を報告してください。"

権限の過剰付与、orphaned users、暗号化設定、監査構成を包括的にチェックし、リスクと改善策を提示。

クエリの最適化

特定のストアドプロシージャが遅く、チューニングが必要。

"このストアドプロシージャのパフォーマンスを分析し、最適化してください。"

実行プラン取得・分析、Missing Index 確認、統計情報鮮度チェック、書き換え候補提案まで自律的に実施。

設定方法

  1. SSMS 22 以降をインストール (最新版推奨)
  2. AI Assistance ワークロードを追加 — Visual Studio Installer で追加
  3. GitHub Copilot サブスクリプションを有効化 — Copilot Free でも利用可能
  4. SSMS で GitHub Copilot にサインイン — 右上の Copilot バッジから
  5. Agent mode を選択 — Chat ウィンドウで Agent mode を選択して利用開始

対応データベース: SQL Server、Azure SQL Database、Azure SQL Managed Instance、Azure Synapse Analytics

制約事項・注意点

⚠️

パブリックプレビュー

本番環境での利用には注意が必要。GA 後の利用を推奨。

⚠️

権限モデル

Agent が実行するクエリはユーザーの権限で実行される。最小権限の原則で制御すべき。

⚠️

人間のレビューが必要

AI による推奨事項は参考情報であり、重要な変更は必ず人間がレビューすること。

推奨される次のアクション

1️⃣

SSMS 22 最新版にアップデート

AI Assistance ワークロードを追加し、最新の機能を利用可能に

2️⃣

GitHub Copilot を有効化

Free プランでも基本的な利用が可能。まずは無料で試用

3️⃣

検証環境で Agent mode を試用

パフォーマンス調査やセキュリティレビューのワークフローに組み込み可能か評価

4️⃣

本番環境は GA 後に

プレビュー段階では検証用途に限定し、本番環境での利用は GA を待つ
