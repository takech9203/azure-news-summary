Application Gateway for Containers In Preview

Application Gateway for Containers:
AKS Automatic マネージドアドオン対応

2026-05-05

概要

Application Gateway for Containers のマネージドアドオンが AKS Automatic クラスターで利用可能になりました（パブリックプレビュー）。Application Gateway for Containers は Kubernetes ワークロード向けのレイヤー 7 ロードバランシングおよび動的トラフィック管理プロダクトです。AKS Automatic はノード管理、スケーリング、セキュリティなどを自動化するマネージド体験を提供します。この統合により運用簡素化の利点を享受しつつ、高度なトラフィック管理機能を活用可能になります。

Before

  • AKS Automatic クラスターでは Application Gateway for Containers マネージドアドオンを有効化できなかった
  • 高度なレイヤー 7 ロードバランシング機能を利用するために AKS Standard に切り替える必要があった
  • デフォルトの Ingress（マネージド NGINX）では対応できない高度なトラフィック管理要件に対応が困難

After

  • AKS Automatic クラスターで Application Gateway for Containers マネージドアドオンを直接有効化可能に
  • フルマネージドなノード管理・スケーリングと高度なトラフィック管理を組み合わせた構成が実現可能に
  • Gateway API や Ingress API を通じた柔軟なトラフィックルーティングが AKS Automatic でも利用可能に

主要機能

🚀

AKS Automatic でのマネージドアドオン対応

AKS Automatic クラスターで Application Gateway for Containers マネージドアドオンを有効化可能。フルマネージドなクラスター運用と高度なトラフィック管理を両立。

🌐

Gateway API および Ingress API サポート

Kubernetes Gateway API v1 に準拠し、従来の Ingress API もサポート。標準的な API を通じた柔軟なトラフィック制御が可能。

⚙

ALB Controller によるライフサイクル管理

ApplicationLoadBalancer カスタムリソースによる Application Gateway for Containers のライフサイクル管理。宣言的な構成管理を実現。

🔀

高度なトラフィック管理機能

トラフィックスプリッティング、ヘッダーリライト、URL リダイレクト、mTLS、gRPC、WebSocket サポートなど、エンタープライズ向け機能を提供。

ユースケース

1. マイクロサービスのカナリアデプロイ

トラフィックを段階的に切り替え、新バージョンへの移行リスクを最小化

YAML HTTPRoute - カナリアデプロイ 
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: canary-route
spec:
  parentRefs:
  - name: my-gateway
  rules:
  - backendRefs:
    - name: app-v1
      port: 80
      weight: 90
    - name: app-v2
      port: 80
      weight: 10

2. マルチテナント SaaS アプリケーション

ホスト名ベースのルーティングと mTLS によるテナント分離

YAML HTTPRoute - テナントルーティング 
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: tenant-routing
spec:
  parentRefs:
  - name: my-gateway
  hostnames:
  - "tenant-a.example.com"
  rules:
  - backendRefs:
    - name: tenant-a-service
      port: 443

制約事項・注意点

  • パブリックプレビュー段階のため、本番環境での利用には注意が必要
  • 委任サブネットに最低 250 の IP アドレスが必要（/24 以上のサブネット）
  • リスナーポートは 80 と 443 のみサポート
  • プレビュー中は SLA が適用されない可能性あり
  • デフォルト Ingress（マネージド NGINX）との共存時の設定に注意が必要
