🔄 概要: Before / After 比較

Azure Event Grid に対して、大規模分散システムの管理を強化し、相互運用性とセキュリティの向上を目的とした複数の新機能が発表されました。GA となった MQTT メッセージオーダリングと、パブリックプレビューのクロステナント Webhook 配信が主要機能です。

Before

  • MQTT ブローカーで QoS 1 使用時もトピック単位・クライアント単位の順序保証なし
  • テレメトリ・時系列データ処理でアプリケーション側での順序制御が必要
  • イベント配信が同一テナント内に限定、マルチテナント Webhook にはカスタム実装が必要
  • 異なるテナント間のイベント連携にセキュアかつ簡便な配信手段が不足

After

  • MQTT v5 で各トピック・各クライアント単位の QoS 1 順序保証付き配信が GA
  • シーケンス整合性が重要なワークフローでアプリケーション側の順序制御が不要
  • マネージド ID とフェデレーテッド認証情報によるクロステナント Webhook 配信が可能
  • Basic / Standard ティアの両方でクロステナント配信をサポート

✨ 主要機能
📨

MQTT メッセージオーダリング (GA)

MQTT v5 でのメッセージオーダリングにより、各トピック・各クライアント単位で QoS 1 メッセージの順序保証付き配信が一般提供として利用可能になりました。テレメトリ、コマンド実行、時系列データなどのシーケンス整合性が重要なワークフローに最適です。

🌐

クロステナント Webhook 配信 (パブリックプレビュー)

マネージド ID とフェデレーテッド認証情報を使用したテナント間イベント配信が可能になりました。Basic ティアおよび Standard ティア (名前空間) の両方でサポートされています。

🔒

TLS 1.2/1.3 必須のセキュア通信

MQTT ブローカーへの接続には TLS 1.2 または 1.3 が必須です。TCP 8883 (直接接続) および TCP 443 (WebSocket) の両ポートに対応しています。

🔄

MQTT クロスバージョン通信

MQTT v3.1.1 と v5 間のクロスバージョン通信がサポートされており、既存デバイスとの互換性を維持しながら新機能を活用できます。

💻

マネージド ID ベースのクロステナント認証

ユーザー割り当てマネージド ID とマルチテナント Microsoft Entra アプリのフェデレーテッド認証情報を組み合わせることで、認証情報のハードコーディングが不要になります。

⚖️ メリット / 制約事項

メリット

ビジネス面

  • マルチテナント環境における組織間イベント連携が容易に
  • IoT ソリューションにおけるメッセージの信頼性が向上
  • カスタム実装の保守負担が不要に

技術面

  • アプリケーション層での順序制御ロジック不要
  • マネージド ID ベースのクロステナント認証で認証情報のハードコーディング不要
  • MQTT v3.1.1 と v5 間のクロスバージョン通信サポート

制約事項

  • メッセージオーダリングは MQTT v5 のみ対応
  • 順序保証は同一トピック・同一クライアント単位に限定
  • クロステナント Webhook 配信はプレビュー段階 (SLA 対象外)
  • QoS 2 は未サポート
  • 最大パケットサイズは 512 KiB に制限
💡 ユースケース
🏭

IoT テレメトリの順序保証付き収集

製造業の生産ラインにおけるセンサーデータを発生順序を保証しながら収集。MQTT v5 のメッセージオーダリングにより、データの時系列整合性を維持できます。

☁️

マルチテナント SaaS プラットフォーム

SaaS プロバイダーが顧客テナントの Azure 環境にあるエンドポイントへイベントを配信。クロステナント Webhook 配信で、カスタム実装なしに実現可能です。

🕹️

コマンド・コントロールシステム

IoT デバイスへのコマンド送信における実行順序保証。QoS 1 のメッセージオーダリングにより、コマンドが正しい順序で処理されます。

🏢

グループ会社間のイベント連携

異なる Azure テナント間での業務イベントリアルタイム連携。マネージド ID とフェデレーテッド認証により、セキュアなテナント間通信を実現します。
