Azure Virtual Machines / Azure Disk Storage

Azure VM Restore Points:
Instant Access

リストアポイント作成直後に即座にディスク復元 - RTO を大幅に短縮

Public Preview 2026-07-01

概要

Azure VM Restore Points に Instant Access 機能がパブリックプレビューとして導入された。これにより、リストアポイント作成後にバックグラウンドデータレプリケーションの完了を待たずに、即座にディスクをリストア可能になる。Premium SSD v2 / Ultra Disk を使用するミッションクリティカルなワークロードにおいて、RTO (Recovery Time Objective) を大幅に短縮する。

Key Points

  • リストアポイント作成直後にディスクを即座に復元可能 (バックグラウンドコピー完了を待つ必要なし)
  • Premium SSD v2 / Ultra Disk (データディスク) が対象
  • アプリケーション整合性を維持 (VSS / プレ・ポストスクリプト対応)
  • Instant Access 期間は 60-300 分で設定可能 (デフォルト 300 分)

Before

  • Premium SSD v2 / Ultra Disk のリストアポイントは、バックグラウンドコピー完了までディスク復元不可
  • 障害発生時の復旧に長時間の待機が必要 (RTO が高い)
  • データコピー完了状況を監視しながら復旧タイミングを判断する運用負荷

After

  • リストアポイント作成直後にディスクを即座に復元可能
  • RTO の大幅な短縮 (バックグラウンドコピー待ち不要)
  • 復元されたディスクはパフォーマンス影響を最小限に抑えて高速にハイドレーション

主要機能

⚡

即時ディスクリストア

リストアポイント作成直後にディスクを復元可能。バックグラウンドデータレプリケーションの完了を待つ必要がなく、RTO を劇的に短縮。

🔒

アプリケーション整合性の維持

VSS (Windows) またはプレ/ポストスクリプト (Linux) を使用し、アプリケーションレベルの整合性を確保した状態でリストアポイントを作成。

⏰

時間制限付き Instant Access

instantAccessDurationMinutes パラメータにより Instant Access 期間を 60-300 分で指定可能。期間経過後はバックグラウンドコピーの状態に依存。

📈

スナップショットアクセス状態の監視

snapshotAccessState プロパティで各ディスクリストアポイントの状態を確認可能。InstantAccess / CopyCompleted / Expired など、現在のステータスを把握。

コードサンプル

PowerShell - プレビュー機能の登録:

Register-AzProviderFeature -FeatureName 'AppConsistentInstantAccessSnapshotForDirectDriveDisks' -ProviderNamespace 'Microsoft.Compute'

Azure CLI - スナップショットアクセス状態の確認:

az resource show --ids <disk-restore-point-id> --query "properties.snapshotAccessState" --output tsv

ユースケース

ミッションクリティカルデータベースの高速復旧

Premium SSD v2 上で稼働する大規模データベース VM に障害が発生した場合、直前のリストアポイントから即座に復旧が可能。バックグラウンドコピーの完了を待つ必要がないため、従来と比較して復旧時間を大幅に短縮できる。
効果: RTO の大幅短縮によりビジネスインパクトを最小化

定期的なリストアポイントによる継続的保護

Ultra Disk を使用する高性能ワークロードで、定期的にアプリケーション整合性リストアポイントを取得。Instant Access 機能により、取得直後から即座にリストア可能な状態が維持されるため、常に高速な復旧手段を確保できる。
効果: 継続的な DR 態勢を維持しつつ復旧時間の SLA を保証
