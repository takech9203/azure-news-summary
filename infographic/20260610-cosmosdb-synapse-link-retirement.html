Azure Cosmos DB Azure Synapse Analytics Retirement

Azure Synapse Link for NoSQL のリタイア

2026-06-10 | Microsoft Fabric ミラーリングへの移行推奨

概要

Azure Synapse Link for Azure Cosmos DB NoSQL は 2029 年 3 月 31 日に完全廃止される。後継ソリューションとして Microsoft Fabric の Azure Cosmos DB ミラーリング への移行が推奨されている。

❌ Before

  • Synapse Link で HTAP 実現
  • 列指向分析ストア経由
  • Synapse Analytics から直接クエリ
  • 新規プロジェクトでも有効化可能

✅ After (移行先)

  • Fabric ミラーリングで同等機能
  • Delta Lake (オープンソース) 形式
  • Fabric SQL / Spark / Power BI
  • より広範な分析エコシステム統合

移行先: Microsoft Fabric ミラーリングの主要機能

🔄

ゼロ ETL リアルタイムレプリケーション

Cosmos DB のデータが Fabric OneLake に自動継続レプリケーション。RU を消費せず、トランザクションワークロードに影響なし。Delta Lake 形式で保存。

🛡️

完全なワークロード分離

トランザクション処理と分析処理が完全に分離。分析クエリが運用ワークロードのパフォーマンスに影響しない。

🧩

統合された分析エコシステム

T-SQL による集計、Apache Spark でのデータ探索・ML、Power BI DirectLake モード、Copilot による生成 AI インサイト取得に対応。

📐

ネストデータのサポート

JSON ネストデータを OPENJSON、CROSS APPLY で展開可能。スキーマ自動推論によるフラット化もサポート。

移行手順

前提条件: Microsoft Fabric 容量が利用可能 / Cosmos DB で継続バックアップ (7 日間) 有効化 / Microsoft Entra ID またはアカウントキー認証

  1. Fabric ワークスペースの準備 — Microsoft Fabric でワークスペースを作成
  2. ミラーリングの設定 — Fabric からミラーリング対象の Cosmos DB データベースを選択
  3. コンテナの選択 — ミラーリングするコンテナを個別に選択
  4. レプリケーション開始 — 初期スナップショット取得後、継続的レプリケーション開始
  5. 分析ワークロードの移行 — Synapse Analytics のクエリを Fabric SQL / Spark に移行
  6. 検証 — データ整合性とクエリ結果を検証
  7. Synapse Link の無効化 — 移行完了後、不要になった Synapse Link を無効化

制約事項・注意点

⚠️

API 対応制限

現時点では NoSQL API のみ対応。Gremlin、MongoDB API は Fabric ミラーリング未対応。

⚠️

複数書込リージョン非サポート

マルチリージョン書込のアカウントは Fabric ミラーリング非サポート。

⚠️

追加コスト

Fabric Capacity が必要で、追加のライセンスコストが発生する可能性がある。

⚠️

読み取り専用

ミラーリングから Cosmos DB への書き戻しは不可 (読み取り専用)。CMK 暗号化は OneLake で非サポート。

推奨アクション

1️⃣

新規プロジェクト

Synapse Link を使用せず、Microsoft Fabric ミラーリングを採用する

2️⃣

移行計画の策定

既存の Synapse Link 環境について 2028 年内の移行完了を目標に計画を策定する

3️⃣

Fabric 導入の検討

Fabric の容量・ライセンスを確認し、組織の分析基盤としての導入を検討する

4️⃣

API 制約の確認

Gremlin / MongoDB API 使用の場合、Fabric ミラーリングは NoSQL のみ対応のため代替手段を検討
