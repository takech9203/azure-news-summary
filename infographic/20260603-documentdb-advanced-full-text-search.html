高度なフルテキスト検索 (Public Preview)

ファジー検索、近接検索、多言語サポート、BM25 ランキングを単一クラスターで実現。ベクトル検索との統合でハイブリッド検索も可能に。

Gated Preview 2026-06-03

Overview

Azure DocumentDB に高度なフルテキスト検索機能が Public Preview として追加された。従来の $text 演算子を置き換える新しい検索エンジンとして、$search アグリゲーションパイプラインステージを通じて BM25 スコアリング、ファジー検索、フレーズ/近接検索を提供する。さらに、ベクトル検索と BM25 を同一コレクション上で統合し、Reciprocal Rank Fusion (RRF) による結果融合を実現する。

Key Points

  • BM25 ランキングによる関連性スコア付きキーワード検索
  • ファジー検索 (Levenshtein 編集距離) でタイプミスに対応
  • フレーズ検索 + 近接マッチング (slop パラメータ)
  • ベクトル検索 + BM25 のハイブリッド検索を RRF で融合
  • 外部検索サービス不要、クラスター料金に含まれる

Before

  • レガシーの $text 演算子のみ、ファジー検索や近接検索に非対応
  • ベクトル検索との組み合わせには外部検索サービスが必要
  • タイプミスへの耐性なし、正確なキーワード一致のみ
  • 単語の順序や近接性を考慮した検索不可

After

  • BM25 ランキングによる関連性スコア付きキーワード検索が可能
  • ファジー検索 (Levenshtein 編集距離ベース) でタイプミス対応
  • フレーズ検索と近接マッチング (slop パラメータ)
  • ベクトル検索 + BM25 を同一コレクションで統合、RRF で結果融合

Azure DocumentDB は同一クラスター・同一コレクション上に BM25 検索インデックスとベクトルインデックスの両方を保持し、書き込み一度で両方のインデックスが更新される。ハイブリッド検索では RRF アルゴリズムにより両方の結果リストを統合する。

🔍

BM25 キーワード検索

$search + text オペレーターによるスコア付き関連性検索。{ $meta: "searchScore" } で BM25 スコアを取得可能。レガシー $text 演算子を完全に置き換え。

⌨

ファジー検索 (タイプミス耐性)

Levenshtein 編集距離に基づくマッチング。fuzzy.maxEdits パラメータで許容する編集距離を制御 (1 または 2)。例: bracXet で bracket にマッチ。

📄

フレーズ検索と近接マッチング

$search + phrase オペレーターで語順を保持した検索。slop パラメータで許容する介在トークン数を指定。slop: 0 で厳密な隣接、slop: 3 で 3 トークンまで許容。

🔄

ハイブリッド検索 (BM25 + ベクトル)

同一コレクションで BM25 検索とベクトル類似検索を並行実行。Reciprocal Rank Fusion (RRF) で結果を融合。追加コストなしでクラスター料金に含まれる。

検索インデックスの作成

// BM25 フルテキスト検索インデックスを作成
db.runCommand({
  createSearchIndexes: "products",
  indexes: [{
    name: "idx_description_fts",
    definition: {
      mappings: {
        dynamic: false,
        fields: { description: { type: "string" } }
      }
    }
  }]
});

ファジー検索の実行

// ファジー検索の実行
db.products.aggregate([
  { $search: {
      index: "idx_description_fts",
      text: { query: "bracXet", path: "description", fuzzy: { maxEdits: 1 } }
  }},
  { $limit: 20 },
  { $project: { title: 1, score: { $meta: "searchScore" } } }
]);

Use Cases

UC1: EC サイトの商品検索 (ハイブリッド検索)

ユーザーが自然言語で商品を検索する際、SKU 番号やブランド名の正確なマッチングと、説明文の意味的な類似検索を両立。BM25 が SKU を正確にマッチし、ベクトル検索が「防水」と「ウォータープルーフ」の類義語を捕捉。

効果: 単独の検索方式より高い再現率と精度を実現

UC2: ナレッジベース・RAG パイプライン

社内ドキュメントの検索で、パラフレーズされた質問でも正しいパッセージを見つけつつ、固有名詞やエンティティ名は正確にマッチ。ベクトル検索のみでは失われやすい固有名詞を BM25 が補完。

効果: RAG の回答品質を向上

UC3: 多言語カタログのタイプミス耐性検索

グローバルな EC プラットフォームで、ユーザーのタイプミスや入力ミスに対応。maxEdits: 2 により「recieve」で「receive」を含むドキュメントがマッチ。

効果: ユーザー体験を損なうことなく検索結果を提供
