概要

Azure Databricks の Mosaic AI Model Serving を通じて、Anthropic Claude Opus 4.7 が一般提供 (GA) された。Claude Opus 4.7 は Anthropic が提供する最も高性能なハイブリッド推論モデルであり、複雑な情報抽出やエージェント型推論タスクにおいて優れたパフォーマンスを発揮する。External Models 機能により、OpenAI 互換の統一 API でアクセスでき、API キーの一元管理、アクセス制御、使用量モニタリングなどのガバナンス機能も併せて利用できる。

Key Points

  • Anthropic 最上位モデル Claude Opus 4.7 が Azure Databricks 上で GA として利用可能に
  • Mosaic AI Model Serving の External Models 機能を通じた統一的なエンドポイント提供
  • OpenAI 互換 REST API により既存のコードベースやツールチェーンをそのまま使用可能
  • AI Gateway によるレート制限、使用量トラッキング、ガードレールの適用
  • Databricks Secrets / Unity Catalog による安全な資格情報管理とアクセス制御

Before

  • Anthropic API への接続を独自に構築・管理する必要があった
  • API キーの管理がアプリ/ユーザーごとに分散しセキュリティリスクが高い
  • LLM の使用量やコストのモニタリングを個別に実装する必要があった
  • 複数の LLM プロバイダーごとに異なる API 仕様に対応が必要

After

  • Mosaic AI Model Serving の統一インターフェースで直接アクセス可能
  • Databricks Secrets による API キーの一元的かつ安全な管理
  • AI Gateway によるレート制限、使用量トラッキング、ガードレール適用
  • OpenAI 互換 REST API で統一されたエンドポイントから呼び出し可能

Architecture

Azure Databricks の Mosaic AI Model Serving が統一的なプロキシとして機能し、ユーザーからのリクエストを AI Gateway を経由して Anthropic の Claude Opus 4.7 に転送する。API キーは Databricks Secrets で安全に管理され、Unity Catalog によるアクセス制御が適用される。

Key Features

🧠

Claude Opus 4.7 ハイブリッド推論

Anthropic の最上位モデルである Claude Opus 4.7 は、通常の応答と深い推論 (extended thinking) を組み合わせたハイブリッド推論モデル。複雑な情報抽出、多段階のエージェント型推論、高度な分析タスクにおいて優れた精度を実現する。

🚀

Mosaic AI Model Serving による統一的なモデル提供

External Models 機能を通じて Claude Opus 4.7 を Databricks のサービング エンドポイントとして公開。OpenAI 互換の REST API を通じてアクセスでき、既存の OpenAI SDK やクライアント ライブラリをそのまま使用可能。

🛡

AI Gateway によるガバナンス

エンドポイントごとの呼び出し回数やトークン数のレート制限を設定可能。モデルの利用状況をリアルタイムで監視し、入力・出力に対する安全性フィルタリング (ガードレール) の適用も可能。

🔑

一元的な資格情報管理

Anthropic API キーを Databricks Secrets に安全に格納し、エンドポイント設定で参照する方式。ユーザーやアプリケーション側に API キーを直接公開する必要がなく、Unity Catalog によるきめ細かいアクセス制御も可能。

Code Samples

エンドポイント作成 (Python SDK / MLflow Deployments)

Python 
import mlflow.deployments

# Databricks デプロイメント クライアントの取得
client = mlflow.deployments.get_deploy_client("databricks")

# Claude Opus 4.7 のサービング エンドポイントを作成
client.create_endpoint(
    name="claude-opus-4-7-chat-endpoint",
    config={
        "served_entities": [
            {
                "name": "claude-opus-4-7",
                "external_model": {
                    "name": "claude-opus-4-7",
                    "provider": "anthropic",
                    "task": "llm/v1/chat",
                    "anthropic_config": {
                        "anthropic_api_key": "{{secrets/my_anthropic_scope/anthropic_api_key}}"
                    }
                }
            }
        ]
    }
)

エンドポイントへのクエリ (OpenAI SDK)

Python 
from openai import OpenAI

client = OpenAI(
    api_key="dapi-your-databricks-token",
    base_url="https://<workspace-url>/serving-endpoints"
)

response = client.chat.completions.create(
    model="claude-opus-4-7-chat-endpoint",
    messages=[
        {"role": "user", "content": "Azure Databricks のアーキテクチャについて説明してください。"}
    ],
    temperature=0.7
)
print(response.choices[0].message.content)

Use Cases

UC1: 大規模データからの複雑な情報抽出

企業の法務部門が、大量の契約書 PDF から特定の条項やリスク要因を自動的に抽出・分類したいケース。Claude Opus 4.7 のハイブリッド推論により、複雑な法的文書の文脈を理解した上での高精度な情報抽出を Databricks 環境内で直接実行できる。

-- ハイブリッド推論による文脈理解で、従来の手動レビューに比べて大幅な効率化と精度向上を実現。

UC2: エージェント型推論による自動分析

データ サイエンティストが、Databricks 上のデータレイクに蓄積されたビジネス データに対して、自然言語による質問に基づく多段階の分析を自動実行するケース。Function Calling を活用し、複数のツール呼び出しを自律的に連鎖させてデータの取得・分析・可視化を自動化する。

-- エージェント型推論能力により、多段階の分析ワークフローを自然言語の指示だけで自動実行可能。

Benefits

Business

  • Azure Databricks のデータ基盤上で最先端 Anthropic モデルを直接活用し、データと AI の統合活用を促進
  • 複数 LLM プロバイダーの統一管理でベンダーロックインリスクを低減
  • AI Gateway のガバナンス機能でコスト管理やコンプライアンス対応が容易
  • 複雑な情報抽出やエージェント型推論タスクの自動化で業務プロセスを効率化

Technical

  • OpenAI 互換 API で既存コードベースやツールチェーンを変更せず導入可能
  • Databricks Secrets による資格情報管理で API キー漏洩リスクを最小化
  • Unity Catalog 統合によるきめ細かい権限管理
  • AI Playground での対話的なモデル評価・プロンプト調整が可能

Limitations / Caveats

  • Anthropic API の利用料金が Databricks の Model Serving 利用料に加えて別途発生する
  • 外部モデルのため、データが Anthropic のインフラストラクチャに送信される点に留意が必要
  • External Models がサポートされるリージョンに制限がある
  • ネットワーク レイテンシーが Databricks ホスト型モデルと比較して大きくなる可能性がある
  • Anthropic API の可用性に依存するため、Anthropic 側の障害が Databricks ワークフローに影響する可能性がある
